| 013 | [013-evict-activate-persist-ordering.md](decisions/013-evict-activate-persist-ordering.md) | `transitionTo` closes the per-direction wake channel AFTER `pool.persist` returns (over moving persist into `lcMu` — deadlocks via `saveLocked` — or serialising transitions through a goroutine — overkill since `Run` is the sole caller); `Activate`/`Evict` drop their already-in-target-state early-return and always wait on the channel so disk and memory agree at every return point |
| 014 | [014-get-or-create-take-or-create.md](decisions/014-get-or-create-take-or-create.md) | `Pool.GetOrCreate` is take-or-create rather than insert-or-error (no `CreateWithID`); validates UUIDv4 shape at the Pool boundary via `ValidID`; holds `p.mu` across register + persist + skip-set + `g.Go` so concurrent same-UUID callers cannot observe a registered entry before its lifecycle goroutine is parked on `activateCh` |
| 015 | [015-update-restart-probe-inline.md](decisions/015-update-restart-probe-inline.md) | `pyry update` daemon-restart probe (`os.Stat` of launchd plist / systemd user unit) lives inline in `cmd/pyry/update.go`, keeping `internal/update/restart.go` a pure decision function with no filesystem calls; restart executor is an injectable function-typed seam on `updateOptions` (mirrors `replace` / `executablePath` pattern) so the integration test asserts argv without exec'ing real `launchctl` / `systemctl` |
| 016 | [016-bootstrap-ignores-persisted-lifecycle-state.md](decisions/016-bootstrap-ignores-persisted-lifecycle-state.md) | Bootstrap session ignores persisted `lifecycle_state` on warm-start: `Pool.New`'s `pickBootstrap` arm forces `lcState = stateActive` regardless of disk because nothing under non-TTY stdin (launchd / systemd / piped wrapper) calls `Pool.Activate` on the bootstrap, so a persisted `evicted` parks `runEvicted` on `activateCh` forever (#202). Non-bootstrap sessions retain persisted state — lazy respawn on attach is correct there |
| 017 | [017-install-script-smoke-check.md](decisions/017-install-script-smoke-check.md) | `install.sh` post-install smoke check is bash + sentinel-grep against `Uptime: 2562047h47m16.854775807s` (over a Go subcommand or daemon-side watchdog); gates on currently-running default-named services only (`launchctl print | grep 'state = running'` / `systemctl --user is-active --quiet pyry`) so deliberately-disabled units aren't auto-started; four exit codes (0 healthy / 2 sentinel / 3 dial-fail / 4 restart-fail) preserve the diagnostic distinction AC #4 names; no manual sleep before the probe — `pyry status`'s built-in dial retry (#199) + 5s deadline IS the grace period |
| 018 | [018-config-overlay-decode.md](decisions/018-config-overlay-decode.md) | `internal/config.Load` initializes `cfg := DefaultConfig()` before `json.Unmarshal(data, &cfg)` so absent JSON fields keep their default values (over a two-pass `map[string]json.RawMessage` merge or pointer-fields distinguishing absent-vs-empty); malformed JSON wraps and surfaces (no silent fallback); error path returns `Config{}` not `DefaultConfig()` so callers that ignore `err` break loudly |
