# Ticket #471 — agent-run: ptyrunner skeleton (spawn + idle wait + WritePrompt + clean shutdown)

Introduces `internal/agentrun/ptyrunner/` — the spawn primitive that drives `claude` as an interactive TUI under [`github.com/pyrycode/tui-driver`](https://github.com/pyrycode/tui-driver), pinned at SHA `b09fe70e60a73d6d52c24707a16802bc1483d532` (first `main` commit after PR #43 landed `Session.WritePrompt`). Sub-issue of [#329](https://github.com/pyrycode/pyrycode/issues/329) (tracking), split from [#468](https://github.com/pyrycode/pyrycode/issues/468). Siblings: [#469](https://github.com/pyrycode/pyrycode/issues/469) (trust pre-write + settings JSON helpers), [#472](https://github.com/pyrycode/pyrycode/issues/472) (JSONL tail + stream-json re-emit + budget + watchdog), [#470](https://github.com/pyrycode/pyrycode/issues/470) (`cmd/pyry/agent_run.go` cutover). This slice is scaffolding only — the package sits alongside `internal/agentrun/streamrunner/`, no production caller imports it yet. The pivot is driven by Anthropic's 2026-06-15 billing-policy classification: "Interactive Claude Code in the terminal or IDE" is explicitly subscription-eligible, but the stream-json subprocess surface that `streamrunner` drives is unclassified; pyrycode pivots proactively to the named-eligible surface before the deadline. The 2026-05-14 reasoning against PTY drive (`/doctor` poisoning, brittle ANSI handling, missing state-detection primitives) no longer applies because `tui-driver` landed comprehensively 2026-05-18 → 19 with PTY lifecycle (`Spawn`/`Session`), state detection (`IsIdle`/`IsThinking`/glyphs), modal handling (`HasTrustModal`/`HasMcpFailureBanner`/`HasNetworkFailure`), `vt10x`-backed render, watchdog tracker, and helpers (`WaitUntil`/`EncodeCwd`/`StripANSI`/`WritePrompt`).

## Implementation

- **Three new files, one production source + two tests** — `internal/agentrun/ptyrunner/runner.go` (266 LOC: package doc + three sentinel error vars + `Config` struct + `Run` + unexported `buildArgs` + unexported `isCtxErr`), `internal/agentrun/ptyrunner/helper_test.go` (89 LOC: `TestMain` dispatcher + `runHelper` fake-claude with five modes), `internal/agentrun/ptyrunner/runner_test.go` (214 LOC: `helperRunCfg` builder + seven `Test…` functions). One `go.mod` line added (`github.com/pyrycode/tui-driver`); one indirect added (`github.com/hinshun/vt10x`). Zero edits to existing files outside `go.mod` / `go.sum`. New knowledge doc (`docs/knowledge/codebase/471.md`, this file).

- **argv shape (per AC #2) is minimal and stripped.** `buildArgs(cfg)` returns the fixed sequence `--session-id`, `--settings`, `--permission-mode default`, `--append-system-prompt-file`, `--model`, `--effort`. The PTY path intentionally omits **`--input-format`**, **`--output-format`**, **`--verbose`** (all three are stream-json mode markers; the interactive TUI rejects them), **`--dangerously-skip-permissions`** (replaced by the deny-default settings JSON from [#469](https://github.com/pyrycode/pyrycode/issues/469)), **`--max-turns`** (the interactive TUI ignores it; [#472](https://github.com/pyrycode/pyrycode/issues/472) enforces the cap pyry-side), and **`--allowed-tools`** (carried as JSON in the settings file). The helper is a pure function returning a fresh slice; one happy-path table assertion in `TestBuildArgs` pins the shape and a forbidden-flags loop ensures none of the stripped flags accidentally re-appear.

- **Three sentinel-shape error vars + `errors.Is` discrimination.** `ErrTrustModalDetected`, `ErrMcpFailureBanner`, and `ErrNetworkFailure` are exported package vars constructed with `errors.New`. Each carries a distinct operator-readable message that names the failing detector — `ErrTrustModalDetected` additionally embeds the substring `"#469's MarkWorkdirTrusted"` so the operator-facing surface (cutover in [#470](https://github.com/pyrycode/pyrycode/issues/470)) can surface the remediation hint by message inspection or by `errors.Is` routing. Detector calls fire in `trust → mcp → network` order at the post-idle snapshot, which is the precise ordering AC #5 pins ("Modal safety check at idle"). Order is not load-bearing for correctness — the detectors are mutually exclusive in practice — but `trust` is first because it's the most actionable (one-shot pre-write fixes it forever).

- **Helper-process mode table** — the fake-claude in `helper_test.go` keys off `GO_PTYRUNNER_HELPER_MODE`:

  | Mode | Bytes written | Detector that fires |
  | --- | --- | --- |
  | `idle` | `❯` + space | `IsIdle` |
  | `trust` | `Quicksafetycheck` + `❯` + space | `HasTrustModal` + `IsIdle` |
  | `mcp_failure` | `1 MCP server failed ` + `❯` + space | `HasMcpFailureBanner` + `IsIdle` |
  | `network_failure` | `FailedToOpenSocket ` + `❯` + space | `HasNetworkFailure` + `IsIdle` |
  | `slow_spawn` | (sleeps 5s before writing anything) | parent's `WaitUntil` ctx-cancels first |

  All modes install a SIGTERM handler so `Session.Close()`'s SIGTERM→grace→SIGKILL sequence resolves on the SIGTERM step (the 3s grace would otherwise dominate test runtime; the seven scenario tests complete in ~3.6s total with this trick). `slow_spawn` is the exception — it sleeps before installing the handler, so the ctx-cancel test exercises the SIGKILL fallback path (still <0.2s elapsed because `SpawnOpts.ShutdownGrace` defaults to 3s and the cancel fires 100ms into the sleep).

- **`WritePrompt` vs `Write` is load-bearing (the slice's headline correctness rule).** [`tui-driver` PR #43](https://github.com/pyrycode/tui-driver/pull/43), merged 2026-05-19, introduced `Session.WritePrompt(text string) error` after empirical verification that `session.Write(promptBytes + "\r")` does NOT commit a multi-line or >1 KB prompt: claude's TUI auto-paste-detects bytes arriving faster than human typing, groups them as `[Pasted text +N lines]` chips, swallows the trailing `\r` into the paste body, and waits indefinitely for an explicit Enter. `WritePrompt` wraps the text in bracketed-paste markers (`\x1b[200~text\x1b[201~`) with a separate trailing `\r` outside the closer — claude's auto-paste-detect honours the explicit markers and treats the trailing `\r` as the commit signal. The package's `Run` calls `sess.WritePrompt(string(cfg.PromptBytes))`; the conversion is the only `[]byte→string` boundary in the call graph. The package doc warns that any future contributor refactoring to `Session.Write` will silently break long prompts — there is no test that exercises a prompt long enough to trip the paste-detect in this slice (paste-detect requires real claude; the fake-claude does not implement it).

- **`TestMain` is required to bypass `go test`'s flag parser.** ptyrunner's `Run` builds its own argv (`--session-id`, `--settings`, …) and passes `os.Args[0]` as the binary so tests re-exec the test binary as fake-claude. The unknown `--session-id` flag would `os.Exit(2)` at `go test`'s own `flag.Parse` if we relied on the streamrunner-style `TestStreamRunnerHelperProcess` sentinel-`if-env-not-set-return`. The fix is a `TestMain` that intercepts `GO_PTYRUNNER_HELPER=1` BEFORE `m.Run()` and dispatches to `runHelper`, which terminates via `os.Exit` and never reaches the flag parser. The streamrunner pattern works because it passes only `-test.run=TestStreamRunnerHelperProcess` + `--` as args (Go-test-known flags); ptyrunner cannot use that pattern because its Config has no `Args` field — argv is internal to the package.

- **`isCtxErr` collapses operator-shutdown into `nil`.** A small helper that returns true if the error wraps `context.Canceled` / `context.DeadlineExceeded` OR if `ctx.Err()` is set. Applied at two sites: `Spawn` failure (claude couldn't start because the parent context was already cancelled) and `WaitUntil` failure (idle never reached before context expired). Mirrors `streamrunner.Run`'s `if ctx.Err() != nil { return nil }` discipline. The double-check is defensive — `tuidriver.WaitUntil` returns `context.Cause(ctx)`, which may be the bare `context.Canceled` OR a wrapped cause; checking `ctx.Err()` directly handles the wrapped case.

- **Logging discipline pinned at the package-doc level.** Only `Warn`-level diagnostics are emitted: spawn failure, close failure, and the three modal-detected events. None of them log `cfg.PromptBytes` or any substring of `sess.Buffer.Snapshot()` — the package doc enumerates the rule explicitly so a future contributor adding a log line is reminded. The Stderr writer is opaque (the package does not parse what passes through it).

- **Forbidden imports verified by `go list -deps`.** The package doc embeds the verifying command:
  ```
  go list -deps ./internal/agentrun/ptyrunner/... | grep -E 'pyrycode/internal/(supervisor|agentrun/(jsonl|streamjson|budget))'
  ```
  Expected output: empty. The three sibling subpackages (`jsonl`, `streamjson`, `budget`) are intentionally absent — they are wired by [#472](https://github.com/pyrycode/pyrycode/issues/472), not this slice. `internal/supervisor` is the long-lived claude wrapper for `pyry start`; ptyrunner is the one-shot agent-run spawn primitive and must not entangle with it. Verified clean post-commit.

- **`MaxTurns` and `Stdout` are declared but unused.** Both fields appear in `Config` with field comments naming the consuming slice (`#472`) and clarifying that `Run` does not touch them today. A casual reader scanning the struct should not be misled into expecting them to do anything in this slice. The precedent is to let the next slice extend `Run` in-place without breaking the public struct shape.

## Patterns established

- **Spawn-primitive package shape (mirrors `streamrunner`).** A spawn primitive in `internal/agentrun/<name>/` exports one `Config` struct + one stateless `Run(ctx, cfg) error` function + sentinel-error package vars (when needed). No constructor, no long-lived object, no exported methods. The package doc carries a three-section structure: purpose, logging discipline, dependency direction (with the verifying `go list -deps` command). `streamrunner` (the stream-json sibling) shipped this shape first; `ptyrunner` adopts it verbatim. Generalises whenever pyrycode needs another claude spawn variant.

- **Forward-compatibility fields declared with `NOT consumed in this slice` field comments.** When a sibling slice (here, [#472](https://github.com/pyrycode/pyrycode/issues/472)) will extend `Run` in-place to consume new Config fields, declare those fields in the struct *now* so the public shape is stable, and mark them with explicit `// declared for #N; NOT consumed in this slice.` comments. This lets the consuming slice add behaviour without breaking field-positional usage (struct-literal callers in tests, etc.), and the inline comment prevents a future reviewer from mistaking the declared-but-unused fields for a bug. Precedent for the next slice (`Stdout` and `MaxTurns` here) to follow if it needs to declare additional forward-compat fields.

- **`TestMain` is the correct dispatcher when fake-child argv collides with `go test` flags.** When a `TestHelperProcess`-style fake-binary is invoked with argv that includes flags `go test` doesn't recognise (anything not prefixed `-test.`), `go test`'s `flag.Parse` will `os.Exit(2)` before the helper function ever runs. The fix is a `TestMain(m *testing.M)` that checks the env-var sentinel BEFORE calling `m.Run()` and dispatches to a non-test helper function that terminates via `os.Exit`. The streamrunner pattern (`if GO_*_HELPER != "1" { return }` inside the test function) only works when the parent passes argv `go test`'s flag parser accepts — which constrains the parent's `Config.Args` to `-test.*` prefixed values. Generalises whenever a spawn primitive owns argv internally and the tests can't influence its shape.

## Lessons learned

- **Forgetting `TestMain` for a TestHelperProcess pattern when the parent owns argv.** Initial draft mirrored `streamrunner/helper_test.go`'s `if !env { return }` shape verbatim. First test run failed with `exit status 2` from every spawn — Go test's `flag.Parse` was rejecting `--session-id` etc. before the helper function got a chance to dispatch. Resolved by introducing `TestMain` that intercepts before `m.Run`. The pre-architect-spec read of the streamrunner pattern missed the subtle precondition: the streamrunner pattern only works because streamrunner's `Config.Args` is caller-controlled (and the streamrunner test passes a flag-parser-safe argv). When the package owns argv internally — as ptyrunner does, per AC #2 — the streamrunner pattern silently breaks. Pattern reinforced: when adopting a test-helper idiom from a sibling package, audit the surrounding contract that makes the idiom work, not just the code shape.

- **`GOPRIVATE` is required to install a private-org Go module via `go get`.** Initial `go get github.com/pyrycode/tui-driver@<sha>` failed with `verifying module: ... 404 Not Found` from `sum.golang.org` because the module is in a private org. Resolved by `go env -w GOPRIVATE='github.com/pyrycode/*'` before the `go get`. The fix lands in the developer's local `~/.config/go/env`, not in the repo — but future developers (and the CI runner) will need the same setting if/when ptyrunner's go.mod entry pulls a new tui-driver version. Pattern: any pyrycode-org private Go dependency requires `GOPRIVATE=github.com/pyrycode/*` in the developer environment. (Worth noting in `CLAUDE.md` if a second private-org dep lands; one-off for now.)

## Files

- `internal/agentrun/ptyrunner/runner.go` — new (+266 LOC). `Config` struct (12 fields incl. `MaxTurns` / `Stdout` forward-compat). `Run(ctx, cfg) error` (validate → exec.CommandContext + EnsureClaudeEnv → Spawn → defer Close → WaitUntil idle → modal/banner detectors → WritePrompt). `buildArgs(cfg) []string` (unexported pure function, six argv pairs). `isCtxErr(ctx, err) bool` (unexported helper for operator-shutdown collapse). Three exported sentinels: `ErrTrustModalDetected`, `ErrMcpFailureBanner`, `ErrNetworkFailure`. Imports: `context`, `errors`, `fmt`, `io`, `log/slog`, `os`, `os/exec`, `github.com/pyrycode/tui-driver/pkg/tuidriver`. Package doc enforces logging discipline + forbidden imports.
- `internal/agentrun/ptyrunner/helper_test.go` — new (+89 LOC). `TestMain` dispatches to `runHelper` when `GO_PTYRUNNER_HELPER=1`, else calls `m.Run()`. `runHelper` writes mode-specific bytes to stdout (UTF-8 anchors satisfying tui-driver's pure-function detectors), drains stdin to `io.Discard` in a goroutine, installs a SIGTERM handler with a 30s fallback timeout, exits 0 on signal.
- `internal/agentrun/ptyrunner/runner_test.go` — new (+214 LOC). `helperRunCfg` Config-builder. `TestRun_HappyPath` (mode `idle`, elapsed <5s). `TestRun_TrustModalDetected` / `TestRun_McpFailureDetected` / `TestRun_NetworkFailureDetected` (each asserts `errors.Is(err, Err*)` + substring on the operator-readable message). `TestRun_CtxCancelDuringSpawn` (mode `slow_spawn`, 100ms cancel, elapsed <8s, returns nil). `TestBuildArgs` (table-light: asserts the six argv pairs + forbidden-flag check loop). `TestRun_MissingRequiredFields` (table-driven, nine subtests, one per required field, each pinning the substring in the wrapped-error message).
- `go.mod` — modified (+1 require: `github.com/pyrycode/tui-driver v0.0.0-20260519122208-b09fe70e60a7`; +1 indirect: `github.com/hinshun/vt10x`).
