# Ticket #606 — Neutral internal turn-event model (`internal/turnevent`)

Introduces `internal/turnevent`, the daemon-owned neutral **outbound**
turn-event model for Phase 2 structured streaming ([EPIC #596]) — pure value
types, stdlib only, no transport or I/O. It is the stable internal contract the
event-stream bridge ([#608]) maps tui-driver `Events()` **into** and the v2 wire
types ([#607]) map **out of**; shaped ~90% like ACP so the future `pyry acp`
adapter ([#600]) is near pass-through, but **owned by us** — so churn in the
external ACP spec stays inside the ACP adapter and never reaches the daemon core
or the mobile wire (the same containment logic the tui-driver substrate seal
applies to claude's screen). This ticket is the **types only**: no `Events()`
draining, no envelope mapping, no consumer wired. It is the foundation [#608] is
blocked on. Full package reference: [features/turnevent-package.md](../features/turnevent-package.md).

## Implementation

Greenfield package, three production files (~150 LOC) + four test files (~180
LOC):

- **`event.go`** — the sealed `Event` sum type (`interface{ isTurnEvent() }`),
  the five event structs (`TextChunk`, `ThoughtChunk`, `ToolStart`, `ToolUpdate`,
  `TurnEnd`), `Location`, value-receiver markers, and compile-time `var _ Event =
  TextChunk{}` assertions. `ToolStart.RawInput` is opaque `json.RawMessage` (never
  parsed); `ToolUpdate.Content` may be `nil` (status-only update); `TurnEnd`
  carries `Reason` only.
- **`taxonomy.go`** — three string-backed enums (`ToolKind`, `ToolStatus`,
  `TurnEndReason`) carrying the **exact ACP taxonomy** strings, grouped/commented
  like `internal/protocol/codes.go`. Each has a `Valid()` predicate that linear-
  scans an **unexported canonical slice** — the single source of truth both
  `Valid()` and the exactness test read, so slice/const/predicate drift is
  structurally impossible.
- **`content.go`** — the sealed `ToolContent` sum type and its three shapes
  (`TextContent`, `DiffContent`, `TerminalContent`); `nil` is a legal "no content
  change".
- **`boundary_test.go`** — AC#5: a deterministic stdlib-only import boundary
  enforced with `go/parser` (see Lessons learned).

The only production import is `encoding/json` (for `json.RawMessage`). `go test
-race`, `go vet`, staticcheck, and the substrate guard are all green (no claude
screen literals appear here).

## Patterns established

- **Sealed marker interface as the closed sum-type seam.** Two seams — `Event`
  (the heterogeneous turn-event stream) and `ToolContent` (a `ToolUpdate`'s
  polymorphic content) — each an `interface{ isXxx() }` with an unexported marker
  on a **value receiver** so `TextChunk{}` (not only the pointer) satisfies it.
  The unexported marker keeps the variant set closed to the package, which is the
  whole point: external ACP-spec churn cannot inject a variant. This is an
  explicit, justified exception to `CODING-STYLE.md`'s "no preemptive interfaces"
  rule — there are already 5/3 concrete implementations and a known consumer
  ([#608]) needing one typed stream element.
- **Single unexported canonical slice = no enum drift.** Per enum, one
  package-private slice (`toolKinds`, …) is both what `Valid()` scans and what the
  exactness test deep-equals against an independent hardcoded ACP literal (+ a
  `len == N` count guard). Adding a const without a slice entry — or vice versa —
  fails CI. Mirrors `internal/protocol`'s drift-detector philosophy without a
  third extra copy.
- **Primitive exposes `Valid()`, not errors or constructors.** The package
  returns no `error` and validates nothing at construction; the seam consumer
  rejects an out-of-taxonomy value via `Valid()` and maps it to its own
  wire/sentinel error. Reaffirms the project convention *"refusal-to-wire-code
  mapping is the consumer's job, NOT the primitive's"* (PROJECT-MEMORY).
- **Internal model is the event-stream shape; ACP-return divergences are the
  adapter's job.** `TurnEnd` carries `Reason` only — ACP's end-of-turn is a
  `stopReason` *return value*, not an event; re-deriving that RPC return is [#600]'s
  job (design-doc divergence 1), not the model's.

## Lessons learned

- **The stdlib-only import boundary is enforced deterministically without a
  third-party linter.** `go/parser.ParseFile(..., parser.ImportsOnly)` over the
  package's non-`_test.go` files, rejecting any import whose first path segment is
  dotted (the signature of a module path) — plus an explicit
  `github.com/pyrycode/pyrycode/` prefix check for a clearer message. Cheap,
  hermetic, and catches *any* external/intra-repo dependency, not an enumerated
  denylist. A `checked == 0` guard prevents the test passing vacuously if the file
  walk ever finds nothing. (Lifted from the PR's Lessons-learned section.)

## Size note (S kept; one red line a documented false positive)

The design declares ~14 exported types, tripping the architect's `>5 exported
types or interfaces` red line. Kept at **S** deliberately: the count reflects how
many variants the ACP **domain** defines (each a ~4-line declaration, no logic,
no untrusted input, no concurrency, no consumer in scope), which is exactly the
recorded `po-data-model-taxonomy-size-red-line` false positive. The decisive test
— *would the PO's pre-authorized A/B split clear the line?* — is **no**: the B
half alone (`ToolStart`, `ToolUpdate`, the two tool enums, the three content
shapes, `ToolContent`, `Location` = 9 types) is still over 5. A split that doesn't
clear the tripped metric proves the metric is the wrong proxy here, so the cut was
**not exercised**. Every other dimension is comfortably S (~330 LOC, 3 production
files, 0 consumer call sites, 0 state-machine reject branches, 5 ACs).
