# Ticket #699 — Remove the dead coarse / non-interactive `message` fan-out (v2)

The deletion that collapses the v2 (Noise) assistant-turn delivery to a **single
path**. Per the 2026-06-22 amendment to [ADR 025] ([#698]), pyrycode ships the app and
daemon together — there is no old-app install base, so every phone always negotiates the
`interactive` capability. That made the **v2 coarse `message` fan-out** dead code: the
coarse emitter gated with `if c.Interactive { continue }` (the exact complement of
[#632](632.md)'s interactive-only structured filter), so its "fan a coarse `message` to a
non-interactive conn" branch was unreachable once no non-interactive conn ever exists.
This ticket deletes the coarse v2 emitter, its wiring, and its tests, leaving the
structured interactive stream ([#632](632.md)/[#633](633.md)) as the **sole** v2
assistant-turn path. It is the terminal step of the path [#589](589.md) opened and
[#634](634.md) re-targeted.

**`security-sensitive`** — "deleting a routing/dispatch gate = changing it." The coarse
path's `if c.Interactive { continue }` was a delivery gate; the security argument is that
removing it preserves the per-conn delivery boundary byte-for-byte (see Security review).
Architect self-review: **PASS, no MUST FIX**.

## Implementation

A purely subtractive change. No new types, no signature changes, no consumer cascade —
the entire production delta is one deleted file plus a wiring removal in `startRelayV2`.

### What we keep (the architect's ADR call — plumbing stays inert)

The ADR/ticket left it to the architect whether to also strip the now-inert capability
plumbing. **Decision: keep all of it inert; `internal/relay` is untouched.**

- **`ActiveConn.Interactive` is not dead** — it is still read by the two surviving
  structured-path guards `if !c.Interactive { continue }`
  (`interactive_turn_v2.go:334`, `session_transition_v2.go:129`). Removing the field
  forces removing those guards *and* the `V2Session.interactive` / `handleActiveConns`
  plumbing — a multi-package refactor into a security-sensitive surface, past `size:s`.
- **The guards are deterministic safety nets** (belt-and-suspenders). They are permanent
  no-ops *only because* every conn is interactive today; removing them would change
  behaviour (a hypothetical non-interactive conn would begin receiving the structured
  stream) with no observed failure demanding it.
- **`TypeMessage` / `MessagePayload` wire constants stay** — still produced by the
  out-of-scope v1 / dispatch-leg coarse bridge (`cmd/pyry/assistant_turn.go`,
  `internal/turnbridge/outbound.go`, `internal/relay/handlers/send_message.go`).
- **`capabilities` round-trips unchanged** (AC#3) — `negotiateCapabilities` is untouched.

Stripping the inert plumbing is a deferred future `internal/relay` follow-up (PO's call;
no functional urgency).

## Patterns established

- **A dead routing branch is still a security-relevant deletion.** The check on
  "deleting a gate = changing it" is: *does the per-conn delivery boundary survive?* Here
  it does, byte-for-byte, **because** the complementary structured-path guards are kept —
  a non-interactive conn still receives nothing (not the coarse `message`, and the
  structured stream still skips it). The change removes one sealed delivery surface and
  opens none. Had the inert guards been stripped too (the rejected refactor), a
  non-interactive conn would suddenly receive the structured stream — a confidentiality
  widening. Subtractive-on-delivery is the safe shape.
- **Relocate shared helpers before deleting their host file.** A test file slated for
  deletion can define helpers that *surviving* tests still call. Move them into their sole
  surviving consumer first; `go test ./internal/e2e/...` compiling is the canary.
- **Grep the delivery assertion, not the gate literal.** A deletion spec's
  `grep -rn "Interactive: false"` catches only unit-test struct literals — it misses e2e
  tests that drive a non-interactive phone through a real handshake. For a fan-out
  cleanup, grep the *delivery* assertion (`TypeMessage` on the v2 leg) to find every site.

## Lessons learned

Lifted from the PR body's **Lessons learned**:

- **The spec enumerated two coarse-fan-out test targets, but four sites asserted v2
  coarse `message` delivery.** The two unlisted ones were test-only and needed
  reconciliation to satisfy AC#4 ("no remaining test asserts a coarse `message`
  fan-out"): the [#642](642.md) structured capstone's B-side asserted B *receives* the
  coarse message (now: B receives nothing), and `TestRelayV2_AssistantTurn_BroadcastsMessageEnvelope`
  (`internal/e2e/relay_v2_daemon_test.go`) was a dedicated coarse-broadcast e2e (now
  deleted). The architect's `grep -rn "Interactive: false"` only catches unit-test struct
  literals, not e2e tests that drive a non-interactive phone via a real handshake — grep
  for `TypeMessage` delivery assertions on the v2 leg instead.
- **`internal/relay`'s `TestPushQueue_Enqueue_MessageIsNeverDrop` references `message`
  but is *not* a fan-out test** — it pins the pushQueue droppability *class* (a still-valid
  wire-type policy in the out-of-scope package), so it was correctly left untouched. The
  v1 coarse-broadcast e2e `TestRelay_AssistantTurn_BroadcastsMessageEnvelope`
  (`relay_assistant_turn_test.go`, no "V2") is likewise the out-of-scope v1 path and stays.
- **Cosmetic residue, deliberately left:** doc comments in `interactive_turn_v2.go` /
  `session_transition_v2.go` still say "mirrors `assistantTurnEmitterV2`", now naming a
  deleted type. Go does not validate comment references; not worth a churn pass.

## Security review (security-sensitive) — PASS

Strictly subtractive on the daemon→phone delivery side. The coarse emitter read the
supervisor cursor, minted a `message`, and `Push`ed it **sealed under the session
CipherState** to every open *non-interactive* conn. Removal deletes that delivery
surface and opens none — no new bytes flow to any phone.

The load-bearing check is **no widening of who receives what**: an interactive conn
already got only the structured stream (the coarse gate skipped it) — unchanged; a
non-interactive conn got the coarse `message` and now gets nothing, and because the
structured-path guards are kept it still gets nothing from the structured stream either —
the per-conn delivery boundary is preserved exactly. `internal/relay` (handshake, token
validation, `V2StateOpen` gating, `ActiveConns` enumeration) is untouched, so the set of
conns eligible for any fan-out is computed as before. Wire shape preserved: `capabilities`
still round-trips; `TypeMessage` / `MessagePayload` retained. No trust boundary relaxed,
no untrusted-input surface added.

## Testing

- **Deletion of the unit + e2e tests is itself AC#4** — no remaining test asserts a
  coarse `message` fan-out to a non-interactive conn.
- **AC#2 (no v2 conn left without assistant output):** covered by the surviving
  `TestTwoPhoneStructured_InteractiveReceivesStream` — A (interactive) receives the full
  structured stream; B (non-interactive) receives nothing. The relocated helpers compiling
  there is the regression guard for the e2e move.
- **AC#3 (`capabilities` round-trips):** unchanged wire behaviour; existing
  `internal/relay` capability-negotiation tests cover it, untouched.
- **Gate:** `make check` green (`go vet`, `go test -race`, `staticcheck`,
  `cmd/substrate-guard`); `go vet -tags e2e ./internal/e2e/...` compiles.

## Out of scope / deferred

- **The pre-v2 dispatch-leg coarse bridge** (`cmd/pyry/assistant_turn.go` /
  `startAssistantTurnBridge`, wired only when `v2Enabled == false`) is a different, older
  mechanism — no capability gate, predates capability negotiation, still produces
  `message`. **Untouched.** Whether the non-v2 relay leg is supported at all is a separate
  question, not this ticket.
- **Stripping the inert capability plumbing** (`ActiveConn.Interactive` + the two
  structured-path guards + `V2Session.interactive`) — a deliberate non-goal here (keeps the
  change in-package and within S; the guards are cheap deterministic safety nets). A future
  `internal/relay` follow-up may strip it; no functional urgency.
- **Per-conversation outbound routing for the surviving v1 coarse bridge** — still fans
  from the bootstrap cursor (pyrycode-mobile#336; the #672-family re-key follow-up).
