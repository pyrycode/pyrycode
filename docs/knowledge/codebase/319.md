# Ticket #319 — `register_push_token`: dispatch.Handler rewrite + wiring

Sibling of #313 (`list_conversations` rewrite). Split from #305; second of
two slices, depends on #318 (per-conn auth slot, landed). Replaces the
pure `handlers.Handle` signature from #250 with a `dispatch.Handler`
factory, registers it in `cmd/pyry/relay.go` alongside
`list_conversations`, and adds the first end-to-end test exercising the
full phone → binary → registry → ack path through the dispatcher.

## Implementation

### Handler factory (`internal/relay/handlers/register_push_token.go`)

```go
func RegisterPushToken(reg *devices.Registry, registryPath string, logger *slog.Logger) dispatch.Handler
```

Returns a closure with signature
`func(ctx context.Context, c *dispatch.Conn, env protocol.Envelope) error`.
`reg`, `registryPath`, and `logger` are captured — no globals, mirroring
[`ListConversations`](../features/relay-package.md#list_conversations-303).
The third (`logger`) parameter diverges from `ListConversations` because
this handler logs on every branch.

Branch order inside the closure:

1. **Nil-device guard** — `dev := c.Auth(); if dev == nil` → reply
   `auth.invalid_token` (Retryable=false) via `replyError`. WARN log
   `event=register_push_token.unauth`, no device-name field (there is
   none). **No registry touch.**
2. **Payload decode** — `json.Unmarshal(env.Payload, &p)` → on error
   reply `protocol.malformed` (Retryable=false) via `replyError`.
   The decode-error text is NOT echoed; only the static
   `msgMalformed = "malformed register_push_token payload"`. WARN log
   `event=register_push_token.malformed` with `err`.
3. **Dedupe** — if `(p.Platform, p.Token, p.DeviceName) == (dev.Platform,
   dev.PushToken, dev.Name)` reply `ack`. DEBUG log
   `event=register_push_token.dedupe`. **No registry touch** — preserves
   the load-bearing optimisation from #250.
4. **Mid-conn-removed race** — `reg.UpdatePushRegistration(...)` returned
   `false` → reply `auth.invalid_token` (same UX as branch 1). WARN log
   `event=register_push_token.gone_mid_conn`.
5. **Save failure** — `reg.Save(registryPath)` returned non-nil → reply
   `server.binary_busy` (Retryable=true, RetryAfterS=nil). WARN log
   `event=register_push_token.save_failed` with `err`. **In-memory is
   still mutated** — post-condition preserved.
6. **Success** — reply `ack`. INFO log `event=register_push_token.write`
   with `device_name=p.DeviceName` and `platform`. Push token never
   appears as a log field.

Two file-local helpers (`replyAck`, `replyError`) marshal the payload
and call `c.Reply(ctx, env, type, payloadJSON)` — the dispatcher stamps
`id` (via `c.NextID()`), `in_reply_to` (echoes `env.ID`), and `ts`
(`time.Now().UTC()`).

### What got removed

- `wrap(connID, inReplyTo, nextID, envType, payload)` — replaced by
  `c.Reply`. The "duplicated `relay.buildResponse`" comment in #250's
  feature doc is now obsolete; the central dispatcher helper is the
  only stamping path.
- `ErrMalformedFrame` sentinel — the dispatcher's `handleOne` has
  already decoded `env protocol.Envelope` from the routing frame before
  invoking the handler. The inner-payload decode survives but now
  replies via `c.Reply` with `protocol.malformed` instead of returning
  a sentinel.
- `routing protocol.RoutingEnvelope`, `device *devices.Device`,
  `nextID uint64` parameters — replaced by `env` (already decoded),
  `c.Auth()`, and `c.NextID()` (called inside `c.Reply`).

### Daemon wiring (`cmd/pyry/relay.go`)

One new register line inside `startRelay`, immediately after
`list_conversations`:

```go
d.Register(protocol.TypeRegisterPushToken, handlers.RegisterPushToken(
    registry, resolveDevicesPath(instanceName), logger))
```

`registry` is the `*devices.Registry` already loaded by `startRelay`
(needed for `authGate`). `resolveDevicesPath(instanceName)` is the
canonical on-disk path — called once at the registration site so the
factory captures a stable string.

### Test seam: `dispatch.NewTestConn` (`internal/dispatch/dispatch.go`)

Handler tests in `internal/relay/handlers` cannot reach the unexported
`setAuth` from a sibling package. The seam is a new exported, non-test
constructor:

```go
func NewTestConn(id string, outbound chan<- protocol.RoutingEnvelope,
                  auth *devices.Device) *Conn
```

Three field assignments; doc-comment opens with **"Test fixtures only —
do not call from production code."** It does NOT mutate auth on a
dispatcher-owned `*Conn`; it only constructs a fresh `*Conn` whose
outbound channel is supplied by the caller. The dispatcher remains the
sole production `*Conn` factory (via `routeConn`).

Returned `*Conn` starts with `nextID = 0` so the first `NextID()`
returns 1. Tests that want to simulate post-`hello_ack` state call
`c.NextID()` once before driving the handler so the first
handler-originated reply lands at id=2.

## Test surface

### Unit tests (`internal/relay/handlers/register_push_token_test.go`, rewritten)

Drives the new signature against a real `*dispatch.Conn` built with
`dispatch.NewTestConn(testConnID, out, dev)`. `newTestConn` returns
`(*Conn, recv func())`; `recv` drains one `RoutingEnvelope`,
unmarshals its `Frame`, and asserts `ConnID == testConnID`. Tests
assert `Type`, `ID == 2` (post-hello_ack accounting), `*InReplyTo ==
testRequestID`, and payload contents.

Seven flat tests, stdlib only, all under `-race`:

- `TestRegisterPushToken_FirstTimeRegister_WritesAndAcks` — happy-path
  write + reload via `devices.Load` and assert the triple.
- `TestRegisterPushToken_ReregisterIdentical_NoWriteAndAcks` — dedupe
  spy: file deliberately not pre-Saved; asserts
  `errors.Is(os.Stat(path), fs.ErrNotExist)` after the call.
- `TestRegisterPushToken_ReregisterChanged_WritesAndAcks` — pre-Save,
  change one field, content-equality post-call (sidesteps CI mtime
  flakes).
- `TestRegisterPushToken_GoneMidConn_EmitsAuthInvalidToken` — empty
  registry + `dev` with a TokenHash absent from the registry, so
  `UpdatePushRegistration` returns false. Asserts
  `Code=auth.invalid_token`, `Retryable=false`.
- `TestRegisterPushToken_SaveFailure_EmitsServerBinaryBusy` —
  regular-file blocker forces `MkdirAll` to fail. Asserts the error
  code/retryable + in-memory-still-mutated post-condition.
- `TestRegisterPushToken_UnauthenticatedConn_EmitsAuthInvalidTokenNoWrite`
  — `auth=nil`; asserts the error shape + no file + unchanged registry.
- `TestRegisterPushToken_MalformedPayload_EmitsProtocolMalformed` —
  `env.Payload = []byte("not-json")`; asserts
  `Code=protocol.malformed`, `Retryable=false`. **New case** —
  replaces the deleted `TestHandle_MalformedFrame_ReturnsSentinel`
  (the dispatcher owns the malformed-frame path now).

The `equalRouting` helper and the malformed-routing-envelope test from
#250 are gone with the sentinel.

### e2e (`internal/e2e/register_push_token_test.go`, new file)

`//go:build e2e`, single test `TestRelay_RegisterPushToken_AckAndPersists`:

1. `RunBareIn(t, home, "pair", "-pyry-name=test", "--name=phone-a")` to
   mint a device; `decodePairPayload` captures the plaintext token.
2. `fakerelay.New(...)`, `StartInWithEnv(t, home,
   []string{"PYRY_ALLOW_INSECURE_RELAY=1"}, "-pyry-relay=...")` boots
   the daemon.
3. Wait up to 5s for `fr.LastBinaryHello(serverID)`.
4. `fakephone.Dial(...)`, send `hello`, assert `hello_ack` with
   `*InReplyTo == 1`.
5. Send `register_push_token` with `ID: 2`. Assert `ack`,
   `*InReplyTo == 2`, **and `env.ID >= 2`** — the dispatcher consumed
   id=1 for `hello_ack` and advanced past it (`runGate`'s
   `_ = c.NextID()`); strictly-greater leaves room for any future
   dispatcher-side replies between `hello_ack` and this ack without
   churning the assertion.
6. Reload `~/.pyry/test/devices.json` via `devices.Load` and assert
   `Platform == "fcm"`, `PushToken == "fcm-token-xyz"`,
   `Name == "phone-a"`.

The test exercises the full chain: gate accept → `c.Auth()` populated
→ handler reads `Auth` → `c.Reply` stamps `id`/`in_reply_to`/`ts` →
relay forwards ack → registry persisted with the new triple.

## Patterns established / reinforced

- **Factory closure for handler dependencies.** Same shape as
  `ListConversations(convReg)` — the closure captures registry +
  on-disk path + logger; no globals, no parameter threading through
  the dispatcher. The third-parameter divergence (logger) is fine; a
  consistent two-param shape is not load-bearing.
- **`c.Reply` is the only stamping path.** Every reply — ack and all
  three error envelopes — goes through `c.Reply`. The per-conn id
  invariant, the `in_reply_to` echo, and the `ts` field are dispatcher
  concerns, not handler concerns. The local helpers (`replyAck`,
  `replyError`) only marshal the payload.
- **Test-only constructor over per-package test seams.** Go does not
  propagate `_test.go` symbols across packages. The handler test sits
  in `internal/relay/handlers` and needs a `*dispatch.Conn` with a
  controllable outbound channel and a synthesised auth. The smallest
  seam is an exported `NewTestConn` in `internal/dispatch` with a
  "test fixtures only" doc-comment; this is preferable to either
  black-box-testing the handler through a full `Dispatcher` (heavier;
  requires `FirstFrameGate`, frames channel, goroutine lifecycle) or
  moving the handler test into the dispatch package (loses package
  isolation).
- **Same-UX auth rejection across two paths.** Nil-device (defence in
  depth against a dispatcher bug) and `UpdatePushRegistration` returning
  false (concurrent revoke between auth-accept and frame arrival) emit
  byte-identical `auth.invalid_token` envelopes. Inline rather than
  extracted — the existing handler does the same, and the architect's
  spec defers the extraction.

## Lessons learned

### `c.NextID()` accounting is the test's responsibility

The dispatcher's gate consumes `id=1` for `hello_ack`; the first
handler-originated reply lands at `id=2`. Unit tests that drive a
handler directly (not through the dispatcher) skip the gate entirely,
so `NextID()` would otherwise return `1` on the first reply — a
divergence from the production state at the moment the handler runs.
`newTestConn` calls `_ = c.NextID()` once before returning, mirroring
the gate's accounting; tests assert `env.ID == testNextID` (= 2) as
the post-handler observable.

If a future test fixture needs to skip the gate-accounting and observe
`id=1`, it can construct the `*Conn` directly via
`dispatch.NewTestConn(...)` without the bump. The bump lives in the
test-file helper, not the constructor, so the discipline is opt-in.

### The malformed-payload test replaces the malformed-frame test

#250's test asserted `errors.Is(err, ErrMalformedFrame)` — a sentinel
return that flagged the dispatcher's responsibility. Now the dispatcher
already decoded `env.Envelope` before invoking the handler, so frame
malformedness can never reach this handler. The only malformedness left
is the **inner payload** — `RegisterPushTokenPayload`. The new test
asserts the wire reply (`Code=protocol.malformed`, `Retryable=false`)
rather than a Go-level sentinel.

The static message `msgMalformed = "malformed register_push_token
payload"` is documented and pinned by the test. The original decode-
error text is deliberately NOT echoed (could reflect attacker-
controlled payload bytes).

### `NewTestConn`'s abuse surface is minimal

The architect's security review flagged the new exported constructor as
"SHOULD FIX in implementation, not in the spec". The realistic abuse
is a future developer reaching for it in production wiring. Mitigations:

- Doc-comment on `NewTestConn` opens with "Test fixtures only — do not
  call from production code."
- The constructor cannot grant fake auth to a real session — it only
  builds a fresh `*Conn` whose outbound channel is supplied by the
  caller, never poking `setAuth` on a dispatcher-owned `*Conn`.
- Code review checks no `cmd/` package calls it.

The same posture as other "test fixture" exports in the tree (e.g.
`transport.newClientForTest`-style seams in `internal/transport`).

## Out of scope (deferred)

- **Per-field length caps on `Platform`/`Token`/`DeviceName`.** The
  architect's security review noted that a 1 MiB inbound frame can
  push ~1 MiB of strings into `devices.json` for a single registered
  device. The existing #250 handler doesn't enforce caps either; the
  rewrite is "preserve, don't worsen". Filed for a follow-up ticket.
- **`replyUnauthorized` helper extraction.** Both the nil-device and
  mid-conn-removed branches emit the same `auth.invalid_token` envelope.
  The architect's spec explicitly defers the extraction; mirrors the
  inline shape from #250.
- **Token revocation propagation to live conns.** A paired device whose
  token is revoked mid-session still holds its `*devices.Device`
  pointer on `Conn.Auth()` until the WS closes. The `gone_mid_conn`
  branch is the closest mitigation: the in-memory registry mutation
  surfaces on the next `register_push_token` for that device. See
  `codebase/318.md` § "Out of scope".
