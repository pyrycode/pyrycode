# Ticket #668 — deterministic transcript-growth commit-confirm for supervised turn delivery

Fixes the mobile "first message after `create_conversation` never reaches claude" bug
(pyrycode-mobile#421 rung-3, the last blocker). The supervised-bootstrap delivery path
([#594](594.md)) returned `nil` from `WriteUserTurn` whenever tui-driver reported
`DeliverResult.Committed == true` — but that signal **false-acks** the short single-line
first mobile turn: the phone is told the turn landed when no turn was written, so it never
retries and no assistant reply ever streams. This ticket replaces the trusted-`Committed`
confirm with a **deterministic transcript-growth** confirm on the bootstrap path:
`WriteUserTurn` returns `nil` only after the resolved claude JSONL actually *grows*;
otherwise it returns the existing `ErrTurnNotCommitted` (a loud, retryable failure) instead
of a false ack.

`security-sensitive`: the delivery path for untrusted, phone-originated turn payloads
(`send_message` → `WriteUserTurn` → live claude). Spec security review: **PASS** (no MUST
FIX) — the change strictly *tightens* the success contract (`nil` only on observed growth)
and adds no new attacker-controlled sink; only `os.Stat` *sizes* are read, never JSONL
content, so the substrate seal stays intact.

## Root cause (confirmed during design)

1. `create_conversation` is pure registry metadata — it never spawns or restarts claude
   ([#666](666.md) § "no session spawn at create time"). So the ~7.5s clean exit +
   `--continue` restart that follows the create is **claude's own black-box behaviour**, not
   a pyrycode-triggered restart. The supervisor cannot and should not try to prevent it — its
   reason to exist is to tolerate the child exiting at any time. **The fix makes turn
   *delivery* robust to a restart racing the send; it does not chase the restart's cause.**
2. The `send_message` arrives while claude is mid-`--continue` resume. The prompt is short and
   single-line, so `DeliverPrompt` uses `TypePrompt` (no bracketed paste → **no `[Pasted
   text]` chip ever renders**). When the turn doesn't commit, tui-driver's `deliverPrompt`
   sees no commit signal *and* no chip and concludes "committed-but-slow" →
   `DeliverResult.Committed = true`. So `deliverViaSession` returned `nil`, the handler sent
   `TypeAck`, and the phone had no reason to retry. The transcript is byte-identical before
   and after.
3. **Why the existing levers didn't catch it.** tui-driver's optional `JSONLPath` commit
   signal is *appearance*-based (`os.Stat` — "the file exists"). Under `--continue` the
   per-session JSONL already exists from the prior turn, so passing `JSONLPath` would make
   it confirm *instantly* — actively worse. **Growth** (size increased) is the only reliable
   on-disk signal, and tui-driver v1.3.0 offers no growth check. It must be done
   pyrycode-side.

## Implementation

Three production files, all in `internal/`. ≤2 production files of real logic + one wiring
line.

### `internal/supervisor/supervisor.go` — the rewrite

- **New `Config.ResolveTranscript` field** — `func(ctx) (path string, size int64, err
  error)`, optional + nil-safe, modelled on `ValidateConversation`. `("", 0, nil)` means no
  transcript exists yet (valid); a non-nil error means the dir is unreadable. The
  production-vs-test seam: set → growth-confirm; nil → #594 `Committed` behaviour
  (foreground mode, unit tests).
- **`deliverViaSession` branches on the resolver.** nil → unchanged #594 body (`WaitReady`
  → `DeliverPrompt` → `!Committed ⇒ ErrTurnNotCommitted` → `nil`). Non-nil → wires the real
  `Session` methods + `Config.ResolveTranscript` into `confirmViaTranscriptGrowth` and
  returns its result. `JSONLPath` in `DeliverOpts` stays empty in **both** modes — we own the
  growth signal now.
- **`confirmViaTranscriptGrowth(ctx, deliverGrowthDeps) error`** — the pure confirm helper.
  Order: `waitReady` → capture baseline (`resolve`) → `deliver` (Committed ignored) → poll
  `resolve` every `transcriptConfirmPoll` (150ms) until `grew()` or `transcriptConfirmTimeout`
  (10s) or `ctx` done. Returns `nil` only on observed growth; `ErrTurnNotCommitted` on
  no-growth-in-window; `ctx.Err()` (matchable) on cancellation.
  - **Baseline is captured *after* `WaitReady` returns idle**, so any JSONL writes claude
    makes *during* the `--continue` resume are already in the baseline and cannot be mistaken
    for the user turn.
  - **Baseline-resolve error → fall back to `Committed`** (log `Warn`, then deliver and gate
    on `committed`). A transient dir-read error must not manufacture a false *negative* that
    drives a duplicate send.
- **`grew(basePath, baseSize, newPath, newSize) bool`** — growth = a larger file (turn
  appended) **or** a different newest file (`/clear` rotation). A still-empty resolution
  (`newPath == ""`) is not growth.
- **`deliverGrowthDeps`** — the testability seam (`waitReady` / `deliver` / `resolve` / `log`
  + injectable `timeout` / `poll`). Tests inject fakes to script readiness, delivery, and
  growth outcomes with **no live claude and no claude-screen literal** — the same "seam one
  level above the screen" pattern as #594's `deliverFn`. `timeout`/`poll` live on the deps
  struct (not package vars) so `t.Parallel()` `-race` tests shrink them without sharing
  mutable global state.
- **Two new unexported constants:** `transcriptConfirmTimeout = 10s` (a documented tuning
  knob, not a contract — generous to cover a slow `--continue` resume drain; too tight risks
  a false negative → phone retry → duplicate turn) and `transcriptConfirmPoll = 150ms`
  (matches tui-driver's `promptCommitPoll`).

### `internal/sessions/reconcile.go` — the resolver

- **`newTranscriptResolver(dir) func(ctx) (string, int64, error)`** — the
  `Config.ResolveTranscript` closure. **Reuses `mostRecentJSONL`** (no third scanner), then
  `os.Stat`s the result for size. Selecting the *same* file `mostRecentJSONL` /
  `reconcileBootstrapOnNew` / the rotation watcher pick is the point: it is **the same file
  the turn-bridge ([#661](661.md)) tails**, so a growth this path confirms is the very turn
  the bridge will stream. A file that vanishes between the scan and the `os.Stat` surfaces as
  the stat error (treated as "no baseline" / "no growth", never a panic). The `ctx` param is
  unused today — it satisfies the field signature and lets a future resolver honour
  cancellation.

### `internal/sessions/pool.go` — wiring (bootstrap only)

- One block after the `ValidateConversation` closure: `if cfg.ClaudeSessionsDir != "" {
  supCfg.ResolveTranscript = newTranscriptResolver(cfg.ClaudeSessionsDir) }`. Bootstrap-only
  by design — the `buildSession` / `--session-id` path stays nil-resolver (see Out of scope).

## Patterns established

- **Belt-and-suspenders means different fabric — applied.** The stochastic tui-driver chip
  heuristic gets a *deterministic* safety net (on-disk transcript growth), not another
  stochastic agent. This is the JSONL-net upgrade [#594](594.md) explicitly deferred: #594
  gated on `Committed` with no downstream net; ptyrunner treats `Committed` as advisory *with*
  a JSONL/watchdog net — this path now gets that net.
- **Appearance is not commit; growth is.** For a resumed (`--continue`) session the per-session
  JSONL already exists, so an `os.Stat`-existence signal confirms instantly and wrongly. The
  reliable on-disk commit signal is file *growth* (size increase, or a rotated newer file),
  measured against a baseline captured *after* the readiness gate so resume-time writes don't
  count.
- **Resolve the same file the downstream consumer reads.** The growth resolver reuses
  `mostRecentJSONL` rather than inventing a parallel scan, guaranteeing by construction that
  the confirmed file == the file the turn-bridge streams. Coherence between the commit-confirm
  and the stream source is what lets the reply actually reach the phone.
- **A false ack on an inbound path is a security defect, not just a bug.** Converting "told
  the phone it succeeded when it didn't" into a loud, retryable failure is a net security
  improvement on an untrusted path — the change can never make `WriteUserTurn` return `nil`
  where #594 returned an error.

## Error handling & delivery model

No new sentinel, no new wire code, no handler edit — the design reuses #594's
`ErrTurnNotCommitted` → handler `default` arm → `server.binary_offline` **retryable** mapping
(`internal/relay/handlers/send_message.go`). Growth observed → `nil` → `TypeAck`; no growth in
window → `ErrTurnNotCommitted` → retryable; `ctx` cancelled mid-poll → `ctx.Err()` → retryable;
baseline-resolve error → fall back to `Committed`.

**At-least-once, not exactly-once.** A turn that commits *after* the 10s window →
`ErrTurnNotCommitted` → phone retry → duplicate turn. Same at-least-once property #594's
deliver-timeout already had; the generous window keeps it rare. Exactly-once / in-driver
re-delivery is tui-driver#124's job, not this ticket's.

## Concurrency model

No new goroutines. `confirmViaTranscriptGrowth` runs synchronously on the same per-conn /
handler goroutine `WriteUserTurn` already uses (post-`sessMu`-release, on the captured
`*Session` — the #594 capture-then-unlock contract is unchanged). The poll is a `time.Timer`
+ `time.Ticker` + `ctx.Done()` `select`, both `Stop`-ped on return; fully bounded by `timeout`
and the handler's 30s `deliverCtx`. The resolver does read-only `os.ReadDir` / `os.Stat` on a
server-owned dir — no shared mutable state. A `setSession(nil)+Close()` racing the captured
pointer stays teardown-safe (#594): a torn-down session surfaces as a `WaitReady` /
`DeliverPrompt` error → loud failure, never a crash or false ack.

## Testing

stdlib `testing`, table-driven, `-race`, `t.Parallel()`.

- **`TestSupervisor_ConfirmViaTranscriptGrowth`** (`supervisor_test.go`) — 9-row table over
  fake `deliverGrowthDeps` (no live claude, no screen literal), with a scripted-`(path,size)`
  counter `resolve` and shrunk `timeout`/`poll` so it runs in ms: growth on first poll →
  `nil`; growth after several polls → `nil` (proves the loop waits); never grows →
  `ErrTurnNotCommitted`; rotation (newer file) counts as growth; empty baseline then file
  appears → `nil`; `waitReady` error wraps + `errors.Is(context.DeadlineExceeded)` survives;
  `deliver` error returned, no panic; baseline-resolve error + `committed=true` → `nil`
  (fallback); baseline-resolve error + `committed=false` → `ErrTurnNotCommitted` (fallback).
- **`TestSupervisor_ConfirmViaTranscriptGrowth_CtxCancelledMidPoll`** — `ctx` cancelled
  mid-poll → non-nil, `errors.Is(context.Canceled)`.
- **`TestNewTranscriptResolver_*`** (`reconcile_test.go`, reusing the `t.TempDir` +
  `mostRecentJSONL` fixtures) — `PicksNewestWithSize`, `EmptyDir` → `("", 0, nil)` (not an
  error), `MissingDir` → propagates the ReadDir error, `IgnoresNonMatching` (non-`.jsonl` /
  non-UUID stems).
- **Existing `internal/supervisor` `deliverFn`-override tests are unaffected** — they override
  `deliverFn` and never reach `deliverViaSession`; the nil-resolver branch they exercise is
  the unchanged #594 path.
- **Gate:** `make check` green — `go vet`, `-race`, `staticcheck`, and **`cmd/substrate-guard`**
  (the resolver reads file *sizes* only, never JSONL content, so no claude-screen literal
  enters pyrycode).

## AC #3 and the cross-repo boundary

AC #1 (transcript grows) and AC #2 (no false ack; loud retryable failure when the turn didn't
land) are **fully satisfied daemon-side** by this ticket. AC #3 (the Kotlin
`InteractiveStreamE2ETest` renders the reply) is **operator-verified, not a Go test** (mirrors
#666's AC #3) and **cross-repo-contingent**: the fix turns the silent false-ack into a
`server.binary_offline` *retryable* reply, and the e2e goes green when the **mobile client
retries** that failure — a pyrycode-mobile behaviour honouring the existing `Retryable` flag
(set since #594), which cannot be a pyrycode child ticket.

This is **not** the tui-driver#124 route-back case. The ticket said route back to PO only if
the investigation found claude *genuinely ready* yet the turn still failed (⇒ only fix is in
tui-driver's chip heuristic). The investigation found the **opposite**: claude is mid-resume
and not genuinely ready, so the deterministic-confirm fix is squarely in pyrycode's domain.
If operator verification shows the phone does *not* retry on `server.binary_offline`, that is a
mobile-side follow-up (the consumer must honour the retryable contract), not a reopen of this
daemon work.

## Out of scope (named)

- **In-driver re-delivery / exactly-once** (clear residue + re-send so the turn lands in a
  single daemon call) — tui-driver#124. Doing it pyrycode-side would nest a retry around
  `DeliverPrompt`'s own and reintroduce the #227 buffered-residue double-paste risk that needs
  claude input-line knowledge (tui-driver's domain). The deterministic confirm here makes the
  *failure* honest; the *single-call landing* is the driver's job.
- **The `buildSession` / `--session-id` path** (`pool.go:946`) stays nil-resolver. Those
  sessions spawn with a pinned UUID and share the workdir's sessions dir, so "newest in dir"
  is ambiguous across peers; and no mobile turn routes there today (lazy-bind on the bootstrap,
  #666). A future per-`--session-id` turn route should build the path from the pinned id
  (`SessionJSONLPath`) instead of scanning.
- **Reading transcript content** to confirm *which* turn landed — we confirm growth (size),
  not identity, keeping the substrate seal. The turn-bridge ([#661](661.md)) owns content
  resolution downstream.
- **Preventing claude's ~7.5s `--continue` restart** — black-box claude behaviour the
  supervisor tolerates by design.

## Lessons learned

- **tui-driver's `JSONLPath` commit signal is appearance-based, not growth-based.** Under
  `--continue` the per-session JSONL already exists from the prior turn, so passing `JSONLPath`
  would confirm a commit *instantly and wrongly*. File-*growth* (size increase, or a rotated
  newer file) is the only reliable on-disk commit signal for a *resumed* session. (From the PR
  body's Lessons learned.)
- **Put the test-only `timeout`/`poll` on the deps struct, not package vars.** The confirm
  tests run `t.Parallel()` under `-race`; shared mutable package vars would race. Per-deps
  fields keep each table row self-contained. (From the PR body's Lessons learned.)
