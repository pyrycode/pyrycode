# Ticket #704 ‚Äî per-conversation inbound message queue + drain on idle/turn-end

The **inbound backlog primitive** under "messages I type while claude is busy are
held and delivered in order, one at a time, as each turn ends" ([EPIC #597] Phase 3
interactive, [ADR 025]). Adds `internal/msgqueue` ‚Äî a per-conversation FIFO with one
serial drain goroutine per active conversation. `Enqueue` is non-blocking and assigns
a stable per-conversation id; the drain delivers in enqueue order, one at a time,
through an injected `DeliverFunc` seam (the shape of `Supervisor.WriteUserTurn`, the
#594 ready-gate ‚Üí commit-confirm ‚Üí recovery path), whose idle-gate blocking **is** the
turn-end pacing. Delivery failures retry the same FIFO head (lossless; bridges a
claude-child respawn). Shipped **unwired** ‚Äî no consumer yet, the same rhythm as the
`turnbridge` producer (#606 unwired, #616 wired). It is the **inbound counterpart of
`internal/eventring`** (#646, the outbound event store). `security-sensitive` ‚Äî it
buffers and releases untrusted phone-originated text into the live claude session.

See [features/msgqueue-package.md](../features/msgqueue-package.md) for the evergreen
package reference (full exported surface, drain pacing, concurrency, security stance).
Highlights specific to how it was built:

- **The drain is paced entirely by the seam ‚Äî no turn-state detector.** Drain trigger
  **option (a)** was chosen over (b): a serial loop calls `deliver` per message, and
  `WriteUserTurn`'s `WaitReady` gate (which blocks while claude is busy, returns on
  idle, then commits) supplies "held until turn ends," "drains promptly when idle," and
  "one in-flight per conversation" all for free. Option (b) ‚Äî a `turnevent.TurnEnd`
  trigger from `turnbridge` ‚Äî was rejected as a redundant second turn-state source.
- **Peek, don't pop.** The drain reads `c.items[0]` under the lock, releases the lock,
  delivers, and advances (`c.items[1:]`) only after a confirmed commit. A delivery error
  leaves the head in place to retry after `RetryInterval` ‚Äî lossless, and what bridges a
  child respawn (`WriteUserTurn` returns `ErrNoLiveSession` during the window).
- **Lazy per-conversation drains.** `Enqueue` get-or-creates the `convQueue`, assigns
  `id = nextID++` (from 1), appends, and spawns a drain only if `started && !closed &&
  !draining && len > 0`. A drain exits when its FIFO empties (clearing `draining` under
  the lock); a later `Enqueue` respawns it. One goroutine per *active* conversation;
  idle conversations hold none.
- **`text` never logged.** The only log is a warn-on-delivery-error carrying
  `conversation_id`, `queued_msg_id`, `queued_at`, and `err` ‚Äî never the untrusted text.

## Patterns established

- **The injected `DeliverFunc` seam keeps the engine unit-testable with no live claude.**
  Reliable delivery arrives as a `func(ctx, convID, payload) error` (the shape of
  `Supervisor.WriteUserTurn`), mirroring `supervisor`'s own `deliverFn` seam and
  `turnbridge`'s `Subscriber` seam. The package imports **no** `internal/*` ‚Äî only
  stdlib ‚Äî so the whole AC matrix runs against a fake. This is the same "shipped unwired
  primitive + injected seam" template `eventring`/`turnbridge` use.
- **The seam's *blocking* is the pacing ‚Äî belt-and-suspenders not warranted.** There is
  one honest turn-end source (`WaitReady` blocking inside `deliver`); adding a second,
  screen-sourced detector for a JSONL/idle-gated invariant would be different-fabric-for-
  its-own-sake with no observed failure to defend. Per the pipeline's Evidence-Based Fix
  Selection, the single source stands.
- **The shutdown join needs a mutex-serialized flag, not a ctx gate.** Beyond the spec
  sketch (which had only `started`), the implementation adds a **`closed bool`** set under
  `q.mu` in `Run` before `wg.Wait()`. Because each drain's `wg.Add(1)` is taken under the
  same lock gated on `!closed`, every `Add` happens-before the `Wait`: a late `Enqueue`
  either adds before `closed` is observed or skips the spawn. A `ctx.Err()`-based gate
  would **not** establish this happens-before (ctx cancellation isn't serialized by the
  mutex) ‚Äî the explicit flag is what makes the join `-race`-clean. (From the PR Lessons
  learned; verified by code review.)
- **Caller-supplied seam ‚Üí error, not panic.** `New` returns an error on a nil `Deliver`
  (it is wiring, surfaced as `error`), in contrast to `eventring.New`'s panic on a
  programmer-constant bound. The distinction tracks *who* supplies the bad value.
- **Lossless = peek-not-pop + retry-same-head + no deadline.** The only loss boundary is
  a full daemon-process restart (in-memory backlog gone) ‚Äî the same boundary as
  `eventring`; reconnect/resync covers it. No on-disk persistence.

## Lessons learned

- **The lazy-spawn/exit-on-empty race and the `Run` shutdown join are both closed by a
  single `closed bool` set under the same mutex the spawn decision (`wg.Add`) takes.**
  Every `Add` either happens-before `wg.Wait` or is skipped because `closed` is observed
  ‚Äî so a drain spawned by a late `Enqueue` can never `Add` to the WaitGroup concurrently
  with its `Wait`. A `ctx.Err()`-based gate would *not* give this happens-before (ctx
  cancellation isn't serialized by the mutex); the explicit flag is what makes it
  `-race`-clean. (From the PR body's Lessons learned.)
- **`advanceLocked`'s capacity compaction (`cap > 2*len ‚Üí copy`) can never fire
  mid-burst.** `c.items[1:]` shrinks `cap` in lockstep with `len`, so the `cap > 2*len`
  guard is only reachable in the *tail* of draining a large burst (e.g. ~14 left of an
  originally-128-cap slice). That's exactly when retaining the big backing array would
  matter, so the logic is correct ‚Äî it just does less work than it looks. (Code-review
  NIT; no change required.)
- **The `convs` map is intentionally never pruned after a conversation fully drains.**
  `items` goes to `nil` but the `*convQueue` stays, preserving `nextID` so ids remain
  stable/monotonic per conversation. The map therefore grows with **distinct** conversation
  ids over the daemon's lifetime ‚Äî bounded by real conversations for a single-operator
  tool, and mirrors `eventring`'s per-conversation map. The *in-conversation* unbounded-
  growth concern is the separately-deferred inbound bound, not this. (Code-review NIT.)

## Out of scope (deferred)

- **Live wiring** ‚Äî the `send_message` handler change (swap synchronous delivery for
  `Enqueue`) + the `cmd/pyry/main.go` constructor + reconciling the now-async ack
  ("delivered" ‚Üí "accepted/queued"). A **separate consumer slice**: folding it in here
  would trip the "new type **and** a `cmd/pyry/main.go` constructor" always-split and the
  `send_message` ack-contract change. This slice does not touch `send_message.go`.
- **Wire / reporting types** ‚Äî `queue_state` / `dequeue_message` and the internal
  `QueueState` / `DropQueued` reporting+removal types are **#705**.
- **Inbound bound / backpressure** ‚Äî deferred on **evidence-based** grounds (paired-device
  auth gate, single-operator tool, unobserved failure, retry shrinks the backlog). The
  cheap deterministic mitigation is pre-specified for the wiring slice: add
  `MaxQueuedPerConversation` to `Config`, change `Enqueue` to `(uint64, error)`, return a
  new `ErrQueueFull` sentinel before appending (**reject**, never drop ‚Äî preserves
  losslessness). The single chokepoint is `Enqueue`; zero call sites today, so deferring
  the signature costs no churn.
- **Daemon-process-restart durability** ‚Äî out of scope (same in-memory boundary as
  `eventring`); resync covers it. No disk persistence.
- **`RetryInterval` tuning** ‚Äî default 1s bridges the respawn window; not load-tested. A
  one-line edit if it needs tuning.

## Testing

`internal/msgqueue/queue_test.go` (`package msgqueue`, stdlib `testing`, `-race`,
`t.Parallel()`, channel-synchronised ‚Äî no sleeps). The fake `DeliverFunc` records
delivery order + payloads, exposes a per-conversation **gate** channel to simulate
"claude busy ‚Üí turn ends", and an **in-flight counter** that fails if it ever exceeds 1
per conversation. Scenarios (RED ‚Üí GREEN, no live claude):

- **Ordered one-at-a-time drain on turn-end** (AC #2/#3/#5): gate every delivery, enqueue
  `m1/m2/m3`, assert only `m1` enters while gated, release per message, assert delivery
  order == enqueue order and max in-flight == 1.
- **Empty-queue no-op** (AC #4): `Run` with no `Enqueue`, zero deliveries, clean cancel.
- **Per-conversation independence** (AC #1): `convA` gated forever, `convB` drains fully.
- **Idle-drains-promptly** (AC #3): non-blocking fake, single enqueue delivered with no
  external signal.
- **Lossless retry / respawn-survival** (AC #4 + restart pin): first 3 attempts fail, 4th
  succeeds; delivered exactly once, 4 attempts observed, max in-flight 1.
- **Stable independent ids** (AC #1): ids monotonic from 1 per conversation, independent
  counters, assigned without `Run`.
- **Clean shutdown / no leak:** cancel while a drain is blocked in `deliver`; `Run`
  returns `context.Canceled` (the `wg.Wait` unblocking proves every drain exited).
- **`New(nil Deliver)` rejects** with a non-nil error (wiring error, not panic).
