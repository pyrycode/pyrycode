# Ticket #639 ‚Äî Bridge & fan-out the stall to phones

Threads the tui-driver `stall_detected` marker ‚Äî previously **dropped at the
daemon** ‚Äî through all three bridge stages to the existing capability-gated push
surface, so a stalled turn (claude gone quiet mid-turn, or the screen-parser
degrading) reaches interactive-capable phones instead of the session silently
appearing to hang. Pure additive wiring: three `case` arms, one per former drop
point, plus the seam tests that prove the end-to-end path. **Part of EPIC #596**
(Phase 2 structured streaming), split from #624 (child B of 2); **depends on
#638** (child A ‚Äî the `turnevent.Stall` internal variant + the
`protocol.StallPayload` / `protocol.TypeStall` wire vocabulary, merged to main
before this slice). Unblocks #373 (mobile stall surfacing + the screen-snapshot
action). `security-sensitive` ‚Äî the fan-out crosses the phone (internet-exposed)
trust boundary; architect + code-review security passes both **PASS**.

**Onset-only ‚Äî no lifecycle mutation.** tui-driver v1.3.0's
`EventKindStallDetected` is a one-shot rising-edge marker (no payload, no clearing
edge, does not repeat while the stall persists). The stall is orthogonal to the
thinking/responding/idle turn lifecycle and not turn-scoped, so it emits with
**zero** lifecycle-state mutation; the phone self-clears on the next turn activity.

## Implementation

The producer‚Üíemitter pipeline was already built and wired (#615 producer, #632
emitter, #633 live wiring). This slice un-drops the marker at the two mapper drop
points and adds the consumer arm; the existing capability-gated `emit()` carries
it the rest of the way unchanged. The three stages are three files, each a thin
echo of an existing turn-event arm.

**Stage 1 ‚Äî tui‚Üíinternal (`internal/turnbridge/mapper.go`).** `mapEvent` gains
`case tuidriver.EventKindStallDetected: return turnevent.Stall{}, true` (was
falling through `default: return nil, false`). Zero-field mapping ‚Äî the marker
carries no payload, so there is no field extraction; the bridge injects
conversation identity later, at wire-mapping time. The package doc comment that
listed the stall marker among the dropped kinds is corrected (only the PTY-state
kinds and `Unknown` still drop).

**Stage 2 ‚Äî internal‚Üíwire (`internal/turnbridge/outbound.go`).** `MapEvent` gains
`case turnevent.Stall: return protocol.TypeStall, protocol.StallPayload{ConversationID: tc.ConversationID}, true`,
mirroring the `TurnEnd` arm. `tc.TurnID` and `tc.Seq` are **ignored** (a stall is
not turn-scoped and not a delta ‚Äî exactly as `BuildTurnState` ignores them);
`StallPayload` has no `turn_id` field, so none can leak.

**Stage 3 ‚Äî consumer fan-out (`cmd/pyry/interactive_turn_v2.go`).** `Handle`'s
type-switch gains `case turnevent.Stall:` ‚Üí `e.emitMapped(ctx, convID, ev)` with
**no lifecycle mutation** (no `startTurnIfNeeded` / `transitionTo` / `endTurn`;
`inTurn`, `turnID`, `seq`, `currentState` all untouched). The case comment is the
AC2 contract in code: the stall is a peer of `turn_state`, flows through the same
`emit()`, and is **not** a droppable delta ‚Äî the droppable set is `assistant_delta`
only (#610). The existing cursor gate at the top of `Handle` (`convID == ""` ‚Üí
drop) covers the stall for free. `eventKind` gains a `stall` arm; `emitMapped`'s
reachability comment adds `Stall` to the reachable set. The stale comment at
`interactive_turn_stream_v2.go:54` ("drives only the dropped stall arm, which the
mapper discards anyway") is corrected ‚Äî the mapper no longer discards it.

Data flow (unchanged pipeline; the stall now survives two former drop points):

```
tui-driver Session.Events()  [EventKindStallDetected ‚Äî rising edge, ~1 per onset]
  ‚Üí mapEvent          STAGE 1   EventKindStallDetected ‚Üí turnevent.Stall{}            (was nil,false drop)
  ‚Üí emitter.Handle    STAGE 3   case turnevent.Stall: emitMapped, NO lifecycle mutation (was default drop)
      ‚Üí emitMapped ‚Üí MapEvent  STAGE 2  Stall ‚Üí (TypeStall, StallPayload{ConversationID}) (was "",nil,false drop)
        ‚Üí emit: for c in ActiveConns where c.Interactive: Push(stall envelope)   [capability gate]
```

The consumer (`Handle`) is reached *before* `MapEvent` ‚Äî `emitMapped` calls
`MapEvent` internally.

## Patterns established

- **A control/state event threads the three-stage bridge with zero new dispatch
  logic.** Because the stall is structurally a peer of `turn_state` (no `seq`, not
  a coalescable delta, rides the same capability-gated `emit()`), un-dropping it is
  three additive `case` arms ‚Äî no new goroutine, lock, channel, error branch, or
  buffering. The "never silently dropped" AC is satisfied by *classification* (the
  stall is outside the droppable-delta set), recorded as a one-line code contract
  on the `Handle` case, **not** by sequencing after the #610 droppable-delta
  optimizer (which was correctly treated as a non-dependency: #610's droppable set
  is `assistant_delta` only).
- **No lifecycle mutation for an onset-only signal.** Emitting the stall without
  touching `inTurn` / `turnID` / `seq` / `currentState` is what lets it ride
  mid-turn without disturbing the open turn (`seq` keeps advancing, turn id
  unchanged) and lets the next content open a fresh turn as if the stall never
  happened. Verified by the mid-turn and no-mutation emitter tests.
- **AC1 is verified by composition across the three seam tests**, matching the
  per-package pattern every other event on this path uses: the mapper flips its
  drop row to `Stall`, outbound adds a payload-bearing row, and the emitter test
  proves fan-out to interactive conns only (the capability gate) + no-mutation +
  mid-turn survival + empty-cursor drop. No new test doubles ‚Äî reuses
  `fakeInteractiveBcast` / `stubCursor`.

None surfaced ‚Äî the PR body carried no Lessons-learned section, code review
returned **PASS** with zero findings (0 MUST / 0 SHOULD / 0 NIT), and the

production consumer of `mapEvent` output, and it now has an explicit
`case turnevent.Stall`, so there is no exhaustive-switch fallout.
