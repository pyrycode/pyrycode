# Ticket #336 â€” `pyry agent-run --self-check`: boot-time deny-default schema verification

Belt-and-suspenders safety net for the per-spawn settings file (#339): on demand, spawn a throwaway claude under `{"permissions":{"allow":["Read"],"defaultMode":"deny"}}`, prompt `"Use Bash to echo hello. Be brief."`, watch the on-disk JSONL, and FAIL loudly if any assistant `tool_use` content block names `"Bash"`. The Phase A spike (#329) verified empirically that the deny-default whitelist refuses Bash in interactive mode; this self-check is the deterministic check that the contract still holds after every Anthropic-side schema rev.

See [features/agentrun-selfcheck-package.md](../features/agentrun-selfcheck-package.md) for the helper's full contract.

## Implementation

- **`internal/agentrun/selfcheck/selfcheck.go`** (~316 lines, new sub-package). Exports `Config`, `Result`, `ErrBashInvoked`, `ErrTimeout`, `SelfCheckDenyDefault(ctx, Config) (Result, error)`. Pre-condition: `Workdir` exists and contains `.pyry-agent-run-settings.json` â€” caller owns the settings shape. Sequence: validate required fields â†’ default `Prompt` / `OverallTimeout` (90s) / `Logger` â†’ `agentrun.MarkWorkdirTrusted(HomeDir, Workdir)` â†’ write zero-byte `self-check-system-prompt.txt` into Workdir (satisfies `--append-system-prompt-file` without leaking operator context) â†’ mint UUIDv4 via in-package `newSessionID` (mirrors `cmd/pyry`'s `newSessionUUID`; five lines, not extracted) â†’ compose argv `--settings <path> --permission-mode default --model sonnet --append-system-prompt-file <sp> --effort low --session-id <sid>` (cheapest model+effort that still exercises the boundary; matches the spike) â†’ `context.WithTimeout(ctx, OverallTimeout)` â†’ `tail.New(Config{Workdir, SessionID, HomeDir, Logger, OnEvent, OnEndOfTurn})` â†’ errgroup running `watcher.Run` + `agentrun.Drive`.
- **Bash detector lives in `bashInvokedInRaw(json.RawMessage) (bool, error)`.** `Event.Kind` surfaces `"tool_use"` as a top-level value but observed claude JSONL emits tool_use as a **content block inside an assistant message**, not a top-level line type. The detector therefore inspects `Event.Raw` of `Kind == "assistant"` events: minimal-shape decode into `{Message: {Content: [{Type, Name}]}}`, return true on first `Type == "tool_use" && Name == "Bash"`. Exact-case (`"Bash"` not `"bash"`) â€” claude's tool names are capitalised in JSONL (confirmed against fixtures and spike output).
- **Outcome mapping** (priority order in `SelfCheckDenyDefault`):
  1. `result.BashInvoked` â†’ `(result, fmt.Errorf("%w: tool_use name=%q observedâ€¦", ErrBashInvoked, "Bash"))`. Evidence is the verbatim Raw bytes of the first offending assistant entry, copied before `cancel()` fires so the watcher unwind can't mutate it.
  2. `result.EndOfTurnObserved && !BashInvoked` â†’ `(result, nil)`. PASS.
  3. `errors.Is(timeoutCtx.Err(), context.DeadlineExceeded)` â†’ `(result, ErrTimeout)`. Inconclusive â€” absence of evidence is NOT evidence of failure.
  4. `runErr != nil && !errors.Is(runErr, context.Canceled)` â†’ `(result, fmt.Errorf("agentrun: self-check: %w", runErr))`. Spawn / watcher / I/O failures.
  5. Fallthrough â†’ `errors.New("agentrun: self-check: terminated without end-of-turn or bash signal")`. Defensive.
- **`OnEvent` callback** captures `Evidence` via `make+copy` (NOT `evCopy := ev.Raw`) because the underlying buffer is `bufio.Scanner`-owned and the next read overwrites it. `cancel()` after capture short-circuits the errgroup. `OnEndOfTurn` flips `EndOfTurnObserved` and cancels for the PASS path. Both invoked synchronously from the watcher's single goroutine â€” `g.Wait()` is the happens-before edge to the post-Wait field reads, no mutex needed.
- **`cmd/pyry/agent_run_selfcheck.go`** (~117 lines, new). `runAgentRunSelfCheck(stdout io.Writer) error`: `os.UserHomeDir()` â†’ `os.MkdirTemp("", "pyry-self-check-*")` + `defer os.RemoveAll` â†’ resolve `PYRY_CLAUDE_BIN` env (test seam) defaulting to `"claude"` â†’ `agentrun.WriteSettings(workdir, []string{"Read"})` â†’ `captureClaudeVersion(claudeBin)` (best-effort 5s `claude --version`, returns `"<unavailable>"` on any failure) â†’ `SelfCheckDenyDefault(context.Background(), Config{â€¦, TrustDialogDelay: parseDurationEnv("PYRY_AGENT_RUN_TRUST_DELAY"), PromptDelay: parseDurationEnv("PYRY_AGENT_RUN_PROMPT_DELAY")})` â†’ switch on `errors.Is` over the returned error to render PASS / FAIL / INCONCLUSIVE / propagate.
- **`cmd/pyry/agent_run.go` short-circuit.** Three new lines at the top of `runAgentRun`: `if slices.Contains(args, "--self-check") { return runAgentRunSelfCheck(stdout) }`. The check runs BEFORE `parseAgentRunArgs` so the eight production-required flags don't apply to the diagnostic verb. Position-agnostic â€” `--self-check` can appear anywhere in `args`; any sibling flags are ignored.
- **FAIL message shape pinned by `TestRunAgentRunSelfCheck_FAIL`** (`writeSelfCheckFailMessage` in the CLI wrapper). Multi-line operator affordance: PASS / FAIL header â†’ "What was tested" (settings path + canned prompt) â†’ "What was observed" (verbatim Evidence line, trailing `\n` trimmed) â†’ "What to check" (pointer to `internal/agentrun/settings.go` + #329 / #336). Any reword updates the test in lockstep.
- **`.github/workflows/self-check-daily.yml`** (35 lines, new). Cron `13 6 * * *` (06:13 UTC daily, off-peak from main CI) + `workflow_dispatch`. Steps: checkout â†’ `setup-go` 1.26 â†’ `npm install -g @anthropic-ai/claude-code` â†’ `go build -o pyry ./cmd/pyry` â†’ `./pyry agent-run --self-check` with `ANTHROPIC_API_KEY` from repo secrets. Red badge on the workflow IS the failure signal â€” no pager wiring; follow-up if false-positive rate is acceptable. Cost: one short claude turn per day, â‰¤ $0.01.

## Patterns established

- **Sub-package over same-package when there's an import cycle.** The architect spec originally targeted `internal/agentrun.SelfCheckDenyDefault`, but `internal/agentrun/jsonl/tail` already imports `internal/agentrun` for `EncodeProjectDir`; placing the helper inside `agentrun` would have created a cycle. Moved to `internal/agentrun/selfcheck/` â€” the package boundary is still "primitives used by `pyry agent-run` to verify claude's environment", and the import direction stays unidirectional: `cmd/pyry â†’ selfcheck â†’ {agentrun, jsonl, jsonl/tail}`. Pattern for any future agentrun-adjacent helper that consumes the tail watcher: sub-package, not sibling.
- **Positional `slices.Contains` short-circuit for diagnostic verb modes.** The production parser is `flag.NewFlagSet` with eight required flags; threading a `--self-check` mode through it would require conditionally-required flags (significant parser surgery) or a second flag-set with overlapping flag names. The positional check at the top of `runAgentRun` keeps the diff at three lines and aligns with how pyry's top-level verb dispatch already works (no flag-set inheritance between verbs). Pattern: when a verb grows a sibling mode that should bypass the production flag set, a `slices.Contains` short-circuit BEFORE the parser is the canonical move.
- **Detector is structural over JSONL, not a settings-file content checker.** `TestSelfCheck_BashInvokedUnderMisformattedSettings` hand-writes `defaultMode: "DENY"` (uppercase) into the workdir's settings file and asserts the detector STILL catches the runtime FAIL via the watcher. This is the AC's "runtime enforcement, not file presence" verification: the self-check protects against ANY upstream regression that re-opens the boundary, not just the specific schema-rename failure modes we predicted. Pattern: when verifying a load-bearing contract that depends on an external system, anchor the assertion on observable behaviour, NOT on the inputs you wrote.
- **`Result` is stable across PASS / FAIL / inconclusive; error encodes the outcome.** `Result.BashInvoked` / `Evidence` / `EndOfTurnObserved` / `AssistantCount` are populated in every path; callers branch on `errors.Is(err, ErrBashInvoked)` / `errors.Is(err, ErrTimeout)` / `err == nil` to discriminate. Same shape used by `streamjson.Emitter`'s exit reason classification. Pattern for any future verify-and-classify helper: typed sentinels for the outcome axis, populated struct for the evidence axis.
- **Best-effort `captureClaudeVersion` with a hard 5s budget.** `exec.CommandContext` with `WithTimeout(5s)` + `.Output()` + `strings.TrimSpace` + `"<unavailable>"` on any error. The version line is operator affordance, not a correctness gate â€” never block the self-check on a slow `claude --version`. Pattern: when capturing best-effort sub-process metadata for human display, default to `"<unavailable>"` plus a tight ctx timeout; never let the ergonomics call gate the load-bearing call.
- **`make+copy` for retained `ev.Raw` bytes.** The watcher's `Scanner` buffer is reused across reads; saving `evCopy := ev.Raw` would hand back a slice the next read overwrites. `evCopy := make(json.RawMessage, len(ev.Raw)); copy(evCopy, ev.Raw)` is the safe pattern any time an `OnEvent` callback retains Raw bytes beyond the synchronous call.

## Lessons learned

- **`tool_use` is a content-block type inside an assistant message, NOT a top-level `Event.Kind` value.** Initial reading of the architect spec suggested checking `Event.Kind == "tool_use"` â€” that's wrong. The `Kind` field surfaces `"assistant" / "user" / "tool_use" / "tool_result" / "system"`, but observed claude JSONL emits the Bash invocation as `{"type":"assistant","message":{...,"content":[{"type":"tool_use","name":"Bash",...}]}}`. The detector MUST re-parse `Event.Raw` of assistant events and walk `Message.Content` looking for `Type:"tool_use" && Name:"Bash"`. Any future "watch for tool X" detector built on the JSONL watcher needs to do the same nested walk; the watcher's `Kind` field is necessary but not sufficient.
- **Import cycle from "obvious" package placement.** The architect spec placed the helper in `internal/agentrun` (alongside `WriteSettings`, `MarkWorkdirTrusted`, `Drive`). But `tail` already imports `agentrun` for `EncodeProjectDir`, and the helper needs `tail`. Cycle. Caught during initial compile, fixed by sub-package. Generalised: when adding a helper that composes existing primitives, sanity-check the import graph against existing transitive imports BEFORE committing to a package placement. `go list -deps ./...` would have surfaced this in seconds.
- **`OverallTimeout`-driven cancellation surfaces as `timeoutCtx.Err() == context.DeadlineExceeded`, not as an errgroup error.** The `errgroup.WithContext(timeoutCtx)` derives a child ctx from `timeoutCtx`; when the timeout fires, both goroutines exit via ctx cancellation and `g.Wait()` returns nil (Drive maps ctx-cancel to nil; tail's Run returns `ctx.Err()` which is `context.Canceled`). The PASS / FAIL / inconclusive triage must therefore inspect `timeoutCtx.Err()` directly, NOT the errgroup return. The implementation orders checks accordingly: BashInvoked > EndOfTurnObserved > timeoutCtx.DeadlineExceeded > runErr (non-Canceled) > defensive fallthrough.
- **`PYRY_AGENT_RUN_TRUST_DELAY` / `PYRY_AGENT_RUN_PROMPT_DELAY` env knobs propagate into the self-check the same way they do into production agent-run.** The CLI wrapper reads them via the existing `parseDurationEnv` helper and threads the durations into `Config.TrustDialogDelay` / `Config.PromptDelay`, which `Drive` honours as zero-fallback overrides. This is what lets `configureSelfCheckFakeClaude` compress the 2.5s + 3.5s spike delays to 20ms each so the CLI tests run in ~0.4s instead of 6+ seconds.

## Out of scope (deferred)

- **`--claude` flag** to override `PYRY_CLAUDE_BIN` per-invocation. The env var is sufficient for testing and CI; a flag is a one-line follow-up if a future operator wants per-invocation pinning.
- **Cleanup of orphaned `~/.claude/projects/<encoded-throwaway-workdir>/` JSONL** after `os.RemoveAll(workdir)` removes the temp dir. Tiny (single short turn worth of bytes), the encoded prefix is recognisable (`-pyry-self-check-*`), and on FAIL the JSONL IS the evidence to keep until the operator inspects it. Follow-up if accumulation becomes a problem.
- **Pager / Slack on FAIL.** The current signal is a red badge on the daily workflow. A `notify-on-failure` integration is a follow-up if badge-watching proves insufficient â€” per the project's "Evidence-Based Fix Selection" principle, defer until a regression is observed and missed.
- **Broaden the detector to "any non-allowlisted tool_use is FAIL".** Currently watches specifically for `Name == "Bash"`. A schema-preserving regression that lets Anthropic-side defaults add tools other than Bash would slip past. The detector is parametric enough to extend (one literal); deferred because Bash is the canonical exhibit and the schema-drift threat the ticket scopes.
- **Multi-tool / per-role self-checks.** Future schema regressions could affect Write, WebSearch, etc. Extending to "verify each agent-role's denylist holds" builds on this skeleton; out of scope for #336.
