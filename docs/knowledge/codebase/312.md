# Ticket #312 — supervisor: WriteUserTurn + conversation_id cursor

Adds the supervisor-side primitive that two future consumers depend on: the
`send_message` handler (sibling slice) and the assistant-turn → `message`
envelope bridge (#311). The supervisor gains a public write surface that
delivers a user-turn payload tagged with a `conversation_id`, and an
externally-readable cursor that tracks which conversation the supervisor was
last told about. No relay wiring, no `Session` accessor — pure primitive.

## Implementation

### New public methods on `*supervisor.Supervisor`

- **`WriteUserTurn(id string, payload []byte) error`** —
  validates → updates cursor → writes to PTY.
  - Calls `Config.ValidateConversation(id)` first (when configured); a
    non-nil result propagates verbatim. **Cursor is NOT mutated on
    validation refusal.**
  - On accept, updates `currentConvID = id` under `convMu` before the PTY
    write so a concurrent `CurrentConversation()` reader never observes a
    PTY effect that lags behind the cursor.
  - Cursor update is NOT rolled back on PTY write failure — the cursor
    reflects intent, delivery is a separate concern (sibling handler
    decides whether to emit `ack`).
  - No active child (between iterations / pre-spawn) → drops the bytes
    silently and returns `nil`. Mirrors `Bridge.Write`'s
    discard-on-unattached behaviour; avoids forcing every handler to
    special-case the backoff window.
  - PTY write failures wrap with `fmt.Errorf("supervisor: write user
    turn: %w", err)` so callers can `errors.Is` the underlying cause.
- **`CurrentConversation() string`** —
  snapshot read under `convMu`. Returns `""` when no `WriteUserTurn` has
  been accepted yet. Persists across child restarts (in-memory state on
  the supervisor, not the iteration).

### `Config.ValidateConversation func(id string) error`

Optional closure handed at construction. When non-nil, runs before any
state mutation. Production wiring (`internal/sessions/pool.go`)
synthesises a closure that calls `Registry.Get` and returns
`conversations.ErrConversationNotFound` for unknown ids — the supervisor
never imports `internal/conversations`; the sentinel travels through the
closure verbatim. When nil, validation is skipped (test ergonomics).

This shape satisfies the "validation at the primitive boundary, not at
the verb handler" rule (`docs/PROJECT-MEMORY.md:24`) without coupling
`internal/supervisor` to the registry package.

### Internal state additions

Two new leaf-only mutex-guarded fields on `Supervisor`:

- **`convMu` + `currentConvID string`** — the cursor.
- **`ptmxMu` + `ptmx *os.File`** — the PTY master fd, registered for the
  lifetime of the current `runOnce` iteration.

Lock order: `convMu` and `ptmxMu` are both leaf-only; `WriteUserTurn`
acquires `convMu` first (briefly), releases, then acquires `ptmxMu` for
the write. The two are never nested. Neither nests with the pre-existing
`mu`/`state` pair.

### Private `setPTY` and `runOnce` edits

`(*Supervisor).setPTY(f *os.File)` is the analogue of `Bridge.SetPTY`:
locks `ptmxMu`, assigns `s.ptmx = f`, unlocks. Two bracketing pairs in
`runOnce`:

- **Service-mode** — `s.setPTY(ptmx)` immediately after
  `Bridge.SetPTY(ptmx)`; `s.setPTY(nil)` immediately before
  `Bridge.SetPTY(nil)` and `ptmx.Close()`.
- **Foreground-mode** — `s.setPTY(ptmx)` after `pty.Start` succeeds;
  `s.setPTY(nil)` immediately before `ptmx.Close()` in teardown.

The `setPTY(nil)` runs **before** the actual `Close` so a `WriteUserTurn`
racing iteration teardown sees `nil` and drops, rather than writing to a
closed fd and surfacing `EBADF`.

### Pool wiring

Two trivial closure plumbings in `internal/sessions/pool.go`:

- `New` (bootstrap supervisor): when `cfg.ConversationsRegistry != nil`,
  populate `supCfg.ValidateConversation` with a closure that consults
  `reg.Get` and returns `conversations.ErrConversationNotFound` on miss.
- `buildSession` (non-bootstrap supervisors): same closure, reading from
  `p.convReg`.

The closure captures the registry by pointer — post-construction
`Create` / `Update` / `Delete` are visible immediately to subsequent
`WriteUserTurn` calls. No cache, no snapshot.

When the registry is nil (test default), the field stays unset and
`WriteUserTurn` skips validation. The Pool does NOT expose a new method
or accessor in this slice; mapping `conversation_id → *Session → *Supervisor`
is the sibling handler slice's concern.

## Test coverage (`internal/supervisor/supervisor_test.go`)

Same-package, stdlib only, all under `go test -race`:

- **Happy path** — validator-nil config; `WriteUserTurn("c-1", "hello\n")`
  delivers to the child (asserted via the `TestHelperProcess` echo
  pattern) and `CurrentConversation()` returns `"c-1"`.
- **Cursor read-back across two writes** — two distinct ids; cursor
  ends on the second.
- **Unknown id returns sentinel and does NOT mutate cursor** — local
  `errTestConvNotFound` sentinel; `WriteUserTurn("c-1", _)` succeeds;
  `WriteUserTurn("ghost", _)` returns the sentinel via `errors.Is`;
  `CurrentConversation()` still equals `"c-1"`.
- **Validator nil = skip** — any id accepted; cursor reflects it.
- **No-PTY drop** — supervisor constructed but `Run` never started;
  `WriteUserTurn` returns `nil`, cursor updates, bytes are dropped.
- **Cursor concurrency** — `t.Parallel`; alternating writers + reader;
  guards against future refactors that drop the mutex.

The local test sentinel keeps the supervisor's test suite independent of
`internal/conversations` (matches the production no-import rule). The
first end-to-end exercise of the Pool-built validator lands in the
sibling handler slice.

## Patterns established / reinforced

- **Closure-at-construction beats direct package import.** The
  supervisor stays decoupled from `internal/conversations`; the sentinel
  travels through the validator return verbatim. Same shape as
  `internal/sessions`'s injected probe/closure seams.
- **Cursor update before PTY write.** Observers that read the cursor
  after observing a PTY effect (e.g. #311's bridge stamping outbound
  envelopes) cannot see a stale cursor. The write may block on PTY
  backpressure; the cursor cannot lag behind that.
- **Cursor reflects intent, not delivery.** PTY write failure does not
  roll back the cursor. The handler decides whether to `ack`. Documented
  on `WriteUserTurn`'s godoc.
- **`setPTY(nil)` before `Close`.** The lock-clear-then-close pattern
  (already used by `Bridge.SetPTY`) extended to the supervisor's own
  ptmx tracking. A racing `WriteUserTurn` sees `nil` and drops, never
  `EBADF`.

## Out of scope (deferred)

- **`Session`-level accessors.** No `Session.WriteUserTurn` /
  `Session.CurrentConversation` delegates. The sibling handler slice
  owns `conversation_id → *Session` resolution and decides whether to
  expose the supervisor directly or wrap it.
- **Wire-code mapping.** The supervisor returns
  `conversations.ErrConversationNotFound` verbatim; mapping to
  `CodeConversationNotFound` is the relay handler's job per
  `docs/PROJECT-MEMORY.md:20`.
- **Foreground-mode consumer.** `WriteUserTurn` works in foreground mode
  but has no real consumer there today (the relay runs in service
  mode). Shared code path keeps the implementation minimal; mode-gating
  can be added later at the entry point if needed.
