# Ticket #348 — `agent-run/jsonl`: line reader + deterministic end-of-turn detector

New subpackage `internal/agentrun/jsonl/` exposing a pure stdlib reader over claude session JSONL output. Surface is one type (`Event`), one config struct, one constructor, three methods (`Next` / `Offset` / `AssistantCount`). Implements the deterministic end-of-turn rule the Phase A spike (#329) discovered — `stop_reason == "end_turn"` AND `sum(len(content[i].text)) > 0` — verified across 1151 real session JSONLs with zero false-positives or false-negatives. No filesystem watching, no path resolution, no goroutines: those concerns belong to siblings #349 (fsnotify wrapper) and #347 (`EncodeProjectDir`). See [features/jsonl-reader.md](../features/jsonl-reader.md) for the full surface.

## Implementation

- `internal/agentrun/jsonl/reader.go` — single production file, ~210 LOC. Package doc-comment carries the verbatim `MUST NOT log file contents at any layer` stance from `internal/agentrun/trust.go:1-9`. Constants: `maxLineBytes = 16 << 20`, `initialBufCap = 8192`. Exported sentinel `ErrLineTooLarge` mirroring `bufio.ErrTooLong` shape.
- **Two-pass JSON decode.** Outer `rawLine` keeps `message` as `json.RawMessage`; inner `rawAssistantMessage` decodes only when `Type == "assistant"`. Single-pass decode would either fail on `user` lines (where `content` is a string, not an array) or require pointer-to-`any` shenanigans. The two-pass shape costs nothing on the hot path (assistant lines decode both passes; non-assistant lines short-circuit before the inner decode).
- **`TextChars` falls out naturally.** `sum(len(c.Text) for c in raw.Message.Content)`. Content blocks of `type == "thinking"` or `type == "tool_use"` have `text` absent → `c.Text == ""` → contributes 0. No type-filter needed; the math matches the spec's literal `sum(len(content[i].text))` wording.
- **Line framing.** Hand-rolled `[]byte` buffer + 4 KiB stack scratch buffer (`scratch [4096]byte` field — avoids per-Read heap alloc). State machine in `Next`: search buf for `'\n'` → if found, slice + decode + return (or skip on parse error / non-assistant); else `src.Read` into scratch, append, repeat. EOF is **not sticky** — only surfaced when `Read` returns `(0, io.EOF)` AND the buffer holds no complete line; a later `Next` may pull more bytes from a growing source.
- **Resume contract.** `Config.StartOffset` is informational (caller must `Seek` first). `Offset()` returns `StartOffset` until the first complete line is consumed; advances past every consumed line's trailing `'\n'` (including silently-skipped non-assistant or malformed lines); never advances into a partial-line buffer. `TestReader_ResumeFromOffset` pins both pre-Next equality and post-EOF stability.
- **Rate-limited malformed-line logging.** `malformedSeen` counter + `if r.malformedSeen == 1 || r.malformedSeen%100 == 0`. Real claude does not emit malformed lines; a flood means catastrophic corruption and the operator needs the FIRST one with enough context, not 50,000 copies.
- `internal/agentrun/jsonl/reader_test.go` — 8 tests, all `t.Parallel()`. Three fixture-driven (`clean.jsonl` / `double_end_turn.jsonl` / `no_end_turn.jsonl`); five synthetic (partial-line via `stepFeeder`, non-assistant skip, malformed-line skip, resume-from-offset, per-line offset advance). Helpers: `newFixtureReader` + `readerHolder` wrap an `*os.File` so `defer closeReader` is a single line; `drainAll` collects events to EOF; `stepFeeder` is a ~30-LOC `io.Reader` that yields chunks one per `Read` call (fsnotify-tail simulation); `recordingHandler` is a ~15-LOC `slog.Handler` that captures records for assertion.
- `internal/agentrun/jsonl/testdata/{clean,double_end_turn,no_end_turn}.jsonl` — byte-identical copies of real pyrycode dev-session JSONLs. Original source paths recorded in the test file's header comment so a future operator can re-pull from the same shape.

## Patterns established

- **Two-pass JSON decode at a polymorphic boundary.** When a discriminator field gates which subsequent shape applies, `json.RawMessage` on the polymorphic field + a second `Unmarshal` once the discriminator is known is cleaner than pointer-to-`any` or `interface{}` + type-assert. Pays the second decode only on the branch that needs it.
- **Stack scratch buffer for read-loop hot path.** `scratch [4096]byte` as a struct field avoids the per-`Read` heap alloc that `make([]byte, 4096)` inside the loop would incur. Free pattern for any reader with a fixed working size.
- **EOF is not sticky on a growable source.** Surfaced only when `Read` returns `(0, io.EOF)` AND no complete line is buffered. Pattern is the standard `tail -f`-on-`*os.File` contract; document explicitly so consumers know to call `Next` again after fsnotify wakes them.
- **Rate-limit error logs from third-party-source noise.** First + every Nth (here N=100) is a four-line counter increment; bounds disk + ops-channel cost when claude misbehaves catastrophically without losing the first-occurrence diagnostic.
- **One-type-per-shape exported surface.** `Event` + `Config` + `Reader` + three methods + one sentinel error. No interface defined here — interfaces belong where consumed (sibling #349 will introduce one if it needs a mock; this package does not pre-define it).

## Lessons learned

- **Architect spec line-counts can drift from observation.** The spec asserted `no_end_turn.jsonl` had 34 assistant entries; the actual file (after the developer copied it into `testdata/`) had 20. The test pins the observed count (`AssistantCount() == 20`), not the spec's 34. Same shape as #347's "ticket bodies on filesystem layout are advisory" lesson (`docs/lessons.md:53` and [codebase/347.md](347.md)) — when a spec quotes a count from a real-world artefact, re-count from the artefact, don't trust the prose. Cheap pin: assert the observed number.
- **`ErrLineTooLarge` is shipped but uncovered by tests.** A 16 MiB line is hard to materialise in unit tests without a meaningful slowdown; the path is exercised via inspection only. Acceptable here because the trigger condition (single line > 16 MiB) is structurally outside claude's behaviour and the failure mode (sentinel error from `Next`) is shallow. If a future bug lands here, add a synthetic test that splices a deliberately-oversized chunk into a `stepFeeder`.
- **`AssistantCount` MUST include transitional empty-content `end_turn` entries.** Spec is explicit; the test (`TestReader_DoubleEndTurn_FirstSkipped` asserting `AssistantCount() == 6` for the 6-assistant double-end_turn fixture) pins it. Easy to "fix" by accidentally gating the increment behind the `EndOfTurn` check — the gate must be on `EndOfTurn` only, never on the counter, so sibling #349's max-turns enforcement counts every model output the user paid for.
