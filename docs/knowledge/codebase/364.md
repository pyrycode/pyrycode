# Ticket #364 — `e2e/realclaude`: prompt-fidelity regression guard

First downstream consumer of the `internal/e2e/realclaude` fixture trio (#361 scaffold + #372 file-system helpers + #373 subprocess invoker). Pins the `pyry agent-run` prompt-handoff contract end-to-end against the real `claude` binary: the literal prompt the helper is asked to send must survive byte-for-byte into the first `user` entry of claude's JSONL session file.

The original `/doctor` prompt-poisoning failure mode (2026-05-14) was made structurally impossible by #391's cutover to the streamrunner envelope; this ticket is a **regression guard** against any future preprocessing, escaping drift, or wiring change that bypasses streamrunner. No production-code change.

See [features/e2e-realclaude.md](../features/e2e-realclaude.md) for the suite contract.

## Implementation

- **`internal/e2e/realclaude/prompt_fidelity_test.go`** (~89 LoC, new). `//go:build e2e_realclaude`, `package realclaude`. One test: `TestRealClaude_PromptFidelity`. No new exports, no helper additions.
  - **Distinctive prompt literal.** `const distinctivePrompt = "INTEGRATION_TEST_PROMPT_PROMPTFIDELITY_2N7Q4R8W"` — ASCII letters+digits only (no characters that JSON would escape) so the substring match against `Event.Raw` is deterministic. Token shape is unique enough that any match in the line implies the real text round-tripped, regardless of whether claude wrote `content` as a string or as `[{type:"text", text:"…"}]`.
  - **Four-phase shape** — worktree allocate → run → run-level asserts → JSONL fidelity assert. Run-level asserts (`ExitCode == 0`, `SessionID != ""`) come BEFORE opening JSONL so a setup/auth/network failure produces a readable message instead of a confusing "no user entry" assertion that buries the real cause.
  - **`RunOpts` shape.** `--max-turns=1`, `--effort=low`, `--model=claude-haiku-4-5` (the literal used in `fixtures_test.go:163`, not the short `haiku` alias) to minimise per-run cost. `SystemPrompt` is non-empty (any value satisfies `validateRunOpts`); the chosen text discourages claude from echoing the prompt back, keeping the door open for future tests that need to distinguish input from response. `AllowedTools: []string{"Read"}` is the minimum non-empty value.
  - **`bytes.Contains(e.Raw, []byte(distinctivePrompt))` against the first `user`-kinded event** — `Event.Raw` is the verbatim line bytes (trailing `\n` stripped, see `internal/agentrun/jsonl/reader.go:134-167`). Substring-matching `Raw` rather than re-decoding `message.content` avoids duplicating the parser's two-shape handling and stays robust if claude switches between string and array content shapes.
  - **`jsonlPathFor(workdir, sessionID)` private helper.** Recomputes the resolved JSONL path for failure-message diagnostics via `os.UserHomeDir()` + `agentrun.EncodeProjectDir(workdir)`. Returns a `<unresolved: …>` sentinel on error rather than failing the test — the assertion has already failed; we just want the best path string we can produce. `ReadJSONL` already opened the file successfully (or the test would have aborted there), so the recompute is a diagnostic-only mirror, not a second source of truth.
  - **Failure-message discipline:**
    - `ExitCode != 0` → fatal with stderr.
    - `SessionID == ""` → fatal with stdout truncated to 1 KiB and a `... (truncated)` suffix when longer (most common headless failure mode is missing API key / network — cheapest to diagnose from the captured stdout).
    - First `user` event found but no substring hit → fatal with the resolved JSONL path AND `string(e.Raw)` so a future debugger sees the actual envelope shape that broke the contract.
    - No `user` event found at all → fatal with the resolved JSONL path AND a comma-joined list of `events[i].Kind` (no event bodies — kind tags alone tell which event class delivered the prompt, while skipping bodies avoids accidentally logging anything sensitive a downstream test might add later).

## Reuses (no re-implementation)

- `realclaude.WithWorktree`, `realclaude.RunPyryAgentRun`, `realclaude.ReadJSONL` (#372/#373) — the entire test body composes the existing helper trio. No new fixture surface.
- `internal/agentrun/jsonl.Event.{Kind, Raw}` — `Raw` exists precisely so consumers can assert against verbatim bytes without re-parsing the envelope's content shape.
- `internal/agentrun.EncodeProjectDir` — the single source of truth for the `~/.claude/projects/<encoded>` segment. The diagnostic path-recompute uses it identically to how `ReadJSONL` resolves the file, so the failure message names the same path the helper opened.

## Patterns established

- **Substring-match `Event.Raw` instead of re-decoding `message.content` for prompt-fidelity assertions.** `jsonl.Event` only decodes the assistant-shaped fields (`StopReason`, `TextChars`, `Usage`); user lines carry `content` as either a string OR `[{type:"text", text:…}]` (see the comment at `internal/agentrun/jsonl/reader.go:137`). `Raw` is the verbatim line — `bytes.Contains` against it is true iff the literal appears anywhere in the JSON, regardless of which shape claude used that day. Generalises to any test that wants "did this exact text round-trip" without owning a second decoder; the tradeoff is that a future migration to binary content blocks would invalidate the assertion shape, and that migration is the correct moment to revisit.
- **Run-level asserts before JSONL asserts in real-binary e2e tests.** `ExitCode == 0` and `SessionID != ""` are checked first so a setup/auth/network failure reports as "ExitCode = 1, stderr: …" rather than as "no user entry in JSONL", which buries the cause two layers down. Generalises to any composed-helper e2e test where the expensive assertion would mask a cheap setup failure.
- **Distinctive ASCII-only prompt literals for substring-match assertions.** The token is JSON-safe by construction (no characters that need escaping), so `bytes.Contains(line, []byte(literal))` is equivalent to "the prompt text appears somewhere in this JSON". Quoting, escaping, encoding choices in the envelope cannot make the match drift. Generalises to any test that asserts a literal survived a JSON round-trip.
- **Per-shape failure-message dump policy: kinds for "missing", body for "found-but-wrong".** When the `user` event isn't there at all, dump kind tags only (which event class delivered the prompt — enough to redirect debugging without bulk noise or sensitive bodies). When the event IS there but the substring doesn't match, dump the full `Raw` line — the diff between expected literal and actual content IS the bug. Two failure paths, two dump shapes; neither dumps more than the assertion needs.
- **Diagnostic path-recompute returns a sentinel on error rather than failing the test.** `jsonlPathFor` mirrors what `ReadJSONL` resolved internally; if the recompute itself errors after the assertion already failed, the diagnostic helper returns `<unresolved: …>` so the operator at least sees the assertion message. Don't escalate diagnostic-helper errors into test failures — the test already has its real failure.

## Out of scope (deferred)

- **Assertions on the assistant reply.** Sibling tickets (#365 and follow-ons) own response-shape coverage. This test deliberately stops at "the prompt round-tripped" so its failure semantics stay narrow.
- **Trailer-derived `session_id` fallback.** `RunResult.SessionID` is init-line only by design (#373); if a future test needs the trailer's value, that's the moment to grow `RunResult.ResultSessionID`, not preemptively.
- **A user-entry helper in `fixtures.go`** (e.g. `FirstUserEntry`). Single consumer today; extract only when a second test needs the same lookup. The current test body is structurally trivial — composing the trio plus a five-line walk-and-match — so the YAGNI bar isn't met.
- **Binary content-block migration.** If claude's wire format ever moves user `content` to non-textual blocks, the substring-against-`Raw` shape stops working; that migration is the correct moment to add per-block extraction, not now.
