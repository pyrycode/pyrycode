# Ticket #255 — `internal/protocol`: envelope, routing, error codes, v1-compat predicate

Phase 3 Track C foundation. Pure-data leaf package declaring the wire-format types for the mobile WebSocket protocol v1 — outer envelope, relay↔binary routing wrapper, error-code constants, type-name constants, and the `IsV1Compatible` predicate. No I/O, no goroutines, no `context`, no `slog`. Spec source-of-truth is `docs/protocol-mobile.md`. Per-type payload catalog (the 16 `*Payload` structs the type discriminators select) is the sibling ticket #256 and slots into `Envelope.Payload (json.RawMessage)` via a second-pass `json.Unmarshal` at the dispatcher.

## Implementation

- **Two production files, stdlib-only.** `internal/protocol/envelope.go` (~94 LOC; behaviour surface — `Envelope`, `RoutingEnvelope`, two sentinels, `IsV1Compatible`, package-private `v1TypeSet`) + `internal/protocol/codes.go` (~62 LOC; pure data — 12 `Code*` error-string constants and 16 `Type*` envelope-type-string constants grouped by spec table order). The ticket body listed four files as illustrative; architect consolidated to two to stay under the "more than 3 new files" red line for no readability gain.
- **`Envelope` field shape pinned to spec § Message envelope.** Six fields, snake_case JSON tags, `omitempty` on `in_reply_to` + `payload_encrypted`. `TS` is `time.Time` (not `string`) so the dispatcher's clock-skew cap (#248) doesn't re-parse on every read; `Payload` is `json.RawMessage` for deferred decode (lets a malformed payload of a known type fail-loud at `protocol.malformed` with the offending envelope's `id` intact). `RoutingEnvelope.Frame` is also `json.RawMessage` so the relay splices byte-for-byte (structural property: relay holds zero per-user state).
- **`IsV1Compatible(env) error` returns Go sentinels, not wire codes.** `ErrUnsupported` when `payload_encrypted=true` (v2 reservation; spec § Reserved for v2), `ErrUnknownType` when `Type` empty/unknown, `nil` otherwise. Check order is pinned: `PayloadEncrypted` first, `Type` second — encrypted-wins-over-unknown is observable through `errors.Is` at the call site and tested by the `encrypted-with-unknown-type` truth-table row. Refusal-to-wire-code mapping stays at the dispatcher (consumer convention pinned by `docs/PROJECT-MEMORY.md` § "Refusal-to-wire-code mapping is the consumer's job"; mirrors `internal/conversations` sentinel idiom).
- **Triple-copy of the 16-entry type list as deliberate drift detectors.** Constants in `codes.go`, `v1TypeSet` map literal in `envelope.go`, two test slices in `compat_test.go`. Three explicit assertions catch "added a `Type*` const, forgot the map" and "fat-fingered `protocol.unkown_type`" regressions. Reflection over `go/types` was rejected — heavier than three explicit assertions for a closed 16-entry set. Revisit only if the v1 type set ever grows past ~50 entries (no plausible path under "additive stays v1, breaking goes v2").
- **Golden round-trip tests against three fixture files.** `testdata/envelope_full.json` (every field populated, `payload_encrypted:false` MUST be omitted on the wire), `testdata/envelope_minimal.json` (required fields only), `testdata/routing_envelope.json` ({conn_id, frame} wrapping a verbatim envelope). `time.Time.Equal` for `TS` to dodge monotonic-clock strip on JSON marshal; `json.Compact` canonicalisation for byte-equal compare. The fixtures double as the cross-language schema reference for mobile clients (no Go binding).
- **Sentinel error strings carry no input bytes and no payload contents.** Static strings only (`"protocol: unknown envelope type"` / `"protocol: unsupported envelope feature"`) — no token-shaped or PII-shaped data ever leaks via the message returned upward.
- **No production consumers in this slice.** Future: `internal/relay-client` (binary→relay WS connection), `internal/dispatch` (#248, calls `IsV1Compatible` + decodes payloads from #256's catalog), `cmd/pyry-relay` (splices `RoutingEnvelope.Frame` byte-for-byte), mobile clients (consume JSON directly).

## Out of scope (deferred)

- Per-type payload structs (#256) — the 16 `*Payload` types that slot into `Envelope.Payload (json.RawMessage)`.
- WS close codes `1000`/`1011`/`4401`/`4404`/`4409` — transport concern, lives with #247 (WSS dial+handshake) and the relay's router, not the JSON envelope.
- Auth/dispatch wiring (`hello_ack`-on-connect, `auth.invalid_token`-on-bad-token, role-based type restriction) — #248–#250.
- `Envelope.ID` monotonicity tracking and dedup — per-connection state, lives with the connection holder.
- `Envelope.TS` clock-skew enforcement (7-day backward / 5-min forward caps) — #248.
- `message_chunk` token-streaming flow + partial-message buffer — #258 or successor.
- `Envelope.Validate()` method, `NewEnvelope(...)` constructor, `AllV1Types []string` exported slice, `go:generate`-driven membership check — alternatives considered and rejected, rationale captured in feature doc.
