# Ticket #673 — fakeclaude grows its transcript on a delivered turn (e2e harness-fidelity fix)

[#668](668.md) tightened the supervised-bootstrap delivery contract: `WriteUserTurn` now
returns `nil` only after the resolved claude session JSONL **grows** past a pre-delivery
baseline (`confirmViaTranscriptGrowth`); no growth within `transcriptConfirmTimeout` (10 s)
returns the loud, retryable `ErrTurnNotCommitted`. The e2e [fakeclaude stub](../features/fakeclaude-binary.md)
writes only a `{}` line at session open / rotation and **never grew the transcript for a
delivered turn**, so the growth poll waited the full 10 s and the `send_message` handler
emitted a retryable error envelope instead of the ack/broadcast the tests assert. This broke
**six** `-tags e2e` tests (6 FAIL / 65 PASS on `main`). This ticket makes fakeclaude grow its
live session JSONL on a delivered turn and aligns five tests' `sessionsDir` so the daemon's
resolver actually scans the file fakeclaude grows.

**Not a production bug — purely a harness-fidelity gap.** Real claude appends the user turn +
assistant reply to its session JSONL at commit time, so growth is guaranteed for any turn that
lands; the [#668](668.md) commit-confirm acks correctly in production. Test/fixture only, no
production code change — the diff is confined to `internal/e2e/**`.

## The failure mechanism (why six tests, why no baseline-resolve WARN)

The daemon **always computes** its sessions dir — there is no flag/env override:
`resolveClaudeSessionsDir(workdir)` → `sessions.DefaultClaudeSessionsDir(abs)` →
`<HOME>/.claude/projects/encode(workdir)`. Because `ClaudeSessionsDir != ""`, the Pool wires
`ResolveTranscript = newTranscriptResolver(dir)` **and** the rotation watcher `os.MkdirAll`s
that computed dir at startup.

The six broken tests are **exactly** the six that (a) set `PYRY_FAKE_CLAUDE_TUI=1` and (b)
drive a user turn through `WriteUserTurn` (`grep -l PYRY_FAKE_CLAUDE_TUI internal/e2e/*_test.go`
returns precisely these six). Of the six, five pointed `sessionsDir` at `tmp/claude-sessions`
(a *different* dir the daemon never scans); one (`TestTwoPhoneStructured_…`) was already
aligned.

- **The 5 unaligned tests** failed because the rotation watcher's `MkdirAll` made the computed
  dir **exist-but-empty**, so the baseline resolve found no `<uuid>.jsonl` and returned
  `("", 0, nil)` — **no error → no baseline-resolve WARN**, so the `berr != nil` `Committed`
  fallback was **not** taken. fakeclaude grew `tmp/claude-sessions` (unseen); the daemon polled
  the empty computed dir → 10 s timeout → `ErrTurnNotCommitted`. (The absent WARN is exactly
  [#668](668.md) AC2's observable.)
- **The 1 aligned test** resolved its baseline to the real pre-created `{}` file but still
  timed out, because fakeclaude never grew *that* file on the delivered turn (its structured
  `JSONL_TRIGGER` append fires only *after* the ack, so it cannot satisfy the ack's own
  growth-confirm).

## Implementation

Two coordinated changes, both inside `internal/e2e/**`.

### Change 1 — fakeclaude grows its live session JSONL on a delivered turn (the shared fix)

A user turn reaches fakeclaude as **stdin bytes** (supervisor `DeliverPrompt` → PTY write),
observed in the `startStdinReader` goroutine. The fix grows the live session JSONL `f` so the
growth-confirm observes it — while preserving the **single-writer-of-`f` invariant** (only the
main goroutine writes `f`, `main.go:187`):

- **`var turnPending atomic.Bool`** (new package var, alongside `stdoutMu`). The stdin reader,
  on every `n > 0` read, calls `turnPending.Store(true)` — a **signal only**, it never touches
  `f`.
- In `main()`'s poll loop: `if turnPending.Swap(false) { appendTurnGrowth(f) }`.
  `appendTurnGrowth` is a ~3-line helper that appends one inert `{}\n` to the **current** `f`
  and `f.Sync()`s, best-effort (mirrors `emitStructuredJSONLIfTriggered`).

All four `f` writers (`openSession`, `emitStructuredJSONLIfTriggered`, the new
`appendTurnGrowth`, the rotation re-open) stay on the main goroutine — **no mutex on `f`**. The
`Swap(false)` collapses chunked stdin into one append per ~50 ms poll cycle and re-arms for a
later turn. The grow only fires when the stdin reader is active (`logPath != "" || tui`,
`main.go:129`); the other 65 e2e tests set neither, so growth never fires for them — blast
radius is exactly the six TUI tests.

### Change 2 — align the 5 unaligned tests' sessionsDir to the daemon's computed dir

For each of the five, replace `sessionsDir := filepath.Join(tmp, "claude-sessions")` with
`filepath.Join(home, ".claude", "projects", encodeWorkdir(home))`, `os.MkdirAll` it, and
**pre-create `<initialUUID>.jsonl` with `{}\n` before the harness start call** — the verbatim
`relay_two_phone_structured_test.go` / `rotation_test.go` pattern. `encodeWorkdir` is the
existing package-level e2e helper; no new helper. Pre-creating before the daemon starts makes
the first growth-confirm baseline resolve at a tiny offset rather than racing fakeclaude's own
open. The sixth test was already aligned, so Change 1 fixes it with **zero test-file edits**.

## Patterns established

- **Single-writer-of-`f` preserved via a signal-only atomic.** Reconciling the on-turn trigger
  with the single-writer invariant — not the byte append — was the real cost. The stdin
  goroutine only flips an atomic; the main goroutine performs every `f` write. This is the
  general shape for any future cross-goroutine fakeclaude trigger: **signal across the boundary,
  write on the owner.**
- **An inert `{}\n` growth line is invisible to every assertion except "the file grew".** It is
  the exact line `openSession` writes; the turnbridge mapper maps an empty/typeless line to
  `(nil, false)` (`mapper.go:72-73`), so the v2/structured producer tails it and emits no event.
  Growth-confirm checks **size only** — so the append carries no claude-TUI substrate glyph and
  loosens no seal (fakeclaude is already on the `cmd/substrate-guard` allowlist from
  [#603](603.md); this reuses the existing append path and adds no allowlist entry — **not
  security-sensitive**).
- **An e2e test that drives a turn must align `sessionsDir` to the daemon's computed dir.** The
  resolver has no env override; a `t.TempDir()` subdir is a dir the daemon never scans. Same
  alignment lesson [#642](642.md) established for the structured producer, now also required for
  the [#668](668.md) growth-confirm.
- **Ordering vs baseline is race-free by construction.** The supervisor captures the baseline
  *after* `WaitReady` and *before* `deliver`; stdin bytes (hence any grow) can only arrive
  *after* `deliver`, so a grow always lands strictly past the baseline.

## Lessons learned

- **The daemon's growth-confirm resolver has no env override** — it always computes
  `<HOME>/.claude/projects/encode(workdir)`. An e2e test that drives a turn must align
  `sessionsDir` to that computed path (not a `t.TempDir()` subdir), or the resolver scans an
  empty dir, finds no growth, and times out with `ErrTurnNotCommitted`. (From the PR body's
  Lessons learned.)
- **An exist-but-empty computed dir produces a clean `("", 0, nil)` resolve, not a WARN.** The
  rotation watcher `MkdirAll`s the computed dir at startup, so the misalignment failure is
  *silent* on the resolver side (no baseline-resolve WARN) and surfaces only as the downstream
  10 s `ErrTurnNotCommitted` — easy to misread as a real regression rather than a harness gap.
- **`#668` broke six tests, not the one originally named.** Two independent bisects of
  `5f8db69~1` vs `main` found the same six green-before / red-after. Fixing only the named test
  would have left five red and trapped a future `-tags e2e` agent — the `grep -l
  PYRY_FAKE_CLAUDE_TUI` blast-radius check is what pinned the true set. (See
  [[architect-verify-regression-blast-radius]]: run the full suite + bisect, don't trust the
  one named test.) The shared fix is a single fixture file, so this stayed **one broadened
  ticket**, not a split (siblings would collide on `fakeclaude/main.go`).

## Out of scope

- **`make check` does not run the `-tags e2e` suite** (which is why [#668](668.md) merged
  green). There is no CI; local `make check` is the gate, and changing it to include the e2e
  tag is an operator decision, explicitly out of scope here.
