# Ticket #378 — `self-check-daily.yml`: header comment refresh for stream-json self-check

Doc-hygiene follow-up to #375. The daily CI workflow (`.github/workflows/self-check-daily.yml`) still ran the correct command (`pyry agent-run --self-check`) and observed the correct exit-code contract, but its header comment described the pre-#375 PTY-interactive / `permissions.defaultMode: "deny"` mechanism. A future on-call reading the file first to triage a red badge would be misled. This ticket rewrites lines 3–10 only; everything else in the workflow (cron, timeout, env, `run:` step, action pins) is byte-identical.

See [codebase/375.md](375.md) for the rewrite that made the old comment stale, and its "Out of scope" entry that named this follow-up.

## Implementation

- **Single edit, `.github/workflows/self-check-daily.yml` header comment (lines 3–10).** Replaces the `#336` / `permissions.defaultMode` / "per-spawn" / "interactive mode" prose with the post-#375 contract statement: claude launched with `--allowed-tools "Read" --dangerously-skip-permissions` must refuse Bash, observed via stream-json stdout (no `tool_use` event with `name=="Bash"`). Reference shape is `#375; supersedes #336`, mirroring the form used in the operator-facing FAIL message at `cmd/pyry/agent_run_selfcheck.go:87-88`.
- **No functional change.** `run: ./pyry agent-run --self-check`, `env: ANTHROPIC_API_KEY`, `timeout-minutes: 5`, `cron: "13 6 * * *"`, `go-version: "1.26.x"`, `actions/checkout@v6`, `actions/setup-go@v6` — all unchanged. The architect verified the 5-minute job budget covers checkout + setup-go + `npm install -g @anthropic-ai/claude-code` + `go build` + `selfcheck.defaultSelfCheckTimeout` (90s) with generous slack.
- **Operator-affordance sentence retained verbatim** (`The job exits 0 on PASS, non-zero on FAIL or inconclusive. Operator monitors the badge; …`) — the exit-code contract is the load-bearing line for a sleepy on-call.

## Patterns established

- **CI workflow header comments are operator triage signage, not documentation.** When a referenced mechanism is rewritten (here: #336 → #375), the header comment is part of the surface that needs to migrate alongside the code, not "we'll catch it later." A future on-call who reads stale prose at 03:00 is structurally worse off than one who reads no prose at all. Treat workflow header comments as belonging to the same change set as the production rewrite they describe; if they didn't migrate in the original PR (as here), file the sibling ticket explicitly rather than letting the drift accumulate.
- **Reference shape `#<current>; supersedes #<predecessor>`** is the canonical form when a ticket replaces an earlier mechanism. Matches the FAIL message in `agent_run_selfcheck.go:87-88` and gives future readers both the active reference and the historical breadcrumb in one line. Use this shape over `#<old>` alone or a free-form sentence.
