# Ticket #659 — surface session-transition signals (clear + eviction) from the pool

The **server-side producer seam** for the v2 `session_transition` wire event
([EPIC #596] Phase 2, [ADR 025]). The two transition sources inside
`internal/sessions` — a `/clear` UUID rotation and an eviction (idle or cap) —
were entirely internal with **no caller-visible signal**. This slice adds an
injectable, in-process **transition observer** to the `Pool` so a `cmd/pyry`-side
consumer ([#657]) can be notified of both transition kinds and map them onto the
wire, **without** `internal/sessions` importing `internal/relay` /
`internal/protocol` (the import cycle that forced the producer to the
`cmd/pyry` boundary).

**In-process plumbing only** — it surfaces the signal; it does **not** build the
wire event or fan it out (that is [#657], blocked on this slice). **Not
`security-sensitive`**: behaviour-preserving, adds no inbound handler, no
dispatch policy, no nonce/crypto/net surface. Split from [#657] (the fan-out
half). See [#656] for the wire type this ultimately feeds.

## Implementation

Three production files; one new file holds all the observer machinery
(one-concern-per-file, matching `id.go` / `reconcile.go`). **3 new exported
types**, **0 changes to external callers** — `cmd/pyry` wiring is deferred to
[#657].

| File | Change |
|---|---|
| `internal/sessions/transition.go` (**new**) | `TransitionReason` (a package-local `string` vocabulary, **not** `protocol`'s wire reason); consts `ReasonClear` / `ReasonEviction`; `SessionTransition{PreviousID, NewID, Reason, OccurredAt}`; `TransitionObserver func(SessionTransition)`. Plus the `Pool` methods `SetTransitionObserver` (the post-construction setter), unexported `notifyTransition` (the nil-guarded leaf callback), and unexported `onRotate` (the clear fire site). |
| `internal/sessions/pool.go` | **+1 field** `transitionObserver TransitionObserver` (zero value nil = disabled; read-only after `SetTransitionObserver`, following the `convReg`/`activeCap` "read-only after New" convention). The `Pool.Run` `OnRotate` closure (`pool.go:810`) now routes through `p.onRotate(...)` instead of calling `p.RotateID` directly. |
| `internal/sessions/session.go` | `runActive` signature → `(TransitionReason, error)`; the eviction reason is surfaced out of its four return paths. `Session.Run`'s `stateActive` case fires the eviction signal **after** `transitionTo(stateEvicted)`, behind a `reason != "" && s.pool != nil` guard. |

**Clear fire site** — `Pool.onRotate(oldID, newID)`: call `RotateID`; on success
fire `SessionTransition{PreviousID: oldID, NewID: newID, Reason: ReasonClear,
OccurredAt: time.Now().UTC()}`; return the `RotateID` error verbatim. Firing only
on success means a failed/no-op rotation emits nothing (the watcher already
logs+continues on an `OnRotate` error).

**Eviction fire site** — `runActive`'s four return paths now carry a reason:

| Path | Returns | Signal |
|---|---|---|
| `<-ctx.Done()` (daemon shutdown) | `("", ctx.Err())` | none (error path; never a transition) |
| `<-runErr` (spontaneous supervisor exit, defensive) | `("", nil)` | none — the wire has no "crashed" reason |
| `<-timerCh` w/ `attached==0` (idle) | `(ReasonEviction, nil)` | eviction |
| `<-s.evictCh` (cap policy) | `(ReasonEviction, nil)` | eviction |

`Session.Run` fires `SessionTransition{PreviousID: s.id, Reason: reason,
OccurredAt: ...}` only when `reason != ""` (skips the spontaneous-exit path) and
`s.pool != nil` (test-constructed sessions lack a pool — mirrors `transitionTo`'s
guard). `NewID` is left empty: an eviction has no successor session.

The evergreen reference is
[features/sessions-package.md](../features/sessions-package.md) § Transition
observer (#659) — updated this phase.

## Patterns established

- **Func type over interface for an injected observer.** `TransitionObserver` is
  `func(SessionTransition)`, matching this package's existing closure-injection
  pattern (`rotation.Config.OnRotate`, `supervisor.Config.ValidateConversation`) —
  the producer owns the type because it stores and invokes it.
- **Post-construction setter, not a `Config` field.** The pool is built via
  `sessions.New(...)` at `cmd/pyry/main.go:460`; the consumer/emitter ([#657])
  comes up later (`startRelay`, `:489`). The observer can't exist at `New` time,
  so it is installed via `Pool.SetTransitionObserver` **before `Pool.Run`**. The
  field is then read-only; the lifecycle + watcher goroutines (both spawned by
  `Run`) read it **lock-free** via `Run`'s goroutine-start happens-before edge.
  A set-after-`Run` call is a programming error the race detector flags.
- **Fire off-lock, post-persist.** Clear fires after `RotateID` returns
  (`Pool.mu` released); eviction fires after `transitionTo` returns (`lcMu`
  released, persist complete). `notifyTransition` takes no lock and the observer
  runs with no `Pool.mu`/`Session.lcMu`/`capMu` held — the leaf, off-lock,
  post-persist point demanded by the `#41`/`#155`/`#169` lessons (lock-order with
  a callback into the host; in-memory state must not flip into a caller's view
  before persist completes). No new lock-order edges.
- **Synchronous fire, no new goroutine.** Fires run on the goroutine that already
  owns the transition (lifecycle goroutine for eviction, rotation-watcher
  goroutine for clear). A per-fire goroutine would add goroutines to paths that
  deliberately have none and could reorder signals (a later eviction overtaking
  an earlier clear). The non-blocking burden is therefore the observer's — [#657]
  hands the signal off to its emitter's buffered outbound. Documented as the
  `TransitionObserver` contract ("MUST NOT block").
- **A package-local reason vocabulary, not the wire's.** `TransitionReason` is an
  `internal/sessions` `string` type so the package imports neither
  `internal/protocol` nor `internal/relay`. [#657] maps it onto the wire
  `{clear, idle_evict, workspace_change}`. (Mirrors the "refusal-to-wire-code
  mapping is the consumer's job" convention.)
- **One eviction reason, idle and cap collapsed.** AC3 permits it and [#656]'s
  wire has no separate cap reason, so distinguishing them here is a defense for an
  unobserved need (evidence-based fix selection). The `string` type leaves room to
  add `ReasonCapEviction` later with **zero signature churn** — a one-line return
  change in the `<-s.evictCh` path.
- **Startup reconciliation rotates directly, so no spurious boot signal.**
  `reconcile.go` calls `Pool.RotateID` **directly** (not via `onRotate`), so the
  reconcile pass fires no clear signal at startup — and in any case it runs before
  any observer is wired. The only production clear path is the live fsnotify
  watcher (`OnRotate → onRotate`). (Surfaced in code review.)

## Lessons learned

- **The "spontaneous-exit fires nothing" test isn't drivable — and that's the
  right call.** The spec's testing strategy listed a `<-runErr`-under-a-live-ctx
  scenario to guard the `reason != ""` gate. It is **not implemented**: `Session.sup`
  is a concrete `*supervisor.Supervisor` whose `Run` returns **only** on ctx
  cancel (no single-shot mode), so the `<-runErr` arm with `ctx.Err()==nil` is
  unreachable in production without adding an interface seam — cross-cutting
  surgery the simplicity-first principle argues against for an unreachable,
  "largely defensive" path. The `reason != ""` gate's correctness is self-evident
  and is exercised **indirectly**: the eviction tests assert exactly one signal,
  and the idle test asserts the subsequent ctx-cancel shutdown fires nothing more.
  A one-line comment in `transition_test.go` records the deliberate omission so a
  future reader doesn't think it was forgotten.
- **The branch diff carries an unrelated realclaude fixture re-record.**
  `internal/e2e/realclaude/testdata/permission_protocol_v2.1.158.json` shows
  ~124 changed lines (fresh `session_id` / `uuid` / temp-path values) — a golden
  fixture re-captured during the code-review phase's real-claude e2e run and swept
  in by its auto-commit safety net (commit `ef4809d`). It is **not** part of the
  transition-signal change; the volatile per-run values are the only delta. (Same
  shape as [#656]'s fixture re-record.)

## Testing

`internal/sessions/transition_test.go` — a `transitionRecorder` (a
`TransitionObserver` that appends each `SessionTransition` under its own mutex,
because fires arrive from different goroutines). Reuses the existing
`helperPool` / `helperPoolIdle` / `helperPoolCap` / `addCapTestSession` /
`/bin/sleep` patterns; settles sessions into `stateActive` before driving
evictions.

- `TestPool_TransitionObserver_ClearFiresOnRotate` — `onRotate(old, new)` fires
  one `ReasonClear` with the old/new ids + non-zero `OccurredAt`, **and** the
  underlying `RotateID` actually moved the entry (`Lookup(old)` →
  `ErrSessionNotFound`, `Lookup(new)` → nil).
- `TestPool_OnRotate_UnknownIDNoSignal` — `onRotate` on an unknown id returns
  `ErrSessionNotFound` and fires **no** signal.
- `TestPool_TransitionObserver_IdleEvictionFires` — short `IdleTimeout`, no
  attaches; one `ReasonEviction` with the evicted id and empty `NewID`; the
  subsequent ctx-cancel shutdown fires nothing more (the indirect `reason != ""`
  guard).
- `TestPool_TransitionObserver_CapEvictionFires` — `ActiveCap=2`, force LRU
  eviction of the oldest peer; exactly one `ReasonEviction` for the evicted peer.
- `TestPool_TransitionObserver_NilIsNoOp` — no `SetTransitionObserver` call;
  rotation still moves the entry and idle eviction still fires, no panic.
- `TestPool_TransitionObserver_RaceConcurrentFires` — N watcher-style `onRotate`
  fires concurrent with a lifecycle idle eviction; asserts N clears + 1 eviction.
  Proves the lock-free observer read is safe under concurrent reads and the
  recorder's mutex keeps it race-clean (`-race`, runs `-count=5` clean).

`go test -race ./...`, `go vet ./...`, `go build ./cmd/pyry` green; real-claude
e2e suite green (PR [#660]).

## Out of scope (deferred)

- **The `cmd/pyry` fan-out + wire emission — [#657]** (`security-sensitive`,
  blocked on this slice): the consumer that wires `SetTransitionObserver`, maps
  `TransitionReason` → the wire `reason`, builds `protocol.SessionTransitionPayload`,
  and emits `TypeSessionTransition` capability-gated.
- **`workspace_change` emission.** No server-side source exists today (`WorkDir`
  is set once at spawn), so this observer never fires a workspace reason. [#656]
  admits `workspace_change` in the wire vocabulary so the mobile decoder stays
  exhaustive; a future workspace-change capability wires it.
- **A distinct cap-vs-idle reason.** Deferred (no observed need; the wire
  collapses both) — addable with zero signature churn.
- **A blocking-observer goroutine defense.** Not built; the contract is documented
  and [#657] owns the non-blocking impl (evidence-based — don't pre-build a defense
  for an unobserved failure).
- **The mobile consumer** — `pyrycode-mobile#336`, which renders the marker as a
  `ThreadItem.SessionBoundary`.
