# Ticket #626 — Capability negotiation on the v2 Noise handshake

The daemon-side trust decision [#607](607.md) deferred. #607 landed the wire
**vocabulary** (`CapabilityInteractive`, the `Capabilities []string` field on both
`HelloClientPayload` and `HelloAckPayload`) but left `handleNoiseInit`
**decoding the phone's advertised capabilities and ignoring them**: the
`hello_ack` echoed nothing, the `V2Session` recorded no capability state, and
`ActiveConnIDs` returned every open conn with no capability filter. This slice
closes that trust boundary — it intersects the phone's advertised set with the
daemon's authoritative supported set (`{CapabilityInteractive}`), echoes exactly
the intersection in `hello_ack`, records the negotiated `interactive` flag per
conn (token-OK path, before `V2StateOpen`), and widens the snapshot funnel to a
capability-aware `ActiveConns` enumeration the later [EPIC #596] structured-stream
fan-out will route on. **`security-sensitive`** — the daemon deciding which
internet-facing phones are *granted* the interactive capability; spec-stage
security review verdict **PASS**. Purely additive, one production file — same
ship-the-primitive-ahead-of-its-consumer pattern as [#571](571.md)/#607.

## Implementation

**One production file: `internal/relay/v2session.go` (+121 / −40, ≈+81 net).** No
wire-type change (#607 owns the fields), no emitter change, no new goroutine, no
new lock — every change runs on the manager's single `Run` dispatch goroutine,
the package's established no-mutex single-owner concurrency model. Five pieces:

- **`supportedV2Capabilities`** — package var `[]string{protocol.CapabilityInteractive}`,
  the daemon's **authoritative** supported set. The negotiation output is built
  from THESE entries only, never from the phone's claims.
- **`negotiateCapabilities(advertised []string) []string`** — pure function (no
  receiver, directly table-testable). Returns advertised ∩ supported by
  **iterating the supported set** and filtering by `slices.Contains(advertised,
  name)`. The iterate-supported-not-advertised shape is the security primitive:
  the result is a subset of supported *by construction*, so a spoofed/unsupported
  advertisement can never appear, duplicates collapse, and order is the daemon's
  echo order. Advertise-nothing / only-unsupported yields `nil` (omitempty then
  drops the ack key, and the recorded flag fails closed to `false`). Loop var
  named `name` to avoid shadowing the `cap` builtin; added the `slices` import.
- **Ack echo** — `negotiated := negotiateCapabilities(helloPayload.Capabilities)`
  computed **before** the `hello_ack` literal (so the same slice is the single
  source of truth for both the echo and the flag), then `Capabilities: negotiated`
  added to the existing `HelloAckPayload{…}`. The ack is sealed via `WriteResp`
  **before** the token check, so it is built on *every* handshake; `omitempty`
  keeps the key absent for a no-capability phone (v1 byte-stability, AC#5).
- **`s.interactive bool` on `V2Session`** — new field beside `device`/`peerStatic`,
  same set-once / single-owner-goroutine doc discipline. Set exactly once in the
  token-OK tail, **between `s.device = &device` and `s.state = V2StateOpen`**, via
  `s.interactive = slices.Contains(negotiated, protocol.CapabilityInteractive)` —
  derived from the same `negotiated` slice the ack echoed, so ack and flag can
  never disagree. The zero value (`false`) is the fail-closed default for every
  non-token-OK path; re-key (`handleRekeyInit`) preserves it by never touching it,
  like `device`/`peerStatic`.
- **Capability-aware enumeration — one funnel widened, not a parallel one.** The
  existing `m.snapshot` funnel's reply type changes `chan []string` → `chan
  []ActiveConn`; `ActiveConnIDs` becomes a thin projection over it. New exported
  `ActiveConn{ConnID string; Interactive bool}` (non-secret routing/decision data
  only — never a `*V2Session`, key, or plaintext). `handleActiveConnIDs` →
  `handleActiveConns() []ActiveConn` (same `V2StateOpen` gate, appends the flag);
  new public `ActiveConns(ctx) []ActiveConn` (structural twin of the old
  `ActiveConnIDs` — funnel send + reply receive, both with the `ctx.Done` escape,
  `nil` on cancel / post-`Run`-exit); `ActiveConnIDs(ctx) []string` reimplemented
  to call `ActiveConns` and map each entry to its `ConnID`, **preserving the
  nil-on-cancel contract** (`nil` in → `nil` out, distinct from a non-nil-empty
  snapshot of an open-session-less manager).

### Data flow

```
phone hello.capabilities ──┐
                           ▼
        negotiateCapabilities(advertised) = advertised ∩ supportedV2Capabilities
                           │
            ┌──────────────┴───────────────┐
            ▼                               ▼
   hello_ack.capabilities          token-OK?  s.interactive = contains(…, interactive)
   (echoed every handshake,                   (set once, before V2StateOpen)
    omitempty)                                          │
                                                        ▼
                              ActiveConns(ctx) → [{conn_id, interactive}, …]  (V2StateOpen only)
                                       ├── ActiveConnIDs(ctx) []string  (projection; #589 consumer)
                                       └── future #596 fan-out: filter Interactive == true/false
```

## Patterns established

- **One funnel widened beats a parallel funnel — when the two reads are the same
  map at two granularities.** `ActiveConns` and `ActiveConnIDs` read `m.sessions`
  for the *same* snapshot at two detail levels, so widening the existing
  `snapshot` reply to `[]ActiveConn` and projecting `ActiveConnIDs` over it is the
  elegant choice. *Contrast* #571's `Push` vs #588's snapshot, which are genuinely
  different operations and correctly got separate funnels. Safe because no test
  references the funnel internals (`snapshotReq`/`handleActiveConns`) by name —
  only the public `ActiveConnIDs(ctx) []string`, whose signature is preserved, so
  the [#589](589.md) consumer compiles untouched.
- **Security by loop shape, not by runtime check.** Iterating the *supported* set
  (not the *advertised* set) makes "a spoofed capability can never be granted" a
  structural property of `negotiateCapabilities` — deterministic, not a guard that
  a later refactor could bypass. The unit matrix + the spoof handshake test pin it.
- **Single source of truth for ack-echo and session-flag.** Both derive from the
  one `negotiated` slice computed before the ack literal, so the echoed capability
  and the recorded flag can never disagree. (Mirrors #607's "one constant, two
  jobs" discipline at the negotiation layer.)
- **Two independent deterministic gates, reused.** The `interactive` flag is
  written **only** on the token-OK branch (after `Devices.Validate`, before
  `V2StateOpen`), **and** `handleActiveConns` filters on `V2StateOpen` — the same
  belt-and-suspenders-of-different-fabric pattern #588/#589 established: even if
  the flag were mis-set, a non-open session is never enumerated.
- **Projection preserves the nil-on-cancel contract explicitly.** `ActiveConnIDs`
  short-circuits `nil` in → `nil` out rather than `make([]string, len(nil))`
  (which would turn a cancelled snapshot into a non-nil-empty one), keeping #588's
  observable contract (nil-on-cancel distinct from empty-manager) intact.

## Security review (verdict PASS)

The grant is built from the daemon's authoritative set (spoofing impossible by
construction), gated on authentication (token-OK only) and on `V2StateOpen` (two
independent deterministic gates), and fail-closed by default. No revision
required.

- **Threat 1 — capability spoofing.** Defeated by the iterate-supported loop
  shape: an advertised `"god-mode"` is never a candidate, so it cannot be echoed
  or flag the session. The AC#3 / ticket-focus property.
- **Threat 2 — granting an unauthenticated peer.** The flag is written only in the
  token-OK branch; every other path leaves it `false`. Second gate: `handleActiveConns`
  filters on `V2StateOpen`, so a non-open session is never enumerated.
- **Threat 3 — the ack echo on the token-fail path.** The `noise_resp` carries the
  ack to a cryptographically-authenticated-but-unauthorized peer that is then
  closed at 4401 and deleted from `m.sessions`. It grants nothing (the session
  never opens, is never enumerated/pushed-to) and leaks nothing
  (`CapabilityInteractive` is public protocol vocabulary). Accepted non-issue.
- **Threat 4 — fail-open default.** Zero-value `interactive` is `false`;
  advertise-nothing / `null` / `[]` / only-unsupported → `negotiated == nil` →
  flag `false`, ack key absent (v1 byte-stable). Default-deny.
- **Threat 5 — DoS via a huge advertised array.** Bounded inside the
  ≤65519-byte AEAD-sealed envelope; O(len(advertised)) linear scan (supported set
  size 1) on the dispatch goroutine — no amplification.

## Testing

White-box tests in `internal/relay/v2session_test.go` (+296 LOC), stdlib only,
`-race`-clean. **`buildHelloEarlyData` (5 callers) and `driveToOpen` (30 callers)
signatures preserved** — a capabilities-bearing `buildHelloEarlyDataCaps` /
`driveToOpenCaps` variant was added, and the capability tests capture the
hello_ack early-data (which `driveToOpen` *discards*) via `Initiator.ReadResp` →
decode `Envelope` → `HelloAckPayload` to assert `.Capabilities`.

- `TestNegotiateCapabilities` — table-driven AC#2/#3 matrix for the pure function:
  `[interactive]`→`[interactive]`; `[interactive, unsupported]`→`[interactive]`
  (drop); `[unsupported]`→`nil` (spoof); `nil`/`[]`→`nil` (advertise-nothing);
  `[interactive, interactive]`→`[interactive]` (dedup).
- `TestV2Session_Handshake_CapabilityNegotiation` — table-driven handshake-level:
  advertise `[interactive]` → ack echoes `[interactive]`; advertise nothing → ack
  has no `capabilities` key; advertise spoof `[interactive, god-mode]` → ack
  echoes only `[interactive]`; advertise `[god-mode]` only → ack has no
  capabilities. Decodes the captured hello_ack.
- `TestV2Session_ActiveConns_MixedInteractive` — two open conns (one interactive,
  one not) → `ActiveConns` reports the correct flag per conn; white-box injection
  mirrors `TestV2Session_ActiveConnIDs_OpenOnly`.
- `TestV2Session_CapabilitySpoof_TokenFail_NeverEnumerated` — the security test: a
  phone advertising `[interactive]` but failing the token is closed at 4401 and
  never appears in `ActiveConns`; the negotiated flag is never observable for an
  unauthenticated peer.
- **AC#5 regression** — the existing `TestV2Session_ActiveConnIDs_*` suite
  (`OpenOnly`, `TornDownSessionAbsent`, `ConcurrentWithDispatch_RaceClean`,
  `EmptyManager`, `CtxCancelled_ReturnsNil`) passes **unchanged** — the `[]string`
  projection preserves the contract. `go test -race ./...`, `go vet ./...`, and
  `go build ./cmd/pyry` all pass.

## Out of scope (deferred to siblings)

- **The structured-stream fan-out that *consumes* `ActiveConns`** to route
  interactive events only to interactive conns — a later [EPIC #596] child,
  replacing #589's coarse `message` broadcast.
- **Any new wire type, emitter, or `Push` change** — #607 / #571 own those.
- **Per-device permission gating of *answering* modals** — Phase 3 / #597,
  orthogonal to capability negotiation.
- **`ActiveConn` multi-capability shape.** The enumeration exposes a single
  `Interactive bool` — the right shape while `supportedV2Capabilities` has one
  member (YAGNI per CLAUDE.md "don't add abstraction it doesn't need yet"). A
  second capability is a deliberate, separately-reviewed change (a sibling bool or
  a migration to a capability set), not pre-built here.
