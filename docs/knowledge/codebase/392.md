# Ticket #392 — agentrun: delete PTY drive code (drive.go, trust.go, settings.go) after stream-json migration

Removes the five exported symbols that drove the legacy PTY implementation of `pyry agent-run` — `agentrun.Drive`, `agentrun.DriveConfig`, `agentrun.MarkWorkdirTrusted`, `agentrun.WriteSettings`, `agentrun.SettingsFilename` — together with their tests, totalling seven files. The stream-json runtime cutover ([codebase/390.md](390.md), [codebase/391.md](391.md)) and the selfcheck rewrite ([codebase/375.md](375.md)) had already moved every production and diagnostic caller off this surface; the symbols survived only as compiled-but-unused exports. No behaviour change is intended — this is dead-code removal, not a refactor. The only edit to a surviving file is a four-line package-doc rewrite in `internal/agentrun/streamrunner/runner.go` that drops the now-stale "headless sibling of internal/agentrun.Drive" framing.

## Lessons learned

- **The architect's deletion manifest treated `trust.go` as wholesale PTY-only, but two of its functions — `ResolveWorkdir` and `EncodeProjectDir` — are workdir-encoding helpers still consumed by `internal/agentrun/jsonl/tail/watcher.go`.** The pre-architect grep only checked the five doomed symbols listed in AC #2; it did not survey every export in each doomed file. Caught at build time (`no required module provides package github.com/pyrycode/pyrycode/internal/agentrun`); resolved by splitting per the issue body's "mixed file" rule — the two workdir helpers moved to `internal/agentrun/workdir.go` (and their tests to `workdir_test.go`), the PTY-only `MarkWorkdirTrusted` was dropped with the rest of `trust.go`. Pattern reinforced: when planning a file-level deletion, grep every exported symbol the file defines, not just the ticket's named targets.
