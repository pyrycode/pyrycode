# Ticket #438 — `internal/keys` X25519 static keypair primitive

Lands the core primitive for the binary side of Mobile Protocol v2 (Noise_IK): one persistent X25519 static keypair per pyrycode daemon, persisted at `~/.pyry/<daemon-name>/static_key.json`. Owns the operator-supplied-daemon-name → on-disk-path mapping behind a restrictive allowlist so a hostile `-pyry-name` argument cannot escape the keystore directory. Filesystem hardening (parent-dir mode rejection, post-`MkdirAll` re-stat, file mode rejection on read, `O_NOFOLLOW`) is intentionally NOT in this slice — it is the sibling primitive of this same package and ships in #439 as a hard prerequisite for any production consumer (#432 QR payload, #433 Noise wrapper).

## Implementation

- **New package `internal/keys`** (`static_key.go` + `store.go` + two same-package test files). Two-file split mirrors `internal/identity` (`server_id.go` + `store.go`): pure type + validator + generator on one side, `LoadOrCreate` + JSON schema + atomic write on the other.
- **`StaticKey` type** with unexported `priv [32]byte` / `pub [32]byte`. Accessors `PrivateKey()` / `PublicKey()` return `[32]byte` by **value**, not by slice or pointer — pinned by `TestStaticKey_AccessorsReturnByValue`, which mutates the returned bytes and asserts internal storage is unchanged.
- **`LoadOrCreate(baseDir, daemonName string) (*StaticKey, error)`** is the single entry point. Package owns the full `(baseDir, daemonName) → filepath.Join(baseDir, daemonName, "static_key.json")` mapping — callers pass the two components, never a precomputed path. A caller-precomputed path would bypass the allowlist and is therefore not part of the API (load-bearing for the security model).
- **Daemon-name allowlist** (`validDaemonName(string) bool`, unexported, white-box tested): length 1..64; every byte in `[a-z0-9_-]`; first byte cannot be `-`. Character-by-character scan, no regex. Rejects all of `""`, `.`, `..`, `/`, `\`, uppercase, whitespace, NUL, multi-byte UTF-8, leading hyphen, and oversize. Reject path returns `fmt.Errorf("keys: invalid daemon name %q: %w", daemonName, ErrInvalidDaemonName)` and the package does NOT touch the filesystem — `TestLoadOrCreate_InvalidDaemonName` asserts `baseDir` has zero entries after every reject row.
- **Key generation via stdlib `crypto/ecdh`.** `ecdh.X25519().GenerateKey(rand.Reader)`; `.Bytes()` and `.PublicKey().Bytes()` give the raw 32-byte material that downstream `flynn/noise` consumes verbatim (no type bridge needed). `newStaticKey()` panics on `crypto/rand` failure, mirroring `identity.NewServerID`'s discipline — silently degrading to a zero-entropy key would break the Noise_IK authentication model.
- **On-disk JSON schema** locked to `docs/protocol-mobile.md` § Static keys — binary side: `{version, algorithm, private_key, public_key, created_at}`. Constants `schemaVersion = 1`, `algorithmName = "Noise_25519"`, `filename = "static_key.json"`. Keys are `base64.StdEncoding` (with padding, 32 bytes → 44 chars) — opposite of `internal/pair`'s `RawURLEncoding`, which is for QR/wire; on-disk JSON gets standard padded base64. Time is RFC 3339 UTC via `time.Time.MarshalJSON` after `t.UTC()` to strip monotonic-clock readings.
- **Atomic-write recipe.** `os.MkdirAll(dir, 0o700)` → `os.CreateTemp(dir, ".static-key-*.tmp")` → **`os.Chmod(tmp, 0o600)` BEFORE write** to defeat umask on macOS → `json.Marshal` body → `f.Write` → `f.Sync` → `f.Close` → `os.Rename(tmp, path)`. `defer os.Remove(tmp)` cleans partial state on any earlier failure. The chmod-before-write is the same umask-defence as `internal/identity/store.go:77`.
- **Existing files are never overwritten on the load path.** Same `LoadOrCreate` invariant as `internal/identity`: paired phones bind to a specific keypair and silent regeneration would invalidate every pairing without operator awareness. Validation failures return `ErrCorruptKeyFile` wrapped with the file path and the on-disk bytes are byte-identical post-call (pinned by `TestLoadOrCreate_CorruptJSONReturnsSentinel`).
- **Seven-step load-side validation** (each returning `ErrCorruptKeyFile`-wrapped, fail-fast):
  1. `json.Unmarshal` syntax check.
  2. `version == schemaVersion`.
  3. `algorithm == algorithmName`.
  4. `base64.StdEncoding.DecodeString(private_key)` succeeds + length 32.
  5. `base64.StdEncoding.DecodeString(public_key)` succeeds + length 32.
  6. `created_at` non-zero (`time.Time.IsZero()` catches both absent and unparseable RFC 3339 values — `encoding/json`'s `time.Time` unmarshaller already enforces RFC 3339 shape).
  7. **Public/private consistency:** `ecdh.X25519().NewPrivateKey(priv).PublicKey().Bytes()` compared against stored `pub` via `crypto/subtle.ConstantTimeCompare`. Strictly unnecessary for equality of public material, but consistent with project hygiene (`internal/devices.VerifyToken`).
- **Three-way switch on `os.ReadFile` error** preserves the I/O-vs-corruption sentinel distinction (mirrors `internal/identity/store.go:38-46`): `nil` → parse; `errors.Is(err, fs.ErrNotExist)` → mint + persist; else → wrapped error that is NOT `ErrCorruptKeyFile`. `TestLoadOrCreate_NonENOENTReadErrorIsNotCorruption` traps this by making `static_key.json` itself a directory so `ReadFile` returns EISDIR.
- **Sentinels:** `ErrInvalidDaemonName`, `ErrCorruptKeyFile` (both `errors.Is`-matchable). Bare wrapped `fmt.Errorf` (no sentinel) covers filesystem I/O errors so an operator can distinguish corruption from permissions/disk-full.
- **Error-message leak resistance.** Error strings include the file path (operator-actionable) but NEVER the file contents, base64 fields, or decoded bytes. Pinned by `TestLoadOrCreate_CorruptJSONErrorDoesNotLeakPrivateKey` — seeds a known base64 private key, triggers `algorithm` mismatch, asserts `err.Error()` substring-free of the known base64. Single defensive assertion against future error-message refactors.
- **`base64.StdEncoding` chosen explicitly over `RawURLEncoding`.** Worth pinning because the project has two flavours in flight: `internal/pair` uses `RawURLEncoding` (no padding, URL-safe) because its output is the QR/wire pairing payload; `internal/keys` uses `StdEncoding` (padded, not URL-safe) because the output is on-disk JSON. The choice matches what most Noise implementations emit; a developer grep for `base64` will see both and the rationale lives in store.go's comment + this codebase note.
- **Daemon-name validator deliberately not consolidated with `cmd/pyry/main.go:sanitizeName`.** `sanitizeName` is a transformer that replaces bad chars with `_` and permits `.` and uppercase; this validator REJECTS and forbids both. Reusing `sanitizeName` here would silently defeat the path-traversal defence. The spec's "Files to read first" section calls this out explicitly so the next implementer doesn't reach for the existing helper. A future hardening ticket may unify the two surfaces, but is not blocked-by anything here.
- **Concurrency.** None. `LoadOrCreate` runs once synchronously on daemon startup before any goroutines fan out. Doc-comment pins "not safe for concurrent use against the same path." Two pyry daemons sharing a HOME is a wider misconfiguration (`sessions.json`, `devices.json`, `server-id` all collide), not this package's concern.

## Files

- `internal/keys/static_key.go` — package doc-comment with SECURITY contract, `StaticKey` type + by-value accessors, sentinel `var`s, `validDaemonName` predicate, `newStaticKey` generator.
- `internal/keys/store.go` — `onDiskKey` JSON envelope, `LoadOrCreate`, `parsePersisted`, `mintAndPersist`, `writeStaticKey` (the atomic-write recipe).
- `internal/keys/static_key_test.go` — allowlist matrix, generator non-zero/uniqueness, accessor by-value contract.
- `internal/keys/store_test.go` — fresh-create + mode assertions, round-trip + mtime-unchanged, allowlist reject + no-filesystem-touch, corrupt-JSON sentinel matrix + no-mutation-on-load, error-message no-leak, I/O-error-is-not-corruption.

## Patterns established

- **Per-daemon JSON-file pattern: fifth instance, recipe stays duplicated.** Sessions / identity / devices / conversations / keys all share the `MkdirAll(0o700)` → `CreateTemp` → `Chmod(0o600)` → encode → `Sync` → `Close` → `Rename(2)` shape. `docs/PROJECT-MEMORY.md` § Project-level conventions named "a fifth registry forces extraction"; the architect re-evaluated at the boundary and concluded that `internal/keys`'s body is small enough (the atomic-write helper is ~25 lines) and its threat model is sensitive enough (private key — extraction would mean one helper shared across stores where the umask defence matters and stores where it does not) that duplication should continue until a sixth forces it. PROJECT-MEMORY.md is read-only for agents so this re-evaluation is recorded here rather than updated upstream — humans maintain that file.
- **Package owns the full path mapping for security-sensitive untrusted inputs.** When an operator-supplied string contributes to a filesystem path, the package that performs the I/O should accept the components and own the `Join`, not the path. `LoadOrCreate(baseDir, daemonName)` is the load-bearing shape; `LoadOrCreate(path)` would let a caller bypass the allowlist by precomputing `filepath.Join(baseDir, "..", "evil")`. This shape diverges from `internal/identity.LoadOrCreate(path)`, which is correct for `identity` because no segment of that path is operator-supplied.
- **Untrusted string validators reject, transformers transform — never share an implementation.** `internal/keys.validDaemonName` rejects on first violation; `cmd/pyry/main.go:sanitizeName` replaces bad chars with `_`. Both consume operator-supplied daemon-name-shaped input; consolidating them would defeat one of the two contracts. The spec's "Files to read first" callout for `sanitizeName` is the institutional memory that prevents grep-driven reuse.
- **Public-key consistency check on load even though the public half is recomputable from the private.** Costs one ECDH `NewPrivateKey().PublicKey()` per load (microseconds, once at startup), catches operator-tampering / disk-corruption that mutates one half without the other, and the constant-time-compare buys nothing for public material but matches the project's hygiene posture. Cheap belt-and-suspenders for a sensitive primitive.
- **Two-file package split: pure type + I/O wrapper.** `static_key.go` has zero non-stdlib imports beyond `crypto/ecdh`+`crypto/rand`+`errors`+`fmt`; `store.go` adds the filesystem and JSON imports. The split lets future readers locate the type, validator, and generator without paging through the persistence recipe — and lets `static_key_test.go` exercise the unexported `validDaemonName` and `newStaticKey` without dragging in `os.TempDir` setup.

## Lessons learned

- **Filesystem hardening was deliberately split into a sibling ticket and the partition is load-bearing for review.** The spec's § Out of scope section names #439 (parent-dir mode rejection, post-`MkdirAll` re-stat, existing-file mode rejection, `O_NOFOLLOW`) as a *hard prerequisite for any downstream consumer*. The architect's security review was able to mark every hardening gap as SHOULD FIX (not MUST FIX) because the issue body, the spec, and the downstream consumer tickets (#432, #433) all name #439 as the blocker — production exposure window is zero. The lesson generalises: a security-sensitive primitive's "out of scope" list earns SHOULD-FIX-not-MUST-FIX classification only when (a) the follow-up is filed, (b) every downstream consumer is structurally blocked on the follow-up, and (c) both facts are written into the spec body so the reviewer can verify them without leaving the document.
- **`time.Time.IsZero()` is sufficient for "missing or unparseable RFC 3339" on the load path.** The architect's spec listed `time.Parse(time.RFC3339, …)` as step 6 of validation; the implementation observes that `encoding/json`'s `time.Time` unmarshaller already returns an unmarshal error on non-RFC3339 strings (step 1's `json.Unmarshal` failure catches `"yesterday"`), so the only residual case is an entirely-absent `created_at` field which surfaces as a zero `time.Time`. One `IsZero()` check covers it. The corrupt-JSON test row `created_at: "yesterday"` still passes — it just fails at step 1 rather than step 6. Worth noting because the spec text and the code don't quite match in shape; the behaviour is identical.

## Out of scope (filed elsewhere or deferred)

- **Filesystem hardening** — parent-dir mode `0700` rejection, post-`MkdirAll` re-stat, existing-file mode `0600` rejection, `O_NOFOLLOW` on read, symlink-swap defence. **Filed as #439**, hard prerequisite for any production consumer.
- **QR payload extension** (`server_static_pubkey` field) — #432.
- **Noise handshake wrapper** consuming `StaticKey.PrivateKey()` — #433.
- **Caller wiring in `pyry pair`** that calls `keys.LoadOrCreate` and feeds the public half into `internal/pair.Payload` — integration ticket on top of #432 + #439 + this one.
- **Key rotation verb** (`pyry rotate-static-key`) — v3 concern per `docs/protocol-mobile.md`.
- **Hardware-backed key storage** (Keychain / Keystore on the binary side) — v3 concern.
- **In-memory zeroisation of the private-key bytes** — Go gives no reliable zeroisation primitives and the bytes live in the `*StaticKey` for the daemon's lifetime regardless; defence-in-depth against process-memory dumps belongs with hardware-backed storage.
- **Aligning `cmd/pyry/main.go:sanitizeName` with this stricter validator** — `sanitizeName` is too permissive for keystore paths but tightening it for `sessions.json` / `devices.json` / `conversations.json` is a separate hardening surface; this ticket does not retroactively touch it.
