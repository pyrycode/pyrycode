# Ticket #722 — `queue_state` producer: fan inbound-backlog changes to interactive phones

The **cmd-side producer** that makes the inbound message backlog visible to a
phone ([EPIC #597] Phase 3 interactive, [ADR 025]). With the queue live ([#721]
wired `msgqueue` into the daemon), the introspection + change seam available
([#719]'s `Snapshot`/`OnChange`), and the wire types defined ([#720]'s
`QueueStatePayload`/`QueuedItem`), this slice emits a `queue_state` to interactive
phones **whenever a conversation's backlog changes** — a message enqueues, drains
to claude, or is removed before draining — so the phone shows what's still waiting
and watches it drain. `security-sensitive`: a new outbound dispatch path on the
internet-exposed surface carrying untrusted, phone-originated `text`; the
`interactive`-capability gate and the per-payload conversation scoping are the
confidentiality decisions. Split from [#705]; sub-issue of [#597].

There is **no** intermediate internal "QueueState" model — the SSOT types live only
in the wire layer ([#720]); the producer maps `[]msgqueue.QueuedMessage` straight to
`protocol.QueueStatePayload`. One new file + two small wiring edits, all confined to
`cmd/pyry` (no `internal/*` edits — queue API, broadcaster, and wire types already
existed). **0 new exported types.**

## Implementation

| File | Change |
|---|---|
| `cmd/pyry/queue_state_v2.go` (**new**, ~212 LOC) | The emitter + the `OnChange` seam factory + the pure mapper. A near-verbatim structural mirror of [`session_transition_v2.go`](657.md) (the #657 template). Surface breakdown below. |
| `cmd/pyry/main.go` | Create the shared `queueChanges chan string` (size `queueStateQueueSize`) **before** `msgqueue.New`; set `OnChange: queueStateNotify(queueChanges, logger)` on the queue `Config`; after `queue` exists build `qse := newQueueStateEmitterV2(queueChanges, queue.Snapshot, logger)` (binds `Snapshot`); thread `qse` as the new trailing arg to `startRelay(...)`. |
| `cmd/pyry/relay.go` | `startRelay` and `startRelayV2` each gain a `qse *queueStateEmitterV2` param (forwarded through; each has exactly one call site). Inside `startRelayV2`, beside the sibling producers, `streamQueueStateCleanup := startQueueStateStreamV2(ctx, qse, mgr)` starts the Run goroutine and is added to the `drain` cleanup before `<-mgrDone`. |
| `docs/protocol-mobile.md` | The `#### queue_state` subsection (Queue (v2) §) gains the emission contract: **when** it fires, the `interactive` gate, the per-payload scoping, and the not-in-replay-ring note (AC-5). |

**`cmd/pyry/queue_state_v2.go` surface:**

- **`queueStateEmitterV2`** — fields: `in <-chan string` (the shared hand-off
  channel; convIDs in, `Run` receives), `snapshot func(convID string)
  []msgqueue.QueuedMessage` (bound to `queue.Snapshot` at construction, read-only on
  `Run`), `logger`, and `nextID uint64` (per-conn envelope-ID counter, read/written
  **only** on the single `Run` goroutine — no atomic). `EventID` is left **nil**.
- **`queueStateNotify(ch, logger) msgqueue.ChangeFunc`** — the `OnChange` seam
  factory: a closure that does a **non-blocking** buffered send of `convID`,
  drop-on-full with a content-free `Warn`. Honors [#719]'s `ChangeFunc`
  **MUST-NOT-BLOCK** contract (fired synchronously from the drain goroutine) and is
  safe for concurrent invocation (channel send). Captures the **channel**, not the
  emitter — so `qse` is never read off the constructing goroutine.
- **`newQueueStateEmitterV2(in, snapshot, logger)`** — plain constructor. Built
  **after** `queue.New` (so `Snapshot` binds) but the channel is created **before**
  `queue.New` and shared with the seam — see § Construction.
- **`Run(ctx, bcast interactiveBroadcaster)`** — drains `in` until `ctx` cancel; per
  `convID` calls `broadcast`. `bcast` is a **goroutine-local parameter** supplied at
  start (when `mgr` exists) — no stored broadcaster field, no race.
- **`broadcast(ctx, bcast, convID)`** — snapshots **only** `convID`'s backlog →
  `toQueueStatePayload` → `json.Marshal` once → fresh `bcast.ActiveConns(ctx)`
  snapshot → **interactive-only** filter (`if !c.Interactive { continue }`) → `Push`
  per conn. Per-conn `Push` error → DEBUG-log + continue (a dropped conn must not
  abort the others); `ctx.Err() != nil` mid-fan-out returns early (teardown).
- **`toQueueStatePayload(convID, items) protocol.QueueStatePayload`** — the **pure,
  unit-testable mapping seam** (mirrors `toWirePayload`). Each `QueuedMessage{ID,Text,
  TS}` → `QueuedItem{QueuedMsgID: ID, Text, TS}`, FIFO order preserved. `Queued` is
  initialised `make([]protocol.QueuedItem, 0, len(items))` so an empty/unknown backlog
  (`Snapshot → nil`) marshals to `[]`, **not** `null` (AC-1; the leaf type can't force
  non-nil, the producer must).
- **`startQueueStateStreamV2(ctx, qse, bcast) func()`** — starts `go qse.Run(ctx,
  bcast)`, returns a cleanup that joins `Run` on ctx-cancel. Takes a **pre-built**
  `qse` (it must exist at `queue.New` time for the shared channel) rather than
  constructing internally — the one shape difference from
  `startSessionTransitionStreamV2`. Cleanup does **not** close `in`.
- **`const queueStateQueueSize = 16`** — buffered hand-off size (copied from
  `sessionTransitionQueueSize`'s rationale: rare human-paced changes; drop-on-full
  bounds memory).

The evergreen [features/v2-session-manager.md](../features/v2-session-manager.md)
§ `ActiveConns` is updated to record `queue_state` as the **third** interactive-only
fan-out consumer (after #632/#634 turn stream, #657 session_transition), and
[features/msgqueue-package.md](../features/msgqueue-package.md) flips its
"consumers deferred" note — `queue_state` (`Snapshot` + `OnChange`) is now live;
only `dequeue_message` (`Remove`, [#723]) remains deferred.

## The binding constraint (why a dedicated goroutine, not inline)

This is the slice's real cost — the factor that held it at **S**, not XS.
`msgqueue.ChangeFunc` fires synchronously on its **caller's** goroutine, and the
callers are a *mix*:

| Trigger | Goroutine |
|---|---|
| **Enqueue** (`send_message`, [#721]) | manager's **Run/dispatch** goroutine (`handleFrame → … → SendMessage → Enqueue`, fully synchronous on Run) |
| **Drain advance** | a per-conversation **drain** goroutine |
| **Remove** (`dequeue_message`, [#723], not wired here) | manager's **Run/dispatch** goroutine |

Two naïve approaches both fail:

1. **Inline `OnChange → ActiveConns` deadlocks** from the enqueue/remove callers —
   `OnChange` runs on the manager's Run goroutine; `ActiveConns` posts to `m.snapshot`
   and waits for Run to service it → Run blocked inside `OnChange` → deadlock
   (`broadcastModalDismissed` documents this exact hazard).
2. **Inline direct `m.sessions` read races** — the deadlock-free alternative is valid
   only on the Run goroutine, but the **drain** caller is not on Run (and it lives in
   `internal/relay`, off-limits this slice).

No single inline implementation is correct for all three callers. The fix —
**decouple `OnChange` from the fan-out with a buffered channel + a dedicated Run
goroutine** — is exactly what [#657]'s `sessionTransitionEmitterV2` does for pool
transitions. `OnChange` does only a non-blocking channel send; the dedicated
goroutine (neither the manager's Run goroutine nor a drain) does the blocking
`ActiveConns` + `Push`, so `ActiveConns` is safe and `nextID` is single-goroutine.

## Construction / ordering (the one wrinkle vs. the #657 template)

`session_transition_v2.go` installs its trigger via `SetTransitionObserver` *inside*
`startSessionTransitionStreamV2` (where `mgr` exists). `queue_state` can't: `OnChange`
is a `msgqueue.Config` field set at `msgqueue.New()` — **before** `startRelay` builds
`mgr`. The fix is to **create the hand-off channel first and share it**, breaking the
chicken-and-egg without any late-bound field:

```
queueChanges := make(chan string, queueStateQueueSize)   // before msgqueue.New
queue, _ := msgqueue.New(msgqueue.Config{ …, OnChange: queueStateNotify(queueChanges, logger) })
qse := newQueueStateEmitterV2(queueChanges, queue.Snapshot, logger)   // after queue (Snapshot bound)
// … threaded through startRelay → startRelayV2 → startQueueStateStreamV2(ctx, qse, mgr)
```

The channel is the **only** shared state between the `OnChange` sender (arbitrary
goroutines) and the `Run` receiver (single goroutine) — channels are
concurrency-safe. `snapshot` is construction-bound; `bcast` is a `Run` parameter. In
practice the channel is empty until a phone connects (enqueue requires a connected
phone's `send_message`; drains only fire on real backlog changes), so no startup
races beyond the benign drop-on-full.

## Patterns established

- **Reuse the fan-out primitive; add no new manager-side path.** `queue_state` is the
  **third** interactive-only consumer of the manager's `ActiveConns`+`Push` surface
  (after the [#632]/[#634] turn stream and [#657]'s `session_transition`). It reuses
  the existing `interactiveBroadcaster` interface (not a fourth copy) and the
  `if !c.Interactive { continue }` capability gate. The buffered `in` channel is the
  *consumer-side* decoupling, not a new fan-out path.
- **Per-payload scoping is structural, not defensive.** `convID` is threaded as **one
  variable** `OnChange(convID) → snapshot(convID) → payload.conversation_id`, so a
  payload's id and its items **cannot desync** (AC-3) — there is no second source for
  either. An arbitrary `convID` only ever yields an empty snapshot (`Snapshot → nil →
  []`); it's a map key, never a path or arg, so no traversal/injection surface.
- **`EventID` left nil; never-drop control event automatically.** `queue_state` does
  **not** join the [#647]/[#649] reconnect-replay ring — it is idempotent full-state
  (only the latest backlog matters; the current state is re-derivable and re-emitted
  on the next change), mirroring every other v2 control event (`session_transition`,
  `modal_shown`/`modal_dismissed`). With `EventID` nil + `omitempty`, the field is
  absent on the wire, so `forwardEnvelope`'s replay-dedup never touches it. Backpressure
  falls out for free: `pushQueue.enqueue` marks only `TypeAssistantDelta` droppable, so
  `queue_state` is a never-drop control event with zero extra code (ADR-025 line 130).
- **Drop-on-full is safe because the seam is edge-triggered.** A dropped `OnChange`
  notification is recovered by the next change (`Snapshot` re-reads current state —
  only the final state matters), so the non-blocking send never violates the
  MUST-NOT-BLOCK contract and never loses a *final* state.
- **Logging discipline carried forward verbatim.** A `// SECURITY:` doc comment
  mirrors the three sibling emitters: only content-free discriminants
  (`event`, `conversation_id`, `conn_id`, `env_id`, Push's transport-sentinel `err`)
  are logged at any level. The untrusted `text`, the marshaled payload bytes, and
  `err.Error()` on the marshal path (where `encoding/json` can quote input bytes into
  its error) are **never** logged.

## Lessons learned

- **The construction-order wrinkle is the only divergence from a faithful copy.**
  Everything else in the slice is a verbatim mirror of [#657]; the single place the
  template doesn't transfer is *when* the trigger is installed. `session_transition`
  installs its observer late (inside the stream-start, where `mgr` exists);
  `queue_state`'s trigger is a `Config` field set at `msgqueue.New` time, which is
  before `mgr` exists. Sharing a pre-created channel (rather than a late-bound setter)
  is the cleaner break — it keeps `snapshot` construction-bound and `bcast` a Run
  parameter, so the emitter has **no** mutable cross-goroutine field beyond the
  channel. Anyone porting this pattern to a *different* trigger seam should first check
  whether the seam is installed eagerly (Config field) or lazily (late setter) — it
  determines whether you share a channel or share the emitter.
- **The envelope `ts` is hoisted out of the per-conn loop** — a deliberate, harmless
  deviation from the `sessionTransitionEmitterV2` template (which recomputes
  `time.Now().UTC()` per conn). Every conn in one fan-out now shares one envelope
  timestamp, which is arguably more correct. (Code review flagged this as a non-issue,
  no change requested.)
- **The `OnChange` seam is wired unconditionally at `msgqueue.New`, even when v2 is
  disabled.** Benign: nothing enqueues without a v2 `send_message`, and the drop-on-full
  send is non-blocking + bounded, so an idle channel costs nothing. (Code-review
  observation, not a finding.)

## Testing

`cmd/pyry/queue_state_v2_test.go` (~422 LOC) — reuses `fakeInteractiveBcast` +
`recordedPush` (from `interactive_turn_v2_test.go`) as the **fake interactive phone**
(AC-4) and a fake `snapshot` func returning scripted `[]msgqueue.QueuedMessage`.
`broadcast` is driven directly (sync) for the fan-out assertions; `Run` +
`queueStateNotify` for the channel/lifecycle ones. `ts` compared via
`time.Time.Equal` (monotonic-clock strip on JSON round-trip).

- **Enqueue → `queue_state` with expected backlog** (AC-1, AC-4): 2-item snapshot →
  exactly one push whose payload matches `{conversation_id, queued:[…×2]}` in order.
- **Drain-advance → updated `queue_state`** (AC-1): a shorter second snapshot → a fresh
  push with the reduced backlog.
- **Empty backlog → `[]` not `null`** (AC-1): `nil` snapshot → marshaled payload
  contains `"queued":[]` (guards the `make([]…,0,…)` requirement).
- **Non-interactive conn receives none** (AC-2, AC-4): mixed `ActiveConns` (one
  `Interactive:true`, one `false`) → only the interactive conn is pushed.
- **Per-conversation scoping** (AC-3, AC-4): a `convID`-keyed snapshot → `broadcast(…,
  "A")` yields `conversation_id=="A"` with **only** A's items; B's text never appears.
- **`toQueueStatePayload` mapping** (pure): `ID→QueuedMsgID`, order preserved,
  empty→non-nil.
- **`queueStateNotify` drop-on-full** does not block / does not panic; logs `queue_full`.
- **`Run` exits on ctx-cancel**; cleanup joins (and is idempotent).
- **`-race`**: fires `OnChange` from several goroutines while `Run` drains — no race on
  `nextID`/the channel.

## Out of scope (deferred)

- **Per-connection conversation isolation.** The fan-out reaches *every* interactive
  conn, each payload stamped with its `conversation_id` (the phone attributes by id).
  There is **no** connection→conversation output binding today (cf. [#679]); a phone
  seeing only "its" conversation's output is undesigned and out of scope, consistent
  with the rest of the structured stream and ADR-025's *ungated-for-any-paired-phone*
  model (a user's paired devices are one trust domain). AC-3 is satisfied at the
  **payload** level — the confidentiality boundary within reach.
- **Reconnect snapshot of `queue_state`.** With `EventID` nil, a phone that reconnects
  after a missed `queue_state` sees the backlog only on the **next** change. If product
  wants the current backlog on reconnect, a future slice adds `queue_state` to the
  `request_snapshot` reply. No AC asks for it here.
- **The `dequeue_message` handler** that calls `queue.Remove` is [#723]. This slice
  only *produces* `queue_state`; it will correctly emit on a Remove-driven `OnChange`
  once #723 wires it, **with no change here**.
- **Inbound queue bound / backpressure.** The unbounded inbound-queue growth vector is
  [#704]/[#721]'s deferred concern (chokepoint pre-specified at `msgqueue.Enqueue`);
  this producer fans the full backlog on every change but adds no new growth surface.
