# Ticket #475 — `internal/agentrun/trust/` subpackage: pre-mark workdirs trusted in `~/.claude.json`

Resurrects the workspace-trust pre-write that [#392](https://github.com/pyrycode/pyrycode/issues/392) deleted when stream-json subprocess mode replaced PTY drive. The 2026-05-19 pivot back to PTY drive (#329 tracking; [`ptyrunner`](../features/ptyrunner-package.md) skeleton in #471; cutover in #470) re-introduces the workspace-trust modal problem — interactive `claude` shows a modal the first time a workdir is opened, and the dispatcher's automated flow has no human to dismiss it. Pre-writing `projects[<realpath(workdir)>].hasTrustDialogAccepted = true` skips the modal entirely.

The new helper is a **strict subset** of the [#341](https://github.com/pyrycode/pyrycode/issues/341) original (see [codebase/341.md](341.md) if it survives). Sub-issue of #329; blocks #470.

## Implementation

- **Subpackage, not sibling file.** Lives at `internal/agentrun/trust/trust.go` (130 LOC) rather than `internal/agentrun/trust.go` as #341 did. The parent `internal/agentrun` package still hosts `ResolveWorkdir` / `EncodeProjectDir` (in `workdir.go`, kept across #392's deletion); the subpackage imports it. New feature doc at [`docs/knowledge/features/agentrun-trust-subpackage.md`](../features/agentrun-trust-subpackage.md).

- **API delta from #341.** Signature changed from `MarkWorkdirTrusted(homeDir, workdir string) error` to `MarkWorkdirTrusted(workdir string) (realpath string, err error)`. The caller (#470's `runAgentRun`) wants the resolved realpath to hand to `ptyrunner.Config.WorkDir`; returning it from the helper eliminates a second `agentrun.ResolveWorkdir` call. `homeDir` moved from explicit parameter to `os.UserHomeDir()` inside the public wrapper; the unexported `markWorkdirTrustedIn(homeDir, workdir)` test seam preserves parallel-test ergonomics (`t.Setenv("HOME", ...)` forbids `t.Parallel`).

- **flock dropped — best-effort write.** #341 held `syscall.Flock` on `~/.claude.json.lock` across the read-modify-write window so concurrent pyry + foreground `claude` writers serialised. The slimmed helper has no lock. Rationale: [`tui-driver`'s `HasTrustModal(snap)`](../features/ptyrunner-package.md#spawn-sequence) runs the post-idle modal detector inside `ptyrunner.Run`; if a concurrent writer clobbers pyry's trust pre-mark, the safety net dismisses the modal at runtime. The lost-update window costs a brief PTY-driven dismissal, not correctness. Per the pipeline's *Evidence-Based Fix Selection* principle: defend an observed failure, don't pre-defend a hypothetical one — flock returns only if a future incident shows the safety net is insufficient.

- **Single-writer atomic-write preserved.** Tempfile + `Sync` + `Close` + `os.Rename` mirrors `internal/devices/registry.go:Save` line-for-line (the canonical pyrycode recipe — convention is duplication-not-extraction until a fifth registry forces it). The crash-safety axis stays: a SIGKILL'd pyry mid-write must not leave `~/.claude.json` torn for the user's own interactive claude. That's distinct from the cross-process serialisation axis (which `flock` was guarding and the safety net now covers).

- **Pass-through preservation discipline unchanged from #341.** `json.Decoder.UseNumber()` is **mandatory** — `~/.claude.json` carries int64-sized values claude wrote (timestamps, counters) that `float64` decoding silently corrupts. Decode into `map[string]any`, refuse to overwrite a non-object `projects` or `projects[realpath]` (return wrapped error, leave file untouched), preserve all sibling top-level + nested fields verbatim. `encoding/json` sorts map keys alphabetically on marshal — combined with `UseNumber`, this gives byte-identical idempotency for repeated calls (pinned by `TestMarkWorkdirTrusted_IdempotentPreservesExtraEntryFields`).

- **File mode policy.** Existing file's mode is captured via `os.Stat` and preserved; absent file is created at `0o600`. Claude defaults to `0o644` in practice; pyry does not silently tighten the user's mode. The mode is informational, not a security boundary (this is single-user data inside `$HOME`). Pinned by `TestMarkWorkdirTrusted_PreservesFileMode`.

- **Workdir resolution delegated to `agentrun.ResolveWorkdir`.** The macOS `/var → /private/var` symlink rule lives in one place (`internal/agentrun/workdir.go`); the trust subpackage imports it. Claude keys `projects` by the symlink-resolved cwd; a non-resolved key never matches the lookup. `TestMarkWorkdirTrusted_WorkdirSymlinkResolvesToRealpath` pins the contract.

- **Logging discipline pinned at package doc-comment.** `~/.claude.json` may contain OAuth tokens or claude-internal state pyry does not own. The helper makes **no** `slog` calls; error wraps name the step (`encode` / `fsync` / `rename` / `stat` / `chmod` / `read` / `parse` / `home dir` / `create temp`) and the path only — never file bytes or unmarshalled values. The eventual caller (#470) logs success at the verb level.

- **Error surface.** Three classes: (1) `fs.ErrNotExist` on the data file → not an error, fresh skeleton; (2) malformed input (unparseable JSON, `projects` not an object, `projects[realpath]` not an object) → wrapped error with file untouched (pinned via `bytes.Equal` pre/post in three tests); (3) I/O failure → wrapped error with step name. Workdir-missing short-circuits via `ResolveWorkdir` BEFORE any `~/.claude.json` access — pinned by `TestMarkWorkdirTrusted_WorkdirMissingReturnsError` (`errors.Is(err, fs.ErrNotExist)` AND `~/.claude.json` not created).

- **Tests: ten cases, nine parallel.** `markWorkdirTrustedIn` (unexported) drives nine parallel behavioural tests covering the seven AC rows plus two #341-era invariants (numeric precision, file-mode preservation) plus two malformed-input rows (`projects` not object, entry not object). One non-parallel smoke test (`TestMarkWorkdirTrusted_PublicSmoke`) exercises the public `MarkWorkdirTrusted` via `t.Setenv("HOME", t.TempDir())` to pin the `os.UserHomeDir` plumbing without duplicating the full behavioural matrix.

## Patterns established

- **"Slim resurrection" pattern when a deleted helper returns with a downstream safety net.** When a helper is deleted (#392 removed `agentrun.MarkWorkdirTrusted` / `WriteSettings` / `Drive`) and later resurrected because the original conditions return, the resurrected version drops any defence that the surrounding system now covers — and documents the descope explicitly with the safety net referenced and the follow-up trigger named. Here: flock dropped because `ptyrunner`'s `HasTrustModal` is the runtime catch; the follow-up trigger is "an observed failure where the safety net is insufficient." Future helpers being resurrected from a deletion should follow the same shape: enumerate what the old version did, identify what the new system covers, descope the now-redundant parts, name the trigger for re-introduction.

- **Test seam for `os.UserHomeDir`-using helpers — explicit-`homeDir` internal sibling.** A helper that calls `os.UserHomeDir()` cannot be exercised in parallel tests via `t.Setenv("HOME", ...)` (Go's testing docs: `t.Setenv` forbids `t.Parallel` ancestors). The fix is a two-line public wrapper that calls `os.UserHomeDir` then delegates to an unexported `*In(homeDir, ...)` sibling tests can drive in parallel with `t.TempDir()`. One non-parallel public-API smoke test pins the wiring. Mirrors the pattern #341 introduced; the same shape applies wherever a public helper takes `$HOME` from the environment.
