# Ticket #373 — `e2e/realclaude`: `RunPyryAgentRun` subprocess fixture helper

Final primitive for the `internal/e2e/realclaude/` suite (after #361's scaffold and #372's file-system helpers): a synchronous `pyry agent-run` invoker that builds the binary lazily, writes prompt + system-prompt files into the caller's workdir, captures all three streams plus exit code, and parses the resolved `session_id` from claude's first stream-json `system`/`init` envelope. Downstream tests (#364–#368) compose `WithWorktree` → `RunPyryAgentRun` → `ReadJSONL` to assert against the real-`claude` trust boundary.

See [features/e2e-realclaude.md](../features/e2e-realclaude.md) for the suite's full contract.

## Implementation

- **`internal/e2e/realclaude/fixtures.go`** (~140 LoC appended, same `//go:build e2e_realclaude` tag, same `package realclaude`). Five new symbols:
  - `RunOpts` (9 fields) mirrors `cmd/pyry/agent_run.go`'s flag surface 1:1 — `Workdir`, `Prompt`, `SystemPrompt`, `AllowedTools`, `MaxTurns`, `Effort`, `Model`, `ExtraEnv`, `Timeout`. **No `SessionID` input** (`pyry agent-run` has no resume flag), **no `Mode` enum** (post-#391, agent-run IS the stream-json forwarder — there is no second mode at this layer).
  - `RunResult` (4 fields) — `ExitCode int`, `SessionID string`, `Stdout []byte`, `Stderr []byte`.
  - `RunPyryAgentRun(t, opts) RunResult` — synchronous. `t.Helper()` → `validateRunOpts` → `ensurePyryBuilt` → write `<workdir>/prompt.txt` and `system.txt` (0o600) → `exec.CommandContext` with the eight required flags in `--flag=value` form → capture into `bytes.Buffer`s → branch on `context.DeadlineExceeded` and non-`*exec.ExitError` for `t.Fatalf` vs. fall-through for normal completion. **Non-zero subprocess exit is NOT fatal** — tests of the real CLI must be able to assert against its error paths.
  - `validateRunOpts(opts) error` — first-violation-wins. Required: `Workdir`, `Prompt`, `SystemPrompt`, `AllowedTools` (non-empty), `MaxTurns > 0`, `Effort`, `Model`. **Does not re-validate the `--effort` enum**; pyry rejects bad values and the helper surfaces that exit code (avoids two places to update).
  - `parseInitSessionID(stdout) string` — `bufio.Scanner` over stdout, returns `SessionID` from the first decoded `{"type":"system","subtype":"init","session_id":"…"}` line with non-empty session_id. Non-JSON / non-init lines are skipped silently. Returns `""` if no init line is found (subprocess died early; callers still see `ExitCode`/`Stderr`).
  - `ensurePyryBuilt(t)` + `pyryBinOnce`/`pyryBinPath`/`pyryBinErr` — verbatim clone of `internal/e2e/harness.go:118-141`, just package-private and using a `pyry-realclaude-*` temp dir. Honours `PYRY_E2E_BIN` short-circuit so CI's pre-built binary skips the `go build`.
- **`internal/e2e/realclaude/fixtures_test.go`** — extended with a `TestMain` that re-execs the test binary as a fake `pyry` when `GO_TEST_HELPER_PROCESS=1` is set, and pins `PYRY_E2E_BIN=os.Args[0]` for every other test so `ensurePyryBuilt` short-circuits to the fake. The fake selects behaviour via `PYRY_E2E_FAKE_MODE` (`happy` / `fail` / `sleep` / `argv`).
  - `TestRunPyryAgentRun_HappyPath` — fake emits init / assistant / result; assert `SessionID`, `ExitCode == 0`, stdout contains both init and trailer.
  - `TestValidateRunOpts_Positive` + `TestValidateRunOpts_Negative` (table-driven across all seven required fields, including positive `MaxTurns = 0` and `-1`) — call the private validator directly so `t.Fatalf` doesn't get in the way.
  - `TestRunPyryAgentRun_InitEventAbsent` — fake exits 2 before emitting stream-json; assert `ExitCode == 2`, `SessionID == ""`, no fatal.
  - `TestRunPyryAgentRun_Timeout` — re-execs itself with `PYRY_REALCLAUDE_TIMEOUT_INNER=1` so the inner branch's `t.Fatalf` is captured as a subprocess failure; assert outer captures both `"timed out"` and the stdout/stderr embeds.
  - `TestRunPyryAgentRun_ArgvContract` — fake echoes argv to stderr; assert each of the eight `pyry agent-run` flags appears with the expected value (substring match, order-independent).
  - `TestParseInitSessionID` — 6-case table over the parser directly (empty / init / empty-session_id / non-init-first / trailer-only / malformed-before-init).

## Reuses (no re-implementation)

- `internal/e2e/harness.go`'s build-cache shape — **structurally cloned, deliberately not extracted.** `internal/e2e` (`//go:build e2e || e2e_install`) and `internal/e2e/realclaude` (`//go:build e2e_realclaude`) are disjoint build-tag worlds; a shared package would need a third tag (multi-tag headers are a known footgun) or be untagged (forcing the agentrun + harness code into every default `go test`). ~20 lines of duplication is cheaper than either alternative. Precedent: `docs/PROJECT-MEMORY.md` § "Resist over-DRY on duplicated registry primitives".
- `cmd/pyry/agent_run.go`'s flag contract — `RunOpts` field names track the flag names verbatim. The argv-contract test is the load-bearing pin: any rename here or there fails the test loudly.
- No dependency on `internal/agentrun/streamjson`. The init-line parser is inline (~14 LoC) and reads claude's output strictly for test introspection. Keeping the dependency direction one-way means the production stream-json reader can change shape without breaking the test fixture's parse contract.

## Patterns established

- **Re-exec the test binary as a fake of the system-under-test, not just as a fake child.** `internal/e2e/harness.go` re-execs to fake **`claude`**; this ticket re-execs to fake **`pyry`** instead. The pattern generalises: when the helper builds + invokes a real CLI binary, the fake substitutes at the binary boundary via `PYRY_E2E_BIN`/equivalent, and the fake selects behaviour from an env var (`PYRY_E2E_FAKE_MODE`) so a single `TestMain` covers happy / fail / sleep / argv-echo modes without separate helper executables. Keeps all test code in one package.

- **Re-exec the outer test process to capture `t.Fatalf` from a fixture helper.** `TestRunPyryAgentRun_Timeout` cannot call `RunPyryAgentRun` directly — the timeout path calls `t.Fatalf`, which would fail the outer test. Instead the test re-execs `os.Args[0]` with a sentinel env var (`PYRY_REALCLAUDE_TIMEOUT_INNER=1`) and `-test.run=^TestRunPyryAgentRun_Timeout$`; the outer call asserts the subprocess failed AND its stdout/stderr contains `"timed out"` plus the embedded captures. Cleaner than wrapping `*testing.T` — no reliance on framework internals, no buffered-T abstraction to maintain. Generalises to any fatal-on-failure helper whose fatal-path needs end-to-end verification (the private-split unit-test trick from #372 is the lighter-weight alternative when the failure is structurally before the exec).

- **`--flag=value` form (not `--flag value`) in argv-contract tests.** The single-token form is unambiguous for substring-match assertions: `bytes.Contains(stderr, []byte("--prompt-file="+workdir+"/prompt.txt"))` matches exactly one token regardless of where in argv it appears. The two-token form requires position-aware parsing and order-coupling.

- **Carve out a private error-returning split (`validateRunOpts`) for any fatal-on-failure helper whose failure path is worth asserting.** Same shape as `resolveAndOpenJSONL` from #372. Public helper calls the split and converts to `t.Fatalf`; tests drive the split directly to assert on the error message. Avoids the "wrap `*testing.T` to capture `Fatalf`" anti-pattern entirely when the failure happens before the subprocess fires.

- **Mirror the SUT's flag surface in the helper's struct field set, then pin it with an argv-contract test.** `RunOpts` has nine fields because `pyry agent-run` has eight flags + a timeout; the argv-contract test asserts each lowering. A future rename in either place fails loudly at exactly one test, with the exact diff in the failure message — no silent regression where the helper still compiles but produces argv pyry no longer parses.

- **Skip the enum re-validation when pyry already gates it.** `--effort` ∈ {`low`,`medium`,`high`,`xhigh`,`max`} is enforced by `cmd/pyry/agent_run.go:parseAgentRunArgs`. Re-validating in the helper would mean two places to update on enum growth and a divergence opportunity if one lags. The helper's job is "put what the caller said onto argv" — pyry owns the meaning. Generalises: when wrapping a CLI that already validates its own inputs, validate only the structural prerequisites the wrapper itself adds (file existence, required strings, positive ints); let the CLI own its own enum gates.

## Lessons learned

- **The ~95 LoC budget under-estimated the structural overhead of two structs + a validator + a build-cache + a parser.** Final production code lands at ~140 non-comment LoC. The spec's prescribed structure (one helper, two exported structs with 9+4 fields, one private validator, one build-cache clone of `internal/e2e/harness.go`, one stream-scanning parser) renders close to this footprint by construction; trimming further would either consolidate the validator into the public function (defeats the unit-test purpose) or collapse the build-cache (defeats the disjoint-build-tag rationale). Flagged at commit time; reviewer accepted with the note. Future S-sized helper specs that include "two exported structs + private split + build cache" should budget closer to ~140 LoC; the ~95 LoC quote was an aspiration the structure doesn't support.

## Out of scope (handled elsewhere)

- **First downstream consumer test (`/doctor` poisoning, etc.)** — #364–#368 build on top of `WithWorktree` + `RunPyryAgentRun` + `ReadJSONL`.
- **Session-resume plumbing.** `pyry agent-run` doesn't accept it. If a future ticket grows it, `RunOpts` can grow a `SessionID` input field at that point.
- **Trailer-derived session_id fallback.** Init-line only by design (lands earliest so failing tests still see the id). A future ticket can add a `RunResult.ResultSessionID` if the asymmetry is ever observed.
