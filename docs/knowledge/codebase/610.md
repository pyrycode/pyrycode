# Ticket #610 — Backpressure + droppable-delta policy on the per-session push queue

Makes [`(*relay.V2SessionManager).Push`](571.md) **non-blocking under relay
backpressure** and adds the **event-class-aware drop policy** ADR 025 §
Backpressure named as the guard for its open risk (decisions/025 line 220:
*"per-session push backpressure under a slow relay must never block the daemon
dispatch goroutine; the droppable-delta policy is the guard"*). Before this
slice `Push` enqueued a `pushReq` onto an **unbuffered** `m.push` channel and
**waited for Run's reply** — which Run sent only after sealing under `s.send`
and forwarding via `m.send` → `Outbound`. Because `transport.Client.Send`
blocks on an unbuffered `sendCh` until the prior `conn.Write` drains (≤ one
`WriteTimeout`), a congested-but-alive relay blocked `Outbound` → blocked Run
inside the seal → blocked the `Push` caller. That caller is [#633](633.md)'s
producer goroutine draining the session JSONL; wedging it is exactly the
ADR-025 open risk. EPIC #596 Phase 2 (structured streaming); `security-sensitive`
(adds outbound push *policy* — drop/admit decisions — on the mobile surface).

`assistant_delta` is loss-tolerant (a later `turn_end` / #611 reconnect replay
reconciles the phone's view); control events (`turn_state`, `tool_use`,
`tool_result`, `turn_end`, `stall`, and the coarse `message`) carry state the
phone cannot reconstruct — never drop them.

## Implementation

**One production file: `internal/relay/v2session.go` (+249 / −65).** The seal
stays single-writer on Run; a bounded buffer + drop policy + a leaf mutex live
in front of it; Run drains incrementally. That reconciliation — not LOC — is the
cost the ticket flagged.

### The buffer + drop policy

- **`queuedEnv{env protocol.Envelope, droppable bool}`** — one buffered
  envelope; `droppable` is precomputed from `env.Type ==
  protocol.TypeAssistantDelta` so the drop scan never re-derives the class under
  the lock. **Envelopes are held UNSEALED** (see Patterns).
- **`pushQueue{items []queuedEnv, dropped uint64}`** — a per-session bounded
  FIFO; `dropped` is an observability counter (no app content). Guarded in full
  by `m.pushMu`; holds no lock of its own.
- **`(*pushQueue).enqueue(env) bool`** — the drop policy, called under `pushMu`,
  returns whether a drop occurred (so `Push` can debug-log after unlock). Five
  cases:

  | state at enqueue | incoming | action |
  |---|---|---|
  | `len < cap` | any | append |
  | `len == cap`, a delta queued | delta | evict oldest queued delta, append (AC#2 — newest text retained) |
  | `len == cap`, a delta queued | control | evict oldest queued delta, append (AC#3 — control admitted by evicting a droppable, never by dropping control) |
  | `len == cap`, all control | delta | drop the incoming delta (loss-tolerant; can't evict control) |
  | `len == cap`, all control | control | **admit past nominal cap** (documented soft overflow) |

  The method only ever *removes* existing entries and appends at the tail → the
  relative order of every survivor is preserved (AC#4). Eviction is
  `slices.Delete(items, i, i+1)` of the first droppable scanning from the front.
- **`const pushQueueCap = 256`** — a count-based bound (envelopes, not bytes), a
  starting value pending the ADR-025 load test. Post-#609 coalescing makes
  deltas arrive per-message/~250 ms, so 256 rides out one `WriteTimeout` window
  without dropping while bounding worst-case per-session memory.

### Manager fields (replacing the #571 funnel)

The `push chan pushReq` field and the `pushReq` type are **removed**.
`V2SessionManager` gains:

- **`pushMu sync.Mutex`** — a **leaf lock** guarding `queues` (the map) and every
  `pushQueue`'s contents. Held only around map lookup + enqueue/pop (both
  O(cap)); never across an `Encrypt`/`m.send`/channel op; never nested with any
  other lock. This is what lets an off-Run `Push` reach the addressed session's
  queue **without** reading Run-owned `m.sessions` or touching `s.send`.
- **`queues map[string]*pushQueue`** — connID → queue. The **key set is
  Run-managed**: created in `handleNoiseInit` at `s.state = V2StateOpen`, deleted
  in `closeWith`; both on Run. `Push` only mutates a queue's *contents*. So
  `queue-exists ⟺ session is V2StateOpen` for any Run-side observer.
- **`drainCh chan struct{}` (cap 1)** — "some queue has work." Non-blocking sends
  (from `Push` and from the drain's re-signal) collapse into at most one pending
  wake.

### `Push` — rewritten (off-Run, non-blocking)

Signature unchanged. (1) `ctx.Err()` short-circuit at entry — the only remaining
ctx dependency, preserving the emitter's teardown branch. (2) under `pushMu`:
`q, ok := m.queues[connID]`; `!ok` → `ErrConnNotFound`; else `q.enqueue(env)`.
(3) non-blocking `select { case m.drainCh <- struct{}{}: default: }`. (4) if a
drop occurred, debug-log (`conn_id`, running `dropped` count, `env.Type` —
**no payload**); return `nil`. `Push` **never** touches `s.send` / `m.sessions`
/ `Outbound` and never blocks on the relay.

### Run-side drain

The `case req := <-m.push:` arm becomes `case <-m.drainCh: m.drainOnce(runCtx)`.
**`drainOnce` pops at most ONE buffered envelope per pass** across all sessions:
under `pushMu` it ranges `m.queues` (Go's map-range randomisation gives rough
fairness across the realistically-tiny conn count), pops one head (FIFO, niling
the slot so the envelope is GC'd), notes whether any queue still has work,
unlocks; then forwards on Run via `forwardEnvelope`; re-signals `drainCh` if more
remains. One `Outbound` per pass is the "a slow `m.send` must not re-block the
producer" guard — Run returns to its `select` between sends, so `ActiveConns` /
inbound frames / wakes are serviced with at most one in-flight `Outbound`
(≤ one `WriteTimeout`) of delay, never a whole-buffer drain.

### `handlePush` → `forwardEnvelope` (rename + shared seal path)

The Run-side seal body (`m.sessions[connID]` lookup → `V2StateOpen` gate →
`s.send.Encrypt` → wrap → `m.send`) is renamed `forwardEnvelope` and reused by
both the drain and the two internal snapshot callers (`handleRequestSnapshot`,
`snapshotReplyError`), which still call it **directly** — their replies are
`InReplyTo`-correlated and must seal in-line, not enter the ordered push stream.
The `V2StateOpen` check stays here; it is the security gate (below).

### Queue lifecycle (Run-owned)

- **Create:** `handleNoiseInit` success tail, at `s.state = V2StateOpen`:
  `pushMu.Lock(); m.queues[s.connID] = &pushQueue{}; pushMu.Unlock()`.
- **Delete:** `closeWith`, alongside `delete(m.sessions, s.connID)`:
  `pushMu.Lock(); delete(m.queues, s.connID); pushMu.Unlock()`. Buffered-but-
  undrained envelopes are discarded — the conn is terminal (#611 reconnect
  replay, not this ticket, reconciles a returning phone). The close envelope
  itself is still sent synchronously via `m.send` (terminal, bypasses the
  buffer).

### Error-contract change (documented; invisible to callers)

Public `Push` now collapses "session not open" into `ErrConnNotFound`: a queue
exists iff the session reached `V2StateOpen`, so a not-open or unknown conn
simply has no queue. `ErrSessionNotOpen` is no longer returned by the public
`Push` (it remains a package sentinel used by `Rekey` and the Run-side forward).
Both production callers ([#632](632.md) emitter, [#589](589.md) coarse bridge)
only debug-log the error, so the collapse is invisible to them.

## Patterns established

- **Drop *before* seal — the load-bearing half of the reconciliation.** The
  Noise send nonce is strictly sequential; the phone's `recv` CipherState expects
  nonces `n, n+1, …` with no gaps. Dropping a *sealed* frame is unrecoverable —
  the next frame MAC-fails and the conn dies at 4421 (indistinguishable from
  tampering). So the queue holds **unsealed** envelopes and the drop policy runs
  **pre-seal**; sealing happens only on the frames actually sent, in order, on
  Run. This composes with re-key for free (the same property `handlePush` always
  had: a forward seals fully under the old or the new key, never torn).
- **The seal stays single-writer; the buffer + leaf mutex sit in front; Run
  drains incrementally.** This is the ticket's named design wrinkle. The #571
  single-writer-funnel idiom (every `s.send.Encrypt` on Run) is preserved
  verbatim — `Push` no longer round-trips through Run, but the *seal* never
  leaves it. The real work is the leaf-mutex registry that lets an off-Run `Push`
  reach the right queue without reading Run-owned `m.sessions`, plus the
  one-per-pass drain that keeps Run responsive. The bounded buffer is the easy
  half.
- **Soft overflow for control in the all-control-saturated case — a deliberate
  trilemma resolution.** *bounded ∧ never-drop-control ∧ never-block-producer* is
  unsatisfiable when the queue fills entirely with control events; one guarantee
  must yield. We yield "strictly bounded": admit the control event past nominal
  `cap`. Reachable only under a connected-but-very-slow relay sustained across
  hundreds of control events with **zero** interleaved text (a true stall
  fast-fails to `ErrNotConnected` and Run drains instantly, so the buffer never
  fills). The phone **cannot drive control volume** (push is server→phone only;
  control is O(1) per turn), so it is unreachable in practice; the alternative
  (drop the control) loses unreconstructable state. A future reviewer preferring
  a hard cap must pick which other guarantee to drop — do not silently change it.
- **`drainCh` cap-1 + re-signal = a self-perpetuating pump with no lost
  wakeups.** A `Push` that enqueues while Run is mid-pass lands its signal into
  the (now-drained) channel and triggers the next pass; whether a signal lands or
  hits `default`, a future drain acquires `pushMu` *after* the enqueue's release
  and therefore observes the envelope — nothing can be stranded. No new
  goroutine — the drain is a Run select arm; orphaned buffers GC with the
  manager, and an in-flight `Push` never blocks on Run, so none wedge on
  shutdown.
- **Queue-exists ⟺ V2StateOpen lets the error contract collapse safely.** Because
  the map key set is mutated only on Run (open/close), `Push`'s lookup that never
  reads `m.sessions` is sound, and the not-open security gate is preserved by
  re-checking `s.state` in `forwardEnvelope` on the drain side — a buffered push
  to a conn that de-authed/closed before drain is dropped there, never delivered
  to an un-authenticated peer. Two deterministic, independent gates (queue
  absence at enqueue + `V2StateOpen` at forward), neither a stochastic rule.

## Lessons learned

*(Lifted from the PR body's Lessons-learned section.)*

- **The buffer must hold unsealed envelopes** and run the drop policy pre-seal.
  The Noise send nonce is strictly sequential, so dropping a *sealed* frame gaps
  the phone's `recv` nonce → MAC failure → 4421 close. Buffering unsealed and
  sealing only the frames actually sent (in order, on Run) is the load-bearing
  half of the single-`Run`-goroutine reconciliation; a naive "buffer sealed
  frames" would silently corrupt the recv stream.
- The two existing concurrency tests (`…InterleavedWithReply`,
  `…ConcurrentWithReplies`) **passed unchanged** across the synchronous-funnel →
  buffered-enqueue rewrite, because `waitForEnvelopes` already polls and the
  assertions decrypt in capture order — the harness absorbs async delivery as
  long as every seal still happens on Run in FIFO order.

## Testing

Stdlib `testing`, `-race`. The drop policy is a pure unit surface (fast,
deterministic, no cipher harness); one integration test covers the end-to-end
non-blocking guarantee.

**Unit — `pushQueue.enqueue`** (table-driven; helpers `pqEnv(type,id)`,
`fillDeltas(q,lo,hi)`, `assertQueue(t,q,want,wantDropped)`):

- `TestPushQueue_Enqueue_UnderCap_AllRetained` — mixed deltas+control below cap →
  all retained, FIFO order, `dropped==0`.
- `TestPushQueue_Enqueue_DropOldestDelta` (AC#2) — cap of deltas + one more delta
  → oldest gone, newest retained, `dropped==1`.
- `TestPushQueue_Enqueue_ControlEvictsDelta` (AC#3) — a control event into a full
  queue holding a delta → a delta evicted, control appended at tail.
- `TestPushQueue_Enqueue_MessageIsNeverDrop` — pins the class decision: the
  coarse `message` is never-drop (treated as control by the `assistant_delta`-only
  predicate).
- `TestPushQueue_Enqueue_ControlNeverDroppedWhenDeltasPresent` (AC#3) — N control
  events into an interleaved full queue → exactly N deltas evicted oldest-first,
  every control present, in order.
- `TestPushQueue_Enqueue_OrderPreservedAcrossDrops` (AC#4) — survivors equal the
  enqueue subsequence with the evicted deltas removed.
- `TestPushQueue_Enqueue_AllControlSoftOverflow` — fill cap with control + one
  more control → admitted (`len==cap+1`, `dropped==0`); a delta into an
  all-control-full queue → dropped, queue unchanged.

**Integration:**

- `TestV2Session_Push_NonBlockingUnderStall` (AC#1) — a stalling outbound double
  (records via an inner `v2Recorder`, but once armed blocks each call on a
  `release` channel) wedges the Run forward; many `Push` calls from the test
  goroutine each return within a tight deadline while outbound is stalled (the
  producer is not wedged); the drop counter engages past cap; release → the
  survivors decrypt **in order** under the phone's `recv` state, proving
  drop-before-seal left no nonce gap end-to-end. `t.Cleanup` releases first so a
  wedged Run goroutine never leaks.
- `TestV2Session_Push_NotOpen_ReturnsErrConnNotFound` (renamed from
  `…ReturnsErrSessionNotOpen`) — a white-box non-open session has no queue →
  `Push` now returns `ErrConnNotFound`.
- `TestV2Session_forwardEnvelope_NotOpen_GateRefuses` (new) — the drain-side
  half: the `V2StateOpen` security gate still refuses a buffered push to a
  session that closed before drain.
- `…InterleavedWithReply_DecryptsUnderRace`, `…ConcurrentWithReplies_NoNonceCorruption`
  — pass **unchanged** (buffered now; `waitForEnvelopes` polls; the 8 pushes are
  well under cap so none drop).
- `…CtxCancelled_ReturnsCtxErr` — still asserts `context.Canceled` (Push checks
  `ctx.Err()` first); the obsolete "no Run goroutine draining `m.push`" comment is
  replaced.

`make check` (vet + `-race` + staticcheck) green; binary builds. No e2e — the
producer→manager seam is unit/integration-covered here; the structured-receive
capstone is #642 (blocked on the harness gap, out of scope).

## Out of scope (deferred)

- **Delta coalescing** ([#609](609.md)) — reduces delta volume at the emitter
  (per JSONL message / ~250 ms); separate Phase-2 slice (shipped).
- **Mid-turn reconnect replay / per-conversation event ring + `last_event_id`
  resync** (#611) — the phone's recovery of dropped deltas. This ticket is only
  the transport-side bounded-queue drop policy.
- **Capacity calibration** — `pushQueueCap = 256` is a starting value pending the
  ADR-025 load test (line 220). Byte-bounding (vs count) is a contained
  refinement if large coalesced deltas later prove a memory concern.
- **Dropped-counter surfacing** — the per-session `dropped` counter is logged at
  debug; exposing it as a metric / snapshot field is deferred (no AC asks, and it
  would widen the surface).
- **`ActiveConns` decoupling** — the producer's per-`emit` `ActiveConns(ctx)`
  still serialises through Run (can wait ≤ one in-flight `Outbound`). This
  ticket's AC is the *enqueue* path; the one-per-pass drain bounds that wait to a
  single send rather than a full-buffer drain.
