# Ticket #603 — Restore `send_message` e2e coverage by teaching fakeclaude the idle/thinking glyphs

Split from [#594]. #594 rewrote `Supervisor.WriteUserTurn` to deliver through
`tuidriver.Session.DeliverPrompt` behind a `WaitReady` idle-gate, acking **only** on a
confirmed commit. `internal/e2e/internal/fakeclaude` renders no claude TUI — it never
emits the idle prompt (`❯`) or thinking spinner (`✻`) — so under the new contract
`WaitReady` never reached idle, the 30 s `sendMessageDeliverTimeout` elapsed, and
`send_message` replied `server.binary_offline` instead of `ack`. Five `//go:build e2e`
flows that drive `send_message → WriteUserTurn` were `t.Skip`ped in #594 to absorb this.
This slice restores them by teaching fakeclaude — **flag-gated** — the two glyphs the
tui-driver detection contract needs, so the harness regression-guards the genuine
delivery path (until now only `deliverFn`-seam unit/handler tested).

**`security-sensitive`** — the change expands the `cmd/substrate-guard` allowlist (the
build-time substrate seal), which is the security-relevant call that earns the label.
Architect self-review **PASS, no MUST FIX** (see the spec's `## Security review`);
code-review independently confirmed the now-allowlisted file carries **only** the two
sanctioned glyphs (no CSI escapes, no `[Pasted text]` chip). **No new pyry
runtime code** (AC #3) — the delta is confined to the e2e test harness, the test files,
and the allowlist data.

### fakeclaude TUI mode

A new env var `PYRY_FAKE_CLAUDE_TUI` (any non-empty value) turns on glyph emission. When
**unset, fakeclaude is byte-identical to its pre-#603 behaviour** — so the TUI-off
callers (`StartRotation`, the rotation tests, the non-skipped
`relay_two_phone_coarse_test.go` from #634) are unperturbed. Opt-in is per-test, **not**
baked into a shared helper.

- **Startup** — write the **idle-prompt glyph `❯` (U+276F)** once to `os.Stdout`. This
  seeds `IsIdle` true so the first `WaitReady` returns immediately. A *single* write
  suffices: tui-driver's `Snapshot()` is a **rolling 4096-byte raw-byte window, not a
  terminal-grid emulator** — the glyph persists in the snapshot until ≥4096 later bytes
  evict it; `\r`/cursor moves do not clear it. No continuous redraw needed.
- **First stdin bytes** — write the **thinking-spinner glyph `✻` (U+273B)** once. The
  `WaitReady`-gated `DeliverPrompt` is the only stdin fakeclaude ever receives; its
  arrival is a faithful "claude started the turn" signal → `IsThinking` true → a **fast
  confirmed commit** (the `DeliverPrompt` commit poll sees `✻` within its 150 ms cadence,
  not the 3 s committed-but-slow fallback).
- **One stdin reader.** `startStdinLogger` was generalised to
  `startStdinReader(logPath, tui)`, started when `STDIN_LOG` is set **OR** TUI is on (a
  second reader would race it for bytes). Per read: append to the log if a path is
  configured (unchanged), and — if TUI on and not yet emitted — emit `✻` once. The
  one-shot `spinnerEmitted` bool lives entirely on this goroutine (no cross-goroutine
  share). TUI mode causes stdin to be read independently of logging, so the design
  doesn't depend on `STDIN_LOG` being set.
- **Never echoes stdin to stdout.** TUI mode writes only the *fixed* glyph, never the
  phone-controlled prompt bytes — holding the trust boundary (no reflection of untrusted
  input onto the observed PTY / `message` stream).
- **One `stdoutMu` serialises every `os.Stdout` write** — the startup `❯`, the `✻`, and
  the assistant chunk (`emitAssistantIfTriggered`). Without it a spinner write could
  interleave mid-assistant-chunk and corrupt the marker the echo tests match on. Glyphs
  are emitted as bare runes (no escapes), kept as local `[]byte` rather than a tui-driver
  import to keep the stand-in zero-dependency.

No "return to idle" / spinner-eviction logic is needed: **every restored flow drives
exactly one idle→thinking transition** (single `send_message`; test 2's respawned claude
is a fresh process that re-seeds `❯` at startup).

### substrate-guard allowlist

fakeclaude's source now carries the raw `❯`/`✻` glyphs, which the guard bans. One path
suffix `internal/e2e/internal/fakeclaude/main.go` is added to `allowlist`, mirroring the
sanctioned `internal/agentrun/ptyrunner/helper_test.go` exemption. The allowlist
*mechanism* is unchanged; the exemption is scoped to the single test-only helper, which
is not importable by / compiled into `pyry`. `make substrate-guard` / `make check` stay
green (AC #4).

## Patterns established

- **Ack-pollution drain (the non-obvious part the ticket under-counted).** tui-driver
  reads the PTY once and fans it to **both** its internal commit-detection buffer (good)
  **and** `Session.MirrorOutput()` → `Bridge.Write` → the assistant-turn observer.
  Because `WriteUserTurn` stamps the supervisor cursor **before** delivery, the
  assistant-turn emitter sees a non-empty cursor and forwards the `✻` chunk as a
  `message` envelope to the same phone conn — **racing the ack** (async emitter queue vs
  the handler's synchronous reply; order is non-deterministic). This is faithful to
  production: real claude's spinner frames are forwarded as `message` chunks too. The fix
  **extends the existing assistant-phase "tolerate prelude chunks" tolerance to the ack
  receive**:
  - **v1 (tests 1–4):** a shared `recvEnvelope(t, phone, want, timeout)` helper in
    `relay_test.go` receives until `Type == want`, skipping (and logging) any other
    envelope, bounded by the test's existing deadline (3 s for 1/3/4; 15 s for 2).
  - **v2 (test 5):** a **decrypt-drain** loop. Every binary→phone frame **must** be
    decrypted in capture order (the receive `CipherState` nonce is sequential), so the
    loop decrypts each frame and skips non-ack types at the **envelope-Type level,
    post-decrypt** — preserving the nonce invariant. (v2 cannot use `recvEnvelope` — its
    frames are Noise-encrypted.)
- **Flag-gated opt-in, not shared-helper bake-in.** TUI off by default keeps every
  existing caller byte-identical; the five flows opt in per-test via
  `StartRotationWithRelay`'s trailing `extraEnv ...string` (tests 1/3/4/5) or the inlined
  `startEvictionHarness` env slice (test 2). **No `harness.go` change.**
- **Fast confirmed commit vs slow fallback is the failure-surfacing seam.** fakeclaude
  must emit `✻` for a *fast* commit; without it `DeliverPrompt` falls through to the 3 s
  committed-but-slow path and the ack arrives ~3 s late, tripping the 3 s `phone.Receive`
  deadline. So a broken spinner emit surfaces as a **deadline failure, not a false pass**
  — and a missing `❯` surfaces as the original 30 s `binary_offline`. Observed acks land
  sub-300 ms (v1) / ~2.3 s post-respawn (test 2), confirming the spinner path fired.

## Lessons learned

Lifted from PR #653's **Lessons learned**:

- The spinner glyph is read once by tui-driver but fanned to **both** the
  commit-detection buffer and `Bridge.Write`, so the commit signal itself surfaces to the
  phone as a `message` envelope racing the ack — restoring these flows required extending
  the existing assistant-phase "tolerate prelude chunks" drain to the **ack** receive.
  The fast (sub-300 ms) ack confirms the spinner path fired, not the 3 s
  committed-but-slow fallback; a broken spinner emit would surface as a deadline failure,
  not a false pass.
- **Incidental noise in the diff:** `internal/e2e/realclaude/testdata/permission_protocol_v2.1.158.json`
  shows ~122 lines changed, but the delta is purely the regenerated golden capture's
  embedded run-specific values (temp-dir paths + `session_id`/`uuid`). It was
  regenerated and auto-committed during the code-review `make e2e-realclaude` verification
  run, not part of this ticket's feature work. (A realclaude golden capture churns those
  fields on every capture run — signal vs noise when reading future diffs.)

## Out of scope / deferred

- **Option 2 (migrate the five flows to the realclaude suite) — rejected.** The
  `internal/e2e/realclaude` suite has no `send_message`/`WriteUserTurn` flow, so option 2
  would be a from-scratch five-flow build that exceeds S, and there is no Noise-phone
  realclaude harness for these flows (per [[architect-structured-stream-e2e-harness-gap]]).
  Option 1 lands as one S ticket confined to the harness + the allowlist.
- **The structured-receive capstone [#642] is *not* fully unblocked by this ticket.**
  #603 clears the `WaitReady → DeliverPrompt → commit` half that all five flows (and #642)
  needed. But fakeclaude still writes only `{}\n` to its session JSONL
  (`openSession`, unchanged), while the structured-stream producer
  (`cmd/pyry/interactive_turn_stream_v2.go`) tails the **real** claude JSONL transcript —
  so an interactive phone under fakeclaude still has no structured stream to receive.
  #642 additionally needs a structured-JSONL-capable harness; #603 alone does not close it.
