# Ticket #702 — remote-permission gate: per-device authorization bit + `--allow-remote-permissions`

The **authorization primitive** for answering a remote permission / trust /
destructive modal from a paired phone ([EPIC #597] Phase 3, [ADR 025]
§ "Security model"). Answering such a modal is the highest-trust action the
mobile head can take, so it is gated separately from everything else a paired
phone does (watch the stream, snapshot, send, interrupt, dequeue — all ungated).
This slice ships the per-device stored bit, the CLI flag that records it at pair
time, and two fail-closed pure predicates. It is **consumed by the modal control
loop ([#703])**, which wires the bit into the live permission-answer path and
owns the deny-on-timeout *timer*; the **audit log** is its sibling [#712].

The gate is a per-device **stored authorization**, NOT a wire capability and NOT
a backward-compat seam — [EPIC #597] retires the coarse path, so there are no old
phones. The wire types ([#701], merged) carry no capability field for it by
design; the bit is set **only** locally at pair time and read off the
already-authenticated `*devices.Device`.

## Implementation

Three additive production files. No state machine, no goroutines, no consumer
cascade (the only consumer, [#703], is a separate ticket).

| File | Change |
|---|---|
| `internal/devices/device.go` | **+1 field** `AllowRemotePermissions bool` on `Device`, tagged `json:"allow_remote_permissions,omitempty"`, mirroring the `Platform`/`PushToken` `omitempty` precedent. The zero value (`false`) is DENY, so an omitted/pre-field on-disk record decodes to denied. `omitempty` keeps the secure default off disk. `Device` stays comparable (`bool` is comparable) so the existing `out != in` round-trip asserts still compile. |
| `internal/devices/auth.go` | **+1 method, +1 func, +1 type, +5 consts** beside `Validate`, in its SECURITY doc voice. `(*Device).MayAnswerRemotePermission() bool` — nil-guarded eligibility (`d != nil && d.AllowRemotePermissions`). `RemotePermissionOutcome` enum (`OutcomeNoAnswer`=iota / `OutcomeAllow` / `OutcomeDeny` / `OutcomeTimeout` / `OutcomeCancel`). `AuthorizeRemotePermission(d *Device, outcome) bool` — the fail-closed decision (`d.MayAnswerRemotePermission() && outcome == OutcomeAllow`). |
| `cmd/pyry/pair.go` | **+1 flag** `--allow-remote-permissions` (default `false`) on `parsePairArgs`'s flagset, threaded through `pairArgs.allowRemotePermissions` into the `devices.Device{...}` literal in `runPairDefault`. Usage string extended. `pyry pair list`/`revoke`/`preflight` unaffected. |

**Data flow** (this ticket owns the CLI→disk→`*Device` path + the two predicates):

```
pyry pair --allow-remote-permissions   (operator, local CLI — the ONLY writer)
  → runPairDefault: devices.Device{..., AllowRemotePermissions: true}
  → registry.Save (atomic temp+rename, 0600)
  → ~/.pyry/<name>/devices.json  ("allow_remote_permissions": true; omitted when false)
  → (daemon) devices.Load → Validate → *devices.Device → dispatch.Conn.Auth()
       └─ #703: Auth().MayAnswerRemotePermission() == false ⇒ reject modal answer w/ error envelope
       └─ #703: AuthorizeRemotePermission(Auth(), outcome) — grant ONLY on eligible × OutcomeAllow
```

## Patterns established

- **Two predicates, two #703 code paths, defense-in-depth.** `MayAnswerRemote­Permission`
  answers "may this device answer *at all*" → [#703] rejects a non-permitted phone
  with an *error envelope* before resolving any answer. `AuthorizeRemotePermission`
  answers "given the outcome, does it grant" → drives the keystroke. The decision
  function **re-checks eligibility internally**, so it denies correctly even if a
  caller skips the upfront gate — the two compose but neither is a single bypassable
  point.
- **The outcome is an enum, not a bare bool — fail-closed is structural.** A bool
  would erase the distinction between no-answer / timeout / cancel (making the AC5
  tests vacuous). The enum makes the safe default live in **one** place: the only
  ALLOW branch is `eligible && outcome == OutcomeAllow`; any future outcome added
  to the enum defaults to DENY unless explicitly mapped. The zero value
  (`OutcomeNoAnswer`) is DENY, so even a default-constructed/forgotten call denies.
  This is the deny-on-timeout model realized in **deterministic code** (the right
  fabric for a safety net — belt-and-suspenders means different fabric), not a
  stochastic agent rule.
- **Nil-guarded pointer receiver = total, fail-closed predicate.** `Conn.Auth()` is
  typed `*devices.Device` and is nil before the first-frame gate accepts; the
  nil-guard (`d != nil && ...`) makes the safe default *structural*, not a convention
  #703 must remember.
- **Ownership split for deny-on-timeout.** [#703] owns the *timer* (detects the
  timeout); this primitive owns the *decision applied on timeout* (`OutcomeTimeout
  → false`). A bug in #703's timer (fires late / never) degrades to "modal stays
  outstanding," never to "auto-grant" — no outcome other than `OutcomeAllow` ever
  grants.
- **`omitempty` polarity is the safe one.** Absence ⇒ `false` ⇒ denied; a `true` bit
  is always written. There is no on-disk shape where an *absent* key reads as
  *granted*. (Contrast: a hypothetical default-true `"deny_remote_permissions"` field
  would be dangerous under `omitempty`.)
- **Pure predicates, no side effects.** No logging, no I/O, no token handling — keeps
  the gate unit-testable in isolation and uncoupled from observability. Audit-writing
  on a decision is [#712]'s primitive, called by [#703], deliberately separate. The
  package SECURITY contract (`device.go:4-10`, never log/serialize the plain token) is
  preserved: the new code touches no token.

## Lessons learned

- `internal/devices/registry_test.go` and `cmd/pyry/pair_test.go` are **gofmt-dirty at
  HEAD under local Go 1.26.2** (struct-literal alignment skew in untouched tests —
  `TestRegistry_FindByTokenHash`, `TestResolveRelay`), confirming
  [[pyrycode-gofmt-dirty-at-head-go1.26]]. The pre-existing skew was left alone; only
  the new additions are gofmt-clean. Check `git show HEAD:<f> | gofmt -l` before
  "fixing" any file you didn't change, or CI's pinned-older Go fights your reformat.

## Out of scope (siblings)

- **Modal control loop — [#703].** Mints/emits modals, routes/validates the inbound
  answer, owns the deny-on-timeout *timer*, and is the sole consumer of both predicates.
  Producer obligations surfaced here: (a) call `MayAnswerRemotePermission` *first* →
  error-envelope-reject a non-permitted phone before resolving; (b) map observed modal
  state (option-kind / timeout / cancel) → `RemotePermissionOutcome`, then
  `AuthorizeRemotePermission`; (c) on reject, no device name in the error
  (anti-enumeration); (d) audit via [#712], never logging modal body text or tokens.
- **Audit log — [#712].** Every remote permission decision → an audit entry; the
  decision-logging primitive #703 calls.
- **Two-heads ownership — [#706].** Stale `modal_id` rejected, first-answer-wins
  (replay/nonce layer on top of this gate).
- **Wire vocabulary — [#701]** (merged). The four modal envelope types; carries no
  capability field by design — see [`codebase/701.md`](701.md).
