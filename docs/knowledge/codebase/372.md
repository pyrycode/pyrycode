# Ticket #372 — `e2e/realclaude`: `WithWorktree` + `ReadJSONL` fixture helpers

Two file-system primitives shared by downstream real-`claude` tests (#364–#368): a HOME-isolated workdir and a typed reader for the session JSONL the run produces. Sibling to #373's subprocess invocation helper; no compile-time link between the two — downstream tests compose all three.

See [features/e2e-realclaude.md](../features/e2e-realclaude.md) for the suite's full contract.

## Implementation

- **`internal/e2e/realclaude/fixtures.go`** (~55 LoC, new). Build tag `//go:build e2e_realclaude`, package `realclaude`. Three exported symbols:
  - `type JSONLEntry = jsonl.Event` — type **alias** (`=`), not a wrapper. Preserves field access (`entry.EndOfTurn`, `entry.Usage.OutputTokens`, etc.) without forcing callers to import `internal/agentrun/jsonl` themselves.
  - `WithWorktree(t) string` — `t.TempDir()` + `t.Setenv("HOME", dir)`, returns the path. No `t.Cleanup` (both stdlib calls register their own). No `MkdirAll(.claude/…)` — claude/pyry create it on first write; pre-creating would mask a regression.
  - `ReadJSONL(t, workdir, sessionID) []JSONLEntry` — delegates the path-resolve-and-open work to private `resolveAndOpenJSONL`, then loops `jsonl.NewReader(f, jsonl.Config{}).Next()` until `io.EOF`. Open/parse failures call `t.Fatalf` with the resolved path in the message.
- **Private split: `resolveAndOpenJSONL(workdir, sessionID) (*os.File, string, error)`.** Pulled out specifically so the missing-file scenario is unit-testable as a returned error rather than via a `*testing.T`-wrapper trick. The public `ReadJSONL` is the only test-friendly entry point; the public surface stays at the three symbols above.
- **`internal/e2e/realclaude/fixtures_test.go`** — five tests, all under `//go:build e2e_realclaude`:
  - `TestWithWorktree_ReturnsExistingHomeIsolatedDir` — return path exists, `os.UserHomeDir()` matches it, and a nested `t.Run` subtest's HOME pin restores cleanly on subtest exit.
  - `TestReadJSONL_HappyPath` — writes two fixture lines (assistant `end_turn` + user) via `writeFixtureLines` helper, asserts `len(events) == 2`, `events[0].Kind == "assistant" && events[0].EndOfTurn`, `events[1].Kind == "user" && !events[1].EndOfTurn`.
  - `TestReadJSONL_EmptyFile` — zero-byte file → empty slice, no `t.Fatalf`.
  - `TestResolveAndOpenJSONL_MissingFile` — drives the private split directly: asserts the returned error wraps `os.ErrNotExist`, the returned path equals the computed `<home>/.claude/projects/<enc>/<sessionID>.jsonl`, and the error message contains that path verbatim.
  - `TestJSONLEntry_AliasCompiles` — compile-only assertion: `var _ jsonl.Event = JSONLEntry{}`.

## Reuses (no re-implementation)

- `internal/agentrun.EncodeProjectDir` — single source of truth for the `~/.claude/projects/<encoded>` segment (`/` AND `.` → `-`). The helper never re-derives the encoding rule; if `EncodeProjectDir` ever changes (e.g. for symlink edge cases), `ReadJSONL` inherits the fix.
- `internal/agentrun/jsonl.{Reader, Event, Config, NewReader}` — parser, event shape, default config. Malformed-line skipping stays internal to `Reader`; `ReadJSONL` only surfaces structural I/O failures.

## Patterns established

- **Test-helper signature convention for file-system fixtures: take `*testing.T`, fatal on failure.** Both helpers follow the `internal/e2e/harness.go` style — caller reads top-to-bottom as a scenario, not as a sequence of `if err != nil` checks. The error-returning split (`resolveAndOpenJSONL`) exists purely for unit-testability of failure paths; it's not the documented public surface. Generalises to: future test fixtures in opt-in suites should default to fatal-on-failure with the offending path/value embedded; carve out a private error-returning split only when a failure path itself needs verification.
- **`t.Setenv` is load-bearing, not interchangeable with `os.Setenv`.** `t.Setenv("HOME", dir)` restores the prior value on cleanup and is documented thread-safe with respect to the framework's cleanup ordering — `os.Setenv` is neither. Mandatory for any test that pins an env var resolved by both the in-test process and a subprocess it later spawns (subprocesses inherit `$HOME` through `os.Environ()`, so the same encoded-projects path resolves on both sides).
- **Type alias (`=`) over wrapper struct for re-exporting parser types from a test fixture package.** `type JSONLEntry = jsonl.Event` keeps `entry.Usage.OutputTokens` accessible without re-exporting `jsonl.UsageBlock` and without forcing downstream tests to import `internal/agentrun/jsonl` directly. Zero-cost ergonomic shim. Generalises: prefer alias over wrapper whenever the only purpose is namespace tidiness and the underlying type's full API should remain reachable.
- **Don't pre-create directories the runtime owns.** `WithWorktree` deliberately does NOT `MkdirAll(.claude/projects/…)`. Pre-creating would mask a regression where the runtime stops creating the directory on first write. Test fixtures should establish only the state the system-under-test isn't responsible for; everything else the runtime creates IS the proof the runtime still works.
- **Carve out a private error-returning split when a public fatal-on-failure helper has a failure path worth asserting.** The "wrap `*testing.T` to capture `Fatalf`" trick is fragile (relies on testing framework internals); refactoring the path-resolve-and-open work into an unexported function and unit-testing _that_ on its returned error keeps the verification deterministic and tag-friendly. Public API surface unchanged; reflection-style test-helper hacks avoided.
