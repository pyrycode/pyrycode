# Ticket #685 — spawn a conversation's session in its canonicalised, trust-marked `Cwd`

The consumer slice of EPIC #672's per-conversation-workdir work (split from #681). Until #685 every conversation's bound claude session spawned in the daemon's shared trusted workdir; the per-conversation `Cwd` (`internal/conversations/conversation.go`, phone-influenceable at create time) was stored inert (see the `SECURITY` comment #666/#677 left at `create_conversation.go:74-85`). #685 makes that `Cwd` a **validated spawn input**: a discussion's session now spawns in its own recorded directory, so discussions targeting different projects are isolated on disk, without the spawned claude wedging on the first-run workspace-trust modal. A `Cwd` that escapes `$HOME` after symlink resolution is rejected as a `create_conversation` error with no half-bound row.

This wires together three pre-existing pieces — it reinvents nothing: `confineWorkdirToHome` + `trustMark` (#670) and `Pool.CreateIn` (#684). The cmd layer is the sole validator; the handler does no path handling.

See the evergreen feature doc: [features/conversation-session-binding.md § `Cwd` is the validated, trust-marked spawn workdir (#685)](../features/conversation-session-binding.md#cwd-is-the-validated-trust-marked-spawn-workdir-685).

## Implementation

4 files, +391/-45 (2 production `.go`, 2 test). No new protocol types, no registry-schema change.

- **Widened the `SessionCreator` seam** (`internal/relay/handlers/create_conversation.go`). `Create(ctx, label) (string, error)` → `Create(ctx, label, spawnDir string) (string, error)`. The doc comment pins the contract: `spawnDir == ""` → the daemon's shared trusted workdir (default, unchanged); a non-empty `spawnDir` is the phone's *raw requested* working directory, validated + trust-marked by the implementation before spawning; an escape is rejected wrapping `ErrSpawnDirRejected`. Bounded cascade — only 4 sites changed (interface decl, `sessionMinter` impl, the handler call site, the `stubSessionCreator` test fake); `startRelay`/`startRelayV2` take the *interface* type and never call `.Create`, so threading stops there.
- **Handler derives `spawnDir` from the raw nullable `p.Cwd`, not the defaulted `cwd`** (`create_conversation.go:143`). `spawnDir := ""; if p.Cwd != nil { spawnDir = *p.Cwd }`. This resolves the ticket's "empty-`Cwd` not visible past the handler" wrinkle *structurally*: the existing `cwd := defaultCwd`-on-null resolution (which still feeds the recorded row + the `conversation_created` reply) is left untouched, and "where to spawn" (`spawnDir`) is kept cleanly separate from "what to record" (`cwd`). So a default conversation records `defaultCwd` yet spawns in `tpl.WorkDir` — byte-identical to today (AC#4).
- **New cmd-layer `resolveSpawnDir(requested string) (string, error)`** (`cmd/pyry/main.go:490`, sibling of `confineWorkdirToHome`), mirroring the daemon bootstrap's own `confine → trust → spawn-in-realpath` sequence (`runSupervisor`, `main.go:547-551`):
  - `""` → `("", nil)`; `trustMark` is **not** called.
  - set → `confineWorkdirToHome(requested)` (canonicalise both sides + confine to `$HOME`, returns the realpath) → `trustMark(realpath)` → return **trustMark's** realpath. Returning trustMark's own return value (not a re-derived path) is what guarantees AC#3: claude's cwd and the trust-marked path are byte-identical, independent of whether `trusted == confined-realpath`.
  - escape (incl. a symlink resolving outside `$HOME`) / unresolvable → confine errors → `fmt.Errorf("%w: %v", handlers.ErrSpawnDirRejected, err)` (confine detail wrapped via `%v` for logs; `errors.Is` matches the sentinel).
  - `trustMark` errors (transient `~/.claude.json` write failure) → returned **plain**, so the handler classifies it retryable.
- **`sessionMinter.Create` collapses to validate-then-mint** (`cmd/pyry/main.go:705`): `resolved, err := resolveSpawnDir(spawnDir); if err != nil { return "", err }; id, err := m.p.CreateIn(ctx, label, resolved)`. The single cmd-layer type-narrowing seam (precedent: `poolResolver`) now also owns spawn-workdir validation.
- **New sentinel + static message + error mapping** (`create_conversation.go`). `var ErrSpawnDirRejected = errors.New("conversation spawn directory rejected")` lives in the consumer/mapper package (cmd/pyry wraps it — no import cycle). `msgCreateConversationCwdRejected = "conversation working directory not allowed"` — static, never echoes the path or `~/.claude.json`. The handler's single mint-failure branch splits in two: `errors.Is(err, ErrSpawnDirRejected)` → `replyError(…, protocol.CodeProtocolMalformed, …, false)` (**non-retryable** — re-issuing the same `Cwd` fails identically); any other error → the existing retryable `server.binary_offline` path. **Both branches still `return` before `reg.Create`** → no half-bound row (AC#2).
- **Rewrote the `SECURITY` comment** (`create_conversation.go:99`). The prior comment stated `Cwd` was "structurally excluded" as a spawn input and deferred validation; #685 reverses that, so the comment now documents the new posture (raw `Cwd` forwarded verbatim as `spawnDir`; validated + trust-marked solely at the cmd-layer adapter; handler stays free of `internal/sessions`/cmd-layer imports). Leaving the stale text would have been a security-doc regression.
- **Tests.**
  - **Handler** (`internal/relay/handlers/create_conversation_test.go`): `stubSessionCreator` extended to record `spawnDirs` and return a configurable wrapped `ErrSpawnDirRejected`. `TestCreateConversation_SetCwd_ThreadsRawSpawnDir` (set `Cwd` → stub records it verbatim, row+reply record the raw value, label stays the minted id — replaces the pre-#685 `CwdNeverReachesMint` whose premise inverts); `TestCreateConversation_NullCwd_EmptySpawnDir` (null → `spawnDir==""`, row records `defaultCwd` — the spawn-vs-record split); `TestCreateConversation_SpawnDirRejected_NonRetryableMalformed` (wrapped sentinel → non-retryable `protocol.malformed`, static message, **0 rows**); plus the existing generic-mint-failure test guarding that a *plain* error still maps retryable.
  - **cmd/pyry adapter** (`cmd/pyry/conversation_spawndir_test.go`, new): mirrors `workdir_trust_test.go` — `t.Setenv("HOME", …)` (non-parallel) and overrides the package-level `trustMark` seam with a recording stub (`installRecordingTrustMark`). Cases: empty → `("", nil)` + trustMark not called; within-`$HOME` → returns trustMark's sentinel return byte-identical, trustMark called once with the confined realpath; outside-`$HOME` → `errors.Is ErrSpawnDirRejected`, trustMark not called; symlink-under-`$HOME`-pointing-out → rejected; trustMark-error → result does **not** match the sentinel (so the handler classifies it retryable).

## Patterns established

- **Validation/trust is a cmd-layer job; the pool primitive receives a pre-resolved realpath.** Reaffirms lesson [[architect-trust-premark-is-cmd-layer-job]] and ADR 025: `Pool.CreateIn` (#684) uses `spawnDir` verbatim and does no validation by contract, so the canonicalise + `$HOME`-confine + trust-mark all happen at `sessionMinter`/`resolveSpawnDir`. The untrusted→trusted boundary is single and explicit; `internal/relay/handlers` never touches the path — it forwards the raw value through the typed seam and maps the sentinel.
- **Reuse the existing confine+trust pair, do not reinvent.** `resolveSpawnDir` is a near-verbatim mirror of the daemon bootstrap's own `confineWorkdirToHome → trustMark → spawn-in-realpath` at `main.go:547-551`. The security posture for a phone-chosen workdir is *identical* to the daemon's own bootstrap workdir — same `$HOME` confinement, same trust pre-mark — which is exactly the "consistent with the daemon's existing workdir-confinement posture" the AC required.
- **Separate "where to spawn" from "what to record."** Reading the raw nullable `p.Cwd` for `spawnDir` while the recorded row keeps the defaulted `cwd` lets a default conversation record `defaultCwd` yet spawn in `tpl.WorkDir`, and round-trips the phone's own input in `conversation_created` (a macOS `/var`→`/private/var` realpath in the reply would surprise the phone). The row's `Cwd` drives nothing downstream — respawns read `supervisor.Config.WorkDir`, captured as the realpath at `CreateIn` time, not re-derived from the row.
- **Reuse `protocol.malformed` over minting a new error code.** A `Cwd`-escape rejection is a deterministic bad request, and `docs/protocol-mobile.md:614` pins application error codes "unchanged from v1"; minting `conversation.cwd_rejected` would be a v2 protocol-vocabulary change (codes.go + spec table + drift detector + mobile handling) out of an S slice's lane. `protocol.malformed` is already this handler's non-retryable channel and the phone already treats it as non-retryable — the correct retry behaviour is what matters. A dedicated code is a deferred protocol follow-up.

## Lessons learned

- **The confine→trust order is load-bearing.** `trustMark` has no `$HOME` bound of its own, so confining first is what keeps an out-of-`$HOME` path from being auto-trusted. A wrong order (trust then confine) would still pass the happy path but silently widen the trust boundary — a happy-path test can't catch it. `TestResolveSpawnDir_OutsideHome_Rejected` asserts trustMark is called **0 times** for an escaping dir, which is the regression guard for the order. (Lifted from the PR's Lessons learned.)
- **A non-existent requested path is rejected, same as an escape.** `confineWorkdirToHome` fails at `EvalSymlinks` for a path that doesn't exist → treated as `ErrSpawnDirRejected` (non-retryable). A phone-named dir that isn't there is a deterministic client error — a deliberate, correct consequence (not an AC, but natural).
- **The confine-time-of-check → chdir-time-of-use TOCTOU window is accepted, not closed.** Between `confineWorkdirToHome`'s `EvalSymlinks` and claude's eventual `chdir`, a path that resolved inside `$HOME` could be swapped to an escaping symlink. This is the **same residual window the daemon's own bootstrap workdir already accepts** (`main.go:547-551` has the identical gap), and the attacker must already control the operator's home to win it. #685 does not widen the window; no new mitigation is in scope.

## Out of scope (deferred)

- **Per-`Cwd` turn bridge** — #686 (`blocked-by #685`): making the outbound reply stream follow the per-conversation transcript when sessions live in distinct directories.
- **Dedicated `conversation.cwd_rejected` error code** for finer phone UX ("directory not allowed" vs generic malformed) — a separate protocol ticket spanning this repo + mobile.
- **Recording the realpath (not the raw requested value) in the row** — deferred; no current consumer needs the row to equal claude's actual cwd. The seam would have to return the realpath to change this; don't without a consumer.
