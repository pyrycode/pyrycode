# Ticket #375 — `agentrun/selfcheck`: rewrite for stream-json mode

Re-implements the boot-time deny-default safety net (#336) against the post-#391 stream-json runtime. The conceptual guard is unchanged — catch silent Claude Code regressions that would let agents call tools we never allowed — but the verification mechanism shifts from "spawn claude in PTY-bridged interactive mode under `permissions.defaultMode: "deny"` and tail the on-disk JSONL" to "spawn claude with `--allowed-tools "Read" --dangerously-skip-permissions` in stream-json mode and parse `tool_use` events directly off stdout."

See [features/agentrun-selfcheck-package.md](../features/agentrun-selfcheck-package.md) for the rewritten contract; this file captures the deltas + lessons.

## Implementation

- **`internal/agentrun/selfcheck/selfcheck.go`** rewritten end-to-end (~263 lines, down from ~316). Single file, single entry point: `SelfCheckDenyDefault(ctx, Config) (Result, error)`. Reuses `streamrunner.Run` (#390) as the spawn primitive and `jsonl.NewReader` (#348) as the parser; the package now owns no exec wiring or fsnotify watching of its own. Argv mirrors production `cmd/pyry/agent_run.go:buildClaudeArgs` less the inputs that don't apply to the diagnostic verb (no `--append-system-prompt-file`, no `--session-id`, no `--settings`); `--allowed-tools "Read"` + `--dangerously-skip-permissions` are the load-bearing pair the watcher protects.
- **Concurrency model** — two goroutines under `errgroup.WithContext(timeoutCtx)` joined by an `io.Pipe`. The spawner calls `streamrunner.Run` with `Stdout: pw`, `Stderr: io.Discard` (SECURITY: stderr is structurally unable to leak into pyry logs). The watcher owns `pr` + a `jsonl.Reader` and loops on `reader.Next()`. **Pipe-close discipline**: the spawner `defer`s `pw.Close()` so the watcher sees `io.EOF` if claude exits without producing end-of-turn nor Bash; the watcher `defer`s `pr.Close()` so a pending `pw.Write` from the spawner fails fast instead of deadlocking. Without those two defers `g.Wait()` would hang indefinitely on the cleanup path.
- **Detector** unchanged byte-for-byte: `bashInvokedInRaw(json.RawMessage) (bool, error)` re-decodes assistant `Event.Raw` into `{Message: {Content: [{Type, Name}]}}` and walks the content array for `Type == "tool_use" && Name == "Bash"` (exact-case). Decode errors return `(false, err)` so the caller logs-and-skips; one malformed line still must not turn a PASS into an inconclusive. The `TestBashInvokedInRaw` table (six rows: Bash hit / Read no-hit / text-only / lowercase "bash" / missing name / invalid JSON) carries over from #336.
- **Outcome priority** unchanged: `BashInvoked` → wrapped `ErrBashInvoked` (FAIL) > `EndOfTurnObserved` → `(result, nil)` (PASS) > `timeoutCtx.Err() == DeadlineExceeded` → `ErrTimeout` (INCONCLUSIVE) > non-Canceled `runErr` → wrapped infra error > defensive `errors.New("…terminated without end-of-turn or bash signal")`. Mechanism shifts (no errgroup-Drive return path) but the post-`g.Wait()` triage is identical, including the `errors.Is(timeoutCtx.Err(), context.DeadlineExceeded)` discriminator.
- **Removed surface** — `ErrUnknownTUI` (PTY-only sentinel, no longer reachable), `Config.HomeDir` (no JSONL tail to address), `Config.TrustDialogDelay` / `Config.PromptDelay` (no PTY, no delays), `canonicalPrompt`'s sibling `systemPromptFile` constant + zero-byte file write (no `--append-system-prompt-file` in argv), `newSessionID` helper (stream-json mints its own session id), `agentrun.MarkWorkdirTrusted` call (`--dangerously-skip-permissions` removes the trust dialog entirely), `agentrun.WriteSettings` call site in the CLI wrapper, the `internal/agentrun/jsonl/tail` import, the `crypto/rand` import. Net deletion: ~80 LoC across two files plus 174 LoC in `selfcheck_test.go` (helper-process PTY fake → simpler stdout-fake).
- **`cmd/pyry/agent_run_selfcheck.go`** narrowed (~89 lines, down from ~117). `runAgentRunSelfCheck(stdout io.Writer) error` body collapses to: `os.MkdirTemp` + `defer RemoveAll` → resolve `PYRY_CLAUDE_BIN` → `captureClaudeVersion` (unchanged best-effort 5s budget) → `selfcheck.SelfCheckDenyDefault(ctx, Config{ClaudeBin, WorkDir})` → `errors.Is` switch rendering PASS / FAIL (`writeSelfCheckFailMessage`) / INCONCLUSIVE / propagate. Drops `agentrun.WriteSettings(workdir, []string{"Read"})`, drops `os.UserHomeDir()`, drops the two `parseDurationEnv` env-knob reads. PASS line stays verbatim (`deny-default whitelist held: N assistant event(s) observed; Bash refused.`); INCONCLUSIVE message stays verbatim (no `defaultMode` references existed there).
- **FAIL message rewritten** — preserves the three-section "What was tested / What was observed / What to check" structure (operator familiarity, pinned by `TestRunAgentRunSelfCheck_FAIL`) but replaces the body of each section. New "What was tested" describes the `--allowed-tools "Read" --dangerously-skip-permissions` argv in stream-json mode. New "What to check" points at `cmd/pyry/agent_run.go`'s `buildClaudeArgs` and references `#329` (Phase A spike), `#336` (predecessor, superseded), `#375` (this rewrite). The "What was observed" body changes "session JSONL" → "stream-json stdout"; the verbatim `Evidence` line render is unchanged (still `strings.TrimRight(evidence, "\n")`). Negative pins in the test forbid `permissions.defaultMode`, `.pyry-agent-run-settings.json`, `per-spawn settings file`, `PTY` in the FAIL output — any future drift back to PTY language will fail this test.
- **`cmd/pyry/agent_run.go`** loses the now-orphaned `parseDurationEnv` helper (`PYRY_AGENT_RUN_TRUST_DELAY` / `PYRY_AGENT_RUN_PROMPT_DELAY` had no other callers). 18-line deletion; the `--self-check` short-circuit at the top of `runAgentRun` is unchanged.
- **Tests** — `internal/agentrun/selfcheck/selfcheck_test.go` rewritten end-to-end (~289 lines, down from ~374). `TestSelfCheckHelperProcess` is now a stdout-fake (write `GO_SELFCHECK_HELPER_STDOUT` verbatim, sleep `GO_SELFCHECK_HELPER_LIFETIME`, exit 0) re-exec'd via a `/bin/sh` wrapper that drops production claude argv (the Go test binary's flag parser would reject `--input-format` etc.). Cases preserved: PASS, BashInvoked, Timeout, MalformedAssistantLineSkipped, ConfigValidation, BashInvokedInRaw table. Removed: `BashInvokedUnderMisformattedSettings` (no settings file to misformat anymore — the runtime-vs-file-presence assertion is now structurally true). `cmd/pyry/agent_run_selfcheck_test.go` rewritten the same way (~155 lines, down from ~174); `configureSelfCheckFakeClaude` writes a wrapper that short-circuits `--version` to `"fake-claude 0.0.0"` (so `captureClaudeVersion` produces a clean line) and otherwise re-execs the test binary in `TestSelfCheckCLIFakeClaude` mode.

## Patterns established

- **`io.Pipe` + paired-defer-Close is the canonical bridge between `streamrunner.Run` and a downstream `jsonl.Reader`.** When you need to consume claude's stream-json stdout in-process rather than forwarding it byte-for-byte to the operator, `streamrunner.Config.Stdout = pw` + a goroutine running `jsonl.NewReader(pr, …).Next()` in a loop is the minimal wiring. The two-defer pattern (spawner closes `pw` on exit so the reader sees EOF; reader closes `pr` on exit so a stalled `pw.Write` fails fast) is load-bearing for clean teardown — without either, `errgroup.Wait` deadlocks. Mirror this shape for any future "spawn claude under a stream-json variant and observe events" diagnostic.
- **Mirror production argv in the diagnostic verb; do not invent flag combinations.** The architect spec forbade extracting a `buildSelfCheckArgs` helper; the six-line inline argv literal in `SelfCheckDenyDefault` IS the single point of truth, and any drift between it and `cmd/pyry/agent_run.go:buildClaudeArgs` is a regression of the safety net (the self-check would then be guarding a different argv shape than production). Pattern for any future "verify claude does X under our argv" probe: copy production argv verbatim, drop only the inputs that genuinely don't apply (here: `--append-system-prompt-file`, `--session-id`, `--settings`), document each omission inline.
- **Operator-string contract is pinned by negative-substring tests, not just positive ones.** `TestRunAgentRunSelfCheck_FAIL` asserts the FAIL message contains `#375` AND does NOT contain `permissions.defaultMode` / `.pyry-agent-run-settings.json` / `per-spawn settings file` / `PTY`. This catches the "developer forgot to update one of three sections" failure mode that a positive-only assertion would miss. Pattern when an AC requires deleting historical terminology from operator output: write the negative-substring assertion alongside the positive one.
- **Sub-package re-exec test fakes drop production argv via a shell wrapper.** The Go test binary's flag parser rejects unknown flags like `--input-format`, so a fake-claude can't be invoked directly with the production argv. The `/bin/sh` wrapper that `exec`s `os.Args[0] -test.run=^TestSelfCheckHelperProcess$` (ignoring all argv after argv[0]) is the standard shape; mirrors `streamrunner/helper_test.go` and the rewritten CLI fake. The `--version` short-circuit in the CLI variant is a small extension for codepaths that probe `claude --version` separately.

## Lessons learned

- **`io.Pipe` close discipline is symmetric — both ends need a defer.** The spawner closing `pw` on exit was obvious (so the reader unblocks on EOF). The reader closing `pr` on exit was less obvious; it matters because if the reader returns first (e.g. on a Bash hit + `cancel()`), `streamrunner.Run` may still be in the middle of a `pw.Write` call. Without `defer pr.Close()`, that `pw.Write` blocks forever waiting for someone to read from `pr`, and the spawner goroutine never returns, and `errgroup.Wait()` hangs. The fix is one line; the symptom (hung self-check, zero output) was minutes of hairloss.
- **`captureClaudeVersion` is a separate process invocation from the self-check spawn — the test fake must handle both.** The first instinct was to write a single fake-claude that handles both `--version` and the stream-json self-check. But `captureClaudeVersion` runs `exec.CommandContext(claudeBin, "--version").Output()` directly (no streamrunner wrapping, no stdin envelope), while the self-check spawn goes through `streamrunner.Run` with the production argv. Two different invocation shapes, both keyed off `PYRY_CLAUDE_BIN`. The shell-wrapper fake handles this by branching on `$1 == "--version"` before re-execing in test-binary helper mode — without the branch, `claude --version` re-execs into `TestSelfCheckCLIFakeClaude` which writes the stream-json fixture instead of `"fake-claude 0.0.0"`, and the PASS test fails on the missing version line.
- **Removing a feature from a CLI codepath can orphan helper functions in adjacent files.** `parseDurationEnv` lived in `cmd/pyry/agent_run.go` but its only callers were the two `parseDurationEnv("PYRY_AGENT_RUN_TRUST_DELAY")` / `parseDurationEnv("PYRY_AGENT_RUN_PROMPT_DELAY")` reads inside `runAgentRunSelfCheck`. Deleting those two lines without checking grep would have left `parseDurationEnv` as dead code, surviving via "compilation passes" until a future code review caught it. The architect spec called this out explicitly ("`parseDurationEnv` … leave it in place" was the original instruction; in fact removing it was correct because no other callers existed). Pattern reinforced: when removing a callsite, grep for the callee's other callers in the same commit, not in a follow-up.
- **`jsonl.Reader` is the right abstraction even when reading from a pipe (not a file).** The architect spec called out that `jsonl/tail` (the fsnotify watcher sub-package) is gone, but `jsonl.Reader` (the parser) carries over verbatim — different sub-packages, different responsibilities. The parser doesn't know or care whether its `io.Reader` is a file, a pipe, or a network socket; the malformed-line resilience, the `Event.Raw` slice copy semantics, the `EndOfTurn` detection rule all apply identically. Pattern for future stream consumers: assume `jsonl.Reader` is the answer; only build a second parser if a structural reason emerges.
