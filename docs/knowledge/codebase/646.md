# Ticket #646 — Event-stream replay: per-conversation event ring keyed by a durable event id

The **storage primitive** under "a phone that reconnects mid-turn catches up
without a gap" ([EPIC #596] Phase 2, [ADR 025] § Backpressure / replay). Adds
`internal/eventring` — a bounded, in-memory, daemon-resident store of the recent
structured turn events the interactive emitter fans out, keyed by a **durable,
connection-independent per-conversation event id** — and wires the emitter to
append to it once per logical event, before the per-conn fan-out. Ships with
tests and **no reconnect wiring**: the per-conn `last_event_id` tracking and the
on-reconnect replay are the consumer slice [#647] (`security-sensitive`) that
reads `After`. Split from [#611]. **Not `security-sensitive`** — local
producer-side plumbing, no inbound trust boundary; it does not touch the per-conn
`nextID` / nonce single-writer invariant (the Technical-Notes condition that would
have re-triggered the label).

## Implementation

One new package (2 files) + a ~+10 LOC additive edit to the emitter. **2 exported
types** (`Ring`, `Event`), **0 constructor call sites changed**.

| File | Change |
|---|---|
| `internal/eventring/ring.go` | **new** (~183 LOC incl. doc comments) — `Event`, `Ring`, unexported `convRing`; `MaxEventsPerConversation = 1024`; `New` / `Append` / `After` / `evictOldest`. |
| `internal/eventring/ring_test.go` | **new** (~320 LOC) — the full scenario table + a `-race` append-vs-query test. |
| `cmd/pyry/interactive_turn_v2.go` | **modified** (~+10 LOC) — `ring *eventring.Ring` field; constructor init; the `emit` hoist-and-append. |
| `cmd/pyry/interactive_turn_v2_test.go` | **additions only** — four emitter-integration tests (existing tests unchanged). |

See [features/eventring-package.md](../features/eventring-package.md) for the
evergreen package reference (full exported surface, eviction, the `After`
contract, durability boundary). Highlights specific to how it was built:

- **`Append` assigns the durable id once per logical event, before fan-out.** In
  `emit()` — the single place envelopes reach the wire — after the one
  `json.Marshal`, before the `ActiveConns` loop. The id is the ring's own
  per-conversation counter (starts at 1, strictly increasing); the per-conn
  envelope `nextID` is **left untouched, not overloaded** (a hard Technical-Notes
  constraint). The returned id is **unused in #646** — #647 surfaces it on the
  wire.
- **The event timestamp is hoisted out of the fan-out loop.** `ts := time.Now().UTC()`
  is now computed once and shared by the ring and every conn's envelope (was a
  per-conn `time.Now()`). Intentional, strictly-more-correct behaviour change
  (one logical event = one timestamp); no existing test asserted per-conn `TS`
  divergence.
- **Append is unconditional — including zero interactive conns.** The ring is the
  replay source for phones that are absent *now* and reconnect later, so an event
  is recorded even when the `ActiveConns` snapshot is empty (proven by
  `TestInteractiveTurnEmitterV2_RingAppendsWithNoInteractiveConns`).
- **Eviction:** hard per-conversation cap; oldest `assistant_delta` dropped first,
  the five control classes retained in preference, oldest-overall dropped only
  under all-control pressure. Slice kept id-ascending (non-contiguous after a
  middle delta eviction — by design, not a gap).
- **`After` is a pure 3-way classification** — caught-up `(nil,false)` /
  gap `(nil,true)` / replay `(events, false)` — plus the unknown-conversation case
  (`afterID==0` caught-up, `afterID>0` gap) that makes the AC-5 signal usable
  across the daemon-restart boundary #646 scopes out.

## Patterns established

- **Emitter-owns-the-ring retires the constructor cascade.** A required
  `*eventring.Ring` parameter on `newInteractiveTurnEmitterV2` would have forced a
  simultaneous edit of all 26 call sites (1 production + 25 test) — the
  constructor-cascade red line that would have split the ticket. Creating the ring
  *inside* the constructor (unexported field, default `eventring.New(MaxEventsPerConversation)`)
  keeps the signature byte-identical and the change additive. The producer is the
  natural owner of "what I emitted, for replay", and being built once in
  `startInteractiveTurnStreamV2` ([codebase/633.md](633.md)) makes the ring
  daemon-resident — satisfying AC-1 durability without injection.
- **The cross-goroutine sharing lives in the ring's own mutex, not the emitter.**
  The ring is a self-contained mutex-guarded object; the emitter keeps its
  documented single-`Run`-goroutine, no-mutex, unguarded-counter invariant
  ([codebase/632.md](632.md) § Concurrency model) **unchanged**. The Technical-Notes
  reconciliation ("adding a shared ring read by another goroutine must reconcile
  with the unguarded-counter invariant") is met by *fabric separation*: the emitter
  stays lock-free, the ring locks itself. `Append` runs only on the emitter's `Run`
  goroutine; `After` (a #647 caller, on the manager goroutine) is the only
  cross-goroutine reader, and it touches only the mutex-guarded ring.
- **Durable replay id ≠ per-conn envelope id.** A second monotonic counter was
  introduced rather than promoting `nextID`, because the replay key must be
  connection-independent and reconnect-stable while `nextID` is per-conn,
  per-envelope, and session-reset ([codebase/632.md](632.md) § Envelope-ID policy).
  Keeping them distinct preserved the single-writer envelope-id invariant and the
  `security-sensitive`-free scoping.
- **Deterministic safety net stays deterministic.** AC-2's bounded memory is a hard
  cap enforced in code (`evictOldest` before append), not an advisory rule; the
  AC-5 gap signal is a pure boolean classification, not a heuristic — both
  unit-pinned.

## Lessons learned

- **`After(convID, 0)` is not a generic "dump all retained" inspector once eviction
  has occurred.** A from-the-start query whose oldest retained id is `> 1`
  correctly reports `gap=true` — the consumer *would* miss the evicted prefix. Tests
  that want to read back the retained tail after eviction must query from the
  eviction boundary (`After(convID, oldestRetained-1)`), and use the `After(_, 0)`
  gap as a *separate* assertion that the hard bound dropped the prefix. This is
  exactly the AC-5 contract working as intended, not a quirk; both `TestAppend_AllControlHardBound`
  and `TestAppend_CapOne` lean on the distinction. (From the PR body's Lessons
  learned.)
- **A dead test helper trips CI's `staticcheck` U1000 gate.** The first push left an
  unused `eventTypes` helper in `ring_test.go`; `staticcheck ./...` (run by both
  `make check` and CI) failed it even though `go vet` and `go test -race` were
  green. Fix was to **remove** the helper, not wire it in — `TestAppend_EvictsDeltasFirst`
  already proves the AC-3 invariant arithmetically (appends exactly 5 control + 5
  delta, asserts `len == cap` and `retainedControl == 5`, forcing all four dropped
  events to be `assistant_delta`), so the helper added no coverage. Run the full
  `make check` (vet + `-race` + **staticcheck** + substrate-guard) before signalling
  green — a `-race`-green build can still be `staticcheck`-red.

## Out of scope (deferred)

- **Reconnect wiring** — per-conn `last_event_id` tracking, the on-reconnect `After`
  query, and the wire mechanism that surfaces the durable id (a new envelope field
  or a separate replay frame). **Sibling [#647]** (`security-sensitive`), which reads
  `emitter.ring` via the seam already reachable in `package main`. No accessor was
  added here (it would be speculative).
- **Disk persistence / surviving a full daemon-process restart.** Explicitly out of
  scope — the ring is purely in-memory (AC-2's bounded-memory framing depends on
  it). That boundary is #647's resync, driven by the AC-5 "you missed some" gap.
- **`MaxEventsPerConversation` calibration.** 1024 is a starting point, not
  load-tested (ADR 025 § Roadmap flags the droppable-delta policy as needing a real
  load test). Kept a named constant so tuning is one edit; #647's reconnect e2e or a
  Phase 2 load test is the right place to calibrate.
- **Global vs per-conversation counter.** #646 uses a per-conversation counter
  (matches AC-1 wording, fully partitions the ring). If #647's wire format prefers a
  global id, that's a #647 decision; `Append`-returns-id is unaffected.

## Testing

`internal/eventring/ring_test.go` (`package eventring`, stdlib `testing`, `-race`,
`t.Parallel()`): id assignment (strictly increasing from 1, independent per
conversation); replay (`After(A,2)` → `{3,4,5}`); caught-up (`After(A,5)`/`After(A,99)`
→ `(nil,false)`); gap when the oldest fell off (distinguishable from caught-up);
conversation isolation; unknown-conversation (`0` caught-up vs `>0` gap);
delta-first eviction (all control retained, total == cap); all-control hard bound
(oldest dropped, ascending preserved); no-fabricated-gap after a middle delta
eviction; cap-1 degenerate ring; `New(0)`/`New(-1)` panic; one append-vs-query
`-race` test.

`cmd/pyry/interactive_turn_v2_test.go` (additions only): ring records the emitted
sequence with ids 1..N (AC-1); id assigned once per logical event, not per conn
(two interactive conns → N ring entries, 2N pushes); append even with zero
interactive conns; no append on an empty cursor.

`make check` (vet + `-race` + staticcheck + substrate-guard) and `make build`
green. No e2e — the ring is a pure unit + an additive emitter behaviour; the
reconnect path that would need an oracle is #647. **Code review PASS** (one
staticcheck regression caught and fixed pre-merge — see Lessons learned).
