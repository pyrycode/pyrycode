# Ticket #741 — Resolve session→conversation binding in the `session_transition` producer and stamp `conversation_id`

The `session_transition` event ships capability-gated to `interactive` phones, but
its payload's `conversation_id` was always `""` ([#740](740.md) added the wire field;
the producer emitted the zero value), so `pyrycode-mobile#336` had no routing key for
the session-boundary marker. #741 is the **read side** of the binding: the producer
now resolves the transitioning session's owning conversation from the maintained
registry binding ([#739](739.md)) and stamps `conversation_id` onto every emitted
envelope — dropping the whole event fail-closed when the binding is unresolvable,
rather than leak an empty or guessed routing key.

Split from #738 (the third sibling: [#739](739.md) write-maintenance, [#740](740.md)
wire-field, #741 read-and-stamp). Spec: [`docs/specs/architecture/741-session-transition-conversation-id-resolve.md`](../../specs/architecture/741-session-transition-conversation-id-resolve.md).
Feature docs updated: [conversation-session-binding.md](../features/conversation-session-binding.md) (§ *Reading the binding to stamp `conversation_id`*), [protocol-package.md](../features/protocol-package.md) (the `SessionTransitionPayload.ConversationID` note — now populated, no longer transiently `""`).
Closes #741; unblocks `pyrycode-mobile#336`.

## Implementation

Two production files, **no new types, no new files**, ~50 production LOC + ~120 test
LOC (`cmd/pyry/relay.go`, `cmd/pyry/session_transition_v2.go`). All single-package
`cmd/pyry`.

- **`conversationForSession(convReg *conversations.Registry, sid string) (string, bool)`** (new, `relay.go`) — the duplicated read scan, the counterpart to #739's `RebindSession` write. Empty `sid` → `("", false)` immediately (mirrors `RebindSession`'s empty-`oldID` guard — an unbound conversation carries `CurrentSessionID == ""` and must never be matched by a stray empty lookup). Otherwise scan `convReg.List()`; a conversation matches iff `c.CurrentSessionID == sid` **or** `slices.Contains(c.SessionHistory, sid)`; first match returns `string(c.ID), true`; no match → `("", false)`. Added `slices` to the import block. The registry exposes **no** by-session-id read method, so the scan is duplicated cmd-side rather than extracted (PROJECT-MEMORY "Resist over-DRY on duplicated registry primitives") — this producer is its only consumer.

- **`sessionTransitionEmitterV2`** (modify, `session_transition_v2.go`) — gains one field `resolveConv func(string) (string, bool)`, set by `newSessionTransitionEmitterV2(bcast, resolveConv, logger)` (new parameter inserted **before** `logger`, keeping logger last per the package idiom). The closure carries the registry, so `session_transition_v2.go` never imports `internal/conversations`.

- **`broadcast`** (modify) — resolve + stamp + drop, inserted between `toWirePayload` and the marshal. After the existing unknown-reason drop: `convID, ok := e.resolveConv(payload.NewSessionID)`; on `!ok` log at Debug (`event: "session_transition.unresolved_conversation"`, `reason` only — no ids) and `return` (whole-event drop, `Run` survives); else `payload.ConversationID = convID` and fan out unchanged. The resolve + stamp happen **once per transition**, before the per-conn loop, so the same `conversation_id` reaches every conn.

- **`startSessionTransitionStreamV2`** (modify) — gains the `resolveConv` parameter, forwarded to the constructor. The caller at `relay.go` passes `func(sid string) (string, bool) { return conversationForSession(convReg, sid) }` — built **at the call site** inside `startRelayV2`, where `convReg` is already an in-scope parameter.

- **SECURITY comment block** (`session_transition_v2.go:38`) updated: the stale assertion that the payload "carries no application content" is replaced with the record that `conversation_id` is a routing key treated as sensitive alongside session ids and `workspace_cwd` — resolved from the maintained binding, stamped on the wire, and never logged (the unresolved-drop line carries `reason` only).

### Why `payload.NewSessionID`, branch-free

`payload.NewSessionID` is the *live* binding id for **both** reasons, so one field resolves both: for `clear` it is `t.NewID` (== `CurrentSessionID` after #739's rebind-before-fan-out); for `idle_evict` `t.NewID` is empty, so `toWirePayload` mirrors the evicted `t.PreviousID` onto both wire id fields (the retained binding id — eviction is binding-neutral). Resolving by this single field matches the wire semantic "`conversation_id` names the conversation owning `new_session_id`" and survives a hypothetical future `SessionHistory` cap (it uses the freshest id). The `SessionHistory` half of the scan covers the async-drain double-rotation race (a second rotation advances `CurrentSessionID` past this transition's `NewID` before `Run` drains it → `NewID` is then in history, still resolving to the same conversation).

### Why no `main.go` threading (the spec's XS downgrade)

PO sized **S** with an explicit "architect may downgrade to XS if the threading is thinner than estimated." It was thinner. The ticket described mirroring `boundHostFunc`'s `main.go → startRelay → startRelayV2 → …` thread, but `boundHost` needs the supervisor+pool (built in `runSupervisor`) whereas this resolver needs **only** `convReg`, which is **already** a parameter of both `startRelay` and `startRelayV2`. So the closure is constructed at the `startSessionTransitionStreamV2` call site — **zero** `main.go`/`startRelay` edits, only `startSessionTransitionStreamV2` and `newSessionTransitionEmitterV2` gain a parameter. Final: 2 production files (under the §4 ≥5-file gate), 0 new exported types.

## Patterns established

- **Inject a resolver closure to keep a pure mapping seam registry-free.** `toWirePayload` stays a pure `SessionTransition → SessionTransitionPayload` mapping; the registry read lives behind an injected `func(string) (string, bool)` set at construction. The producer file sees only the `func` type and never imports `internal/conversations`. This is about *purity*, not an import cycle — `cmd/pyry` is package `main` and may import anything; the genuine cycle constraint is on `internal/sessions`, which is why `toWirePayload` already lives cmd-side. Same shape as the `boundHostFunc` resolver (`interactive_turn_stream_v2.go`).
- **Build the closure at the deepest call site where its capture is already in scope.** When the captured dependency (`convReg`) is *already* a parameter of the function that makes the consuming call, construct the closure there — don't thread a new parameter down from `main.go` just to mirror a sibling's threading shape. This is what collapsed the estimate from S to XS; the `boundHost` mirror only needed `main.go` threading because *its* capture (supervisor+pool) isn't in scope at the relay layer.
- **Fail-closed on an unresolvable outbound routing key.** A missing binding (race: session torn down before drain) drops the **whole event** — no conn receives a boundary with a guessed or empty `conversation_id` — rather than emit-with-empty. Identical whole-event-drop shape to the existing unknown-reason drop; `Run` survives to the next transition. For an internet-facing dispatch decision a mis-resolution is a cross-conversation confidentiality leak, so the producer never guesses. Mirrors the reply-stream resolver's "retries-never-falls-back" confidentiality discipline (`resolveTarget`, `interactive_turn_stream_v2.go`).
- **Resolve once per event, before the per-conn fan-out.** The `conversation_id` is a per-transition value, not a per-conn one, so resolve + stamp happen once before the loop; the per-conn `Push` resilience (a dropped conn must not abort the others) is untouched.
- **Duplicate the registry scan cmd-side rather than over-DRY.** #739 owns the *write* (`RebindSession`) and exposes no reusable read scan; #741's read scan is its only consumer, so the duplication is tolerated until a third consumer forces an extraction (PROJECT-MEMORY "Resist over-DRY on duplicated registry primitives"). The empty-`sid` guard is duplicated too — it defends the read path's own stray-empty-lookup catastrophe (matching the first unbound conversation), the mirror of `RebindSession`'s empty-`oldID` guard.

## Security (security-sensitive — verdict PASS)

The outbound dispatch decision — what `conversation_id` to stamp on a phone-bound frame — is the trust boundary; a mis-resolution is a cross-conversation confidentiality leak. It is explicit and single-point: `resolveConv(payload.NewSessionID)` in `broadcast`, returning a value from the authoritative `conversations.Registry` binding, never guessed/defaulted/caller-supplied. No untrusted input reaches the resolver — `sid` comes from the pool's own `SessionTransition` via `toWirePayload`, not the network. The single-owner invariant (a session id binds exactly one conversation for life) guarantees the scan cannot resolve to a *different* conversation; the **fail-closed drop** handles the unresolvable race (deleted/torn-down session) with no envelope rather than a guessed/empty key; the empty-`sid` guard prevents matching an unbound conversation. AC#5 enforced: `conversation_id` is never logged — the new unresolved-drop line carries `event` + `reason` only, and the stale SECURITY comment was updated to record the field as sensitive. Concurrency: no new locks or goroutines; the `List()` snapshot is race-safe against concurrent `RebindSession` appends (append writes at/above `len`, or reallocates — neither overlaps the captured `[0,len)` read). Spec security review (self-review per `architect/security-review.md`, 2026-06-23): **PASS**, no MUST-FIX.

## Lessons learned

- **An estimate that mirrors a sibling's threading can over-count when the captured dependency is already in scope deeper down.** The ticket sized the work around `boundHostFunc`'s full `main.go → startRelay → startRelayV2` thread, but `convReg` was already a `startRelayV2` parameter, so the closure built at the call site needed **zero** `main.go`/`startRelay` edits — collapsing S → XS. When a spec says "mirror sibling X's threading," check whether *this* closure's capture is in scope at a shallower layer than X's before assuming X's parameter count.
- **QA `go test -race ./...` flakes were pre-existing, not regressions.** The full run hit two reds — `internal/sessions` (PTY-bind timing) and `internal/transport` (`wssclient_test.go`, the open flake [[internal-transport-wssclient-timing-flake]] tracked by #291) — both outside this diff (a pure `cmd/pyry` change with no causal path to either package) and both passing in isolation at `-count=1`. Documented in the PR body, no rework routed. See also [[realclaude-sigterm-test-is-timing-flake]].
- **The code-review auto-commit touched only an unrelated e2e fixture.** Commit `24c04cf` ("auto-commit uncommitted changes for #741") re-stamped `internal/e2e/realclaude/testdata/permission_protocol_v2.1.158.json` (random `session_id` / `cwd` from a realclaude test run) — incidental to #741's logic, not a documentation concern.
