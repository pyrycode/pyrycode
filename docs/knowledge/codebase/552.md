# Ticket #552 — agent-run: TUI session flight-recorder behind `PYRY_RECORD_DIR`

Opt-in flight recorder for the interactive-TUI claude session driven by
[`internal/agentrun/ptyrunner`](../features/ptyrunner-package.md). When the operator sets
`PYRY_RECORD_DIR`, mirror every PTY byte of the session to an asciinema-v2 `.cast` file so a
failed run can be replayed (`asciinema play`) or diffed offline — the control channel (the
rendered PTY screen) is otherwise never persisted, and nearly every ptyrunner failure lives in
that un-persisted channel. **OFF by default — byte-identical to today when the env var is
unset/empty.** Pyrycode-side consumer wiring of the `tuidriver.NewCastRecorder` API published by
tui-driver#125; library-first, no edit-and-cp. Unblocked by [#553](553.md) (chip-gated repaste —
landed in the same file) so the same-files conflict risk is gone.

## Implementation

Two-helper-and-a-block addition, all in `internal/agentrun/ptyrunner/runner.go`; the rest of the
diff is test surface (`.gitignore` + `go.mod`/`go.sum` are one-liners).

- **Env read lives inside `Run`, not a `Config` field.** `Run` reads `os.Getenv("PYRY_RECORD_DIR")`
  directly — the first `os.Getenv` in the package. Rationale (decided, not to be re-litigated): the
  env knob is read *next to the seam it gates* (the `SpawnOpts.Mirror` seam lives in `Run`), it has
  the smallest blast radius (one production file + one test file), and the recorder is an internal
  behaviour of `Run`, not a caller-supplied input, so no new public API and `cmd/pyry/agent_run.go`
  is untouched. This mirrors the `TUIDRIVER_*` env-driven pattern. (`PYRY_CLAUDE_BIN` /
  `PYRY_USE_STREAMJSON` are read in `cmd` because they substitute a flag value / select the runner
  path — neither shape applies here.)
- **No baked-in default directory.** The env var unifies switch + location: set → ON and write
  there; unset/empty → OFF. `~/.local/share/pyry-recordings/` is *documented operator guidance*
  (sibling of `~/.local/share/pyry-artifacts/`, outside Obsidian Sync / Time Machine reach), never
  a fallback the code falls back to.
- **`Run` signature changed to a named return** `func Run(ctx, cfg) (err error)` so the finalize
  defer reads the run's *final* result (`-ok` / `-err`) when it fires, not nil at registration.
  Non-invasive: every existing `return X` still assigns and returns; no existing defer assigns to
  `err`; transparent to callers (a named return doesn't change the function type).
- **Control-flow block** inserted after `tuidriver.EnsureClaudeEnv(cmd)` and **before**
  `tuidriver.Spawn(...)`; `mirror` defaults to a pure nil `io.Writer` so `SpawnOpts{Mirror: nil}` is
  byte-for-byte equivalent to the pre-#552 `SpawnOpts{}` when `dir == ""` (AC #1 holds by
  construction — no typed-nil trap):
  1. `os.MkdirAll(dir, 0o700)` — **fail-fast**.
  2. `pruneOldRecordings(dir, logger)` — **best-effort** (see below).
  3. `createRecordingFile(dir, cfg.SessionID)` — **fail-fast**.
  4. **register the finalize defer NOW** as a wrapping closure (before `defer sess.Close()` below).
  5. `tuidriver.NewCastRecorder(f, int(DefaultPtyCols), int(DefaultPtyRows))` + `rec.WriteHeader()`
     — **fail-fast**.
  6. `mirror = rec`.
- **cols/rows source = `tuidriver.DefaultPtyCols` / `DefaultPtyRows` (120 × 40).** `StartPTY` sets
  every PTY tui-driver spawns to exactly those, and ptyrunner spawns with empty opts, so the live
  PTY *is* 120×40. Recording with the same exported constants makes the cast's dimensions match the
  bytes' origin and track any upstream change — no magic numbers. (Resolves the ticket's one open
  design point.)
- **Filenames.** Temp at create: `<UTC-stamp>-<sessionID>.cast`, where stamp =
  `time.Now().UTC().Format("20060102T150405Z")` (sortable, e.g. `20260530T182431Z`), opened
  `O_CREATE|O_EXCL|O_WRONLY, 0o600`. Session-identifiable *from creation*, so a crash / SIGKILL
  before the rename still leaves a session-tagged, replayable file. Final on close: the outcome is
  inserted **inside the stem** — `strings.TrimSuffix(tmpPath, ".cast") + "-" + outcome + ".cast"` →
  `<stem>-ok.cast` / `<stem>-err.cast`. The suffix goes before the extension so the file **stays
  `*.cast`** (required by both prune and `asciinema play`). This is the precise meaning of the AC's
  "only adds the suffix": annotate the stem, keep the extension.
- **Three unexported helpers + one const** (`runner.go`): `createRecordingFile`,
  `finalizeRecording`, `pruneOldRecordings`, `const recordingMaxAge = 7 * 24 * time.Hour`.
  `MkdirAll` + `WriteHeader` are inline (they read clearly in place).
- **`outcome` mapping:** nil run error → `"ok"`; non-nil → `"err"`. Maps the fake-claude modes
  exactly (`jsonl` → `-ok`; `trust` / `network_failure` → `-err`). Operator-shutdown ctx-cancel
  collapses to nil → `-ok`. On panic the defer still runs and renames to `-ok` (named return is
  nil) — no panic-specific branch (no observed panic failure mode to defend against).
- **Dependency / housekeeping deliverables:** `go.mod` bumped
  `v0.0.0-20260523181457-c2dcd1e49992` → **`v0.0.0-20260531143940-6bec180ad34c`** (the
  pseudo-version that publishes `NewCastRecorder`); `*.cast` added to `.gitignore` (belt-and-
  suspenders — the production default is outside the repo, but an operator pointing
  `PYRY_RECORD_DIR` inside the tree never commits recordings); flag-site `SECURITY:` block comment
  warns that enabling writes **full session content (prompt + claude output + ALL tool output, incl.
  file contents and possible secrets)** to disk.

### Defer placement — the load-bearing invariant

The finalize defer is the **strict tail** of `Run`'s documented LIFO cleanup chain:

```
cancel() → wg.Wait() → counter.Stop() → emitter.Close() → sess.Close() → finalizeRecording()
```

Registering it *before* the existing `defer sess.Close()` (which it is, since the recorder block
sits before `Spawn`) makes it run **last**. That matters: `finalizeRecording` must close + rename
the file only **after** tui-driver's PTY reader goroutine — the *sole* `Mirror` writer — has
stopped. `sess.Close()` guarantees that: it closes the PTY then blocks on `<-s.readerDone`
(happens-before). So the close + rename can never race a mirror write. Verified under `go test
-race`. The `Run` doc-comment's Cleanup-ordering block was updated to append `finalizeRecording()`
and one sentence on why.

### Fail-fast vs best-effort

- **Setup fails fast** (`MkdirAll` / `createRecordingFile` / `WriteHeader`) → returns a wrapped
  `ptyrunner: <op>: %w` error before `Spawn`. The operator opted in explicitly; silently continuing
  without a recording would betray the opt-in, and nothing is lost (no session has started). If
  `WriteHeader` fails, the already-registered finalize defer closes + renames the partial file to
  `-err` — no leaked temp.
- **Prune is best-effort.** `pruneOldRecordings` globs `filepath.Join(dir, "*.cast")`, stats each
  hit, and `os.Remove`s those with `ModTime().Before(now - recordingMaxAge)`. Glob/stat/remove
  errors are Warn-logged (path + err only) and never abort the run — a stale-cleanup hiccup must not
  block recording a fresh session. The fresh file's mtime is `now`, far inside the window, so it
  survives its own run's prune.
- **Rename is best-effort.** A failed rename leaves the session-tagged temp in place — still a
  valid, replayable cast — Warn-logged with the path only.

## Patterns established

- **Env knob read next to the seam it gates.** When a debug/behavioural knob gates one specific
  seam, read its env var right beside that seam rather than threading a `Config` field down from
  `cmd`. Justified here because ptyrunner is an application-internal package (not a reusable
  library) and the knob is cohesive with the `Mirror` seam. Contrast `PYRY_CLAUDE_BIN` /
  `PYRY_USE_STREAMJSON`, read in `cmd` because they substitute a flag value / select a runner path.
- **The consumer owns the recorder's file lifecycle; the library owns none.** `NewCastRecorder`
  takes an `io.Writer` and frames bytes; closing/renaming the underlying `*os.File` is the
  consumer's job. The idiom: a **wrapping-closure `defer` registered before `defer sess.Close()`**
  so LIFO runs it after the sole writer-goroutine has joined — the close/rename is then race-free
  without any new lock.
- **OFF-by-default opt-in via one env var carrying both switch and location.** Set → ON + write
  there; unset/empty → OFF. No baked-in default directory (which would risk ambient recording of
  sensitive content). `mirror` defaults to a pure nil `io.Writer` so the disabled path is provably
  byte-identical.
- **Session-identifiable temp filename from creation; clean close only appends an outcome suffix
  inside the stem.** A crash before the rename still leaves a session-tagged, replayable file; the
  rename keeps the `.cast` extension so prune + replay still work.
- **Fail-fast on opt-in setup, best-effort on housekeeping.** Setup failures (dir/create/header) of
  an explicitly-requested artifact surface loudly before any work starts; cross-run housekeeping
  (prune) and cleanup (rename) must never corrupt the primary job of driving claude.

## Lessons learned

- **Named return + wrapping-closure defer is the only correct shape for an outcome-tagged
  finalizer.** `func Run(...) (err error)` + `defer func() { finalizeRecording(f, tmpPath, err,
  logger) }()` reads `err`'s final value at fire time. A bare `defer finalizeRecording(f, tmpPath,
  err, logger)` would capture the *nil* `err` at registration and mis-tag every run `-ok`. Classic
  Go gotcha; the close+rename must be wrapped *and* the function must name its return. (Lifted from
  PR #557 Lessons.)
- **`CastRecorder.Write` writes straight to the file (no internal buffer), so the bare `f.Close()`
  in finalize cannot drop the tail.** Confirmed by code review — the data is already flushed by the
  reader goroutine before `sess.Close()` returns, so a best-effort (ignored-error) close is safe.

## Testing

Four tests added to `runner_test.go` (the single edited test file), reusing `helperRunCfg` + the
existing fake-claude modes. All drive `os.Getenv("PYRY_RECORD_DIR")` via `t.Setenv`, so they **must
not** call `t.Parallel()` — Go runs serial tests to completion before the parallel batch, so an
env-setting serial test never overlaps a parallel one. The `.cast` is always non-empty: every
helper mode writes `❯ ` to the PTY → ≥1 mirrored `"o"` event after the v2 header.

- `TestRun_RecordOff_NoFileCreated` (AC #1) — `PYRY_RECORD_DIR=""`; asserts nil return, the success
  trailer is present (run unaffected), and `filepath.Walk` finds zero `.cast` under the home +
  workdir temp dirs.
- `TestRun_RecordOn_CreatesOkTaggedValidV2` (AC #2 + AC #3-ok) — exactly one `.cast` named
  `^\d{8}T\d{6}Z-<sid>-ok\.cast$`, mode `0600`, non-empty, parses as asciinema v2 (header
  `version == 2`, width 120 / height 40; ≥1 event line is a 3-element `[]any{float64, "o", string}`).
- `TestRun_RecordOn_ErrTagged` (AC #3-err) — table-driven over modes `network_failure` /
  `trust`; asserts `errors.Is(err, ErrNetworkFailure)` / `ErrTrustModalDetected` and the single
  recording ends `-err.cast`.
- `TestRun_RecordPrune_Scoped` (AC #4) — seeds `old.cast` (`os.Chtimes` to now − 8d), `keep.cast`
  (now), and a stale `decoy.log`; after a run, `old.cast` is gone, `keep.cast` + `decoy.log` remain
  untouched, and the run's own `-ok.cast` survives prune.

Green (run in the documentation worktree):

```
go build ./...                              # clean
go vet ./internal/agentrun/ptyrunner/       # clean
go test ./internal/agentrun/ptyrunner/
ok  github.com/pyrycode/pyrycode/internal/agentrun/ptyrunner  4.094s
```

The recording tests exercise the reader-goroutine ↔ finalize handoff and are race-clean under
`-race` (per the spec's requirement and the code-review's confirmation).

  doc-comment's logging paragraph (`runner.go:19-27`) was **not** updated to add a one-line
  recording-artifact carve-out (the spec's AC #5 asked to mirror it there). This is a documentation
  defect *inside the code comment*, not a behavioural one — the actual no-content-logging discipline
  *is* preserved (the new Warn lines carry only path + err, never recording content). Code review
  rated it below the FAIL threshold and recommended fixing it "in a follow-up or before final human
  merge." Recorded here so the dangling pointer isn't lost; it's a one-line code comment fix, out of
  the documentation phase's lane.
- **Second `Mirror` consumer.** If a future ticket adds another `Mirror` use, wrap with
  `io.MultiWriter`. Not needed now — the recorder is the sole consumer.
- **Install / rebuild.** Handed to the operator (inode-cache gotcha + the ~2026-06-14 billing
  deadline). Rollback is trivial: the flag is off by default, so an unset `PYRY_RECORD_DIR` is
  already the safe state.
