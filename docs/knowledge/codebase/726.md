# Ticket #726 ‚Äî safe-answer seam: abstract modal choice ‚Üí tui-driver keystroke

The **inbound actuator** foundation for the daemon-side modal bridge (EPIC #597
Phase 3, [ADR 025](../decisions/025-mobile-remote-head-interactive-session.md)).
tui-driver v1.3.0 (vendored via #619) ships the safe-answer keystroke primitives
(`Session.AcceptTrust()` ‚Üí `"1\r"`, `Session.Answer(choice)` ‚Üí `choice + "\r"`,
`Session.SendEsc()` ‚Üí `0x1b`), but no pyrycode code called them. #726 adds the
supervisor-side wrapper that captures the live `Session` and drives those
keystrokes, so higher layers can resolve a modal **without importing tui-driver
or reaching into the child PTY directly**.

The primitive carries **no trust decision** ‚Äî it routes whatever keystroke it is
told to. Authorization (per-device gate, nonce validation, idempotency,
`option_id` ‚Üí choice mapping) lives entirely in the consumers: #717 (gated
`modal_answer` ‚Üí `Answer`/`AcceptTrust`), #727 (interception/cancel ‚Üí `SendEsc`),
#725 (deny-on-timeout ‚Üí the deny keystroke). Peeled from #717's 3-way split.

This is the **inbound** counterpart to the outbound half documented in
[features/modalbridge-package.md](../features/modalbridge-package.md) (#716): that
slice turns a detected modal into a `modal_shown` event pushed to the phone; #726
is the verb the phone's answer eventually actuates back against claude's PTY.

## Implementation

One new file + two small edits to `supervisor.go`. **Purely additive** ‚Äî no
existing call site changes, 0 consumer sites updated. ~113 prod + ~144 test LOC.

- **`internal/supervisor/modal.go` (new).**
  - `type modalKey int` ‚Äî **unexported** (`keyAcceptTrust`/`keyAnswer`/`keyEsc`,
    iota). The exported surface is the three verb methods, *not* the enum. Its
    `String()` ("accept trust" / "answer" / "send esc") feeds the error-wrap
    prefix.
  - Three thin methods on `*Supervisor`: `AcceptTrust() error`,
    `Answer(choice string) error`, `SendEsc() error` ‚Äî each a one-liner
    `return s.sendModalKey(keyXxx, choice)`. The verb names are one level up from
    tui-driver's concrete methods and are pinned by #717's body, so no
    consumer-side rename is ever needed.
  - `sendModalKey(k modalKey, choice string) error` (helper) ‚Äî the
    **capture-then-release** block: lock `sessMu`, copy `s.sess`, unlock, then
    actuate on the captured pointer. No lock held across the PTY write. A nil
    captured session ‚Üí `fmt.Errorf("supervisor: %s: %w", k, ErrNoLiveSession)`
    and `keystrokeFn` is **never invoked** (writes nothing, AC-3); a keystroke
    error is wrapped with the same per-verb prefix, underlying error preserved
    for `errors.Is`.
  - `sendModalKeystroke(sess, k, choice) error` ‚Äî the production `keystrokeFn`:
    a `switch modalKey` to the matching `sess.AcceptTrust()`/`Answer(choice)`/
    `SendEsc()`. The `default` (unreachable from the exported API) **returns a
    loud error, not a panic** ‚Äî per CODING-STYLE, panic is reserved for genuinely
    unreachable code; a bad key is a defensively-returned programmer bug.
- **`internal/supervisor/supervisor.go` (modified, +9).**
  - New `Supervisor` struct field `keystrokeFn func(sess *tuidriver.Session, k modalKey, choice string) error`,
    immediately after the existing `deliverFn`. Documented as set-once in `New`,
    **immutable post-`New`** (read lock-free by the modal methods), overridden
    only in tests.
  - One wiring line in `New`: `s.keystrokeFn = sendModalKeystroke`, right after
    `s.deliverFn = s.deliverViaSession`.
- **No `context.Context`.** Unlike `WriteUserTurn` (which blocks for seconds on
  `WaitReady`+`DeliverPrompt`), a keystroke is a single non-blocking `pty.Write`
  with nothing to cancel. A ctx would advertise a cancellation contract the
  method cannot honor; it is deliberately absent and documented inline so it is
  not cargo-culted.
- **No new sentinel, no new exported type, no cursor/`convMu` touch.** Reuses
  `ErrNoLiveSession`. These are not user turns ‚Äî they carry no `conversation_id`,
  so `currentConvID`/cursor stamping is untouched. No relay, registry, gate,
  audit, broadcast, logging, or `WaitReady` gate.

### Tests (`internal/supervisor/modal_test.go`, new)

In-package, table-driven, `t.Parallel()`, built via `New(helperConfig("exit0"))`
+ `setSession(&tuidriver.Session{})` + `keystrokeFn` override (the
`supervisor_test.go` `deliverFn`-seam template).

- **Verb dispatch (AC-1/AC-4 "correct call fires").** Each of `AcceptTrust()` /
  `Answer("2")` / `SendEsc()` delegates the right `(modalKey, choice)` to the
  seam (`keyAcceptTrust`/`""`, `keyAnswer`/`"2"`, `keyEsc`/`""`) and returns nil.
- **No live session (AC-3/AC-4 "sends nothing").** No `setSession`: each verb
  returns `errors.Is(err, ErrNoLiveSession)` with the per-verb wrap prefix and
  the seam is **never called** (asserted via a `called` flag).
- **Keystroke-error wrap.** Seam returns sentinel `boom` ‚Üí verb wraps with the
  prefix, `errors.Is(err, boom)` holds. Mirrors
  `TestSupervisor_WriteUserTurn_DeliverErrorFailsLoud`.
- **Unknown-key guard.** `sendModalKeystroke(&tuidriver.Session{}, modalKey(99), "")`
  returns an error without panicking (the `default` branch never touches the
  nil-PTY session).

## Patterns established

- **`keystrokeFn` is the `deliverFn` injection seam, copied one-for-one.** The
  real tui-driver keystroke methods nil-deref the PTY on a zero-value
  `&tuidriver.Session{}`, so verb dispatch could not otherwise be unit-tested
  without a live claude. The unexported, set-once fn-field lets a test record
  `(modalKey, choice)` and assert the abstract dispatch ‚Äî the identical
  testability boundary as `deliverViaSession`, whose real-claude path is only
  exercised by the heavier real-spawn test. **This boundary is stated, not
  closed:** the one-line `modalKey ‚Üí real tui-driver call` switch plus
  tui-driver's own `keys_test.go` byte assertions cover the rest; a real-spawn
  integration test is **not** required by the ACs and was not added.
- **Capture-then-release reused for a non-blocking actuator.** Same `sessMu`
  copy-then-unlock-before-write discipline as `WriteUserTurn`/`ScreenSnapshot`.
  The teardown race is already handled by tui-driver and matches `WriteUserTurn`'s
  documented contract: a captured `sess` racing a concurrent `setSession(nil)+Close`
  writes into tui-driver's teardown-safe PTY-error path (`writeRaw` ‚Üí `pty.Write`
  on a closed `*os.File` returns a non-nil error, never panics) ‚Äî so a torn-down
  session surfaces here as a **loud wrapped error, never a crash or a false
  success**. No new mutex, no new goroutine.
- **An abstract verb layer one level up from tui-driver.** Consumers act on a
  modal in pyrycode vocabulary (`AcceptTrust`/`Answer`/`SendEsc`) and never
  import `pkg/tuidriver` or touch the PTY. The trust decision stays in the gated
  consumer; the supervisor is a mechanical actuator.
- **No ctx on a non-blocking primitive.** Don't advertise a cancellation contract
  the method cannot honor just because a sibling (`WriteUserTurn`) has one.

## Lessons learned

- **The three verb methods take no `context.Context` ‚Äî and that divergence from
  `WriteUserTurn` is deliberate, not an omission.** A keystroke is a single
  non-blocking `pty.Write` with nothing to cancel; `WriteUserTurn` only carries a
  ctx because it blocks for seconds on `WaitReady`+`DeliverPrompt`. A ctx here
  would promise cancellation the method can't deliver, so it is left out and the
  reasoning is documented inline to stop it being cargo-culted from
  `WriteUserTurn`. (Lifted from the PR's *Lessons learned*.)

## Out of scope / incidental

- **Consumers (deferred):** #717 (gated `modal_answer`), #727 (interception/cancel),
  #725 (deny-on-timeout). `option_id` ‚Üí abstract-choice mapping is #717's, not here.
- **Not security-sensitive.** A mechanical keystroke actuator carrying no trust
  decision: it loosens no control and accepts no untrusted input directly (the
  gate lives in #717). The architect's spec confirms this.
