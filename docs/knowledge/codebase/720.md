# Ticket #720 — queued-backlog v2 wire types: `queue_state` + `dequeue_message` (SSOT)

The wire **vocabulary** for the queued-message backlog over the encrypted mobile
wire ([EPIC #597] Phase 3, [ADR 025]). A phone that types while `claude` is busy
has its turn buffered in `internal/msgqueue` (the engine — [#704], extended
[#719]). The phone needs to **see** that backlog and **cancel** an entry it no
longer wants. This slice adds the two v2 envelope types + their payload shapes so
the producer ([#722]) and handler ([#723]) build on one ADR-025-pinned contract
instead of each inventing queue shapes ad hoc:

- **View** — `queue_state` (daemon → phone) = `{conversation_id, queued:[{queued_msg_id, text, ts}]}`, the wire form of `msgqueue.Snapshot(convID)`.
- **Cancel** — `dequeue_message` (phone → daemon) = `{conversation_id, queued_msg_id}`, driving `msgqueue.Remove(convID, id)`.

**Wire shape only** — two type constants, three payload structs, the v2-partition
classification, three round-trip tests, and the `docs/protocol-mobile.md` SSOT.
**No runtime ships here.** No producer, no inbound handler, no live queue, no
fan-out, no policy. Same SSOT-first rhythm as [#656] (`session_transition` vocab)
→ [#657] (producer) and [#701] (modal vocab) → [#703]/[#706]/[#702] (runtime).

**`security-sensitive`: no** (label absent) — matching [#656], **not** [#701].
The contrast is the point: [#701] earned the label because `modal_id` is an
unguessable nonce and `modal_answer` opens a *gated* inbound control surface for a
high-consequence action. Here `queued_msg_id` is a **plain per-conversation
counter** from `internal/msgqueue` (not a security primitive), and **dequeuing is
ungated for any paired phone** (ADR 025 § Security model) — no nonce, no
per-device gate. The trust work — the `conversation_id` → authorized-conversation
binding (cross-conversation confidentiality), the capability-gated fan-out, and
the live inbound handler accepting an untrusted phone frame — all live in the
labelled siblings [#722]/[#723]/[#721].

## Implementation

Two production files, both additive. **2 new constants** (`TypeQueueState`,
`TypeDequeueMessage`), **3 new exported types** (`QueuedItem`,
`QueueStatePayload`, `DequeueMessagePayload`) — below the architect's `≤5
exported types` / `<5 files` S ceilings. **0 construction sites changed** —
nothing fans out, imports, or dispatches it yet.

| File | Change |

The three payload structs (all in `messaging.go`):

```go
type QueuedItem struct { // one element of QueueStatePayload.Queued
    QueuedMsgID uint64    `json:"queued_msg_id"` // plain per-conversation counter (≥1), NOT a nonce
    Text        string    `json:"text"`          // untrusted, phone-originated transit content
    TS          time.Time `json:"ts"`            // enqueue time, RFC3339Nano
}
type QueueStatePayload struct { // binary → phone; wire form of msgqueue.Snapshot(convID)
    ConversationID string       `json:"conversation_id"` // daemon's own resolved id (#722), never attacker-derived
    Queued         []QueuedItem `json:"queued"`          // ordered FIFO/enqueue order (the options []ModalOption precedent)
}
type DequeueMessagePayload struct { // phone → binary, inbound control (intercepted pre-dispatch.Route, no handler)
    ConversationID string `json:"conversation_id"` // untrusted phone input; #723 resolves to an authorized conversation
    QueuedMsgID    uint64 `json:"queued_msg_id"`   // the id to remove (msgqueue.Remove(convID, id))
}
```

Test files (excluded from the production count): `messaging_test.go` gains
`TestQueueStatePayload_RoundTrip` (asserts `len(Queued)==2` + positional
`QueuedMsgID`/`Text` + per-item `TS` via **`.Equal`**, then byte-equal
round-trip), `TestDequeueMessagePayload_RoundTrip`, and a table-driven
`TestDequeueMessagePayload_Malformed`; plus two single-line fixtures under
`testdata/` (`queue_state.json` with N=2 items, `dequeue_message.json`) authored
in **struct-field order** (`canonical()` compacts but does not sort keys).

## Patterns established

- **A vocab slice is labelled by its *shape*'s trust surface, not by "it's queue
  data."** [#701] (modal) carried `security-sensitive`; this mirror of it does
  **not**. The discriminators: (1) `queued_msg_id` is a plain monotonic counter,
  not an unguessable nonce like `modal_id` — there is no secrecy property to
  review; (2) dequeuing is **ungated** for any paired phone (ADR 025 § Security
  model), where answering a permission-class modal is gated. The new inbound
  control surface alone does not earn the label — `modal_answer` earned it because
  it injects a *gated, high-consequence* decision; `dequeue_message` only cancels
  the phone's own queued turn. The convID→authorized-conversation binding (the one
  real trust boundary) lives in the labelled handler [#723].
- **Queue backlog is daemon *state*, not part of claude's *turn stream* — so no
  `turnevent`/`turnbridge` neutral hop.** This is the architect's load-bearing
  mechanism decision (the original split body proposed routing neutral
  `QueueState`/`DropQueued` types through `turnevent` → `turnbridge.MapEvent`; the
  spec **declined** them). The producer ([#722], by analogy to [#657]) builds the
  `protocol.QueueStatePayload` directly from engine state (`msgqueue.Snapshot`),
  and the inbound frame ([#723], by analogy to [#703]'s `modal_answer`) is
  intercepted at `v2session.go`'s `dispatchAppFrame` and decoded to
  `protocol.DequeueMessagePayload` — **neither routes through `turnevent` or
  `turnbridge`**. That `turnevent → turnbridge.MapEvent` path is reserved for
  tui-driver turn-stream events (`assistant_delta`, `tool_use`, `turn_end`,
  `stall`, permission). The `session_transition`/`modal` precedent (state, not
  turn-stream), not the interactive-event precedent, is the one to follow.
- **`turnbridge` has no inbound wire-frame decoder.** The original AC's
  "`turnbridge` decodes `dequeue_message` → `DropQueued`, exhaustive decoder" does
  not match the code: `mapper.go` maps tui-driver `Event` → `turnevent.Event`;
  `outbound.go`'s `MapEvent` maps `turnevent.Event` → wire payload. Inbound v2
  control frames are intercepted at `dispatchAppFrame` (as `modal_answer` /
  `modal_cancel` are), not bridged. The inbound decode + handling is [#723]'s job.
- **If neutral `turnevent` types are ever warranted, they split across two
  opposite-direction sealed sums.** `turnevent` has **two** sums: outbound `Event`
  (`isTurnEvent`) and inbound `Inbound` (`isInbound`) — not one. The correct homes
  would be outbound `QueueState` → `Event` (like the internal-only `Stall`
  variant), inbound `DropQueued` → `Inbound` (the `turnevent/permission.go` package
  doc already names `Prompt`/`Cancel`/`DropQueued` as future `Inbound` members). Do
  **not** put `DropQueued` on the outbound `Event` sum. Out of scope here; recorded
  so a future producer slice does not re-derive it wrong.
- **The mixed inbound+outbound cluster lives in one const block with one rationale
  comment** — the `request_snapshot`/`screen_snapshot` and modal precedent.
  `TestTypeConstants_V1V2Partition` is grouping-independent, so block membership is
  pure readability; the deterministic gate is the partition test.
- **Empty-backlog `[]` vs `null` is a producer decision the leaf type cannot
  force.** `[]QueuedItem(nil)` marshals to `"queued":null`; a non-nil empty slice
  to `[]`. The contract here is only "`queued` is always present" (no `omitempty`);
  the docs **recommend** the producer (#722) emit `[]` so the mobile decoder keeps
  `queued` a plain array — but both round-trip, so the type does not encode an
  enforcement it cannot carry.

## Lessons learned

- **JSON `null` into a non-pointer numeric is a no-op (no error), so it is NOT a
  valid malformed test case.** The spec suggested a second malformed shape of
  `queued_msg_id: null` into the non-pointer `uint64` field. But Go's
  `encoding/json` treats JSON `null` into a non-pointer number as a **no-op** —
  the field keeps its zero value and `Unmarshal` returns **nil error**. That case
  would *not* reject, vacuously passing the "rejected cleanly" assertion in the
  wrong direction. The developer substituted a **negative number** (`-1`, out of
  `uint64` range) as the second malformed shape; both `queued_msg_id` as a JSON
  string and as a negative number reject cleanly with a non-nil error and no
  panic. General gotcha for any "malformed rejection" test against a stdlib
  `json.Unmarshal` into a non-pointer field: type-mismatch and range-overflow
  reject, but `null` is silently absorbed — assert against a shape that actually
  fails the target type. (Counters the spec's proposed-but-wrong case; the rest of
  the spec was a clean #656/#701-template scale-out.)
- **`internal/protocol` is gofmt-clean at HEAD** under local Go 1.26.2 — no
  `git show HEAD:<f> | gofmt -l` triage was needed for the touched files. The
  repo-wide dirtiness in [[pyrycode-gofmt-dirty-at-head-go1.26]] does not affect
  this package (as already noted on [#701]).
- **Code review left no MUST/SHOULD finding** that shaped the shape. The
  `code-review` auto-commit reshuffled an unrelated `testdata/` permission-protocol
  fixture (working-tree noise swept up by the safety net); it carries no queue
  content and is not part of this slice.

## Out of scope (deferred to siblings, all `security-sensitive`)

- **The `queue_state` producer — [#722]:** emit on queue change, build the
  `QueueStatePayload` from `msgqueue.Snapshot(convID)` with the daemon's own
  resolved `conversation_id`, capability-gated fan-out. Decides the empty-backlog
  `[]` vs `null` rendering (recommend `[]`).
- **The `dequeue_message` handler — [#723]:** add the `case
  protocol.TypeDequeueMessage` arm to `v2session.go`'s `dispatchAppFrame`, resolve
  the untrusted `conversation_id` to an authorized conversation, apply
  `msgqueue.Remove(convID, QueuedMsgID)`.
- **`send_message` queue routing — [#721]:** route a phone turn through the queue
  when claude is busy.
- **No neutral `turnevent`/`turnbridge` types** (deliberately declined — see
  Patterns established).
- **The mobile consumer** — the phone-side backlog render / cancel UI ([#429]),
  which must decode `queued_msg_id` as an integer (not a string).
