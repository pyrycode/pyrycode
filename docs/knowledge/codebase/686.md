# Ticket #686 — turn bridge resolves a conversation's transcript from its per-`Cwd` JSONL dir

The final slice of EPIC #672's per-conversation-workdir chain (`#684` primitive → [`#685`](685.md) spawn-in-validated-`Cwd` → **#686** per-`Cwd` bridge). [#685](685.md) made each conversation's session spawn in its own validated `Cwd`, so claude now writes that session's transcript under `~/.claude/projects/<encoded-cwd>/<session-id>.jsonl` instead of the daemon's single shared sessions dir. The outbound reply stream's by-id resolver ([#679](679.md)) still tailed the *shared* `claudeSessionsDir`, so a per-`Cwd` conversation's `<id>.jsonl` was never found there → its reply never streamed. #686 makes the resolver follow the transcript into the conversation's own directory, keyed by the bound session id, while a default (shared-workdir) conversation keeps resolving from the startup shared dir byte-for-byte unchanged.

The cross-conversation **confidentiality** property #679 established is preserved and strengthened: #679 keyed the *filename* off the bound server-minted session UUID; #686 keys the *directory* off the bound session's spawn workdir, closing the remaining axis. `security-sensitive` (cross-conversation confidentiality on the internet-exposed phone reply surface).

See the evergreen feature docs: [features/turnbridge-package.md § Which JSONL](../features/turnbridge-package.md#which-jsonl-and-surviving-clear-rotation) and [features/conversation-session-binding.md § Edge cases & limitations](../features/conversation-session-binding.md#edge-cases--limitations).

## Implementation

3 production `.go` files + 2 test files; +90/-31. No protocol change, no `internal/sessions`/`reconcile.go` change, no `turnbridge.SessionHost` interface change. The non-additive edit (`boundHostFunc`'s return arity) destructures at exactly one site.

- **`Supervisor.WorkDir()` accessor** (`internal/supervisor/supervisor.go`) — returns `s.cfg.WorkDir`, the working directory the supervised claude was spawned in. `cfg` is set once in `New` and never mutated (only `state` is mutex-guarded), so the read is **lock-free** — unlike `State()`, whose shape it otherwise mirrors. One production caller (`boundHost`). This is the authoritative, byte-exact spawn cwd: claude writes `<session-id>.jsonl` under `~/.claude/projects/<encoded-WorkDir>/`.
- **`perConversationSessionsDir(sessionWorkDir, bootstrapWorkDir, sharedDir) string` pure helper** (`cmd/pyry/interactive_turn_stream_v2.go`, beside `boundHostFunc`/`resolveTarget`). The discriminate-and-encode core, isolated so it is unit-testable without `runSupervisor`:
  - `sessionWorkDir == bootstrapWorkDir` → `sharedDir` (**AC3**: a default/null-`Cwd` session spawns in the bootstrap workdir, so it keeps resolving from the startup `claudeSessionsDir` byte-for-byte — *including* the latent abs-vs-realpath skew that dir already carries, see Open questions).
  - `sessionWorkDir == ""` → `sharedDir` (defensive default; a supervisor built with no `WorkDir` inherits the process cwd).
  - otherwise → `sessions.DefaultClaudeSessionsDir(sessionWorkDir)` (**AC1/AC2**), the single source of truth for the `~/.claude/projects/<encoded-cwd>/` encoding — **called, not hand-rolled**. Returns `""` only when it can't encode (no `$HOME`); the caller treats `""` as unresolvable.
- **`boundHostFunc` return arity grows by one** — `(host, sessionID, ok)` → `(host, sessionID, dir, ok)`. `boundHost` (`cmd/pyry/main.go`, built in `runSupervisor` capturing `trustedWorkdir` + `claudeSessionsDir` from scope) computes `dir := perConversationSessionsDir(sess.Supervisor().WorkDir(), trustedWorkdir, claudeSessionsDir)` after the existing `Get`/guard/`Lookup`; an underivable `dir == ""` is a **miss** (`ok=false`), never a silent fallback. All existing miss paths gain the extra `""` dir return.
- **`resolveTarget` feeds the returned dir to the bound branch** (`cmd/pyry/interactive_turn_stream_v2.go`). The bootstrap branch (`convID == ""`) keeps its `dir` param (= `claudeSessionsDir`) for `resolveLatestSessionJSONL` (**AC4**, unchanged). The bound branch now passes the per-conversation `convDir` to `resolveBoundSessionJSONL(convDir, sessionID)` instead of the shared dir param. `resolveBoundSessionJSONL` itself is **unchanged** — it already takes `(dir, sessionID)` and tails `<sessionID>.jsonl` under `dir`, validating the stem against `jsonlStemPattern` before `filepath.Join`.

### Design decision — derive `dir` from spawn `WorkDir` (B), not re-canonicalise `conv.Cwd` (A)

The architect chose **(B) read the session's captured `supervisor.Config.WorkDir`** over **(A) re-run `confineWorkdirToHome(conv.Cwd)` at resolve time**:

- **Byte-exact single source of truth.** `WorkDir()` is the *exact* string claude spawned with (`buildSession`, `pool.go`), so `DefaultClaudeSessionsDir(WorkDir())` is byte-identical to where claude writes — for both per-`Cwd` and default sessions. (A) re-runs `Abs`+`EvalSymlinks` at resolve time, which can drift from the spawn-time realpath if a symlink changed in between (a TOCTOU re-resolution).
- **Keeps #685's invariant intact.** #685 established that the recorded `conv.Cwd` "drives nothing downstream — respawns read `supervisor.Config.WorkDir` … not re-derived from the row" ([codebase/685.md](685.md) § Patterns established). (B) extends the *same* rule to the bridge; (A) would resurrect the raw row `Cwd` as a resolve-time path input, reversing it.
- **No re-validation on the hot path.** (B) reads an immutable string; (A) re-runs `EvalSymlinks` syscalls on every (re)subscription.

### Tests

- `TestSupervisor_WorkDir` (`internal/supervisor/supervisor_test.go`) — accessor returns `Config.WorkDir` verbatim; empty config → `""`.
- `TestPerConversationSessionsDir` (`cmd/pyry/interactive_turn_stream_v2_test.go`) — pure table test (no `t.Setenv`): default (== bootstrap) → shared; empty → shared; per-`Cwd` → `DefaultClaudeSessionsDir(W)`; plus a two-distinct-`Cwd` isolation assertion (two workdirs → two distinct dirs, neither the shared dir — AC1/AC2 at the derivation layer).
- `TestResolveTarget_BoundSessionWhenRouted` extended — `boundHost` now returns a *distinct* per-conv `convDir`; a **same-stem decoy** `<uuidA>.jsonl` is planted in the bootstrap `dir` param, so a regressed bound branch that tailed the param instead of `convDir` would resolve to the decoy. Asserts the resolved path is under `convDir` (AC1) and not the newer sibling in the same dir (AC2). The three `boundHost` fakes gained the `dir` return.

## Patterns established

- **Read the captured spawn `WorkDir`, never re-derive a path from the raw row.** The bridge consumes `supervisor.Config.WorkDir` — the realpath already confined-to-`$HOME` + trust-marked by #685's `resolveSpawnDir` at spawn time — keeping #685's `resolveSpawnDir` the **single validation boundary** and the raw phone-supplied `conv.Cwd` inert. This is the security-relevant call: design (A) would have re-crossed the untrusted→trusted boundary at resolve time.
- **Confidentiality is layered: filename (UUID) × directory (spawn workdir).** #679 made the *filename* the confidentiality boundary (`<bound-sessionID>.jsonl`, a server-minted UUID); #686 makes the *directory* a second axis. A misderived directory fails **closed** — `os.Stat` misses → retryable error → no stream — and can never surface another conversation's content, because no other conversation's transcript shares this session's UUID filename. The change is fail-safe by construction.
- **`DefaultClaudeSessionsDir` is the sole encoder; `Supervisor.WorkDir()` is the sole spawn-cwd source.** Don't hand-roll the `/`+`.`→`-` cwd-encoding rule and don't re-canonicalise — reuse the exported encoder and read the immutable captured workdir. (Verified by code-review: the `WorkDir()` read needs no lock; the blast radius of the arity change is one destructure site.)

## Lessons learned

- **Deriving the per-`Cwd` dir from `supervisor.Config.WorkDir` (the captured realpath) rather than re-canonicalising `conv.Cwd` at resolve time keeps the raw phone-supplied row inert and avoids a resolve-time `EvalSymlinks` TOCTOU re-resolution** — the same single-validation-boundary posture #685 established. (Lifted from the PR's Lessons learned.)
- **`AC4`'s "a `Cwd` that no longer resolves yields a retryable error, never a fallback" is enforced at the `Stat` layer, not a derivation re-check.** Under design (B) the dir is derived from an always-present captured string, so "no longer resolves" manifests as the *directory being absent on disk* → `resolveBoundSessionJSONL`'s `os.Stat` miss → retryable error, no fallback. The dir-derivation itself cannot fall back: every non-default branch goes through `DefaultClaudeSessionsDir`, and a non-derivable dir is a hard miss. The intent (deleted `Cwd` → retry, never cross-stream) holds.
- **An empty `sharedDir` is reachable-in-principle and handled fail-closed (code-review NIT).** If `resolveClaudeSessionsDir` couldn't resolve `$HOME` at startup, `sharedDir == ""`; `perConversationSessionsDir` returns it → `boundHost` sees `dir == "" → ok=false` → retry, no fallback. Correct behaviour, no change needed.

## Out of scope (deferred)

- **Latent abs-vs-realpath skew on the *default* path.** The startup `claudeSessionsDir = DefaultClaudeSessionsDir(filepath.Abs(*workdir))` uses `Abs` (no symlink resolution), while the bootstrap session spawns in `trustedWorkdir = realpath(*workdir)`. On a symlinked `-pyry-workdir` these differ, so the default/bootstrap stream already looks in the wrong dir today. #686 **deliberately preserves** that (AC3 "unchanged") and **does not inherit** it on the per-`Cwd` path (which uses `DefaultClaudeSessionsDir(realpath)` end-to-end). Canonicalising `resolveClaudeSessionsDir` to the realpath is a separate bootstrap-path ticket.
- **Recorded `conv.Cwd` stays inert** — neither read nor written by this slice, consistent with #685. A future consumer needing the row to equal claude's actual cwd is #685 Open-Question 1's deferred seam.
- **Two-conversation / multi-operator structured e2e confidentiality** — still infeasible without a per-conversation fakeclaude harness (lesson [[architect-structured-stream-e2e-harness-gap]]); pinned by the deterministic by-id + per-`Cwd`-decoy unit tests, as #679/#685 were. Multi-operator cross-conversation isolation (two phones each viewing a different conversation) remains deferred — the structured stream fans out to all interactive conns by capability with no connection→conversation binding for output.
