# Ticket #582 — retire the binary↔relay `hello`/`hello_ack` handshake (treat WS upgrade as established)

The binary's outbound relay connection (`internal/relay/connection.go`) ran a v1
ceremony on every fresh conn: it sent a `hello` advertising
`ProtocolVersions: ["v1"]` and blocked up to 5s for a wrapped `hello_ack`. The
production relay (`pyrycode-relay`) is **content-blind** under v2 — it registers
the binary's server-id from the `x-pyrycode-server` request header and claims the
slot on WS upgrade, and **never sends a `hello_ack`** (under v2 a `hello_ack`
would be AEAD-sealed application data the relay holds no key for). So a real
binary against the live relay failed with `hello_ack timeout after 5s` and
recycled the conn forever. This ticket deletes the ceremony: the conn is treated
as established the moment `transport.Client.Connected()` fires, and `run()` goes
straight to `forwardFrames`. This was the lone remaining blocker to a real `pyry`
completing its relay leg against the live relay (the 2026-05-29 Noise spike only
worked around it with an in-process fakerelay that synthesised the ack).

## Implementation

Deletion-heavy. One production file changed (`internal/relay/connection.go`); the
rest is test rework and one doc amendment.

- **`connection.go` — the cut (AC #1).** Deleted `handshake()` in full (the `hello`
  send, the 5s `Receive`, the `hello_ack`-timeout error, and the
  `protocol.HelloServerPayload` construction), the `handshakeTimeout` package
  `var`, and the conn-recycle-on-handshake-failure path
  (`Logger.Warn` + `client.DropConn()` + `continue`). The `run()` loop's
  `case <-c.client.Connected():` branch now logs one `Info("relay: conn
  established", "server_id", …)` breadcrumb (ops parity with the deleted
  `"relay: handshake complete"` log; also the observable signal for AC #2) and
  calls `c.forwardFrames(ctx)` directly.
- **`DropConn` now has no caller in `internal/relay`** — expected and fine. It
  remains a public method on `transport.Client` for future callers; not deleted.
- **Conflict path preserved with zero production change (AC #3).** The
  4409 → `ErrServerIDConflict` mapping in `classifyTransportErr` rides the
  transport's `FatalCloseCodes: [4409]` + `errors.Is(err, transport.ErrFatalClose)`
  — it never depended on the hello round-trip, so it survives untouched.
  Confirmed by `TestServerIDConflict_FatalNoReconnect` (still returns
  `ErrServerIDConflict`, `ConnCount == 1`).
- **Doc comments updated** (package doc, `Frames`, `Send`): no longer describe a
  relay-originated `hello`/`hello_ack` handshake — reconnects are transparent,
  frames resume on the new conn directly. `v2session.go`'s `rekeyInterval`
  comment, which name-dropped `handshakeTimeout` as the test-substitution idiom,
  was de-referenced in the same change (see Lessons learned).
- **Concurrency strictly simplifies.** Removing `handshake()` removes one
  synchronous `Receive` with a derived `context.WithTimeout` from `run()` — no new
  goroutines, channels, or locks. The shutdown sequence is unchanged.

### Tests

- **`connection_test.go` — test relay rebuilt as a content-blind forwarder.** The
  old relay impersonated v1 (read `hello` → send `hello_ack`); it now registers on
  upgrade and forwards frames, never reading a hello or sending an ack. Behaviors
  enum collapsed to `behaviorForward` (default), `behaviorCloseImmediately4409`,
  and `behaviorDropOnConnect` (`CloseNow` immediately after upgrade →
  transport reconnects); `behaviorSilentNoAck`/`behaviorSendBadType` deleted
  (nothing to be silent about / no handshake frame to corrupt). New
  `waitConnCount(t, relay, n, timeout)` helper replaces the `HelloEnv(0)` poll
  loops; `helloEnv`/`HelloEnv`/`shortenHandshakeTimeout` removed.
- **AC #2 regression guard:** `TestConnect_HappyPath` (which asserted the hello
  envelope's contents) repurposed into **`TestConnect_ReachesForwardingNoAck`** —
  against a relay that never sends a `hello_ack`, a frame pushed via
  `relay.outboundFrames` arrives on `c.Frames()` **and** `ConnCount == 1` (no
  recycle). Frame delivery + single conn together pin "reaches frame-forwarding,
  no ack, no recycle."
- **Deleted (AC #4):** `TestHandshake_AckTimeout`, `TestHandshake_UnexpectedFrame`.
  **Re-synced** (readiness off the vanished hello, onto `waitConnCount`):
  `TestTransportDropOnConnect_Reconnects`, `TestTransportDropPostConnect_Reconnects`,
  `TestFrames_AfterConnect_InOrder`, `TestClose_ShutsDownCleanly`,
  `TestContextCancel_ShutsDownCleanly`.
- **e2e:** deleted `TestRelay_Hello` (`internal/e2e/relay_test.go`) — its entire
  subject was the binary↔relay hello the binary no longer sends. `TestRelay_4409`
  and `TestRelay_1011` stay (header/`WaitBinary`-based, no hello dependency).

## Patterns established

- **The binary↔relay leg has no application-layer handshake; it is established at
  the transport upgrade.** The relay is content-blind: its only check on this leg
  is header-based server-id registration (`x-pyrycode-server`, first-claim-wins,
  4409 on conflict). The real authentication boundary — per-phone-conn Noise_IK +
  device-token validation — lives downstream of `internal/relay` and is unchanged;
  reaching `forwardFrames` without a relay-leg ack admits no unauthenticated
  traffic (every forwarded `RoutingEnvelope` still passes the downstream
  Noise+token state machine). Removing the redundant plaintext hello also slightly
  *reduces* relay-visible metadata.
- **Strangler Fig completed.** This is the middle slice of #569's three-step
  removal: **[#581](581.md) migrated the readers** (e2e readiness off
  `LastBinaryHello` onto header-based `WaitBinary`) → **#582 retires the
  handshake** → **#583 removes the now-dead `LastBinaryHello` accessor + fakerelay
  hello-receiving path.** Migrating consumers first turned a risky cross-cutting
  removal into independently-green steps — retiring the handshake here cascaded no
  `t.Fatal`s because no readiness gate still keyed off the hello.

## Lessons learned

- **Deleting a `package var` that other files reference only in *doc comments*
  leaves dangling symbol references that compile and pass `go vet`/`staticcheck`
  but mislead readers.** Here `handshakeTimeout` was name-dropped in
  `v2session.go`'s `rekeyInterval` comment ("the same `t.Cleanup` save-and-restore
  idiom `handshakeTimeout` uses") and in an internal `forwardFrames` comment.
  Grep the removed identifier across the whole package — not just call sites — and
  fix the stale comments in the same change. *(Lifted from the PR's Lessons-learned
  section.)*
- **`protocol.HelloServerPayload` is now an orphaned type.** After this deletion it
  has **no consumers anywhere in the repo** — it is only defined and self-tested in
  `internal/protocol/handshake.go` + `handshake_test.go`. (The spec anticipated it
  surviving alongside `HelloAckPayload`/`TypeHello`/`TypeHelloAck`, but those are
  the types the *phone↔binary* leg keeps — `auth.go` and `v2session.go` use
  `HelloAckPayload`/`TypeHello`/`TypeHelloAck`/`TypeError`, none use
  `HelloServerPayload`.) It is a clean candidate for the named follow-up type
  cleanup; left in place here to keep this diff a pure behavioural deletion.
- **Incidental testdata churn:**
  `internal/e2e/realclaude/testdata/permission_protocol_v2.1.158.json` changed in
  the code-review auto-commit — identifier-only (`session_id`, message/`toolu` ids,
  timing/cost), no structural change. It belongs to a realclaude test, not to this
  ticket; not a #582 behaviour change. (Same incidental-churn footnote as #581.)

## Testing

- `go build ./...`, `go vet ./...`, `staticcheck ./...` clean — no unused imports
  after the deletions (`protocol`/`time`/`errors`/`encoding/json` retain other
  uses in `connection.go`).
- `go test -race ./internal/relay/` — reworked unit suite green; AC #2 pinned by
  `TestConnect_ReachesForwardingNoAck`, AC #3 by `TestServerIDConflict_FatalNoReconnect`.
- `go test -race -tags e2e ./internal/e2e/` — `TestRelay_4409` / `TestRelay_1011`
  and the v2 round-trip tests pass after the binary stops sending a hello (the
  fakerelay registers on upgrade, so `WaitBinary`-based readiness is unaffected);
  `TestRelay_Hello`'s deletion left the file compiling. The `e2e`-tagged
  `relay_v2_handshake_test.go` (not named in the spec, exercises the surviving
  phone↔binary Noise_IK leg) was run explicitly and passed — the relay-leg
  retirement is transparent to it.
- **Ultimate validation (manual, out-of-CI per AC #2):** a real `pyry` binary
  completes its relay leg against the live `pyrycode-relay` with no
  `hello_ack timeout` and no recycle.
