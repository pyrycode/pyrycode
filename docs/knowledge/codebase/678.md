# Ticket #678 — `send_message` routes the turn to the conversation's bound session

The consumer half of EPIC #672 ("per-conversation sessions"), direct sibling of [#677](677.md). #677 made every conversation eagerly mint + bind its own session (`Conversation.CurrentSessionID`), but `send_message` still routed **every** turn to the single bootstrap session — the `ConversationID` reached `Supervisor.WriteUserTurn` only to validate the conversation and stamp the outbound cursor, never to *select* a session. #678 makes the handler resolve the bound session and deliver the turn **there**, so each discussion's turns land in its own claude and never bleed across conversations. This is the slice that actually realises the isolation #677 set up.

See the evergreen feature doc: [features/conversation-session-binding.md § Routing](../features/conversation-session-binding.md#routing-send_message-consumes-the-binding).

## Implementation

- **New consumer-declared `SessionRouter` interface** (`internal/relay/handlers/send_message.go`) — `Route(conversationID string) (TurnWriter, error)`. It *returns* the existing `TurnWriter` rather than any `internal/sessions` type, so `handlers/` stays import-clean (mirrors the sibling `SessionCreator` from #677). `Route` is deliberately **ctx-free**: it is a pure in-memory lookup (registry read + field check + pool lookup), non-blocking, with no cancellation surface; the blocking `Activate`/`WriteUserTurn` run later under the handler's unchanged budgets.
- **Handler change is one prelude, nothing else.** `SendMessage(router SessionRouter, logger)` prepends `w, routeErr := router.Route(p.ConversationID)` with a two-arm error switch, then runs the **byte-identical** two-phase Activate→WriteUserTurn block against the resolved `w`. AC#5 ("only the session-selection step is new") is honoured — the two 30s budgets, the ack envelope, and the activate/delivery error mapping are untouched.
- **`sessionRouter` + `boundSession` adapters** (`cmd/pyry/main.go`, beside `sessionMinter`/`poolResolver` — the only package importing both `internal/conversations` and `internal/sessions`):
  - `sessionRouter{pool, convReg}.Route` resolves in a **load-bearing order**: `convReg.Get` miss → `conversations.ErrConversationNotFound`; **`conv.CurrentSessionID == "" → errNoBoundSession` *before any Lookup*** (see Lessons); `pool.Lookup(id)` error (`ErrSessionNotFound`) flows straight through; else wrap in `boundSession`.
  - `boundSession{pool, sess, id}` satisfies `TurnWriter`. `*sessions.Session` already satisfies `TurnWriter` directly — this wrapper exists **only** to redirect `Activate` through the cap-enforcing `Pool.Activate(ctx, id)` instead of `Session.Activate` (see Lessons). `WriteUserTurn` passes straight through.
  - `errNoBoundSession` is an unexported sentinel (no wire surface); the handler maps any non-`ErrConversationNotFound` `Route` error → retryable `server.binary_offline`.
- **Error mapping (no new wire code).** Unknown conversation → `conversation.not_found` (not retryable, new `send_message.unknown_conversation` log at routing time); empty binding / dangling id → `server.binary_offline` (retryable, new `send_message.no_bound_session` WARN log). The deliver-switch `ErrConversationNotFound` arm stays — it now defends the TOCTOU where the conversation is deleted between `Route` and delivery (the bound session's own `ValidateConversation` re-fires).
- **Threading (mechanical, compiler-checked).** `sess handlers.TurnWriter` → `router handlers.SessionRouter` in `startRelay` + `startRelayV2` (`cmd/pyry/relay.go`), threaded into both `handlers.SendMessage(router, logger)` registration sites (v1 dispatcher, v2 manager map). `main.go` passes `sessionRouter{pool, convReg}` where it previously passed `bootstrap`; `bootstrap.Supervisor()`/`bootstrap.Bridge()` (the still-bootstrap-tapped assistant-turn bridge) are unchanged. Clean swap, no transitive cascade — `dispatch.Handler` return type is unchanged.
- **Tests.** New `stubSessionRouter` test double + `routeTo(w)` helper; the 7 existing handler tests are rewired through `routeTo(stub)` and stay green as the AC#5 regression guard. New handler cases: `RoutesToBoundSession` (AC#1), `TwoConversations_NoCrossDelivery` (AC#2), `UnknownConversation_RoutingTime` + `NoBoundSession_EmitsBinaryOffline` (AC#4). New `cmd/pyry/session_router_test.go` (`TestSessionRouter_Route`) pins the resolution against a **real** `*sessions.Pool` + `*conversations.Registry`, including an explicit `Lookup("")`-returns-bootstrap precondition assertion — the one check a pure handler stub cannot make.

## Patterns established

- **A consumer-declared interface that *returns* another consumer-declared interface.** `SessionRouter.Route → TurnWriter` composes two narrow seams without `handlers/` ever importing `internal/sessions`. The resolution that bridges `conversations` + `sessions` lives in the single `cmd/pyry` adapter layer, exactly where `sessionMinter` (#677) and `poolResolver` already sit.
- **Honour a "behaviour-unchanged" AC by prepending, not threading.** AC#5 demanded the two-phase block behave exactly as today. The implementation resolves the write surface in a prelude and runs the *unchanged* block against it — a swap of `w`'s source, not a rewrite. The code-review verified the block is byte-identical rather than trusting the spec.
- **Test the resolution where the real types live.** The AC#4 "never routes to bootstrap" guarantee depends on `Pool.Lookup("")`-returns-bootstrap — a behaviour a handler-level stub can't exhibit. The proof lives in a `cmd/pyry` test against a real `*sessions.Pool` (`sessions.New` builds the bootstrap entry without spawning claude; `os.Args[0]` is the guaranteed-executable fake `ClaudeBin`).

## Lessons learned

- **`Pool.Lookup("")` returns the *bootstrap* session — so the empty-`CurrentSessionID` guard MUST fire before any `Lookup`.** This is the load-bearing line of the whole slice and the AC#4 confused-deputy hazard: without rejecting an unbound conversation up-front, a phone's turn would be silently delivered into the shared bootstrap claude — an isolation break. The guard returns a non-`ErrConversationNotFound` error → retryable `server.binary_offline`, never the bootstrap. Pinned by `TestSessionRouter_Route` with an explicit precondition assertion that `Lookup("")` does return bootstrap.
- **`boundSession` exists only to funnel `Activate` through `Pool.Activate`.** `*sessions.Session` already satisfies `TurnWriter`, so the wrapper looks redundant — but the bootstrap path used `Session.Activate` (the bootstrap is special: always active, never cap-evicted), whereas per-conversation sessions are full `ActiveCap` citizens. Activating one may need to LRU-evict a peer, which only happens inside `Pool.Activate`. Calling `Session.Activate` directly would bypass the cap and break the invariant the idle-evict follow-up #680 relies on. A mid-flight removal degrades safely: `Pool.Activate` re-Lookups internally and returns `ErrSessionNotFound` (→ retryable) rather than touching a freed pointer.
- **Inbound now routes per-conversation; outbound still taps the bootstrap PTY.** The assistant-turn → `message` bridge (#311) still reads `bootstrap.Bridge()`, so replies fan out from the bootstrap session's PTY while inbound turns land in per-conversation claudes. This inbound/outbound asymmetry is a deliberate, documented phased-migration decision — out of #678's scope — and the real-claude e2e confirmed the full phone→claude→phone round-trip is intact. The outbound half is a #672-family follow-up; recorded as a limitation in [conversation-session-binding.md](../features/conversation-session-binding.md#edge-cases--limitations).
- **gofmt skew at HEAD, again.** `send_message_test.go` is gofmt-flagged at HEAD on the pre-existing `stubTurnWriter` struct (local Go 1.26 vs CI's pinned-older Go alignment skew). Left untouched per the known skew lesson — only the new additions are gofmt-clean. Don't "fix" pre-existing dirtiness; it sprays spurious diffs and fights CI.
