# Ticket #361 — `internal/e2e/realclaude/` scaffold + build tag + make target

Pure plumbing. Creates a build-tag-gated home for the real-`claude`-binary integration tests that #362 / #363 will populate. Nothing in this ticket runs under `make test` / `make check`.

Background: Phase C broke on 2026-05-14 because `internal/agentrun/drive_e2e_test.go` uses `TestHelperProcess` fakes that stop at the trust boundary with the real `claude` binary — exactly the class of bug `/doctor` prompt-poisoning sits in. The fake-claude harness can't catch it; we need a separate suite that crosses that boundary.

See [features/e2e-realclaude.md](../features/e2e-realclaude.md) for the suite's full contract.

## Implementation

- **`internal/e2e/realclaude/smoke_test.go`** (~24 lines, new). Build tag `//go:build e2e_realclaude` (single tag, no alternation — deliberately distinct from `e2e` / `e2e_install` so a future `make e2e` won't pick up real-claude tests). Package `realclaude` (not `realclaude_test`). One test: `TestClaudeBinaryAvailable` asserts `exec.LookPath("claude")` succeeds, then runs `exec.CommandContext(ctx, "claude", "--version")` under a 10 s `context.WithTimeout`. `CombinedOutput()` is captured for the failure message; the version string is **not** parsed — content checks belong to #362+.
- **`Makefile`** — adds `.PHONY: e2e-realclaude` between `test:` and `staticcheck:` invoking `$(GO) test -tags e2e_realclaude ./internal/e2e/realclaude/...`. No `-race` (I/O-bound trust-boundary check, not goroutine-stress). `make check` unchanged — CI continues to run only `make check`; this suite is developer-local opt-in.

## Patterns established

- **`e2e_<purpose>` build-tag naming for opt-in sibling test packages.** `e2e_install` already established this; `e2e_realclaude` is the second instance. Each sibling package owns one tag, no alternation with the umbrella `e2e` tag — keeps each suite's intent legible at the file header and prevents accidental pickup by a future broad `make e2e` target.
- **Fatal-not-skip when the suite is invoked explicitly.** `exec.LookPath` failure inside the smoke test is `t.Fatalf`, not `t.Skip`. The developer opted in by typing `make e2e-realclaude`; silently skipping would hide the misconfiguration. Generalisable to any future opt-in-via-make-target suite: skip behaviour belongs to suites that run automatically, fatal behaviour to suites the developer asks for by name.
- **No `-race` by default on I/O-bound suites.** The make target deliberately omits `-race`. `-race` adds noise (and overhead) without value when the test surface is `exec.Command(...).Run()` synchronously. If a later test in the directory does spin goroutines, the author can add `-race` to that specific invocation; we don't bake it into the directory default. Same rationale would apply to any future "real external binary" suite.

## Verification

- `make e2e-realclaude` passes on a developer machine with `claude` on PATH.
- `make test 2>&1 | grep realclaude` is empty after this lands — unsatisfied build tags drop files at the build stage, so the package compiles to an empty test binary under the default tag set. Done-when #5.
- `make check` (vet + test + staticcheck) passes — confirms the tagged file doesn't trip vet/staticcheck under default tags.

## Out of scope (deferred to #362+)

- Any actual prompt-poisoning / trust-boundary assertion. The scaffold's only job is "the directory exists, the tag works, the make target runs".
- Wiring the suite into CI. Stays developer-local until #362/#363 prove out the contract.
