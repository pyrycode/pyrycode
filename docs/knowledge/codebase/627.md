# Ticket #627 — Outbound bridge: `turnevent.Event` → v2 wire-envelope mapper (pure adapter)

The symmetric **outbound** half of the Phase 2 structured-event bridge ([EPIC #596],
[ADR 025] § Phase 2). Where #615's `mapper.go` maps a live tui-driver event **into**
the neutral internal `turnevent.Event` model ([#606]), this slice maps that model
**out** to the matching v2 interactive wire payload ([#607]) — given explicit turn
context (conversation id, turn id, per-turn seq). It is a **pure value-to-value
adapter**: no I/O, no state, no envelope-ID minting, no clock read, no sealing — all
of which belong to the turn-lifecycle integration consumer (`cmd/pyry`, building on
[#616]'s capability-gated fan-out). This is the pure-mapping cut #616's body
pre-authorized (#616 deferred the actual event→envelope mapping to this slice).
Package reference (updated this ticket): [features/turnbridge-package.md](../features/turnbridge-package.md).

## Implementation

One new production file in the existing `internal/turnbridge` package, alongside
`mapper.go` (`outbound.go` ~158 LOC; `outbound_test.go` ~289 LOC, table-driven, not
counted toward the production budget). It adds the package's **first
`internal/protocol` import** — dependency direction stays clean
(`cmd/pyry → internal/turnbridge → {tuidriver, turnevent, protocol}`; only
`outbound.go` reaches `protocol`, only `mapper.go`/`producer.go` reach `tuidriver`).

Exported surface: **2 types** (`TurnContext`, `TurnState`), **2 functions**
(`MapEvent`, `BuildTurnState`), **3 state constants** (`StateThinking` /
`StateResponding` / `StateIdle`). Plus the unexported `const maxSummaryLen = 200`.

- **`MapEvent(ev turnevent.Event, tc TurnContext) (typ string, payload any, ok bool)`**
  — a pure type-switch over the sealed `Event`, the exact mirror of `mapEvent`'s
  `(value, ok)` idiom. Carries every field verbatim from `tc` + the event:

  | `ev` concrete type | `typ` | `payload` | `ok` |
  |---|---|---|---|
  | `TextChunk` | `TypeAssistantDelta` | `AssistantDeltaPayload{tc.ConversationID, tc.TurnID, tc.Seq, ev.Text}` | true |
  | `ToolStart` | `TypeToolUse` | `ToolUsePayload{…, ToolUseID: ev.ToolCallID, Name: ev.Title, InputSummary: inputSummary(ev.RawInput)}` | true |
  | `ToolUpdate` | `TypeToolResult` | `ToolResultPayload{…, ToolUseID: ev.ToolCallID, IsError: ev.Status == ToolStatusFailed, ResultSummary: resultSummary(ev.Content)}` | true |
  | `TurnEnd` | `TypeTurnEnd` | `TurnEndPayload{…, StopReason: string(ev.Reason)}` | true |
  | `ThoughtChunk` | `""` | `nil` | **false** (drop) |
  | nil / unknown | `""` | `nil` | false (drop) |

  `payload` is `any` because the four payload structs share no marker interface; the
  consumer `json.Marshal`s it directly (same path as `protocol.MessagePayload`).
  Zero-value-safe: a nil `ev` falls to the default → drop, exactly like `mapper.go`.
  Internal-only fields with no #607 wire home (`ToolStart.Kind`/`Locations`,
  `*.MessageID`) are correctly **not** forwarded.

- **`BuildTurnState(conversationID string, state TurnState) (typ string, payload protocol.TurnStatePayload)`**
  — shapes a `turn_state` payload (thinking / responding / idle). Concrete return
  type (not `any`) because it is monomorphic; no type assertion needed downstream.
  The consumer's lifecycle machine decides *which* state applies and calls this; the
  adapter only shapes the payload. `TurnState` is string-backed (enum-safe call site;
  the wire field itself stays a plain string per #607).

- **Summary derivation (AC3).** Three pure helpers produce the bounded, single-line
  précis the wire envelopes carry (a précis, *not* the raw input/output):
  - `inputSummary(json.RawMessage)` — `json.Compact` (whitespace → one line) then
    `truncate`. Empty/nil and invalid-JSON both → `""` (best-effort/opaque
    `RawInput` posture from #606, mirroring `mapper.go`'s `rawInput`).
  - `resultSummary(turnevent.ToolContent)` — **exhaustive** over the sealed
    `ToolContent` sum type: `nil`→`""`, `TextContent`→text, `DiffContent`→`Path`,
    `TerminalContent`→`"terminal <id>"`, each truncated.
  - `truncate(s, max)` — rune-aware (counts/cuts on `utf8.RuneCountInString` /
    `[]rune`, not bytes), appends `"…"` when over bound, so multibyte text never
    splits mid-rune.

- **`is_error` derivation.** `IsError == (Status == ToolStatusFailed)`;
  `completed`/`pending`/`in_progress` all map to `false`. Round-trips with
  `mapper.go:toolStatus` (failed↔error, completed↔success).

- **`ThoughtChunk` treatment (AC4, confirmed against ADR 025).** #607 defines **no**
  thought-text envelope, and ADR 025 classes thinking as screen-sourced/brittle and
  surfaces it as a `turn_state` transition. So `MapEvent` **drops** `ThoughtChunk`
  (`ok == false`) — the thought *text is not forwarded* (a test row pins "no thought
  text leaks"). The thinking **state** surfaces via `BuildTurnState(convID,
  StateThinking)`, which the **consumer's** lifecycle machine calls when it observes
  a `ThoughtChunk`. Deciding "a ThoughtChunk means we are thinking" is a lifecycle
  decision (state transitions are the integration slice's, per ADR 025 phase
  scoping), so it stays out of the pure mapper — the mapper supplies the *builder*,
  the consumer owns the
  *decision to call it*. Emitting `turn_state{thinking}` from `MapEvent` itself would
  embed a lifecycle rule and break the pure value-to-value seam.

`make check` (vet + `-race` + staticcheck + substrate-guard) green; real-claude e2e
green (`ok internal/e2e/realclaude 139.213s`).

## Patterns established

- **A symmetric-mirror file in a shared package must disambiguate its test names.**
  `outbound.go` joins `mapper.go` in `package turnbridge`, whose `mapper_test.go`
  already owns `TestMapEvent`. The outbound table is `TestMapEventOutbound` — a name
  collision is a **compile error**, not a runtime surprise, but it is the first thing
  that bites when you mirror a file into an existing package. (Lifted from the PR's
  Lessons-learned section.)
- **The pure-mapper seam, mirrored.** `outbound.go` is the exact mirror of
  `mapper.go`: a pure `(typ, payload, ok)` type-switch with small pure helpers below,
  **no logger, no I/O, no state**. `ok == false` means "no wire representation"; the
  **consumer** owns the single `log.Debug` drop-log, the ID/TS/marshal/seal/push.
  This is what keeps the adapter table-testable and isolates it from the lifecycle
  state machine — the same "pure mapper, drop-logging in the caller" posture #615
  established for the inbound direction.
- **Exhaustive switch over a sealed sum type guards against silent variant loss.**
  `resultSummary` handles all of `nil`/`TextContent`/`DiffContent`/`TerminalContent`
  even though the current producer (`mapper.go:toolResultContent`) only ever emits
  `TextContent` or `nil`. The Diff/Terminal arms are unreachable today but handled —
  kept deliberately minimal (`Path`, `"terminal <id>"`) — so a future producer
  variant (an ACP adapter [#600], a refinement) cannot vanish off the wire. ~2 lines
  + 2 test rows each; cheap insurance from the closed type.
- **A précis is "bounded + single-line", which is what makes it not the raw value.**
  `inputSummary`/`resultSummary` derive a `maxSummaryLen`-bounded one-liner. The bound
  is a **phone-display** choice (200 runes), not a wire constraint (the envelope cap
  is far larger) — tunable if the mobile view asks.
- **The upstream "no omitempty" contract reaches boundary zero-values to the wire.**
  Because #607's payloads carry no `omitempty`, `seq:0` and `is_error:false` are
  always serialized; the `Seq == 0` test row proves the boundary value reaches
  `assistant_delta.seq` instead of being dropped as a zero.

## Lessons learned

- **Mirroring a file into an existing package collides on test names.** See the
  pattern above — `TestMapEventOutbound` disambiguates from `mapper_test.go`'s
  `TestMapEvent`. A compile-time failure, caught immediately, but the predictable
  first snag of the symmetric-mirror approach. (From the PR body's Lessons-learned.)
- **`truncate`'s `max` parameter shadows the `max` builtin (code-review NIT,
  non-blocking).** Go 1.21+ has a built-in `max`; this module is go 1.26.
  `truncate(s string, max int)` shadows it — harmless because the body never calls
  the builtin, but `maxRunes`/`n` would avoid the shadow. Left as-is (non-blocking);
  flagged here so the shadow is a known, deliberate cost if the helper grows.

## Scope (S confirmed)

1 production file (`outbound.go`, under the ≥5 split gate), **2** new exported types
(`TurnContext`, `TurnState` — under the >5 red line; the three `State*` are
constants, not types, same accounting as #607's `Type*`), **0** consumer call sites
(greenfield-additive: `MapEvent`/`BuildTurnState` are new exports with no collision
against the unexported `mapEvent`, and no caller until the integration slice wires
them). Tests scale linearly with the mapping table and don't count toward the
production budget. One cohesive slice: type-switch → summary helpers → tests.

## Out of scope (deferred to consumer / siblings)

- **Envelope `ID` minting, `TS` (clock read), `json.Marshal`, AEAD sealing, `Push`,
  and the drop-log** — the turn-lifecycle integration consumer (`cmd/pyry`, building
  on [#616]'s capability-gated fan-out). See `cmd/pyry/assistant_turn_v2.go` for the
  existing shape that wraps a payload into an `Envelope`.
- **The turn-lifecycle state machine** — when to transition thinking/responding/idle,
  turn-id assignment, delta-seq advancement, coalescing — the integration slice. This
  adapter consumes those decisions as inputs (`TurnContext`); it never derives them.
- **Richer `input_summary` (per-tool salient-field extraction:** Read→`file_path`,
  Bash→`command`, Grep→`pattern`) — would couple the adapter to claude tool-input
  schemas; #615 deliberately kept tool metadata minimal and this slice holds that
  line. Revisit only if the compact-JSON précis proves insufficient in the live phone
  view (evidence-based: no observed need yet).
- **`DiffContent`/`TerminalContent` précis shape** — handled minimally; the descriptor
  format can be refined against a real consumer when a producer (ACP adapter [#600])
  actually emits them.
