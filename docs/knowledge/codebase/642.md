# Ticket #642 — Event-stream bridge: live two-phone structured-receive e2e capstone

The **live** confirmation of the Phase 2 capability-gated dual-path ([EPIC #596],
[ADR 025] § Phase 2). #632/#633/#634 shipped and **deterministically** proved the
dual-path (structured stream fans only to `interactive` conns; coarse `message`
confines to `!interactive` conns — two complementary filters over one set-once flag,
plus the #626/#632 unit matrices). What was *not* yet proven was the **live** run: one
daemon, two real Noise phones, where the `interactive`-granted phone actually **receives
and decrypts** the structured envelopes (`turn_state` / `assistant_delta` / `tool_use` /
`turn_end`) while the non-interactive phone receives only the coarse `message`. Every
upstream slice deferred this capstone (#589/#634 prove only coarse; #632 shipped no e2e;
#633 drove a *fake* subscriber). This ticket lands it. **Test/harness only — no
production change**; the dual-path production code already shipped.

**`security-sensitive`** — the live end-to-end proof that a non-`interactive` phone
**never** receives the structured stream on the internet-exposed mobile surface (#607's
deferred gated-fan-out review). The deterministic oracle still lives with #634/#632; this
is the live confirmation, guarded against a **vacuous pass** (below). Architect
self-review **PASS, no MUST FIX**; code-review **PASS** (two non-blocking NITs).

## The harness decision: option (b), reuse — no carve

The ticket flagged a hard blocker (a structured-stream-capable e2e harness does not yet
exist) and handed the architect a build-vs-reuse design call: **(a)** real-claude JSONL
under a Noise-phone handshake suite, or **(b)** `fakeclaude` replaying captured
claude-format JSONL. The architect chose **(b), extend the existing suite — no carve, one
S ticket**:

- **(a) rejected** — it would fuse the `e2e_realclaude` suite (real `claude` binary,
  network, API key, non-deterministic output/timing) with the `e2e` Noise-phone suite.
  That is multi-file foundational work, inherently flaky, and cannot run in the standard
  `-tags e2e` CI column.
- **(b) chosen** — the structured production stack is **already wired live** in the relay
  harness (`startInteractiveTurnStreamV2`, gated `bridge != nil && claudeSessionsDir !=
  ""` at `cmd/pyry/relay.go`), so the structured events flow through the **real**
  production path (tui-driver JSONL parse → `turnbridge` mapper → #632 emitter → Noise
  seal → phone decrypt). Only the claude *process* is the fakeclaude stand-in — exactly as
  it is for every other e2e in the suite. The Noise two-phone harness already exists
  (`fakephone` + `fakerelay`, the v2 handshake drivers, #634's
  `TestTwoPhoneCoarse_NonInteractiveOnly` as a near-complete template). Only **two** small
  pieces were missing: sessions-dir alignment and a fakeclaude structured-JSONL emitter.

So this capstone stays a **single concern**: the live two-phone test plus the ~20-LOC
fakeclaude extension that feeds it. The PO's pre-pass (`[[po-dont-precarve-when-scope-is-architect-design-call]]`)
correctly left the carve to the architect; the architect's design run that reached "no
carve" was not wasted.

### fakeclaude: `PYRY_FAKE_CLAUDE_JSONL_TRIGGER` (the structured-stream feeder)

Mirrors `emitAssistantIfTriggered` exactly, but appends to the **live session JSONL** (the
file the daemon's structured producer tails) instead of stdout. When the trigger path
exists: read its contents (capped at `assistantMaxBytes`), append them **verbatim** to the
open session `*os.File` (already `O_APPEND`), `f.Sync()`, then `os.Remove` the trigger.
The trigger file's **contents ARE the claude-format JSONL lines to append** (same
"contents are the payload" shape as the assistant trigger). Errors silenced — the e2e
asserts downstream, and a missing trigger is the steady state. Only the main goroutine
writes `f`, so it never races the stdin reader. See
[features/fakeclaude-binary.md § JSONL-trigger mode](../features/fakeclaude-binary.md).

The `f.Sync()` is **load-bearing**: the daemon's tail is a separate process and macOS APFS
otherwise defers cross-process visibility (same reason the stdin reader fsyncs per write).

### The test choreography (`TestTwoPhoneStructured_InteractiveReceivesStream`)

Lifts #634's two-phone structure and flips A's assertion to "receives structured":

1. **Align the sessions dir.** `sessionsDir = filepath.Join(home, ".claude", "projects",
   encodeWorkdir(home))` (reuse the `e2e`-package `encodeWorkdir` from `rotation_test.go`).
   `resolveClaudeSessionsDir` has **no env override** — it always computes
   `<HOME>/.claude/projects/encode(workdir)` from `HOME`+workdir — so pointing fakeclaude
   at the same computed dir makes the producer tail exactly what fakeclaude writes,
   **alignment by construction**. (#634 deliberately used a throwaway dir, which is why its
   coarse proof came off the PTY chunk, not the JSONL.)
2. **Pre-create `<initialUUID>.jsonl` (`{}\n`) BEFORE the daemon starts** — the
   `rotation_test.go` pattern, and the fix for the prior run's cold-start race (see
   Lessons learned). The producer's first resolve then succeeds immediately at a tiny
   offset, captured **seconds before** the post-ack fixture append, so every appended line
   lands inside the tailed range.
3. **Pair A and B** against instance `test` (distinct bearer tokens, shared per-instance
   server static key); seed `conversations.json` with `knownConvID` so the
   `ValidateConversation` gate in `WriteUserTurn` passes and the cursor stamp lands.
4. **Start the daemon TUI-on:** `StartRotationWithRelay(..., "PYRY_MOBILE_V2=1",
   "PYRY_FAKE_CLAUDE_TUI=1", "PYRY_FAKE_CLAUDE_JSONL_TRIGGER="+jsonlTrigger)`.
5. **Handshake A interactive** (`driveHandshakeToOpenDaemonInteractive`, which asserts A's
   hello_ack **echoes** the grant) and **B non-interactive, pinned**
   (`driveHandshakeNonInteractivePinned`, new local helper — asserts B's hello_ack
   **lacks** `CapabilityInteractive`).
6. **Drive the turn from A and await the sealed ack** (decrypt-drain; fail loud if A
   receives a `TypeMessage` — A is interactive, so the coarse path must skip it). The ack
   is the **deterministic fence**: it confirms `WriteUserTurn` ran → the cursor is stamped
   → the structured `Handle` will see a non-empty cursor when the fixture tails.
7. **After the ack, drop the JSONL trigger** with four claude-format fixture lines
   (modeled on `turnbridge/mapper_test.go`'s oracle): assistant text → `turn_state(responding)`
   + `assistant_delta`; assistant `tool_use` Read → flush `assistant_delta` + `tool_use`;
   user `tool_result` → `tool_result`; assistant text + `stop_reason:end_turn` → flush
   `assistant_delta` + `turn_end` + `turn_state(idle)`.
8. **Assert A (AC1 + vacuous-pass guard).** One decrypt-drain under a single 20 s deadline
   (fakephone closes the WS on a timed-out `Receive`, so a single long deadline with
   back-to-back reads, **never** a short poll); collect structured types; break once all
   four required types are seen; **fail loud on any `TypeMessage`** (A gets no coarse);
   every structured payload's `ConversationID == knownConvID`. If A's structured set is
   empty/partial, `failVacuous` `t.Fatal`s naming the harness-produced-nothing failure
   mode.
9. **Assert B (AC2/AC3).** B receives the coarse `message` (assistant role, `knownConvID`,
   valid-UUID message-id, `InReplyTo == nil`) and **zero** structured envelopes (any
   structured frame to B → `t.Fatalf` "capability-gate leak"). B's coarse marker is the
   `✻` spinner chunk (TUI mode → PTY → bridge → coarse emitter → `!interactive` conns) —
   **no separate assistant-trigger needed**. Once the coarse marker is in hand the deadline
   tightens to 2 s for a prompt trailing leak-check.

## Patterns established

- **Live oracle for a deterministically-proven invariant — the capstone confirms, it does
  not re-prove.** #634's complementary filters (`c.Interactive` structured vs
  `!c.Interactive` coarse over one set-once flag) + the #626/#632 unit matrices are the
  *correctness* oracle; this test is the *live* confirmation over real sealed sessions. A
  live e2e is the right tier for "does the whole stack actually behave," not for the gate's
  primary proof — splitting it out of #634 (`[[po-descope-in-place-vs-parent-closing-split]]`)
  did not weaken the `security-sensitive` contract.
- **Vacuous-pass guard as an ordered precondition (the headline `security-sensitive`
  requirement).** "B never receives a structured envelope" is **meaningless unless the
  structured path is live and observed on A in the same run** — otherwise a harness that
  produced *no* structured events at all would pass the negative vacuously. So A's positive
  (≥1 each of the four required types) is asserted **first**, with a dedicated `t.Fatal`
  naming the empty-set failure mode; B's negative runs only after. The guard is structural,
  not advisory: without the live producer (sessions-dir alignment + fakeclaude emission)
  A's set is empty and the test fails. This is the architect-owned guard the label requires.
- **Pin both ends of the capability gate.** A's grant is pinned by the existing
  `driveHandshakeToOpenDaemonInteractive` hello_ack echo; B's **non**-grant is now pinned
  by `driveHandshakeNonInteractivePinned` (hello_ack lacks `CapabilityInteractive`). A
  mis-granted B would already fail loud (it would receive a structured envelope → fatal),
  so pinning is robustness, not a correctness gate — but it closes the gap between "B
  advertised nothing" and "B was not *granted* interactive," making the negative airtight
  at the gate boundary.
- **Drive from the interactive phone → one turn produces both signals.** Driving the turn
  from A keeps A's ack stream coarse-free (A is interactive → the coarse emitter skips it),
  and the **same** turn's `✻` spinner chunk (forwarded on first stdin, #603) is fanned to
  non-interactive B as *its* coarse `message`. So one `send_message` yields A's structured
  stream (from the appended fixture) **and** B's coarse marker — no second assistant-trigger
  wiring.
- **TUI mode is load-bearing twice here.** Unlike #634 (TUI-off, which ate the 30 s
  `WaitReady` floor — see [634.md](634.md) Lessons), this test runs **TUI-on**, which buys
  *both* a prompt sealed ack (no 30 s floor) *and* B's coarse spinner marker. The #603 glyph
  contract is a prerequisite of this capstone, not just of the `send_message` flows it
  restored.
- **Harness-extension over harness-build.** The whole proof rides the *shipped* production
  stack; the only new harness surface is the ~20-LOC fakeclaude JSONL feeder. The
  foundational build the ticket warned about (option a) was the wrong shape — the existing
  fakeclaude `e2e` suite already had every other piece.

## Lessons learned

Lifted from the PR body's **Lessons learned** (the developer flagged three non-obvious
surprises). The first is also the root cause of the **prior developer run's timeout**
(2026-06-08), recorded on the issue and resolved here:

- **Cold-start producer-subscribe race → pre-create the session JSONL before the daemon
  starts.** The structured producer captures its tail offset at the **first resolve**.
  Without a pre-existing `<initialUUID>.jsonl`, that first resolve finds an empty dir,
  Warn-logs "no session jsonl found", and retries ~500 ms later — and that retry can land
  **after** the post-ack fixture append, capturing an EOF offset *past* every event, so the
  producer tails from beyond the fixture and A receives **zero** structured envelopes (the
  vacuous-pass guard then fires correctly). Pre-creating the file (the `rotation_test.go`
  pattern) makes the first resolve succeed at startup, seconds before the append, so every
  fixture line lands inside the tailed range. The prior run had already verified
  sessions-dir alignment, the fixture append itself, and trigger consumption — the gap was
  purely producer-subscribe / cursor-stamp / append **ordering**.
- **Drive the turn from the interactive phone, and let the TUI spinner be B's coarse
  marker.** One turn drive produces both A's structured stream (from the fixture) and B's
  coarse `message` (the `✻` spinner forwarded to non-interactive conns) — no separate
  assistant-trigger wiring (see Patterns above).
- **A's drain breaks at `turn_end` (the 4th required type, the 6th of 7 emitted
  envelopes), before the trailing `turn_state(idle)`** — so the observed structured count
  is **6, not 7**. Expected, not a dropped event: the loop's exit condition is "all four
  required types seen," which `turn_end` satisfies, so it stops one envelope early.

## Testing

One new `//go:build e2e` test asserting all four ACs in a single run; **no production test
added** (no production change to cover).

- **AC1** — A decrypts ≥1 each of `turn_state` / `assistant_delta` / `tool_use` /
  `turn_end` under its session keys, all with `ConversationID == knownConvID`.
- **AC2 / AC3 (mutual exclusivity, live)** — A receives **zero** `TypeMessage`; B receives
  the coarse `TypeMessage` and **zero** structured envelopes.
- **AC4 (no app output logged)** — inherited from #589; the production emitters log only
  content-free discriminants and this test adds no logging, so it holds **by construction**
  (no assertion needed).
- **Vacuous-pass guard** — A's structured set asserted non-empty before B's negative.

Gates green: `go test -race ./...`, `go vet ./...`, `staticcheck ./...`, `make
substrate-guard`, `go build ./cmd/pyry`, and `go test -tags=e2e -race -count=3 -run
TestTwoPhoneStructured ./internal/e2e/` — **deterministic across 3 runs (~3 s each)**,
A live-receives all four required types (6 envelopes), B coarse-only. The realclaude e2e
column (`make e2e-realclaude`) PASSed in the code-review verification; this ticket adds
nothing to that suite.

**Note (CI coverage gap, surfaced by code-review):** QA's gates do **not** pass `-tags=e2e`,
so this test file is **never compiled or run upstream of code-review** — code-review is the
first place it builds. Code-review verified compile (`go vet -tags=e2e`) and ran it green.

**Code-review NITs (both non-blocking, not fixed):**

- The JSONL trigger is written **once** via `os.WriteFile`, not re-armed. A cross-process
  partial/empty read by fakeclaude's 50 ms poll (landing between the test's implicit
  `O_TRUNC` and its `write`) would append nothing, remove the trigger, and lose the fixture
  → A receives no structured stream → the vacuous-pass guard fires as a **flake**.
  Probability is tiny; the architect explicitly deferred re-arming (spec § Open questions).
  An `os.Rename`-from-temp would make the write atomic **if CI ever shows it** — note, don't
  build preemptively.
- `driveHandshakeNonInteractivePinned` duplicates ~40 lines of handshake boilerplate from
  `driveHandshakeToOpenDaemon`. This mirrors the existing
  `driveHandshakeToOpenDaemonInteractive` (which also copies rather than wraps, because the
  base helper doesn't surface hello_ack capabilities) — consistent with the established
  `e2e` idiom. The local-definition placement (not `relay_v2_handshake_test.go`) also avoids
  the in-flight `feature/449` overlap, mirroring #634's local helpers.

## Out of scope / deferred

- **Per-conversation subscription routing** — the coarse fan-out still reaches *every*
  non-interactive conn (the #589 shape), not a `conversation_id`-subscribed subset. Not a
  leak today (one supervised claude, one cursor, payloads carry `conversation_id` for
  client-side filtering). Inherited #589/#632/#634 deferral to **pyrycode-mobile#336**.
- **Atomic trigger write / re-arm** — deferred (spec § Open questions). The once-write is
  fine; harden to `os.Rename`-from-temp only if CI flakes.
- **The structured-stream-capable harness this builds also unblocks the live two-phone
  reconnect-replay e2e** that [#647]/[#649] deferred on the *same* gap (fakeclaude emits no
  structured JSONL). Their `internal/relay` unit tests remain the behaviour oracle; a future
  slice can now lift the `PYRY_FAKE_CLAUDE_JSONL_TRIGGER` + sessions-dir-alignment +
  pre-create-JSONL recipe here to drive that e2e live. (Their per-ticket notes stay frozen;
  this is a forward pointer.)
