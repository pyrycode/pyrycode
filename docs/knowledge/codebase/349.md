# Ticket #349 â€” `agent-run/jsonl/tail`: fsnotify watcher wiring the line reader to the encoded project dir

New sibling subpackage `internal/agentrun/jsonl/tail` that ties #347 (`EncodeProjectDir`) and #348 (`jsonl.Reader`) together. Waits for `~/.claude/projects/<encoded-cwd>/<sid>.jsonl` to appear via fsnotify, opens it (with bounded retry to absorb the CREATE-before-ready race), seeks to `StartOffset`, and drives `jsonl.Reader` until the deterministic end-of-turn signal fires or the caller cancels. Two exported types (`Config`, `Watcher`), three exported funcs/methods (`New`, `Run`, `Offset`). ~150 production LOC, 6 tests. See [features/jsonl-tail-watcher.md](../features/jsonl-tail-watcher.md) for the full surface.

## Implementation

- `internal/agentrun/jsonl/tail/watcher.go` â€” single production file, 294 LOC. Package doc-comment carries the verbatim `MUST NOT log file contents at any layer` stance from the parser package and `internal/agentrun/trust.go`. Constants: `probeRetryDelays = []time.Duration{0, 50*ms, 200*ms}` mirroring `internal/sessions/rotation/watcher.go`'s schedule (250ms total worst-case retry).
- **Project-dir existence resolved by `MkdirAll(0o700)` at `New` time** â€” placed BEFORE `fsnotify.Add(dir)` so the watch always lands on a real directory. Same shape as `rotation.New`. Idempotent if claude already created the dir; the architect-flagged "may not exist on first run" question is closed with no two-level watch needed.
- **Single goroutine, no inner errgroup.** `Run` defers `fsw.Close()` and the opened file's close; New leaves both pre-armed (fsnotify added, no file open). Caller MUST call Run for cleanup â€” same contract as `rotation.Watcher`. The lifecycle has only three state transitions: pre-stat â†’ file-not-yet-opened (event-loop) â†’ file-opened-and-draining; ev.Op is matched against `Create` for the first transition, `Write|Create` for further drains.
- **Initial `os.Stat(expectedPath)` covers the pre-existing-file race.** `fsnotify.Add` was called in New, so any CREATE that races with the stat is queued in `fsw.Events` and replayed by the loop â€” no double-open. If stat returns `fs.ErrNotExist`, the event loop takes over; any non-ENOENT stat error bails.
- **`openWithRetry` for the CREATE-before-content race.** Walks `probeRetryDelays`, opening with `os.Open` each step. `ctx.Done` during a `time.After` sleep returns `ctx.Err()` so cancellation interrupts mid-retry. Retry exhausted with `fs.ErrNotExist` is logged Warn and the event loop continues (a later WRITE may surface the file); non-ENOENT errors bail.
- **`drain` is the synchronous bridge to `jsonl.Reader`.** Loop on `reader.Next()`: each `(Event, nil)` invokes `cfg.OnEvent`; `ev.EndOfTurn` then invokes `cfg.OnEndOfTurn` and returns `(done=true, nil)`. `io.EOF` returns `(false, nil)` â€” file hasn't grown yet, wait for the next fsnotify event. Any other error wraps as `tail: read %s: %w` (errors.Is-able through to `jsonl.ErrLineTooLarge`).
- **Resume contract wired through two places** â€” `f.Seek(StartOffset, io.SeekStart)` on open AND `jsonl.Config.StartOffset` on reader construction. Together they make `Offset()` report absolute file positions for durable resume; the watcher itself doesn't persist anything (caller responsibility).
- `internal/agentrun/jsonl/tail/watcher_test.go` â€” 6 tests, all `t.Parallel()`. Test scaffolding lives at the top of the file: `eventRecorder` (mutex-wrapped events + endTurns counter), `startWatcher(t, cfg)` helper (Run in a goroutine + `t.Cleanup` cancels and asserts exit within 2s), `expectedEncodedDir(t, home, workdir)` (calls `agentrun.EncodeProjectDir` so tests stay synchronised with the production encoder), `waitForEndOfTurn(t, rec, deadline)` (poll with 20ms sleep â€” same pattern as `internal/sessions/rotation/watcher_test.go`), `writeLineByLine(t, path, lines, delay)` (O_CREATE|O_WRONLY|O_APPEND between writes, Sync after each).
- **Test rows pin each AC scenario** â€” see the table in [features/jsonl-tail-watcher.md#tests](../features/jsonl-tail-watcher.md#tests). Key coverage shapes: validation row-table for the four New-time errors; darwin-only realpath assertion that the watched dir-name is `-private-var-folders-...` (NOT `-var-folders-...`); late-create that writes a 3-line synthetic JSONL (tool_use + user + end_turn) AFTER Run is in its event loop with a 50ms head-start sleep so fsnotify is armed; existing-file that pre-creates the JSONL before Run starts and asserts the initial-stat path picked it up without any fsnotify event needed; fixture-integration that replays `internal/agentrun/jsonl/testdata/clean.jsonl` line-by-line via `bufio.Scanner` (buffer cap `1<<20` initial, `16<<20` max â€” covers the parser's `maxLineBytes`) with 3ms inter-write delay; context-cancellation that asserts `errors.Is(err, context.Canceled)` within 2s of cancel.

## Patterns established

- **fsnotify wrapper subpackage shape.** Mirrors `internal/sessions/rotation/` exactly: a small, single-purpose subpackage with one production file + one test file; one exported `Watcher` type wrapping `*fsnotify.Watcher`; `MkdirAll` before `Add`; single `Run` goroutine with `select { ctx.Done / fsw.Events / fsw.Errors }`; bounded retry-probe on CREATE-before-ready; `Run` defers both the fsnotify close and any opened file's close; caller MUST call Run for cleanup. New filesystem-watcher subpackages should copy this shape rather than reinventing.
- **Initial-stat-then-event-loop covers all timing paths.** `fsnotify.Add` in the constructor + `os.Stat` as the first action in Run + event-loop afterward is the canonical pattern for "watch a file that may already exist OR may appear later". The pre-existing-file CREATE race is structurally absorbed by queuing in `fsw.Events`; the late-create race by the event loop; spurious CREATE by retry-then-continue. Three races, one shape.
- **`MkdirAll(0o700)` before `Add` resolves "may-not-exist parent" without a two-level watch.** Cheaper than dynamically adding watches on parent dir + waiting for child-dir CREATE. Trade-off: pyry creates the dir if claude hasn't yet, which is empirically fine for claude. Apply this pattern whenever the watched dir is owned-by-some-other-process and may be lazily created.
- **Callback API + caller-owned offset persistence over channel API.** `OnEvent` / `OnEndOfTurn` invoked synchronously from the Run goroutine, `Offset()` callable after Run returns. Callers that want a channel can send from inside the callback; the watcher stays single-threaded. Avoids channel-buffer sizing, slow-consumer back-pressure, and goroutine-leak audits.
- **Subpackage import direction discipline.** `tail` imports `jsonl` and `agentrun`; neither imports `tail`. Keeps the parser (`jsonl`) testable against `io.Reader` fixtures with zero fs / fsnotify dependencies â€” same discipline as keeping `agentrun` pure-stdlib pre-spawn primitives. Future filesystem-orchestration code over the parser belongs in `tail` or another sibling.

## Lessons learned

- **Caller-MUST-call-Run is a quietly-load-bearing contract.** `fsnotify.NewWatcher` allocates an inotify/kqueue resource immediately in `New`; the only cleanup is `fsw.Close()` and the only call site is `defer` inside `Run`. If a caller constructs a `Watcher` then drops the reference without calling Run, the resource leaks until process exit. Same as `rotation.Watcher`. Mitigation in tests is `t.Cleanup` + 2s exit deadline in `startWatcher`. For production callers, the doc-comment is the only enforcement. A future hardening could move `fsnotify.NewWatcher` into Run, but that complicates error handling (Run becomes the first place a "couldn't watch the dir" failure surfaces) â€” defer until the first observed leak.
- **`probeRetryDelays = {0, 50ms, 200ms}` is a copy-paste from rotation, not a measured value.** The rotation watcher's schedule was tuned for the "claude opens a new JSONL after `/clear`" race; this watcher faces the "claude opens its session JSONL after spawn" race. Both are the same family of failure (CREATE before content). Empirically the 0ms first attempt usually succeeds; the 50ms + 200ms tail cover slow filesystems. If a future operator hits a real exhaustion, the warn-log carries enough context (`path`) to retune. Don't change the schedule without evidence.
- **Tests that depend on fsnotify timing need an explicit arm-up sleep.** `TestWatcher_LateCreate` and `TestWatcher_FixtureIntegration` both `time.Sleep(50 * time.Millisecond)` after `startWatcher` returns but before the first write. Without it, the first write can race the watcher's transition from `New` returning to `Run`'s `os.Stat` + `select` arming â€” on a fast machine, the file appears, the stat misses (file isn't there yet because the test hasn't written it), then the CREATE event fires while the goroutine hasn't entered `select`; depending on scheduling, the test passes ~95% of the time. The explicit sleep is a structural fix: the test guarantees the watcher is in `select` before the file appears. Same idiom as `rotation_test.go`.
- **`os.Stat` is preferred over a blind `os.Open` for the initial existence check.** Open-then-handle-ENOENT works too but allocates an `*os.File` we'd just close on the miss path. Stat is cheap and the success path drops into `openAndDrain` which does its own (retried) Open â€” no race, no double-close, no fd leak on the miss path.

- Max-turn enforcement â€” consumes `OnEvent` + `AssistantCount` + `OnEndOfTurn` from above.
- Two-level watch (`~/.claude/projects/` + per-session) â€” deferred unless claude is observed to refuse writing into a pre-created dir. Not observed; per the project's "Evidence-Based Fix Selection" principle, defer.
- Multi-session multi-watcher orchestration â€” one `Watcher` per session is the documented unit; aggregation belongs to the verb.
