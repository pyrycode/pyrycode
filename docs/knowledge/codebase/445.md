# Ticket #445 — `internal/relay` v2 inner-frame handshake + token gating

Lands the binary-side per-`conn_id` state machine for [Mobile Protocol v2](../../protocol-mobile.md): receive `noise_init`, validate the AEAD-sealed device-pairing token piggybacked in IK message 1 early-data, emit `noise_resp` (Noise_IK message 2 carrying `hello_ack`) on success, and refuse every out-of-state or unauthenticated inner frame at the WS-close layer. No `noise_msg` application dispatch — the open-state handler chain in `internal/relay/handlers/` is **never reached** from any code path added in this slice (the load-bearing invariant the gating test pins). Follow-on to [#433](433.md) (`internal/noise` Responder wrapper); preconditioned by [#432](432.md) (QR `server_static_pubkey`) and [#438](438.md) (`internal/keys` static keypair). Hand-off to **#446** for the open-state handler wiring and the production cutover in `cmd/pyry/relay.go`.

## Implementation

- **New file `internal/relay/v2session.go`** holds the state machine, the two new exported close-code constants, the wire types' decode helper, and the AEAD-sealed-error-then-close emission helper (~600 LOC including doc-comments and structured-log fields). One file by design — the manager's state transitions, the close-code constants, and the decode/seal helpers are co-located so the contract is readable in one buffer and the AEAD-then-close atomicity is structurally obvious.
- **Two new exported close-code constants** (`StatusProtocolMismatch = 4421` / `StatusHandshakeFailure = 4426`) declared alongside the file's preamble, mirroring `auth.go`'s `StatusUnauthorized = 4401` style. 4401 is **reused unchanged** from `auth.go` — the v2 manager does NOT redefine an aliased constant; it imports the same symbol the v1 gate uses.
- **Six exported types** keeps the slice at the size:S boundary: `V2SessionState` enum + `V2Session` (per-conn state) + `V2SessionConfig` (manager parameterisation) + `V2SessionManager` + `NewV2SessionManager` constructor + `(*V2Session).State()` accessor + `(*V2SessionManager).Run(ctx)` driver. The `V2Session` fields are unexported; same-package tests in `v2session_test.go` reach in directly (no `// for tests only` exports needed).
- **`V2SessionState` has four states**: `V2StateAwaitingInit` → `V2StateHandshakeComplete` → `V2StateOpen` → `V2StateClosed`. The `handshakeComplete` substate is **observably distinct** from `open` even though both can be set inside the same `noise_init` handler back-to-back. The externally-controlled `state` field exists so the gating test pins the "handler chain unreachable from `handshakeComplete`" invariant deterministically (AC #4) and so any future refactor that splits the dispatch loop cannot silently remove the invariant. `State()` is a plain field read — atomic-/mutex-promotion is deferred until a cross-goroutine reader appears (Open Question 3 in the spec).
- **New file `internal/protocol/v2envelope.go`** (~28 LOC) carries the `V2Version = 2` const, the three `TypeNoise{Init,Resp,Msg}` discriminator constants, and the `InnerFrameV2` struct. Pure data; no methods, no validation. The `internal/protocol` package is the canonical home for wire-format types — v1's `Envelope` lives there and the v1 `Type*` constants live in `codes.go`; v2 follows the same pattern. Shape-checking happens at the manager (`decodeInnerFrameV2`).
- **One-line modification to `internal/protocol/handshake.go`**: appends `Token string \`json:"token,omitempty"\`` to `HelloClientPayload`. v1 round-trip stays byte-identical (`omitempty` + absent value), so existing v1 fixtures and `relay_auth_test.go` are unaffected. The field is the in-band carrier of the device-token under v2 — the spec moves the token from the WS-header / routing envelope into AEAD-sealed early-data (`docs/protocol-mobile.md` § Authentication line 420). A SECURITY doc-comment on the field explicitly names it plaintext credential material and forbids logging at any level.
- **The v1 `RoutingEnvelope.Token` field is NOT removed.** The v1 dispatcher still consumes it; v2's manager deliberately ignores it (spec line 600 says binary MUST ignore in v2). Removal lives in the eventual v1→v2 hard-cutover slice.
- **Maximum decoded `Data` length is `maxNoisePayloadBytes = 65535`** (the Noise framework's per-message limit, `docs/protocol-mobile.md` § Wire shapes). Enforced at the JSON-decode boundary in `decodeInnerFrameV2` so oversized payloads never reach `Responder.ReadInit`. Outer `RoutingEnvelope` byte cap is inherited from `internal/transport`'s 1 MiB `SetReadLimit` ceiling.
- **Single dispatch goroutine.** `Run` is the only goroutine the manager owns; it reads `cfg.Frames`, looks up `m.sessions[env.ConnID]` (creating the session lazily on the first frame for a conn_id, in `V2StateAwaitingInit`), and processes the frame synchronously. `m.sessions` is mutated exclusively by `Run`; no lock. Intentionally simpler than `internal/dispatch.Dispatcher`'s per-conn fan-out — v2 in this slice runs no application handlers, every operation is a synchronous Noise call (~100 µs) or sub-µs JSON decode, so single-goroutine is correct and obviously safe. Per-conn fan-out is a #446 concern.
- **`V2Session` carries no mutex.** `flynn/noise`'s `CipherState` holds a mutable 64-bit nonce counter; concurrent access would be UB. The per-conn dispatch loop IS the lock — adding a mutex would mask the structural ownership contract.
- **AEAD-error-then-close is one outbound envelope.** The 4401 token-failure path AND the gating-row noise_msg-in-handshakeComplete path both emit the sealed `auth.invalid_token` error envelope **and** the `CloseCode = 4401` in a **single** `RoutingEnvelope` via `Outbound`. This is the atomicity claim: the phone observes the error envelope before the WS close, because `(*relay.Connection).Send` writes one WS frame per call and the relay processes `CloseCode` after the frame is acknowledged. Two-call sequencing (`Send` then a separate `CloseConn`) would race the relay's output paths. Mirrors v1 dispatcher's pattern at `cmd/pyry/relay.go:38` + `dispatch.go:498-512`'s `Response.CloseCode = outcome.Code`.
- **`Outbound` is a function, not a channel.** Two reasons: (1) production wiring forwards to `(*relay.Connection).Send`, which is already function-shaped and returns `transport.ErrDisconnected` / `ErrNotConnected` that the manager should log-and-drop, not pause on; (2) tests substitute an in-memory recorder without standing up a channel. Contract: a non-nil error from `Outbound` is treated as "frame lost, conn now unhealthy" — log at debug, continue. Mirrors v1's `cmd/pyry/relay.go:163` forwarder posture.
- **`noise_resp` from a phone is rejected as a state-machine violation in every state.** Only the binary writes `noise_resp`; a phone-side `noise_resp` is structurally invalid → close(4421). Single dedicated branch in `handleFrame`'s switch instead of a generic "unknown type" fallthrough so the reject-reason log field is specific (`reason=noise_resp_from_phone`).
- **Lazy `Responder` construction.** `noise.NewResponder(cfg.StaticPriv)` runs at the **first** `noise_init` frame for a conn (not at session-creation time, not at `NewV2SessionManager`). Why: it lets `awaitingInit` sessions accept zero key-derivation cost for connections that never send a frame (broken phones, exploration probes). The cost paid is one X25519 key derivation per real handshake — the same cost we'd pay anyway.
- **`noise_msg` in `V2StateOpen` is a silent drop.** The gating row of the table (`noise_msg` in `handshakeComplete`) is exercised by tests; the open-state branch in this slice is `_ = inner; return`. Application dispatch lives in #446. The handler chain in `internal/relay/handlers/` is unreached structurally — `handleNoiseMsg` has no edge to it.
- **`fakerelay` adds a `/v2/server` mux route.** One-line `mux.HandleFunc("/v2/server", s.handleBinary)` at `internal/e2e/internal/fakerelay/fakerelay.go:144`. The relay-side wire (binary↔relay `RoutingEnvelope`) is unchanged in v2, so the handler is shared. Phone-side `/v2/client` is NOT registered — tests dial `/v1/client` (the routing wire is shared; v2 only changes the *inner* frame).
- **`fakerelay.binaryRecvPump` treats `null` raw-message frames as nil.** `json.RawMessage` marshals nil as the literal token `"null"` (4 bytes). The close-only 4421/4426 paths emit `RoutingEnvelope{ConnID, CloseCode: code}` with `Frame: nil`, which round-trips through the WS as `"frame": null` — without this guard the fakerelay would attempt to forward `null` as a phone-side WS text payload. Matches the production relay's close-only envelope contract (`connection.go:CloseConn` + `protocol/envelope.go` field docs).
- **`fakephone` grows `SendBytes(data []byte)` / `ReceiveBytes(timeout)` byte-oriented siblings** of `Send(env)` / `Receive(timeout)`. The wire shape inside `RoutingEnvelope.Frame` under v2 is an `InnerFrameV2` (not a `protocol.Envelope`), so the test driver builds the v2 frame as raw bytes and bypasses the envelope marshal/unmarshal. The original `Send` / `Receive` delegate to the byte variants so v1 behaviour is byte-identical.
- **The v2 manager is NOT wired into `cmd/pyry/relay.go`.** The production daemon continues to run the v1 `internal/dispatch.Dispatcher` against `/v1/server`. The two consumers of `Frames()` cannot coexist on a single connection at runtime; this slice's manager is reachable only through test wiring today, and the production cutover is a follow-up — likely #446 (which lands `noise_msg`-in-open-state handler dispatch) or a sibling. The architecture spec § "Wiring" makes this explicit.
- **Log policy mirrors v1's `auth.go` posture.** ACCEPT logs `event=v2.handshake.accept` with `conn_id` + `device_name` (operator-actionable). REJECT logs `event=v2.{handshake.reject.invalid_token,handshake.reject.ik_failure,state.reject}` with `conn_id` + `close_code`. **NO `device_name` on reject** — anti-enumeration of paired-device names from binary logs. `cfg.StaticPriv`, `HelloClientPayload.Token`, raw `Frame` bytes, AEAD ciphertext bytes, plaintext payload bytes, and base64-encoded forms thereof are all MUST-NOT-log at any level (including `%w`-wrapped errors).

## Patterns established

- **AEAD-error-then-close is one routing envelope, not two.** The atomicity comes from `(*relay.Connection).Send` writing one WS frame per call and the relay processing `CloseCode` after the frame is acknowledged. Two-call sequencing (`Send` then `CloseConn`) would race the relay's output paths. The pattern: when a wire-level "send this frame and then close" sequence must be atomic, the routing envelope `{Frame, CloseCode}` IS the atomicity primitive — co-emit, never split. Same shape as the v1 dispatcher's `outcome.Code` → `Response.CloseCode` mapping; this slice extends the pattern to AEAD-sealed contents.
- **The single dispatch goroutine IS the lock.** `V2Session` holds two `*noise.CipherState` values whose embedded nonce counters are not goroutine-safe; the manager's `Run` is the sole writer of session state across all conns. A mutex would mask the structural ownership contract (it would let a second goroutine attempt mutation and silently corrupt the nonce). The pattern recurs from [#433](433.md)'s `CipherState`-is-single-goroutine contract — both slices push the serialisation responsibility to the per-conn-dispatch boundary rather than per-CipherState locks. When per-conn fan-out lands in #446, the per-conn goroutine becomes the new lock — but a single lock per conn, not per CipherState.
- **The transition table is the contract; the code is one decode/seal path.** A single `decodeInnerFrameV2` helper handles every inbound frame's structural validation (JSON, version, type discriminator, base64, size cap), and a single `sealError` helper handles every outbound AEAD-sealed error envelope (`auth.invalid_token` only today, easily extensible). Keeping the decoder/sealer centralised is the reason the gating invariant is *structurally* enforced — there is only one decode path to be wrong and one seal path that can run only when CipherStates exist. The pattern: state-machine cells should reference a single, central code path for each operation class, not duplicate the operation per cell.
- **An observably-distinct intermediate state is a future-proof invariant anchor.** `V2StateHandshakeComplete` is set and immediately advanced (or rejected) inside the same `noise_init` handler in this slice — no inbound-frame flow naturally observes the state. The substate exists for a different reason: the *gating test* directly assigns it to drive the "handler chain unreachable" row, and future refactors that defer token validation (e.g. network-backed registries) become observable to inbound frames between handshake completion and validation completion. Encoding the gating row in the state machine *today* defines the behaviour for the *future* world, structurally pinning the invariant. The pattern generalises: when a load-bearing security invariant has a "today this is unreachable but tomorrow it might be" shape, encode the unreachable cell now — the gating test becomes the regression guard for any future refactor that adds the missing edge.
- **Lazy responder construction at first-frame.** `noise.NewResponder(cfg.StaticPriv)` runs at the first `noise_init` for a conn, not at session-creation time. Zero handshake-cost for connections that never send a frame (broken phones, exploration probes); one X25519 derivation per real handshake (which we'd pay anyway). The same lazy-init shape applies any time a per-conn cryptographic primitive is expensive to construct relative to the no-frame case. Compare with [#247](features/transport-package.md)'s eager dial-on-Connect — appropriate there because the transport conn IS the resource, here the responder is a derived value of a precious key.

## Lessons learned

- **The architect spec calls out a step-5 reconciliation that bit the implementation.** Spec § *State-machine transition table* step 5 originally read "decode early-data as `protocol.Envelope`; on err OR on `env.Type != TypeHello`: take the handshakeComplete-non-hello path" — i.e. AEAD-seal an `auth.invalid_token` and close at 4401. The spec's own next paragraph then walks that back: at step 5 we have not yet called `WriteResp`, so CipherStates do **not** exist; there is no key under which to seal an error envelope. The reconciled rule is "early-data decode failure or non-hello at handshake-time is a 4421 handshake-layer protocol violation (close-only); the AEAD-sealed-error path is reserved for after step 7 when CipherStates exist." Implementation follows the reconciled rule. The lesson generalises: when a security mandate has a "send sealed error before close" shape, the precondition is *"a key exists to seal under"* — and that precondition has to be checked structurally at every transition that could in principle take the sealed-error path. Document the reconciliation in the spec, in the code comment, and in the lessons file so the next slice that touches the state machine doesn't reintroduce the version that seals without a key.
- **`json.RawMessage` marshals nil as the literal token `"null"`, not as JSON omission.** `RoutingEnvelope.Frame` is a `json.RawMessage`; close-only envelopes (`{ConnID, CloseCode: code}` with `Frame: nil`) marshal to `{"conn_id": "...", "frame": null, "close_code": 4421}` on the wire. Fakerelay's `binaryRecvPump` was reading `env.Frame` and forwarding it to the phone-side `sendCh` — without the `string(frame) == "null"` guard, the phone would observe a `null` WS payload before the WS close. Worth knowing because the production relay's implementation of the close-only contract is "send the close frame with no preceding WS payload"; fakerelay had to learn the same lesson. The lesson generalises: any test double that mediates between two systems and uses `json.RawMessage` to splice frames byte-for-byte will see the `"null"` literal and must decide whether to treat it as "forward nothing" or "forward the literal bytes" — explicit handling beats an accidental wire-shape divergence.
- **The fakephone byte-oriented API is the natural fit for protocol-versioned test drivers.** v1 tests construct a `protocol.Envelope`, call `Send(env)`, and let the harness marshal. v2 tests construct an `InnerFrameV2`, marshal manually, and call the new `SendBytes(data)`. Extracting the byte path from `Send` / `Receive` and reshaping the envelope path as a thin wrapper kept v1 tests byte-identical and v2 tests minimal. The lesson generalises: when a protocol version changes only the inner wire shape but reuses the outer transport, the harness's API should expose both the byte boundary and the typed boundary — versioned drivers consume the byte API; type-checked test code consumes the typed wrapper.
- **An "S" slice with seven type exports lands at the size:S boundary, not over.** The architect spec's § *Scope self-check* counts: 5 exported types (`InnerFrameV2`, `V2SessionState`, `V2Session`, `V2SessionConfig`, `V2SessionManager`) + 2 constructors/methods (`NewV2SessionManager`, `Run`) + 1 accessor (`State`) + 5 constants (`V2Version`, three `TypeNoise*`, `StatusProtocolMismatch`/`StatusHandshakeFailure` — wait, that's six) at the S ceiling. Constants don't count toward the "new exported types" budget; the type count is what gates the size estimate. Worth knowing for the next slice's scope self-check: count types and constructors separately, and use constants liberally to keep the type budget loose. (The dispatcher hit max_turns on this slice, but the surface count was on-budget — the time cost came from the e2e test harness extensions, not from the production surface.)

## Out of scope (filed elsewhere or deferred)

- **`noise_msg` application dispatch in the `open` state** — **#446**. Wires the `internal/relay/handlers/` chain to v2, hooks the manager into `cmd/pyry/relay.go`, and decides the v1↔v2 daemon-side selector (URL path or config flag).
- **Re-key timer + `rekey_request` envelope handling** — **#435**.
- **Pre-flight release-flag gate** for the production cutover — already shipped as [`pyry pair preflight`](pyry-pair-command.md) (#436); the actual cutover gating slice is a follow-up that flips `cmd/pyry/relay.go` to dial `/v2/server`.
- **Per-phone-conn 10s handshake timeout** (`docs/protocol-mobile.md:366`) — requires a relay→binary "phone connected" signal that does not exist in the v2 wire today. Tracked for a future protocol amendment + binary slice. The state machine in this slice processes frames synchronously, so a phone that sends nothing causes no resource use; the bound is "relay-side WS-idle close", which the relay already enforces.
- **`V2Session` cleanup on phone-initiated WS close** — same root cause; relay→binary "phone disconnected" forward signal does not exist. State entries linger until the binary↔relay leg recycles. Bounded by per-binary-leg lifetime, not a real leak.
- **Production wiring of `V2SessionManager` into `cmd/pyry/relay.go`** — the v1 dispatcher continues to own the wire there today. Cutover decision lives with #446 (handlers) or a sibling.
- **Encrypted echo round-trip e2e and tampered-`noise_msg` AEAD-failure → 4421 teardown** — follow-up slice (#446; no open-state dispatch path exists here to fail).
- **Removal of the v1 `RoutingEnvelope.Token` field** — v1 dispatcher still consumes it; removal lives in the eventual v1→v2 hard-cutover slice.
- **Phone-side and relay-side endpoint changes** — the production relay's `/v2/server` route is its own team's work; only the fakerelay test double is in scope here.
- **`payload_encrypted` flag removal from `internal/protocol.Envelope`** — v2 makes the field structurally redundant per [ADR 024](../decisions/024-noise-ik-mobile-e2e.md); removal is part of the v1 retirement slice, not this one.
