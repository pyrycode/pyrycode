# Ticket #511 — `streamjson.Emitter` consumes `tuidriver.JSONLEntry`

Consumer-side swap on the JSONL → stream-json seam: `streamjson.Emitter.Emit`'s parameter changes from the local `jsonl.Event` shape to `github.com/pyrycode/tui-driver/pkg/tuidriver.JSONLEntry`. Four of the five reads in `Emit` (`Raw → RawLine`, `Kind → Type`, `StopReason → Message.StopReason`, `EndOfTurn → tuidriver.IsEndTurn(entry)`) remap to direct tuidriver accessors; the fifth (Usage) gets a ~25 LOC private `readUsage` helper that walks `entry.Message.Raw["usage"]` as a `map[string]any` — the same shape `encoding/json` decodes into and the same shape tuidriver's package-internal `parseMessage` produces (numeric fields surface as `float64`). The watcher's `OnEvent func(jsonl.Event)` Config field is **deliberately not changed** — `ptyrunner/runner.go`'s closure gains a small `eventToEntry` adapter that wraps `jsonl.Event` into the `JSONLEntry` shape the emitter consumes. Throwaway code: sibling [#512](https://github.com/pyrycode/pyrycode/issues/512) deletes the adapter when the watcher pivots to `tuidriver.TailJSONL` and feeds entries directly. Sibling-ticket of [#510](510.md) (watchdog migration); together they close out the consumer-side migrations against the 2026-05-23 tui-driver pin without touching `budget.Counter`, `selfcheck`, or `internal/e2e/realclaude/fixtures.go` — those remain on `jsonl.Event` until #512 sweeps them.

## Implementation

- **`Emit`'s signature now takes `tuidriver.JSONLEntry`; the function body shape is preserved.** The state-update block at `emitter.go:184-203` keeps its ordering — counters/flags first, then the write — so totals stay consistent on a broken-pipe write. The five field reads remap as: `entry.Type == "assistant"` (was `ev.Kind`), `entry.Message.StopReason` read **inside** the assistant branch with a defensive `entry.Message != nil` guard (was `ev.StopReason` field), `tuidriver.IsEndTurn(entry)` (was `ev.EndOfTurn` field — function call, not field), `readUsage(entry)` returning `(usageBlock, bool)` (was `ev.Usage != nil` gate), and `entry.RawLine` for the byte-passthrough write (was `ev.Raw`). The `append([]byte(nil), entry.RawLine...) + '\n'` shape is unchanged because `RawLine`'s contract ("byte-identical to what parseEntry consumed, with the trailing `\r\n` or `\n` stripped") matches `jsonl.Event.Raw`'s verbatim. The single mutex held for the full body and the sticky `writeErr` short-circuit are untouched.

- **`readUsage` is the private map-walk helper that replaces the typed `*UsageBlock` field gate.** `entry.Message.Raw["usage"].(map[string]any)` plus four `m[k].(float64)` extractions truncated to `int`. The two-return idiom (`u, ok := readUsage(entry)`) preserves "absent vs. present-but-all-zero" semantics — a legitimate zero-cost assistant entry would silently drop from aggregation if the helper inferred presence from "all four fields zero". Four absent paths short-circuit to `(zero, false)`: nil `entry.Message`, nil `entry.Message.Raw`, missing `"usage"` key, non-map value at `"usage"`. The reviewer flagged `entry.Message.Raw == nil` as technically redundant (indexing a nil map returns `(nil, false)` from the assertion regardless) and explicitly kept it as a defensive readability nudge — see the PR #521 code review comment.

- **`usageBlock` mirrors `jsonl.UsageBlock`'s four-field shape internally; the wire shape is unchanged.** The accumulator (`Emitter.aggUsage usageTotals`, `emitter.go:98-103`) is untouched — `readUsage` returns the same four `int` fields the old `*UsageBlock` gate aggregated. Both `usageBlock` and `usageTotals` stay private to `streamjson`; this is internal aggregation state, not API surface. A future `tuidriver.Usage(entry) (usageBlock, bool)` helper is a tui-driver follow-up if a second consumer surfaces — not gating here.

- **`tuidriver.IsEndTurn` is the contractual replacement for `jsonl.Event.EndOfTurn`.** Library docstring + `pkg/tuidriver/jsonl.go:297-305`: `assistant ∧ Message.StopReason == "end_turn" ∧ AssistantText(e) != ""`. Semantically equivalent to the old `EndOfTurn` field computation at `internal/agentrun/jsonl/reader.go:62-64`. No behaviour change at the watcher's `OnEndOfTurn` fan-out — the emitter still sets `endOfTurnSeen = true` on the same set of envelopes.

- **`ptyrunner/runner.go`'s `OnEvent` closure gains the `eventToEntry` adapter; the watcher Config field signature stays.** Path (a) per the spec — `watcher.OnEvent func(jsonl.Event)` is unchanged, the adapter lives at the call site (one `OnEvent` closure body change + ~30-LOC `eventToEntry` helper). The adapter populates `Type` from `ev.Kind`, `RawLine` from `[]byte(ev.Raw)` (direct cast, same underlying slice), and a minimal `Message{StopReason, Raw{"usage": ...}}` for assistant entries with usage — the `Raw["usage"]` sub-map carries the four counter fields as `float64` so `readUsage`'s type-assertion path matches tuidriver's own `parseMessage` output. The alternative path (b) would have forced `budget.Counter.OnEvent` to migrate in this same ticket (the spec explicitly tied them via the shared callback fan-out at `runner.go:368`), fanning out across `budget.go`, `budget_test.go`, and matching watcher_test.go fixtures — out of bounds for an S-sized ticket.

- **Synthetic `Content[]{Text: "x"}` on end-of-turn entries is load-bearing for `IsEndTurn` semantic equivalence.** `tuidriver.IsEndTurn` walks `AssistantText(entry)` which iterates `Message.Content` and requires non-empty text — but the adapter constructs `Message` from `jsonl.Event`'s flat fields without a content slice. On `ev.EndOfTurn == true` the adapter populates a single `ContentBlock{Type: "text", Raw: map[string]any{"text": "x"}}` so `IsEndTurn(entry)` returns the same boolean the old `ev.EndOfTurn` field carried. The single `"x"` byte is arbitrary; any non-empty string passes `AssistantText`'s `!= ""` check. The synthetic content is **never re-emitted** — `Emit`'s byte-passthrough uses `RawLine` not `Content`, so `make e2e-realclaude` byte-equivalence (#506) is structurally preserved. The semantic equivalence is sound because `jsonl.Event.EndOfTurn`'s own contract already required `StopReason == "end_turn" ∧ TextChars > 0`; whenever the adapter sees `ev.EndOfTurn == true`, the source `jsonl.Event` had non-empty text upstream and the synthetic placeholder is faithful to that.

- **`eventToEntry` is throwaway code by design — #512 deletes it.** The adapter exists solely to keep the watcher Config field signature (`OnEvent func(jsonl.Event)`) stable while the emitter migrates ahead of the watcher. After #512 pivots the watcher onto `tuidriver.TailJSONL`, both the adapter AND the entire `jsonl` package disappear in one cut. Documenting "this is throwaway" in the doc-comment is the explicit signal a future contributor needs — without it, the synthetic-content trick reads like a load-bearing invariant when in fact #512 evaporates it.

- **`emitter_test.go` rewrite: two file-scope helpers + four absent-path tests + captured-fixture replay.** The `entry(t, rawLine) tuidriver.JSONLEntry` and `assistantEntry(rawLine, stopReason, usage *usageBlock, endOfTurn bool) tuidriver.JSONLEntry` helpers mirror the structural shape `ptyrunner.eventToEntry` produces — drift between them would silently break ptyrunner-driven coverage. Reviewer NIT flagged that `entry`'s first parameter shadows the conventional `t *testing.T` name (could read as `kind` instead) — kept as-is, not blocking. Four new single-purpose tests (`TestReadUsage_NilMessage`, `TestReadUsage_NoRawMap`, `TestReadUsage_NoUsageKey`, `TestReadUsage_NonMapUsage`) pin each absent path: each emits one assistant entry with the matrix and asserts `tr.NumTurns == 1` (state machine still updates) AND `assertZeroUsage(t, tr)` (the four counter totals all zero). `TestNonMapUsage` is the most defensive — it asserts the `m["usage"].(map[string]any)` assertion doesn't panic on a string value.

- **`TestCapturedFixture_ByteEquivalence` drops `jsonl.NewReader` via a local `lineToEntry` helper.** The captured-fixture replay test (730+ LOC, the load-bearing byte-equivalence assertion for #506) used to feed lines through `jsonl.NewReader(...).Next()` to produce the `Event` shape — which couples the test to a package #512 deletes. The new path constructs `tuidriver.JSONLEntry` values directly via `lineToEntry(line []byte) tuidriver.JSONLEntry`, which duplicates ~25 LOC of tuidriver's package-internal `parseEntry` + `parseMessage` logic: `json.Unmarshal(line, &raw)` → set `Type, Message` from `raw["type"]` / `raw["message"]` → walk `m["content"].([]any)` for each block's `type` field. The duplication is accepted with a one-line comment pointing at the future `tuidriver.ParseEntry` export (tui-driver follow-up). #512's watcher pivot to `tuidriver.TailJSONL` makes both the duplicate AND the helper itself disposable.

- **No `go.mod` bump; same 2026-05-23 tui-driver pin as #509 / #510.** `v0.0.0-20260523181457-c2dcd1e49992` already exports `JSONLEntry`, `EntryMessage`, `ContentBlock`, `IsEndTurn`, `AssistantText` from tui-driver #101. Pre-pickup verification was `go doc github.com/pyrycode/tui-driver/pkg/tuidriver JSONLEntry` (non-empty). #511 is the third consumer of the 2026-05-23 pin (after #509's `SessionJSONLPath`/`WaitForSessionJSONL` and #510's `RunWatchdog`/`ParseSpinner`/`WatchdogOpts`/`DefaultWatchdogTick`).

## Patterns established

- **Throwaway adapter at the call site beats a cross-package signature change when the deleting-ticket is already filed.** Path (a) — keep the watcher's `OnEvent func(jsonl.Event)` Config field, wrap inside the closure — keeps the blast radius for #511 to two files (`emitter.go`, `runner.go`) and ~370 LOC total. Path (b) — change the watcher's Config field signature to `func(tuidriver.JSONLEntry)` — would have pulled `budget.Counter`, `budget_test.go`, and the watcher's own test fixtures into the same diff because the shared callback fan-out at `runner.go:368` ties them together. The throwaway adapter is ~30 LOC; #512 deletes it whole. Generalises whenever a migration spans multiple consumers fed from one fan-out: **migrate the consumers one at a time, absorb the impedance mismatch at the call site as throwaway code, and explicitly document the deletion ticket in the adapter's doc-comment**. The doc-comment is what stops a future contributor from mistaking the adapter for a permanent shim and "improving" it. Compare with [#510](510.md)'s preserved `runWatchdog` signature — same principle (boundary preserved so the next ticket's diff stays minimal), different mechanism (function signature preserved vs. data-shape adapter).

- **A synthetic placeholder in a throwaway adapter is sound only when the source contract already guarantees the placeholder's truth.** The `eventToEntry` adapter populates `ContentBlock{Type: "text", Raw: {"text": "x"}}` whenever `ev.EndOfTurn == true` so `tuidriver.IsEndTurn(entry)` fires. This is sound because `jsonl.Event.EndOfTurn`'s upstream contract at `internal/agentrun/jsonl/reader.go:62-64` already required `StopReason == "end_turn" ∧ TextChars > 0` — whenever the boolean was true on the input, the source line genuinely had non-empty assistant text, and the synthetic single-byte placeholder is a faithful (if degenerate) reconstruction. The placeholder would NOT be sound if the source contract was weaker (e.g. if `EndOfTurn` could be true on an assistant entry with empty text); in that case the adapter would be inventing data that didn't exist upstream. Generalises whenever adapting between two type shapes where the target has a derived predicate (here `IsEndTurn(entry)` derives from `Content + StopReason`) and the source carries that predicate as a precomputed field (here `EndOfTurn bool`): **verify the source contract guarantees the target's structural prerequisites before fabricating placeholder data**; document the equivalence in the adapter's doc-comment so a future contributor doesn't grep for a real source for the `"x"` text and find none.

- **`map[string]any` walks decode JSON numbers as `float64`, not `int` — the type assertion must match the decoder's output, not the wire shape's semantic type.** `readUsage` reads each counter as `m[k].(float64)` then truncates to `int`. The first instinct is `m[k].(int)` because the wire shape's `input_tokens` is an integer-valued field — that compiles, type-asserts cleanly with `ok := false`, and silently returns zero on every entry. The bug is silent: the test that catches it is `TestReadUsage` over real fixture data, not a unit test on the helper in isolation. Generalises whenever extracting numeric fields from a `map[string]any` produced by `encoding/json.Unmarshal`: **always type-assert to `float64`, never to `int`** (even for fields the JSON schema documents as integers — `encoding/json` doesn't know that). The same shape applies to tuidriver's `Message.Raw` because tuidriver itself uses `encoding/json` to populate it (`pkg/tuidriver/jsonl.go:251-266`). The duplicate-parser test helper `lineToEntry` follows the same rule by construction — it's `json.Unmarshal` into `map[string]any`, identical to what tuidriver does upstream.

- **Reviewer-flagged redundancy can be kept for defensive readability when the cost is one line and the explicit check matches the helper's documented absent paths.** The `entry.Message.Raw == nil` check at `emitter.go:231` is technically redundant — indexing a nil `map[string]any` returns a nil `any`, and the subsequent `.(map[string]any)` returns `(nil, false)` regardless. The code-review pass flagged this and explicitly kept it because (a) the helper's doc-comment names "no Message.Raw" as one of the four absent paths the helper handles, and (b) explicit-is-better-than-clever for a helper whose contract is "all four absent paths short-circuit to `(zero, false)`". Generalises whenever a defensive check is structurally redundant but matches a documented contract surface: **keep the redundancy if it makes the contract literal**. The line of redundant code is cheaper than the cognitive load of mapping "this is one of four absent paths" to "but only three are actually checked, the fourth is implied by the second's behaviour". Compare with [#510](510.md)'s deleted `ok && IsThinking` AND — that was also redundant once the library tightened its contract, but the redundancy had no documentation function and was deleted.

## Out of scope (follow-ups)

- **#512 — delete `internal/agentrun/jsonl` and pivot watcher to `tuidriver.TailJSONL`.** Sweeps `budget.Counter.OnEvent(ev jsonl.Event)` → `tuidriver.JSONLEntry`, `selfcheck`'s independent `jsonl.NewReader` consumer, `internal/e2e/realclaude/fixtures.go`'s `type JSONLEntry = jsonl.Event` alias, the watcher's `OnEvent func(jsonl.Event)` Config field, and deletes both `eventToEntry` (in `runner.go`) and `lineToEntry` (in `emitter_test.go`) as the alias/adapter layer evaporates. Captured-fixture replay test simplifies back to a direct line-driver loop after `tuidriver.ParseEntry` is exported (or stays as-is if the helper keeps its purpose).

- **tui-driver follow-up: export `Usage(entry)` accessor.** `readUsage` lives in `streamjson` for #511 because it's the only consumer of `Message.Raw["usage"]` so far. If a second consumer surfaces (likely `budget.Counter` after #512), the helper should be promoted upstream to `tuidriver.Usage(entry) (Usage, bool)` so both consumers share one implementation. The spec explicitly flags this as "not gating here".

- **tui-driver follow-up: export `ParseEntry(line []byte) (JSONLEntry, bool)`.** `emitter_test.go:lineToEntry` duplicates ~25 LOC of tuidriver's package-internal parser. An exported variant of that parser would let the captured-fixture replay test drop the duplication. Filed in the spec under "Out of scope (handled by future tickets)".
