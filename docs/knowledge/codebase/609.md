# Ticket #609 — Delta coalescing on the bridge (per JSONL message or ~250 ms)

The buffer-and-flush layer that makes the [#632](632.md) structured emitter honor
ADR 025 § Phase 2's *"`AssistantText` → coalesced `assistant_delta` (per JSONL
message or ~250 ms, not per token)"* contract. Before this slice the emitter
mapped **every** `turnevent.TextChunk` to its own `assistant_delta`; the
[#615](615.md) producer (live-wired by [#633](633.md)) yields one `TextChunk`
per JSONL `assistant` line, and a single streamed message can surface as several
same-`MessageID` lines — so the phone received a burst of tiny deltas for one
logical message. This slice inserts coalescing between `Handle`'s `TextChunk`
arm and `emit`, keyed on `TextChunk.MessageID` ([#606](606.md)).

**Structured (`assistant_delta`) path only.** It does **not** touch #589's coarse
`message` path ([#634](634.md)), the capability gate, or the per-conn fan-out in
`emit` — those stay exactly as #632 shipped them. Coalescing sits *upstream* of
the gate and makes no routing/trust decision, so the label is **not**
`security-sensitive` (re-affirmed at refinement; mirrors the unlabeled #633
plumbing — see the project's "label tracks design, not lineage" rule).

## Implementation

Three production files, ~50 LOC, **zero new exported types** (two additive
`turnbridge.Config` fields, not types).

### Emitter: the buffer + flush (`cmd/pyry/interactive_turn_v2.go`)

Four new plain fields on `interactiveTurnEmitterV2`, read/written only on the
single `Handle`/flush goroutine — the same unguarded-but-race-free contract as
#632's lifecycle fields:

| field | role |
|---|---|
| `deltaBuf strings.Builder` | accumulated assistant text for the open (un-flushed) delta |
| `deltaMsgID string` | `MessageID` of the buffered text; meaningful only while `deltaBuf.Len() > 0` |
| `deltaConvID string` | conversation cursor captured when buffering began; the flush emits against it |
| `flushTimer *time.Timer` | the ~250 ms latency timer; **owned here, its channel selected by the producer** |

New package-level constant `coalesceWindow = 250 * time.Millisecond` (ADR 025's
"~250 ms"; a single named constant, not exposed as config — no AC asks for it).

- **`flushDelta(ctx)`** — emits the buffered text as **one** coalesced
  `assistant_delta` (current `turnID` + `seq`), advances `seq` **once**, clears
  the buffer, stops the timer. **No-op on an empty buffer**, which is what makes
  every flush trigger (boundary, non-text, turn-end, timer-on-no-pending-text)
  harmless. Reconstructs a synthetic `turnevent.TextChunk{MessageID, Text:
  concatenated}` and reuses the existing `emitMapped` — **no new payload path**
  (`MapEvent` reads only `Text` for `assistant_delta`; `MessageID` is the
  coalescing key, not a wire field). **Invariant: `seq` advances once per emitted
  coalesced delta, NEVER once per buffered `TextChunk`.**
- **`TextChunk` arm — rewritten.** (1) message-boundary flush: if
  `deltaBuf.Len() > 0 && chunk.MessageID != e.deltaMsgID`, `flushDelta` first;
  (2) `transitionTo(responding)` (a no-op mid-text-run — see ordering note);
  (3) capture `wasEmpty` **before** appending, record `deltaConvID`/`deltaMsgID`,
  `deltaBuf.WriteString(chunk.Text)`; (4) `if wasEmpty { flushTimer.Reset(...) }`
  — arm the window from the **oldest** unflushed chunk, never re-arm on a
  same-message append, so a long-streaming message is bounded to one window.
- **Every non-text arm flushes first.** `ThoughtChunk` / `ToolStart` /
  `ToolUpdate` / `TurnEnd` / `Stall` each call `flushDelta(ctx)` as their first
  emitting action (after the `startTurnIfNeeded`/`inTurn` guard, before their
  `transitionTo`/`emitMapped`). This preserves wire ordering: buffered text
  emits before the interleaved `turn_state` / `tool_use` / `tool_result` /
  `turn_end` / `stall` it logically preceded. `TurnEnd` is the turn-boundary
  flush AC#3 requires (delta before `turn_end`, before the `idle` `turn_state`);
  `Stall` keeps its no-lifecycle-mutation shape and only gains the leading flush.

### Producer: a generic flush arm (`internal/turnbridge/producer.go`)

`Config` gains two **additive, optional** fields (zero values = disabled; all
existing call sites and tests compile unchanged):

```go
FlushSignal <-chan time.Time  // selected in drain; each receive invokes OnFlush
OnFlush     func()            // runs on the single Run goroutine, like OnEvent
```

`drain` adds a third `select` arm using the **nil-channel idiom** (a nil
`flushSignal` is never ready, so the disabled path is one branch, no duplicated
select):

```go
case <-p.flushSignal:
    if p.onFlush != nil { p.onFlush() }
```

The producer stays **generic** — it knows "select a flush signal and call back on
the Run goroutine," not "coalesce assistant deltas." All coalescing policy lives
in the emitter.

### Wiring (`cmd/pyry/interactive_turn_stream_v2.go`)

`startInteractiveTurnStreamV2` adds `FlushSignal: emitter.flushC()` and
`OnFlush: func() { emitter.flushDelta(ctx) }` to the `turnbridge.Config` — the
symmetric partner of the existing `OnEvent` closure, both capturing the relay
lifecycle ctx. No other wiring change.

## Patterns established

- **A timer reconciled with a passive single-goroutine emitter — no second
  goroutine, no lock.** This was *the* design wrinkle and the reason the ticket
  was S, not the buffering. The emitter **owns** the `*time.Timer` and arms /
  resets / stops it (from `Handle`/`flushDelta`), but **never reads its own
  channel** — the producer's `drain` select reads `flushTimer.C` and routes the
  fire back into `flushDelta` on the **same single Run goroutine**. So the timer's
  runtime fire is not "a second goroutine touching emitter state," and #632's
  passive no-goroutine/no-mutex/no-atomic contract is preserved **verbatim** —
  `Handle` and the timer-driven flush never overlap (the select processes one arm
  at a time). The reusable shape: *to give a passive single-goroutine consumer a
  timer, let it own the `*Timer` but hand the channel to whatever owns its
  goroutine to select — the consumer drives reset/stop, the owner drives the
  callback.*
- **Reset-on-flush belongs to the component that knows a flush happened.** AC#2
  wants a full window after **every** flush (so a boundary-flush isn't chased by a
  near-empty timer-flush). Only the emitter sees its own message-boundary /
  non-text flushes, so `Reset`/`Stop` live there — the producer is deliberately
  blind to *why* a flush fired. This is why the generic `FlushSignal`/`OnFlush`
  seam (producer-side) and the timer ownership (emitter-side) are split the way
  they are.
- **Go 1.26 timer semantics make the single-goroutine arm correct without a
  channel drain.** `go.mod` is `go 1.26.2`, where after `Stop`/`Reset` returns no
  stale value is delivered on the channel. The legacy `if !t.Stop() { <-t.C }`
  drain is **omitted on purpose** — and would be *wrong* here, since the channel
  is read by the producer's select, not by the resetting code. Invariant the
  whole scheme rests on: **`flushTimer` is armed iff `deltaBuf` is non-empty.**
- **Flush-before-non-text is correctness, not tidiness.** Without it, buffered
  text would emit *after* a `tool_use`/`turn_state` that logically followed it and
  `seq` ordering would break. `transitionTo(responding)` *before* buffering in the
  `TextChunk` arm is safe by construction: during a text run the state is already
  `responding` so it's a no-op; it only emits on the first content of a turn, when
  the buffer is necessarily empty — no `turn_state` ever interleaves between
  coalesced deltas.
- **The flush reuses `emitMapped` — no new wire path, no new error path.** The
  capability gate, per-conn fan-out, marshal/`Push` error contracts, and the
  session-monotonic `Envelope.ID` (`nextID`) are all reached unchanged through
  `flushDelta`→`emitMapped`→`emit`. Coalescing reduces the *count* of
  `assistant_delta` envelopes, not the counter's monotonicity. Assistant text is
  still **never** logged at any level (#632/#589 contract inherited).

## Lessons learned

- **With coalescing, a scripted event sequence that ends on a `TextChunk` leaves
  its final delta buffered** — no flush trigger follows. Several existing
  emitter/wiring tests that asserted a trailing delta needed an explicit `TurnEnd`
  appended (or a direct `flushDelta` call modelling the timer). This is the
  deliberate **"no flush on drain exit"** design choice surfacing in tests, not a
  regression. (From the PR body's Lessons-learned.)
- **The coalescing key assumes a non-empty `MessageID` per logical message.**
  (Code-review NIT, non-blocking.) Two *distinct* logical messages that both
  carried an **empty** `MessageID` would coalesce into one delta (the boundary
  check `v.MessageID != e.deltaMsgID` is false for `"" == ""`). Harmless in
  practice — the #606 producer populates `MessageID` from the JSONL message id for
  every assistant line, so empty ids only appear as unit-test defaults, and even
  then the concatenated text is still correct.

## No flush on drain exit (deliberate scope limit)

When `drain` returns (session restart via channel-close, or ctx cancel) a partial
buffer is **not** force-flushed. On teardown a flush would no-op (cancelled ctx →
`ActiveConns` nil / `Push` returns `ctx.Err()`). On a mid-turn restart the turn is
already broken; the partial text flushes lazily on the next message boundary or
the next timer arm after re-subscription. The ACs do not require flush-on-restart,
and inventing one would defend an unobserved failure (evidence-based fix
selection). The `flushTimer` holds no goroutine and is GC'd with the emitter — no
explicit disposal; an explicit stop on relay teardown would be a one-line wiring
addition if ever wanted.

## Testing

`cmd/pyry/interactive_turn_v2_test.go` (new scenarios) + `producer_test.go` (one
new arm), stdlib `testing`, `-race`. Timer fires are modelled by a **direct
`e.flushDelta(ctx)` call** to stay deterministic (no real-time waits).

- **`SameIDCoalesce`** (AC#1) — `[Text(m1,"Hel"), Text(m1,"lo"), TurnEnd]` → one
  `assistant_delta{"Hello", seq:0}` before `turn_end`.
- **`NewIDBoundaryFlush`** (AC#1) — `[Text(m1,"A"), Text(m2,"B"), TurnEnd]` → two
  deltas `{"A",seq:0}` then `{"B",seq:1}`; the `m2` chunk flushes `m1` first.
- **`TimerFlushMidMessage`** (AC#2) — `[Text(m1,"A"), Text(m1,"B")]`, direct
  `flushDelta`, `[Text(m1,"C"), TurnEnd]` → `{"AB",seq:0}` then `{"C",seq:1}` (a
  long message split across the window keeps one rising `seq`).
- **`FlushBeforeNonText`** (AC#3) — table over tool / thought / stall: the
  `assistant_delta{"A"}` precedes the `tool_use` / `turn_state{thinking}` /
  `stall` in push order.
- **`FlushAtTurnBoundary`** (AC#3) — `[Text(m1,"A"), TurnEnd]` → delta before
  `turn_end`, before `turn_state{idle}`.
- **`OneSeqPerCoalescedDelta`** (AC#3) — three same-id chunks + `TurnEnd` → one
  delta at `seq:0`.
- **`TestDrain_FlushSignalInvokesOnFlush`** (producer) — `&Producer{…,
  flushSignal: ch, onFlush: f}`; one send → `OnFlush` runs on the Run goroutine;
  nil `flushSignal` (existing tests) never fires the arm.

**Existing #632 scenarios re-verified (real work, not a rubber stamp):**
single-text-chunk scenarios are unchanged (flush-before-non-text / flush-at-turn-end
emits the delta at the *same* wire position). `PerTurnSeqReset` given **distinct**
`MessageID`s + trailing `TurnEnd`s (now 4 deltas at seq 0,1,0,1); `InterleaveDeDup`
correctly left untouched (asserts `turn_state` order only); stall/mid-turn
scenarios add an explicit `flushDelta` to model the flush; the no-app-output-log
test still drives the heaviest (`push_err`) path.
