# Ticket #632 — Event-stream bridge: structured turn-event emitter (per-turn lifecycle + `turn_state` machine + interactive-gated fan-out)

The **stateful structured emitter** at the heart of Phase 2 ([EPIC #596],
[ADR 025] § Phase 2). It is the consumer that #627's pure outbound adapter
deliberately left to its caller: it consumes neutral [`turnevent.Event`](../features/turnevent-package.md)s
one at a time, derives the `turn_state` lifecycle **statefully**
(responding / thinking / idle), maps each content event to a v2 wire envelope via
the merged #627 adapter (`MapEvent` / `BuildTurnState`), and fans each envelope
only to v2 conns that were **granted** the `interactive` capability (#626). It
mirrors #589's coarse-path fan-out shape but consumes structured events instead of
PTY chunks, derives `turn_state`, and gates on the `interactive` grant.

It does **NOT** wire the live production producer (sibling [#633] — the emitter is
exercised here against an injected event source so it unit-tests without a live
claude) and does **NOT** touch #589's coarse `message` path (its own sibling slice
re-targets that). **`security-sensitive`** — the capability-gated dispatch on the
internet-exposed mobile surface (#607 deferred the gated-fan-out review onto this
slice).

## Implementation

One new production file, `cmd/pyry/interactive_turn_v2.go` (~254 LOC incl. tests
references; `package main`, all unexported, **zero new exported types** — mirrors
#589). No edits to any existing production file: every reuse seam is referenced,
not modified (`cursorReader` from `assistant_turn.go`; `relay.ActiveConn`/`Push`/
`ActiveConns`; `turnbridge.MapEvent`/`BuildTurnState`/`TurnContext`/`TurnState`/
`State*`; `turnevent.*`; `protocol.Envelope`; `conversations.NewID`).

### `interactiveBroadcaster` — the consumer-declared fan-out surface

A new minimal interface, **distinct** from #589's `v2Broadcaster` (which uses the
capability-agnostic `ActiveConnIDs`):

```go
type interactiveBroadcaster interface {
    ActiveConns(ctx context.Context) []relay.ActiveConn   // #626 capability-aware snapshot
    Push(ctx context.Context, connID string, env protocol.Envelope) error // #571 sealed delivery
}
```

`*relay.V2SessionManager` satisfies it. Declared at the consumer (CODING-STYLE) so
the unit tests drive a fake without a real manager.

### `interactiveTurnEmitterV2` — a single-goroutine state machine

Holds the broadcaster, the cursor reader, the logger, and the **lifecycle state** —
all plain fields, **no atomic, no mutex** (read/written only on the single goroutine
that calls `Handle`):

| field | role |
|---|---|
| `inTurn bool` | whether a turn is currently open |
| `turnID string` | current turn's id, minted at turn start |
| `seq int` | per-turn assistant-delta counter; **reset to 0 at each turn boundary** |
| `currentState turnbridge.TurnState` | last-emitted `turn_state`, for transition de-dup |
| `nextID uint64` | **session-monotonic** envelope-ID counter; **never reset across turns** |

### `Handle(ctx, ev)` — the entry point

Driven one event at a time. Reads the conversation cursor once
(`sup.CurrentConversation()`); on `""` drops the event with a DEBUG log and returns
(mirrors #589's no-cursor drop). Otherwise it type-switches:

| `ev` concrete type | Lifecycle action | Emitted envelope(s), in order |
|---|---|---|
| `ThoughtChunk` | `startTurnIfNeeded`; `transitionTo(thinking)` | `turn_state{thinking}` (if transitioning). **Thought text never forwarded.** |
| `TextChunk` | start; `transitionTo(responding)`; `MapEvent`→emit; `seq++` | `turn_state{responding}` (if transitioning), then `assistant_delta{…, seq}` |
| `ToolStart` | start; `transitionTo(responding)`; `MapEvent`→emit | `turn_state{responding}` (if transitioning), then `tool_use{…}` |
| `ToolUpdate` | start; `transitionTo(responding)`; `MapEvent`→emit | `turn_state{responding}` (if transitioning), then `tool_result{…}` |
| `TurnEnd` | if `inTurn`: `MapEvent`→emit; `transitionTo(idle)`; `endTurn` | `turn_end{stop_reason}`, then `turn_state{idle}` |
| nil / unknown | drop, DEBUG log | — |

- **`startTurnIfNeeded`** — no-op if `inTurn`. Else mint a turn id via
  `conversations.NewID()` (UUIDv4 / `crypto/rand`, the same generator #589 uses for
  `MessageID`); on its rare error, WARN and return `false` (turn stays closed, next
  event retries). Sets `seq=0`, `inTurn=true`, `currentState=""` (no state emitted
  yet).
- **`transitionTo(ctx, convID, state)`** — de-dup: returns if `currentState == state`.
  Else sets `currentState`, calls `BuildTurnState(convID, state)`, and emits. **State-
  change-based emission** is a superset of "first content → responding" and naturally
  handles interleaving (`thinking → text → thinking` re-emits each transition).
- **`endTurn`** — `inTurn = false`. The next content/thought event re-mints a fresh
  turn id and resets `seq`/`currentState`.
- **`seq++` happens ONLY after a `TextChunk` emit**, so tool / turn_end events carry
  the current seq (ignored by their #627 mappings) and never advance it.

### `emit(ctx, typ, payload)` — the fan-out primitive (the ~25-LOC #589 echo)

The one place envelopes reach the wire, with the envelope-ID policy **mirrored from
#589 exactly**:

1. `json.Marshal(payload)` once; on error, DEBUG log (no payload, **no
   `err.Error()`** — `encoding/json` echoes invalid input bytes into its error
   string, the #589 lesson) and return.
2. Snapshot `bcast.ActiveConns(ctx)` **fresh per envelope** (a conn that joined
   mid-turn is included next emit; a dropped conn is absent or surfaces as a `Push`
   error).
3. For each conn: **`if !c.Interactive { continue }`** — the capability gate (see
   § Security) — then `e.nextID++`, build the `protocol.Envelope{ID, Type, TS, Payload}`,
   and `Push`.
4. A per-conn `Push` error: if `ctx.Err() != nil` return (teardown); else DEBUG log
   (`conn_id`, `env_id`, `conversation_id`, `turn_id`, `err`) and `continue` — a
   dropped conn must not abort the turn for the others (AC#4).

### Envelope-ID policy (AC#3) — preserved from #589

`nextID++` runs **per conn per envelope**, so the same logical envelope gets a
distinct `env.ID` on each conn. It is **session-monotonic with no reset across
turns** — each conn still sees a strictly-increasing subsequence. This is the basis
for #611's mid-turn-reconnect resync and is deliberately preserved on the structured
path. Promotion to a manager-owned per-session counter is #611's call, not this
slice's (inherits #589's open question).

`make check` (vet + `-race` + staticcheck) and the real-claude e2e suite green
(`ok internal/e2e/realclaude 148.249s`). Code review **PASS**, one non-blocking NIT.

## Patterns established

- **The pure-mapper / stateful-consumer seam, realised.** #627 supplies the
  *builder* (`MapEvent`/`BuildTurnState`); this slice owns the *decision to call it*
  — the turn-id mint, per-turn `seq`, and responding/thinking/idle derivation. The
  ownership boundary #627's doc named ("the consumer owns the decision to call
  `BuildTurnState`") is exactly the line this emitter implements. `MapEvent` drops
  `ThoughtChunk`, so thinking surfaces **only** as a `turn_state{thinking}`
  transition — the thought text is never mapped and never forwarded.
- **State-change emission de-dups for free.** Emitting a `turn_state` only when
  `currentState` actually changes means no duplicate same-state envelope, and the
  interleave `thinking → responding → thinking → responding` falls out without a
  special case. A "first content → responding" rule would have needed extra state;
  "transition on change" is the smaller, more general primitive.
- **Capability-gate as a single explicit fail-closed check, two deterministic nets
  behind it (`security-sensitive`).** The load-bearing boundary is *which conns
  receive the structured stream*; it is the one line `if !c.Interactive { continue }`
  in `emit`, backed by two different-fabric deterministic gates the emitter does not
  re-implement: `ActiveConns` enumerates only `V2StateOpen` (token-authenticated)
  sessions (#626/#588), and `Push` independently re-gates `V2StateOpen`
  (`ErrSessionNotOpen`, #571). The emitter never constructs a `conn_id` from event
  data — it only iterates the manager's snapshot — so hostile claude output can never
  redirect delivery. Belt-and-suspenders, different fabric (the #588/#589 pattern,
  inherited verbatim).
- **A passive state machine spawns no goroutine.** Contrast #589, which needed a
  buffered queue + `Run` goroutine because its source (`Bridge.Write` → observer) ran
  on the PTY-drain goroutine and a slow fan-out would wedge claude. Here the source is
  the producer's Run goroutine draining `Session.Events()`; a slow consumer does
  **not** wedge claude (tui-driver drains the PTY independently and `TailJSONL` reads
  a file — #593/#615), so no decoupling queue is required. Zero goroutines spawned →
  nothing to leak (strictly better than #589).
- **No app output logged, ever (AC#5).** Every log branch carries only the event-type
  discriminant (`eventKind` returns the variant *name*, never content), `conversation_id`,
  `turn_id`, `env_id`, `conn_id`, and `Push`'s transport-sentinel `err`. Assistant
  text, thought text, and tool title/input/result summaries are never logged at any
  level. The marshal-error branch omits both the payload and `err.Error()`. The
  per-conn `Push`-error branch logs `err` verbatim — a **deliberate carve-out**
  (documented so code-review does not flag it as a regression): `Push` returns only
  transport sentinels (`ErrConnNotFound`/`ErrSessionNotOpen`) or seal/marshal
  sub-errors over the already-validated `json.RawMessage` this emitter produced, so no
  app content is reachable through it.

## Lessons learned

- **`gofmt` re-aligns a contiguous block of trailing field comments together**, so a
  struct that mixes long and short field-comment lines needs `gofmt -w` before commit
  even when each line looks individually fine. (From the PR body's Lessons-learned.)

## Concurrency model

- **No goroutine is spawned by this slice** — see Patterns above. No queue, no `Run`,
  no cleanup func (contrast #589).
- **`Handle` is not safe for concurrent use; it is designed for the producer's single
  Run goroutine**, which invokes `OnEvent` serially (turnbridge `producer.go`
  contract). All emitter state (`inTurn`/`turnID`/`seq`/`currentState`/`nextID`) is
  therefore read/written on one goroutine — no atomics, no mutex (mirrors #589's plain
  `nextID`). **Named assumption for [#633]:** the wiring closure must invoke `Handle`
  from that one goroutine only — concurrent invocation would race the unguarded
  counters.
- **`ActiveConns`/`Push` funnel through the manager's single Run goroutine**, each
  with a `ctx.Done` escape arm. A `Handle` in flight during teardown passes a
  cancelled ctx → `ActiveConns` returns `nil` (fan out to nobody) and `Push` returns
  `ctx.Err()` → `emit` returns early. The emitter never blocks on a winding-down
  manager.

## Testing

`cmd/pyry/interactive_turn_v2_test.go` (new, `package main`, stdlib `testing` only,
`-race`). Drives `Handle` with scripted `[]turnevent.Event` sequences against a
`fakeInteractiveBcast` (records `(connID, env)` pushes, supports a per-conn injected
`Push` error and a scripted-per-call `ActiveConns` snapshot) and reuses #589's
package-level `stubCursor` / `testConvID` / `discardLogger` helpers (from
`assistant_turn_test.go`, same `package main`). Scenarios:

- **`TransitionOrder`** (AC#2) — `[Thought, Text, Tool, TurnEnd]` →
  `turn_state{thinking}`, `turn_state{responding}` + `assistant_delta`, `tool_use`,
  `turn_end` + `turn_state{idle}` in exact order.
- **`InterleaveDeDup`** (AC#2) — `[Thought, Text, Thought, Text]` →
  `thinking, responding, thinking, responding` (proves no duplicate same-state emit).
- **`PerTurnSeqReset`** (AC#1) — two turns × two `TextChunk`s → `assistant_delta.seq`
  is `0,1` in each turn; the two turns carry **distinct, canonical UUIDv4** turn ids
  (`conversations.ValidID`).
- **`MonotonicEnvIDAcrossTurns`** (AC#3) — `env.ID` strictly increasing across the
  turn boundary, no reset.
- **`FanOutOnlyInteractive`** (AC#4 + § Security) — mixed snapshot `{a:interactive,
  b:non-interactive}` → every push targets `a`, none target `b`, across every envelope
  kind.
- **`MidTurnJoin`** (AC#4) — `{a}` for the first event, `{a,b}` after → `b` receives
  only from the second event onward, never the pre-join `thinking`.
- **`PushErrorDoesNotAbortTurn`** (AC#4) — three interactive conns, the middle one's
  `Push` returns `ErrConnNotFound` → the other two still receive every envelope.
- **`NoAppOutputLogLeak`** (AC#5) — a full turn carrying distinctive thought /
  assistant / tool-title / tool-input / tool-result strings, with the push_err DEBUG
  branch fired for every envelope (most log-heavy path) → none of those substrings
  appear in any log line; thought text never appears in any pushed `env.Payload`.
- **`DropsWhenCursorEmpty`** (AC#1) and **`TurnEndOutsideTurnDropped`** — the two drop
  guards.

No e2e in this slice (no production caller until [#633]); the unit scenarios are the
security/behaviour oracle. **Code-review NIT (non-blocking):** `Handle`'s nil/unknown
`default` arm has no test — the `turnevent.Event` sum type is sealed (unexported
marker), so "unknown" is unconstructable outside the package and only a `nil` event
can reach the arm; a purely defensive guard.

## Out of scope (deferred to siblings)

- **Production wiring** — constructing the `turnbridge.Producer` with
  `NewSessionSubscriber` + the JSONL resolver + live-`/clear` re-subscription,
  attaching `OnEvent: func(ev){ emitter.Handle(relayCtx, ev) }`, and the foreground
  gate / cleanup in `cmd/pyry/relay.go`. **Sibling [#633].** (The ctx-less `OnEvent`
  seam is bridged by a closure capturing the relay lifecycle ctx — keeping
  `context.Context` out of the emitter struct.)
- **Retiring #589's coarse `message` path** — its own sibling slice re-targets that;
  untouched here.
- **Backpressure / delta coalescing / droppable-delta policy** and **mid-turn
  reconnect replay (#611) / event ring** — separate Phase 2 children per ADR 025
  § Backpressure. This slice emits synchronously and unbuffered.
- **Per-conversation routing.** The fan-out broadcasts to *every* interactive conn
  (the #589 shape), not a `conversation_id`-subscribed subset — not a confidentiality
  leak in the single-operator / single-`CurrentConversation()` model (each payload
  carries `conversation_id` for client-side filtering). A true `conversation_id →
  conn_id` subscription map is deferred to **pyrycode-mobile#336** (the #589 deferral,
  inherited).
