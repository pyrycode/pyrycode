# Ticket #712 — remote-permission decision audit log (the audit-sink primitive)

The **forensic sink** for the remote-permission security model ([EPIC #597]
Phase 3, [ADR 025] § "Security model" item 6 "Audit"). Answering a security-class
modal (permission / trust / destructive) from a paired phone is the highest-trust
action the mobile head can take; every such decision must leave a local record so
an operator can reconstruct *which device was allowed or denied the right to
answer, for which modal, with what outcome, from where.* This slice ships the
writer **primitive** — a new zero-dependency leaf package `internal/audit` — that
the modal control loop ([#703]) calls **once per resolved decision**. It does not
own the loop, the deny-on-timeout timer, the modal nonce, or the decision logic.

Peeled out of [#702] per that ticket's pre-authorized seam ("split the audit log
if it exceeds S"). [#702] (merged) owns the **gate** (the per-device
authorization bit + the fail-closed predicates); this ticket owns the **audit
sink** for the decisions that gate produces. The two are deliberately decoupled:
`audit.Log` records the already-decided outcome and never consults the gate.

## Implementation

One additive production file — a new leaf package. No state machine, no
goroutines, no consumer cascade (the only consumer, [#703], is a separate
ticket). Imports **only `log/slog`** — it depends on neither `internal/devices`
nor `internal/protocol`.

| File | Change |
|---|---|
| `internal/audit/audit.go` (~81 LOC) | New package. `Entry` struct (the four AC fields + ADR §6's class): `DeviceHash`, `DeviceLabel`, `ModalID`, `ModalClass`, `Outcome`, `Source` — **all non-secret, plain strings**. `Outcome` string-enum (5 consts: `allowed` / `denied_unauthorized` / `denied_timeout` / `cancelled` / `denied`) — the ticket's self-contained vocabulary. `Source` string-enum (3 consts: `remote` / `timeout` / `local`) mirroring the wire `ModalDismissedPayload.Source` closed set. `Log(logger *slog.Logger, e Entry)` — one `slog.Info` record, message `"audit: remote permission decision"`, fixed six-attr set; nil-logger guard → `slog.Default()`. |
| `internal/audit/audit_test.go` (~186 LOC) | Five tests: one-per-outcome (5-row table, asserts exactly one record + outcome/source); field-completeness (all six attrs + `level=INFO` + `msg` + automatic `time`); exact-key-set (the structural no-leak guard); no-secret-leak (sentinel token absent + hash present); nil-logger-no-panic. |

**Data flow** (this ticket owns only the boxed `audit` primitive):

```
#703 modal control loop (sole caller) resolves a decision
  ─┬─ inbound modal_answer → gate predicates (#702) → Outcome + SourceRemote
  └─ deny-on-timeout fires → safe-deny           → OutcomeDeniedTimeout + SourceTimeout
   → builds audit.Entry{DeviceHash: dev.TokenHash, DeviceLabel: dev.Name, ModalID, ModalClass, Outcome, Source}
   → audit.Log(logger, entry)  →  one slog.Info record  →  daemon's local slog sink
                                                            (never on the wire — ADR 025 §6)
```

## Patterns established

- **No-leak by construction, not by discipline.** The `Entry` type has *no field
  that can hold a plain device token* — the leak the `internal/devices` SECURITY
  contract (`device.go:4-10`) forbids is *impossible*, not merely avoided. The
  package never imports `devices`, so it cannot even reach a plain token;
  `device.PushToken` likewise has no field. This is the same "make the safe
  default structural" fabric as [#702]'s nil-guarded fail-closed predicate —
  belt-and-suspenders means *different fabric* (a structural guarantee, not a
  second stochastic rule).
- **The structural no-leak test is an *exact-key-set* assertion.** The decisive
  guard isn't the sentinel-substring scan — it's pinning the emitted slog
  attribute keys to *exactly* `{device_hash, device_label, modal_id, modal_class,
  outcome, source}`. A sentinel scan only catches a *known* secret; the key-set
  test catches *any* new field, secret-bearing or not, the moment it is added.
  (See Lessons learned.)
- **Self-contained outcome vocabulary, decoupled from the gate's input enum.**
  `audit.Outcome` does not import `devices.RemotePermissionOutcome`; #703 maps
  onto it. This keeps the sink independent of the authorization check (AC4) and
  lets the audit name distinctions the gate input doesn't — e.g. *allow rejected
  for lack of a bit* (`denied_unauthorized`) vs an authorized phone's explicit
  *deny* (`denied`). The writer records the already-classified value verbatim and
  never re-derives it, so the audit transcribes #703's decision rather than
  computing a second, possibly-divergent verdict.
- **Source vocabulary mirrors the wire, deliberately.** `Source` reuses the
  `protocol.ModalDismissedPayload.Source` closed set `{remote, local, timeout}`
  so #703 passes *one* source value to both the wire dismissal and the audit —
  no second, divergent vocabulary to keep in sync.
- **Named string-backed types for an in-process value type.** Wire payloads use
  plain `string` (untrusted decode, may-grow set); `audit.Entry` is built
  in-process and never decoded from the network, so it gets type-safe named
  enums while the `string` backing keeps the serialized record stable and
  human-readable regardless of constant order.
- **Carry ADR-minimum fields even when the AC under-specifies.** The AC frames
  modal identity as the opaque `modal_id` nonce alone; ADR 025 §6 names *"device
  id, class, decision, time."* The entry carries **both** `modal_id` (instance
  key) and `modal_class` (so an operator can tell a *benign* grant from a
  *destructive* one) — reconciling the AC with the governing ADR rather than
  picking one. `time` is satisfied implicitly by slog's automatic record
  timestamp (no explicit field).
- **Total, panic-free primitive.** `Log` returns nothing (an audit write has no
  caller-actionable failure) and the nil-logger guard defaults to `slog.Default()`
  — so an audit write can never become a crash that drops the very decision it
  was meant to record. Durability / rotation / append-only integrity is the
  daemon's slog-sink configuration concern, out of scope here (ADR §6 requires
  only "logged locally").

## Producer obligations surfaced (carry into [#703])

The security review flagged these as #703's responsibility, not gaps in this
primitive (the shape supports all four and forbids the unsafe ones structurally):

- **(a)** Call `audit.Log` on **every** resolved decision branch (allow /
  unauthorized-reject / explicit-deny / cancel / timeout). Call-site completeness
  is #703's to test; this primitive makes each call one total, panic-free write.
- **(b)** Supply the `Outcome`/`Source` per the mapping table in
  [`features/audit-package.md`](../features/audit-package.md) so the record
  matches the keystroke actually sent (audit says `allowed` ⟺ it granted).
- **(c)** Populate `ModalClass` from the surfaced modal (`""` if unavailable).
- **(d)** Construct `Entry` from `dev.TokenHash` + `dev.Name`, never the plain
  token; never pass modal body text (`Title`/`Prompt`/`Options`) into any field.

## Lessons learned

- **Pin the *exact attribute key set*, not just a sentinel substring.** The
  load-bearing no-leak guard is `TestLog_ExactKeySet`, which asserts the emitted
  slog record carries *exactly* `{time, level, msg, device_hash, device_label,
  modal_id, modal_class, outcome, source}` and nothing else. A sentinel-substring
  test (`TestLog_NoSecretLeak`) only proves a *specific known* token is absent —
  it would silently pass if a future edit added an unrelated new field that
  happened to carry a secret. The exact-key-set test fails the moment *any* field
  is added, forcing a deliberate review. Both tests are kept (the sentinel test
  also positively asserts the hash *is* present, proving hash-not-token is the
  recorded identity); the key-set test is the structural backstop. *(Lifted from
  the PR's Lessons-learned note.)*

## Out of scope (siblings)

- **Modal control loop — [#703].** The sole consumer. Mints/emits modals,
  routes/validates the inbound answer, owns the deny-on-timeout *timer*,
  constructs the `Entry`, and calls `audit.Log` per decision. The
  `RemotePermissionOutcome` → `Outcome` mapping is #703's (documented in the
  feature doc for vocabulary alignment).
- **Authorization gate — [#702]** (merged). The per-device bit + the fail-closed
  `MayAnswerRemotePermission` / `AuthorizeRemotePermission` predicates. The audit
  is decoupled from it (`Log` never consults the gate). See
  [codebase/702.md](702.md).
- **Wire vocabulary — [#701]** (merged). The four modal envelope types
  (`ModalShownPayload.ModalID`/`.Class`, `ModalDismissedPayload.Source`) the
  entry's identity/source fields map from. See [codebase/701.md](701.md).
- **Two-heads ownership — [#706].** Stale-`modal_id` rejection / first-answer-wins;
  the `SourceLocal` path a desktop-TTY resolution would audit.
- **Audit-log durability / integrity** (append-only, tamper-evidence, rotation)
  is the daemon's slog-sink configuration concern, not this primitive's.
