# Ticket #657 — emit `session_transition` events on the v2 wire (cmd-side producer + capability-gated fan-out)

The **cmd-side producer** that completes the session-boundary feature ([EPIC #596]
Phase 2, [ADR 025]). [#656] shipped the `session_transition` wire **vocabulary**
(`protocol.TypeSessionTransition` + `SessionTransitionPayload`); [#659] surfaced
the pool-side **transition signal** (`/clear` rotations + idle/cap evictions) via
an injectable `sessions.TransitionObserver`. This slice is the bridge between them:
it installs that observer, maps the session-local reason onto the wire `reason`,
builds a `protocol.SessionTransitionPayload`, and fans a `TypeSessionTransition`
envelope to **capability-gated interactive phones** through the established [#571]
push surface — the same way structured turn events flow today. So the phone can
interleave `SessionBoundary` delimiters between message runs instead of rendering
one flat run with no session structure.

**`security-sensitive`**: new outbound dispatch policy / capability-gated frame
routing on the internet-exposed v2 push surface (#659 is internal plumbing,
unlabelled). Split from [#655] → [#656] (wire type) / producer further cut on the
package boundary into [#659] (session-side signal, `internal/sessions`) + **this**
(cmd-side bridge + fan-out, `cmd/pyry`). Unblocks mobile `pyrycode-mobile#336`
(it needs the relay to actually emit).

## Implementation

Two production-file edits + one new file; **0 new exported types** (the one new
type, `transitionObserverSink`, is unexported), **0 changes to the
`V2SessionManager` surface** — this is a pure consumer of the existing
`ActiveConns`/`Push` fan-out primitive.

| File | Change |
|---|---|
| `cmd/pyry/session_transition_v2.go` (**new**, ~223 LOC) | The emitter + wiring. A near-verbatim structural mirror of `assistant_turn_v2.go`, with the capability filter **inverted** to interactive-only. See the surface breakdown below. |
| `cmd/pyry/relay.go` | `startRelayV2` gains a `transitions transitionObserverSink` param; wires `streamTransitionsCleanup := startSessionTransitionStreamV2(ctx, transitions, mgr, logger)` **unconditionally** after the `startInteractiveTurnStreamV2` block (~`:345`) — **no `bridge != nil` gate** (see Patterns) — and adds it to the `drain` cleanup before `<-mgrDone`. `startRelay` threads the same param through to `startRelayV2` (`:142`); the v1 branch accepts but ignores it (`session_transition` is v2-only). |
| `cmd/pyry/main.go` | Passes `pool` as the new trailing arg to the `startRelay(...)` call at `:489`. `*sessions.Pool` satisfies `transitionObserverSink` structurally; main.go already imports `internal/sessions`, so no new import. |

**`cmd/pyry/session_transition_v2.go` surface:**

- **`sessionTransitionEmitterV2`** — fields: `bcast interactiveBroadcaster` (the
  **existing** interface reused, not a third copy — `*relay.V2SessionManager`
  satisfies it), `logger`, buffered `in chan sessions.SessionTransition` (size
  `sessionTransitionQueueSize = 16`), and `nextID uint64` (per-conn envelope-ID
  counter, read/written only on the single `Run` goroutine — no atomic). `EventID`
  is left **nil**: this producer does not append to the [#647] replay ring (no
  replay AC).
- **`Enqueue(t)`** — the `sessions.TransitionObserver` callback. Non-blocking
  buffered send, drop-on-full with a WARN log → honors [#659]'s **MUST-NOT-BLOCK**
  contract (invoked synchronously from the pool's lifecycle/watcher goroutine, no
  lock held).
- **`Run(ctx)`** — drain loop; per transition calls `broadcast`.
- **`broadcast(ctx, t)`** — `toWirePayload` → marshal once → fresh `ActiveConns`
  snapshot → **interactive-only** filter (`if !c.Interactive { continue }`) →
  `Push` per conn. A per-conn `Push` error is DEBUG-logged and the loop continues
  (a dropped conn must not abort the others, AC#2); `ctx.Err() != nil` mid-fan-out
  returns early (teardown). No cursor read, no `EventID` mint (the contrast with
  `assistant_turn_v2`).
- **`toWirePayload(t) (protocol.SessionTransitionPayload, bool)`** — the **pure,
  unit-testable mapping seam**. `internal/sessions` must not import
  `internal/protocol` (import cycle), so [#659]'s `TransitionReason` → wire
  `reason` string mapping lives cmd-side:

  | #659 `TransitionReason` | wire `reason` | `previous_session_id` | `new_session_id` | `workspace_cwd` |
  |---|---|---|---|---|
  | `ReasonClear` | `"clear"` | `t.PreviousID` | `t.NewID` | `null` |
  | `ReasonEviction` | `"idle_evict"` | `t.PreviousID` | `t.PreviousID` (mirrored — no successor) | `null` |
  | anything else | — | **drop** (`ok=false`, DEBUG) | | |

  `OccurredAt` is propagated verbatim (#659 already UTC-stamps it). `WorkspaceCwd`
  is **always nil** → literal JSON `null`.
- **`transitionObserverSink interface { SetTransitionObserver(sessions.TransitionObserver) }`**
  — the narrow consumer-declared interface (CODING-STYLE: declare at the consumer)
  so `relay.go` threads the value through **without** importing `internal/sessions`.
  `*sessions.Pool` satisfies it.
- **`startSessionTransitionStreamV2(ctx, sink, bcast, logger) func()`** — mirrors
  `startAssistantTurnBridgeV2`: build the emitter, `sink.SetTransitionObserver(emitter.Enqueue)`
  (installs the observer — must run **before** `Pool.Run`; the call site guarantees
  it), start `Run` on a goroutine, return a cleanup that waits for `Run` to exit on
  ctx-cancel. **Cleanup does NOT close `in` and does NOT clear the observer** — the
  observer field is read-only after `Pool.Run` (cannot be cleared), and a late
  `Enqueue` racing teardown is panic-safe precisely because `in` is never closed (a
  non-blocking send to an open-but-full channel just drops).

The evergreen `internal/relay` reference —
[features/v2-session-manager.md](../features/v2-session-manager.md) § `ActiveConns`
— is updated this phase to record `session_transition` as the **second**
interactive-only fan-out consumer alongside the #632/#634 turn stream.

## Patterns established

- **Reuse the fan-out primitive; add no new path.** The producer is the **second
  interactive-only consumer** of the manager's `ActiveConns`+`Push` surface ([#571],
  non-blocking since [#610]) after the [#632]/[#634] structured turn stream — it
  reuses the existing `interactiveBroadcaster` interface (don't declare a third
  copy) and the `if !c.Interactive { continue }` capability gate. The ticket's "do
  **not** add a new channel or fan-out path" prohibits a *manager-side* primitive
  parallel to `Push`/`ActiveConns` — which this doesn't add. The buffered hand-off
  `in` channel is the *consumer-side* decoupling [#659]'s observer contract
  **mandates** (because `ActiveConns` blocks and the observer must not), the same
  shape `assistant_turn_v2` already uses — reuse, not a new path.
- **Unconditional wiring — no `bridge != nil` gate.** The coarse bridge and the
  structured turn stream gate on `bridge != nil` because both tap the **PTY output
  observer**, which exists only in service mode. The transition producer has **no
  PTY dependency** — it consumes pool transitions, which fire in any mode — so it
  is wired whenever the v2 manager exists. The capability filter (`ActiveConns` →
  `Interactive`) is the real delivery gate: no interactive phone connected ⇒ the
  fan-out reaches nobody. Do not cargo-cult the bridge gate. (Clear transitions
  additionally require the rotation watcher, which `claudeSessionsDir == ""`
  disables independently; eviction fires regardless.)
- **The reason → wire mapping is a pure helper (`toWirePayload`).** The
  unit-testable seam, cmd-side because of the [#659] import-cycle boundary. **Cap
  eviction collapses to `idle_evict`** (mobile #336's data model has no separate
  cap reason). **Unknown reason drops** (`ok=false`, the defensive `default`): a
  future [#659] reason can never silently emit a malformed envelope onto the wire.
- **Eviction mirrors the evicted id onto *both* wire id fields.** An eviction has
  no successor (`t.NewID == ""`); per mobile #336 the evicted id is written to both
  `previous_session_id` and `new_session_id` rather than emitting an empty
  `new_session_id`.
- **`EventID` left nil; `nextID` single-goroutine.** A session boundary is not a
  turn-stream event — it carries no `event_id` and does not join the [#647] replay
  ring. `nextID` is mutated only on the serial `Run` goroutine, so no atomic.
- **Logging discipline carried forward verbatim.** A `// SECURITY:` doc comment
  mirrors `assistant_turn_v2.go` / `interactive_turn_v2.go`: only content-free
  discriminants are logged at any level — `event`, `reason`, `conn_id`, and `Push`'s
  transport-sentinel `err`. The marshaled payload is **never** logged. The payload
  carries no application content (session ids + reason + timestamp), so there is
  nothing sensitive even on the defensive paths — but the discipline is kept so a
  future field addition can't quietly start leaking through a log line. (Session ids
  are non-secret routing identifiers — `session_id` is already a standard field
  across `internal/sessions`.)

## Lessons learned

- **"Do not add a new fan-out path" vs [#659]'s "hand off to a buffered channel"
  is only an *apparent* contradiction.** They operate at different layers: the
  ticket's prohibition is on a new *manager-side* fan-out primitive; the observer
  contract's mandate is on a *consumer-side* buffer to satisfy MUST-NOT-BLOCK
  (because `ActiveConns` does a blocking Run-goroutine round-trip). Both are honored
  by routing the fan-out through the single existing push surface while keeping the
  buffered hand-off — the established `assistant_turn_v2` shape.
- **The branch diff carries an unrelated realclaude fixture re-record.**
  `internal/e2e/realclaude/testdata/permission_protocol_v2.1.158.json` shows 108
  changed lines (54/54) — a golden fixture re-captured during the code-review
  phase's real-claude e2e run and swept in by its auto-commit safety net (commit
  `667b8ac`). It is **not** part of the producer change; the volatile per-run
  `session_id` / `uuid` / temp-path values are the only delta. (Same shape as
  [#656]'s and [#659]'s fixture re-records.)

## Out of scope (deferred)

- **`workspace_change` emission.** There is no server-side workspace-change source
  (`WorkDir` is set once at spawn; nothing mutates it mid-session). [#656] keeps
  `workspace_change` a valid wire reason so the mobile decoder stays exhaustive;
  this producer **never** emits it and always renders `workspace_cwd: null`. A
  follow-up wires it if/when a workspace-change capability lands — do not invent a
  source.
- **Reconnect replay of `session_transition`.** `EventID` is nil — this producer
  does not append to the [#647] replay ring. A phone that reconnects mid-stream and
  replays via `last_event_id` will **not** receive a missed session boundary. For
  #336's rendering completeness this is a gap, but it is squarely a [#647]-replay
  concern (whether transitions join the per-conversation ring, and under what
  durable id), not this ticket's. **Recommend a follow-up** if reconnect replay must
  include session boundaries.
- **A distinct cap-vs-idle wire reason.** Collapsed to `idle_evict` (mobile #336
  has no cap reason; [#659] already collapses idle and cap into `ReasonEviction`).
- **The mobile consumer** — `pyrycode-mobile#336`, which decodes the marker into
  `ThreadItem.SessionBoundary`.
