# Ticket #323 — `send_message` e2e roundtrip with fakeclaude stdin observation

Split from #313. Test-only slice that closes the wire-path gap on #322:
unit tests in `send_message_test.go` cover all four handler branches
against a `stubTurnWriter`, but nothing exercised the live path **phone
WS frame → fakerelay → binary → dispatcher → handler →
`Supervisor.WriteUserTurn` → PTY → child stdin** under `-race`. This
slice adds the one happy-path e2e that does, plus the fakeclaude
observability needed to make it deterministic. **No production package
outside `internal/e2e/` is modified.**

## Implementation

Three additive pieces.

### 1. fakeclaude stdin logging (additive, opt-in)

`internal/e2e/internal/fakeclaude/main.go` grew one optional env var,
`PYRY_FAKE_CLAUDE_STDIN_LOG`:

- **Default off** — unset/empty: `os.Stdin` is never read, no log file
  is opened. Existing rotation/restart tests are byte-identically
  unaffected.
- **When set** — `startStdinLogger(path)` opens the file
  `O_WRONLY|O_APPEND|O_CREATE`, mode `0o600`, then launches a
  goroutine that loops on `os.Stdin.Read(buf[:4096])`. On `n>0`,
  writes are followed by `f.Sync()`; on EOF or any read/write error
  the goroutine returns silently (the supervisor closes the PTY at
  shutdown — that's not a fault).

Why `Sync()` per write: a sibling test process polls the file via
`os.ReadFile`. Without the fsync, page-cache-only writes can be
invisible across processes for an indeterminate window (Linux is
generally fine; macOS APFS less so). One fsync per envelope is
negligible for an e2e budget. Why raw `Read` (not a `bufio.Scanner`):
`Scanner` would buffer arbitrarily before the test sees bytes; raw
`Read + Sync` keeps the visible-byte boundary tight.

The unit test `TestFakeClaude_StdinLog_AppendsBytes` builds the binary,
exec's it with `cmd.Stdin = strings.NewReader(want)`, polls the log
for `bytes.Contains(got, want)` with a 2s deadline, SIGTERMs, and
asserts a clean signaled exit. Mirrors the rotation unit test's
process-launch shape.

### 2. `StartRotationWithRelay` helper (`internal/e2e/harness.go`)

```go
func StartRotationWithRelay(t *testing.T, home, sessionsDir, initialUUID,
    trigger, stdinLog, relayURL string) *Harness
```

Composes `StartRotation`'s fakeclaude env+claude-bin shape with
`StartInWithEnv`'s relay-flag + insecure-relay-env shape in a single
`spawnWith` call. Concretely:

- `claudeBin = ensureFakeClaudeBuilt(t)`, `claudeArgs = []string{}`
- `extraFlags = ["-pyry-workdir=<home>", "-pyry-relay=<relayURL>"]`
- `extraEnv = ["PYRY_ALLOW_INSECURE_RELAY=1",
   "PYRY_FAKE_CLAUDE_SESSIONS_DIR=…", "PYRY_FAKE_CLAUDE_INITIAL_UUID=…",
   "PYRY_FAKE_CLAUDE_TRIGGER=…", "PYRY_FAKE_CLAUDE_STDIN_LOG=…"]`
- Populates `Harness.ClaudeSessionsDir`, registers `t.Cleanup(teardown)`,
  blocks on `waitForReady` — same contract as the rest of the harness.

A new helper rather than extending `StartRotation` because
`StartRotation` has positional params and no `extraFlags`/`extraEnv`
hooks — threading two new positional args would force every existing
caller to opt out of the relay wiring. The focused helper is
lower-risk and self-documenting at the call site.

### 3. `TestRelay_SendMessage_AckAndPTYDelivery`

`internal/e2e/relay_send_message_test.go` (build tag `//go:build e2e`).
Single linear test (no table-drive — refusal branches stay unit-level):

1. **Pair** — `RunBareIn(t, home, "pair", "-pyry-name=test", "--name=phone-a")`,
   `decodePairPayload` from `register_push_token_test.go`.
2. **Seed `conversations.json`** — write
   `{"conversations":[{"id":<knownConvID>, "cwd":<home>, …}]}` to
   `<home>/.pyry/test/conversations.json` (parent dir already exists
   from `pyry pair`). The seed is load-bearing:
   `Supervisor.WriteUserTurn` calls `ValidateConversation`, which
   returns `conversations.ErrConversationNotFound` on miss. Without
   the seed, the handler would surface the refusal envelope from
   #322 instead of the ack.
3. **Allocate paths** — `sessionsDir`, fixed `initialUUID`, never-created
   `trigger`, `stdinLog` all under `t.TempDir()`.
4. **fakerelay + daemon** — `fakerelay.New(relayTestLogger())` then
   `StartRotationWithRelay(...)`.
5. **Wait for handshake** — `readPersistedServerID(t, home)` then
   poll `fr.LastBinaryHello(serverID)` with a 5s deadline.
6. **Phone hello → hello_ack** — `fakephone.Dial`, send envelope id=1
   `TypeHello`, assert `Type==TypeHelloAck`, `*InReplyTo==1`.
7. **send_message → ack** — id=2 `TypeSendMessage` with
   `SendMessagePayload{ConversationID: knownConvID, MessageID: "m-1",
   Text: knownText}`. Assert `Type==TypeAck`, `*InReplyTo==2`,
   `ID>=2`, payload decodes as `AckPayload{}`.
8. **Marker on stdin** — poll `os.ReadFile(stdinLog)` for
   `bytes.Contains(got, []byte(knownText))` with a 3s deadline.

`knownText = "e2e-323-marker:hello world\n"`; a fixed UUIDv4
`knownConvID = "33333333-…"`. The test file's header comment warns
**not to paste real secrets into `knownText`** — the log captures it
verbatim and the failure message echoes the file contents.

### Conversation-id association — implicit, not asserted explicitly

`Supervisor.WriteUserTurn` only reaches the PTY after
`ValidateConversation(p.ConversationID)` returns nil. Therefore:
observing **both** the wire ack AND the marker bytes on stdin
proves the supervisor accepted the seeded `knownConvID` before
writing. No cursor-readback or separate id-association assertion
is needed (and adding one would just couple the test to
internal/sessions internals).

## Patterns established / reinforced

- **Additive-by-env-var observability for test scaffolding.** When
  `PYRY_FAKE_CLAUDE_STDIN_LOG` is unset, fakeclaude behaviour is
  byte-identically unchanged. Same posture as the existing
  `PYRY_FAKE_CLAUDE_TRIGGER` / `PYRY_E2E_BIN` env vars — opt-in,
  default-off, no flag flips required at existing call sites.
- **Compose harness helpers via fresh entry points, not positional
  extension.** `StartRotation` had no `extraFlags`/`extraEnv` seam, so
  growing it to add relay wiring would have ripple-touched every
  caller. A new helper sharing the same `spawnWith` core is the lower-
  risk shape. Reinforces #301's `StartInWithEnv` pattern.
- **Wire-path e2e covers the happy path only; refusal branches stay
  unit.** `send_message_test.go` already pins the four `WriteUserTurn`
  outcomes against a `stubTurnWriter`; duplicating the
  `conversations.not_found` / malformed / wrapped-error branches at
  the e2e tier would slow CI for no marginal coverage. The e2e proves
  the wires; the unit suite proves the branches. Matches the e2e
  discipline in the existing relay tests.
- **Fsync visibility across processes.** `os.Stdin.Read + f.Write +
  f.Sync` is the minimum surface for a one-way bytes-observable
  channel across process boundaries on macOS APFS. Useful template
  for future test-scaffolding observers.
- **Implicit-proof assertion.** When two observable events are
  causally chained via production code (here: ack ⇐ ValidateConversation
  pass ⇐ WriteUserTurn ⇒ PTY write ⇒ stdin marker), asserting both
  endpoints transitively proves the middle without exposing internal
  invariants. Avoids the test coupling to internal `Supervisor` /
  `Pool` state.

## Lessons learned

### `ValidateConversation` is wired by the production path, not bypassable in-test

The naive e2e shape ("just send a `send_message` and look for the
bytes") misses that `Supervisor.WriteUserTurn` calls
`ValidateConversation(id)` before writing — wired by
`pool.go:355-362` against the conversations registry. With
`convReg == nil` the validator is skipped, but the prod daemon (and
the e2e, by design) always passes a registry. So the seeded
`conversations.json` is load-bearing: without it the test sees a
`conversations.not_found` error envelope instead of an ack and fails
opaquely. The seed is one `os.WriteFile` call before daemon spawn
and is documented inline in the test.

### `pyry pair` creates the registry parent dir as a side-effect

Convenient: the `<home>/.pyry/<name>/` directory exists after `pyry
pair` runs, so the test can `os.WriteFile` `conversations.json`
directly into it without a prior `MkdirAll`. Order matters — pair
first, then seed the registry, then spawn the daemon. Reversing
those steps would either need an explicit `MkdirAll` or fail with a
"no such directory" write error.

### macOS APFS visibility — fsync per write, not per envelope

The first natural shape for the stdin logger was "buffer in a
`bufio.Writer`, flush on close". That made the sibling-process
`os.ReadFile` polling flaky on macOS — page-cache writes were
invisible for tens to hundreds of ms. Switching to `f.Sync()` after
every `Read+Write` was the deterministic fix; the cost is one fsync
per kernel-delivered chunk (≪ once-per-envelope at typical e2e
volumes). Worth remembering for future cross-process observation
helpers.

### `bufio.Scanner` is wrong for byte-level observation

The test polls for `bytes.Contains(content, marker)` — there's no
line-framing semantics. A `Scanner` would buffer arbitrarily
upstream of the visible file boundary, defeating the determinism
goal. Raw `os.Stdin.Read` into a `[4096]byte` is the right shape.

### Out-of-band proof beats internal-state assertion

The temptation was to add a `Harness.LastConversationCursor()` accessor
on the supervisor for stronger id-association assertions. That would
have crossed the "no production package outside `internal/e2e/`"
constraint AND coupled the e2e to `Supervisor` internals. The
implicit-proof shape (ack-and-bytes ⇒ id-was-accepted) ships zero
production surface for the same coverage.
