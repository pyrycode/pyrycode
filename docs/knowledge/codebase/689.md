# Ticket #689 — e2e fixtures bind their conversation to the bootstrap session (#678 regression)

[#678](678.md) changed `send_message` from writing the per-conn bootstrap surface to resolving
the conversation's **bound** session via `sessionRouter.Route(conversationID)`, and deliberately
removed the bootstrap fallback (its AC#4). `Route` rejects an empty `current_session_id` **before
any pool `Lookup`** (`cmd/pyry/main.go:686`), returning `errNoBoundSession`, which the handler
maps to a retryable `server.binary_offline` (`internal/relay/handlers/send_message.go:121-132`).
Seven `internal/e2e` tests seed `conversations.json` **without** a binding, relying on the
now-removed fallback — so each got a fast (~3 s) error envelope where it asserts a `TypeAck`. The
`-tags e2e` suite went red on `main`. This ticket binds each affected conversation to its
bootstrap session id via a shared `seedBoundConversation` helper. **Test + fixture only, zero
production-code change.**

**Not a production bug — a fixture gap against #678's new contract.** Production conversations are
created via `create_conversation` (`handlers/create_conversation.go`), which mints a session and
sets `CurrentSessionID` at creation time; only the hand-seeded test fixtures skipped the binding.
Same family as [#673](673.md) (which greened the same suite against [#668](668.md)'s contract
change). `internal/e2e` is **not in `make check`**, so #678 merged green while breaking it — the
second time today (#670, then #679) a developer agent wedged on this red suite that was not its
own bug.

## The non-obvious constraint (corrects the ticket's Technical Note)

The ticket suggested *"a helper that **reads** the bootstrap id from `sessions.json` and seeds
`current_session_id`."* **That mechanism does not work**, and the developer correctly did not
build it:

1. The daemon loads `conversations.json` **once** at startup into an in-memory
   `*conversations.Registry` (`main.go:520`); `Route` reads that in-memory copy — there is **no
   reload**. Writing `conversations.json` *after* the daemon starts has no effect on routing.
2. So the binding must be on disk **before** the daemon starts — but the bootstrap session id is
   only minted/reconciled *during* startup.

These reconcile only because the bound id is **predictable, not read-back**:
`reconcileBootstrapOnNew` (`internal/sessions/reconcile.go:118-148`) rotates the bootstrap entry's
id to the most-recent `<uuid>.jsonl` in the daemon's **computed** sessions dir
(`<HOME>/.claude/projects/encode(workdir)`). Six of the seven tests already pre-create exactly one
`<initialUUID>.jsonl` there before start, so after reconciliation `Pool.Default().ID() ==
initialUUID`. The fixture binds `current_session_id = initialUUID` — a compile-time constant the
test already owns. No `sessions.json` parsing, no read-back.

## Implementation

All changes confined to `internal/e2e/**`.

### Shared helper — `seedBoundConversation` (`harness.go:362`)

```go
func seedBoundConversation(t *testing.T, home, convID, boundSessionID string)
```

Writes `<home>/.pyry/test/conversations.json` (`0o600`) with one row `{id, cwd: home,
current_session_id: boundSessionID, is_promoted: false, last_used_at: "2026-01-01T00:00:00Z"}`,
`t.Fatalf` on write error. The diff against the prior inlined literal is exactly "added the
`current_session_id` field + extracted to a helper" — byte-identical field order otherwise. Its
doc comment centralises the load-bearing invariant (bound id == reconciled bootstrap id ==
`initialUUID`, on disk before start) in one place so a future test edit can't silently re-break the
binding. No new imports (`os`/`filepath`/`testing` already present). Non-affected
conversation-seeding tests (v2 list-conversations / snapshot, which never call `send_message`) were
**not** migrated — touch only what's necessary.

### Per-test change (six tests)

Each replaces its inline `convJSON`-write block with `seedBoundConversation(t, home, knownConvID,
initialUUID)`, placed *after* the pre-created `<initialUUID>.jsonl` write and *before* the harness
start call.

### The coarse outlier (`relay_two_phone_coarse_test.go`)

Unlike the other six, this test used `sessionsDir = filepath.Join(tmp, "claude-sessions")` (a
private temp dir the daemon never scans) and pre-created **no** JSONL — so reconciliation found the
computed dir empty and never rotated, leaving the bootstrap id a random pool mint that
`initialUUID` would not match. It was brought onto its structured sibling's pattern
(`relay_two_phone_structured_test.go:141-158`): `sessionsDir` → the computed
`<home>/.claude/projects/encodeWorkdir(home)` (`os.MkdirAll`'d), pre-create `<initialUUID>.jsonl`,
then `seedBoundConversation`. `tmp` is retained for `rotateTrigger`/`stdinLog`/`asstTrigger`; only
`sessionsDir` moved. The coarse test awaits no ack and asserts nothing about JSONL content, so the
move is inert to its assertions — its only effect is making reconciliation set the bootstrap id to
`initialUUID`.

## Patterns established

- **A bound test fixture must commit the binding to disk *before* the daemon starts, using the
  predictable reconciled id — not a read-back.** The registry is load-once-at-startup with no
  reload, and the bootstrap id is minted during startup, so reading `sessions.json` after start is
  too late and reading it before start is impossible. The only handle the test has is that
  `reconcileBootstrapOnNew` makes `Pool.Default().ID()` equal the single pre-created
  `<initialUUID>.jsonl` — a constant the test already owns. This is the general shape for any
  future e2e fixture that must reference the bootstrap session id.
- **An e2e test that drives a turn must align `sessionsDir` to the daemon's computed dir.** The
  same alignment lesson [#673](673.md)/[#642](642.md) established — reconciliation (like the
  growth-confirm resolver) scans only `<HOME>/.claude/projects/encode(workdir)`, never a
  `t.TempDir()` subdir. The coarse outlier is the last `send_message`-driving test that still
  pointed elsewhere; it now matches its peers.
- **The `-tags e2e` suite now proves the [#396](../features/idle-eviction.md) silent-outage
  contract under #678's bound-session routing** (see AC#3 below), not just under the old bootstrap
  path. The whole suite must be re-run at merge: #674 greened one path while siblings re-broke the
  rest because the full suite was never re-run.

## AC#3 product check — the one case that could have escalated

`TestE2E_IdleEviction_RespawnsOnSendMessage` exercises an idle-**evicted** bound session. The
concern (ticket AC#4): does `Route` *refuse* an evicted-but-bound session rather than letting
`Activate` respawn it? **No — verified green with the binding alone, no production change.** An
idle-evicted session stays in the pool parked in `runEvicted`, so `Route → pool.Lookup(boundID)`
resolves it and `boundSession.Activate → pool.Activate` (`main.go:712-714`) respawns it; the
inbound `send_message` yields a `TypeAck`. The #396 contract holds under bound-session routing — no
`Route` refusal, **no escalation needed**. (Had it failed, the ticket directed splitting it out as
a `security-sensitive` product-regression ticket rather than masking it in the test; it did not
fail.)

## Lessons learned

- **`relay_assistant_turn_test.go` is gofmt-dirty at HEAD under local Go 1.26.2** (a trailing blank
  line CI's pinned-older Go accepts). The flagged diff is pre-existing and untouched by this
  change — left as-is rather than reformatting, which would spray a spurious diff and fight CI.
  (From the PR body's Lessons learned; the recurring skew of
  [[pyrycode-gofmt-dirty-at-head-go1.26]].)
- **`make check` does not run the `-tags e2e` suite**, so both #668 and #678 merged green while
  breaking it. The operator decision is **no CI** — adding it is out of scope here. Process note:
  turn-delivery and session-binding changes should run `go test -tags e2e ./internal/e2e/...
  -count=1` before merge. (See [[architect-verify-regression-blast-radius]]: run the full suite,
  not the named tests in isolation.)
- **The coarse two-phone test legitimately takes ~31 s** (the pre-existing non-TUI 30 s `WaitReady`
  timeout, unchanged by this work) — don't mistake it for a hang.
