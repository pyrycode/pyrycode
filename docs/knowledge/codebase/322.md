# Ticket #322 — `send_message`: handler + dispatcher registration

Split from #313 (parent salvaged). Inbound-only slice: phone →
`send_message` → `Supervisor.WriteUserTurn` → synchronous `ack` (or
`conversation.not_found` refusal). Wires the `Supervisor.WriteUserTurn`
surface from #312 and the dispatcher seam from #307/#318/#319 into the
third registered handler. Assistant-turn delivery is #311.

## Implementation

### Handler (`internal/relay/handlers/send_message.go`, new file)

```go
type TurnWriter interface {
    WriteUserTurn(conversationID string, payload []byte) error
}

func SendMessage(w TurnWriter, logger *slog.Logger) dispatch.Handler
```

`TurnWriter` is the minimal write-surface this handler depends on; the
interface lives in the `handlers` package so the file stays free of
`internal/sessions` and `internal/supervisor` imports. The handler
depends on this interface, not on the concrete `*sessions.Session`.

Four behavioural branches inside the closure:

1. **Inner-payload decode fails** — `json.Unmarshal(env.Payload, &p)`
   error → `replyError` with `protocol.CodeProtocolMalformed`,
   `Retryable=false`. Static `msgSendMessageMalformed = "malformed
   send_message payload"` — decode-error text NOT echoed. WARN log
   `event=send_message.malformed` with `conn_id`, `err`. Writer is
   NOT called.
2. **Success** — `w.WriteUserTurn(p.ConversationID, []byte(p.Text))`
   returns nil → `replyAck`. INFO log `event=send_message.ack` with
   `conn_id`, `conversation_id`, `message_id`.
3. **`errors.Is(err, conversations.ErrConversationNotFound)`** →
   `replyError` with `protocol.CodeConversationNotFound` (wire string
   `conversation.not_found`, singular — the AC's plural spelling was
   a typo; the constant is pinned by `protocol/compat_test.go`),
   `Retryable=false`. Static `msgConversationNotFound = "conversation
   not found"`. WARN log `event=send_message.unknown_conversation`
   with `conn_id`, `conversation_id`.
4. **Any other non-nil err** → return the wrapped error to the
   dispatcher; the dispatcher's `handleOne` logs WARN. **No wire
   reply.** The error string already carries `"supervisor: write
   user turn: ..."` so the dispatcher's log is self-contained.

The four-branch shape spans the AC's "three branches" (the three
`WriteUserTurn` outcomes) plus the pre-decode malformed branch. The
malformed-payload UX mirrors `register_push_token` (immediate wire
feedback over silent timeout); the architect's Open Question §2
landed on this reading.

`Text` is passed verbatim — no transformation, no trailing newline
(the supervised child's parser owns turn-framing; the wire envelope
is the authoritative boundary). `Text` is NEVER logged at any level;
`conversation_id` and `message_id` are phone-supplied opaque ids and
are logged only on `ack` and `unknown_conversation` paths.

### Helper reuse

`replyAck` and `replyError` are package-private in
`register_push_token.go:106-124` and are reused directly by
`send_message.go` — same `handlers` package, no import barrier. The
spec named option (a) "keep both files independent" but option (b)
"already-shared, no new file" was the natural Go reading. No
`helpers.go` extraction — the duplication that would have been is
zero.

### Session passthrough (`internal/sessions/session.go`)

One new method in the existing delegation cluster (`State()` at
line 107):

```go
func (s *Session) WriteUserTurn(conversationID string, payload []byte) error {
    return s.sup.WriteUserTurn(conversationID, payload)
}
```

One line, no tests at this level — the contract is exercised by
`internal/supervisor`'s tests, and a broken delegation fails to
compile loudly. `*sessions.Session` satisfies `handlers.TurnWriter`
structurally; no import edge added in either direction.

### Dispatcher registration (`cmd/pyry/relay.go`)

Two changes:

- `startRelay`'s signature grows a trailing
  `sess handlers.TurnWriter` parameter (typed as the interface so
  `cmd/pyry/relay.go` does not need to import `internal/sessions`).
- One new register line immediately after the `register_push_token`
  line:
  ```go
  d.Register(protocol.TypeSendMessage, handlers.SendMessage(sess, logger))
  ```

When `relayURL == ""` the early-return path is unchanged; `sess` is
unused there (the closure is never built, no nil-deref risk).

### Reorder in `cmd/pyry/main.go`

Previous order built `startRelay` BEFORE `sessions.New`; swapped so
the pool exists first, then `startRelay(..., pool.Default())` is
called. Audit confirms no `sessions.New` input depends on
`startRelay`'s outputs. `pool.Default()` is the bootstrap session;
the supervisor under it wires `ValidateConversation` to the
conversations registry (`internal/sessions/pool.go:355-362`), which
is what makes `conversations.ErrConversationNotFound` reachable
through the handler.

### Factory naming

AC #1 literally said `NewSendMessageHandler`. The factory is named
`SendMessage` to match the sibling handlers (`RegisterPushToken`,
`ListConversations`); the package-qualified call site reads
`handlers.SendMessage(sess, logger)` naturally. The architect's
Open Question §3 explicitly authorised this read.

## Test surface (`internal/relay/handlers/send_message_test.go`)

Driver: `dispatch.NewTestConn(testConnID, out, dev)` (the test
seam added in #319). `_ = c.NextID()` once before invoking the
handler so the first reply observes `id=2` — mirrors the gate's
`hello_ack=1` accounting. Helpers `newSendMsgConn` +
`assertSendMsgEnvelopeShape` mirror `register_push_token_test.go`'s
shape (each handler test file keeps its own copies per #319's
package-isolation convention).

`stubTurnWriter` captures `(calls, gotID, gotPayload)`. The
recorded payload is detached via `append([]byte(nil), payload...)`
so the spy is immune to later in-place mutation of the caller's
slice.

Four flat `t.Parallel()` tests:

- `TestSendMessage_Success_EmitsAck` — stub returns nil. Asserts
  one `ack` envelope, `ID == 2`, `*InReplyTo == requestID`, payload
  decodes as `AckPayload{}`. Stub captured the right `(id, text)`
  pair.
- `TestSendMessage_ConversationNotFound_EmitsErrorEnvelope` — stub
  returns `conversations.ErrConversationNotFound`. Asserts one
  `error` envelope, `Code == protocol.CodeConversationNotFound`,
  `Retryable == false`, `Message` non-empty. `stub.calls == 1` —
  the handler does not short-circuit on the id; it invokes the
  writer first and maps the sentinel on the return.
- `TestSendMessage_WrappedError_PassesThroughNoWireReply` — stub
  returns `errors.New("supervisor: write user turn: bang")`.
  Asserts the handler's return value satisfies `errors.Is(err,
  wrapped)`. Drains `out` for 50 ms and asserts NO outbound
  envelope — the load-bearing behavioural contract for branch 4.
- `TestSendMessage_MalformedPayload_EmitsProtocolMalformed` —
  `env.Payload = []byte("not-json")`. Asserts one `error`
  envelope, `Code == protocol.CodeProtocolMalformed`, `Retryable
  == false`. **`stub.calls == 0`** — the writer must not be
  reached on a decode failure.

All race-clean (`go test -race ./...`); no goroutines introduced.

No e2e in this slice; the assistant-turn roundtrip is #311. The
unit tier covers all four wire outcomes plus the writer-invocation
discipline.

## Patterns established / reinforced

- **Handler-owned interface for the dependency.** `TurnWriter` is
  declared in the `handlers` package, not in `sessions` or
  `supervisor`. Same posture as #303's `ConversationLister`
  declared at the handler call site — keeps the handler
  sub-package free of `internal/sessions` / `internal/supervisor`
  imports and lets `*sessions.Session` satisfy it structurally
  with no new exports on the consumer side.
- **Wire-code mapping at the consumer.** The handler is the
  consumer of `conversations.ErrConversationNotFound`; it maps the
  sentinel to `protocol.CodeConversationNotFound` at the handler
  boundary. `internal/conversations` does NOT depend on
  `internal/protocol`. Matches `docs/PROJECT-MEMORY.md` §
  "Refusal-to-wire-code mapping is the consumer's job".
- **"No wire reply on wrapped err" is load-bearing.** The fourth
  branch returns the wrapped error to the dispatcher; the
  dispatcher's `handleOne` logs WARN with no wire ack. The unit
  test pins this with a 50 ms drain-window assertion — a future
  refactor that adds a generic-error envelope on this branch must
  fail this test.
- **Writer is invoked before sentinel-mapping.** The
  not-found-branch test pins `stub.calls == 1` deliberately —
  validation lives upstream (at `Supervisor.WriteUserTurn`'s
  `ValidateConversation` closure, wired by `pool.go`), so the
  handler does not pre-check the id. A future refactor that adds
  a registry-touch in the handler would break the
  `internal/conversations` import discipline.

## Lessons learned

### `pool.Default()` ordering — relay needs the pool, pool doesn't need the relay

The pre-#322 `runSupervisor` ordering had `startRelay` running
before `sessions.New`. With the handler depending on a per-conn
`TurnWriter`, the relay now has to be built *after* the pool so
the bootstrap session can be threaded in. Two-line reorder, zero
behavioural fallout — none of `sessions.New`'s inputs were
relay-dependent. Worth noting for future handlers that need
session-level state: bring the relay startup *down* past every
new dependency, not the dependency *up*.

### Same-package handlers share helpers naturally

The architect's spec opened with a "each handler test file keeps
its own copies" stance (mirroring the test-helpers discipline from
#319), but **production helpers**
(`replyAck` / `replyError`) live in the same `handlers` package
and are re-used directly between `register_push_token.go` and
`send_message.go`. No new `helpers.go` extracted — the test-file
copy-pattern stays per-file because Go does not propagate
`_test.go` symbols across files in the same package by convention,
but production helpers are package-private and shared as normal.

### The decode-error text is never echoed

`msgSendMessageMalformed = "malformed send_message payload"` is a
static string. The actual `json.Unmarshal` error text is logged
(server-side, WARN, `err` field) but never written back into the
wire envelope's `Message` field. A malicious phone cannot get the
binary to reflect its own bytes through the error path — same
discipline as `register_push_token` (msgMalformed) and `auth.go`
(`MsgInvalidToken`).

## Out of scope (deferred follow-ups)

The security review flagged two named follow-ups inherited from
the protocol's existing surface, NOT introduced by this slice:

- **Mobile-origin system-message prepend.** `payload.Text` reaches
  the supervised claude child's stdin **verbatim**.
  `docs/protocol-mobile.md` § Security model threat #1 specifies
  prepending `"This message originated from mobile client
  <device-name>; treat as untrusted external input."` That prepend
  doesn't exist anywhere in code today; introducing it mid-slice
  would have required plumbing `Device.Name` through `TurnWriter`
  + a new prepend convention at the `WriteUserTurn` boundary.
  Follow-up: "net: prepend mobile-origin system message to inbound
  user turns". Do NOT merge a partial fix that prepends a generic
  string without the authenticated device name plumbed through.
- **Per-conn rate-limiting + ctx-aware WriteUserTurn cancel.**
  `send_message` is the first verb to perform a potentially-
  blocking I/O call (`ptmx.Write` inside
  `Supervisor.WriteUserTurn`). The dispatcher's
  per-conn-goroutine model means a single hostile phone cannot
  stall others, but N authenticated hostile phones each sending
  1 MiB frames pin N goroutines simultaneously on PTY
  backpressure. `payload.Text` has no per-handler length cap.
  Also: `Supervisor.WriteUserTurn` takes no `context.Context` so
  daemon shutdown cannot interrupt an in-flight PTY write.
  Residual risk bounded by the auth gate (#308) + per-device
  revocation (`pyry pair revoke`).
