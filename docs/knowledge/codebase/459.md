# Ticket #459 — `internal/control`: rekey verb wire + `Rekeyer` interface

Slice A of the [#451](https://github.com/pyrycode/pyrycode/issues/451) split. Lays the control-socket wire contract (`VerbRekey` + `RekeyPayload{ConnID}` + `ErrCodeConnNotFound`) and the seam (`Rekeyer` interface, `Server.SetRekeyer`, `handleRekey` dispatcher, `control.Rekey` client helper) the upcoming `pyry rekey <conn_id>` subcommand will plug into. Ships no production caller — until slice B ([#460](https://github.com/pyrycode/pyrycode/issues/460)) lands, every production-path `VerbRekey` returns `"rekey: no rekeyer configured"`. The slice exists as a separate piece because the original end-to-end sketch in #451 would have changed `control.NewServer`'s signature across 1 production + 10+ test call sites — `SetRekeyer` is the seam that keeps that signature frozen while still letting the daemon install a `*relay.V2SessionManager` between `Listen` and `Serve`. Mobile Protocol v2 (`docs/protocol-mobile.md` § Re-key) reserves `payload.reason = "manual"` for this operator path; the verb is part of the v2 contract and this slice unblocks it.

## Implementation

- **Three production source files modified, one new test file** — `internal/control/protocol.go` (+33 LOC: `VerbRekey` constant + `RekeyPayload` struct + `ErrCodeConnNotFound` + `Request.Rekey` field), `internal/control/server.go` (+106 LOC: `Rekeyer` interface, `ErrConnNotFound` sentinel, `Server.rekeyer` field, `SetRekeyer` method, `case VerbRekey` dispatch arm in `handle`, `handleRekey` method), `internal/control/client.go` (+33 LOC: `Rekey` helper). New `internal/control/rekey_test.go` (+359 LOC: 9 tests + `fakeRekeyer` + `startServerWithRekeyer` helper). No existing tests modified; `NewServer`'s signature unchanged.

- **`VerbRekey = "rekey"` is a single-word, no-namespace verb.** Unlike the `sessions.*` family which shares an `internal/sessions.Pool` consumer (so the `sessions.` namespace mirrors a single Go package), the rekey trigger is its own operation on a v2 conn — there is no "rekey.*" family in scope. Doc-comment names slice B (#460) as the eventual operator-side consumer and locks the production state (`"rekey: no rekeyer configured"`) until then.

- **`Request.Rekey *RekeyPayload \`json:"rekey,omitempty"\`** keeps every existing verb's wire bytes byte-identical (the same back-compat property `SessionsPayload`'s addition preserves). Single-purpose payload struct alongside `SessionsPayload` / `ResizePayload` / `AttachPayload`; protocol.go's "one payload struct per verb family" idiom is preserved.

- **`RekeyPayload.ConnID` carries the camelCase JSON tag `"connID"`** — matches the control-socket convention (`SessionsPayload.ID`, `AttachPayload.SessionID`, `ResizePayload.SessionID`). Deliberately distinct from `RoutingEnvelope.ConnID`'s snake-case `"conn_id"` which is the mobile-WS wire and unrelated to the operator-facing control socket. Doc-comment locks both halves of that asymmetry so a future contributor doesn't "normalise" one to match the other. No `omitempty` — empty ConnID is invalid input, and the server-side guard rejects it before calling Rekeyer (an absent field decodes to `""` and hits the same guard).

- **`ErrCodeConnNotFound = "conn_not_found"`** is the wire token analogue of `ErrCodeSessionNotFound`. Snake-case to match the existing wire-token convention (`session_not_found`, `cannot_remove_bootstrap`). The Go sentinel `ErrConnNotFound = errors.New("rekey: conn not found")` is **owned in `internal/control`** because the `Rekeyer` contract is defined here; slice B's `*relay.V2SessionManager.TriggerRekey` will wrap its internal not-found condition with `%w` against this sentinel. Mirrors how `internal/control` consumes `sessions.ErrSessionNotFound` — the import-direction is flipped because slice B's package owns the producer; either ownership shape works (slice B can choose to define `relay.ErrConnNotFound` and have the wiring layer wrap-translate), and slice A's dispatcher is byte-stable either way because it uses `errors.Is`.

- **`Rekeyer` is free-standing — NOT embedded into `Sessioner`.** The existing `Sessioner` aggregate embeds `Remover` / `Renamer` / `Lister` / `GetOrCreator` because they share a single producer (`*sessions.Pool`). Slice B's `*relay.V2SessionManager` is a different concrete type, so embedding `Rekeyer` into `Sessioner` would either force `Pool` to grow a stub `Rekey` method or force a covariant adapter; both are noise. The decoupled-interfaces shape matches the "defined at the consumer, satisfied at the producer" pattern this package already uses for per-verb sub-interfaces — each is independently addressable; `Sessioner` aggregates only the ones `Pool` happens to satisfy.

- **`Server.SetRekeyer(r Rekeyer)` installs the implementer between `Listen` and `Serve`.** This is the AC2-load-bearing decision: threading `rekeyer` through `NewServer` would cascade across 1 production call site (the daemon's `cmd/pyry/` wiring — slice B will identify) plus 10+ `_test.go` files that already pass long argument lists to `NewServer`. The setter sidesteps the cascade entirely and keeps the wire contract independent of the constructor's shape. Doc-comment explicitly names "passing nil clears the previously-installed Rekeyer (used by tests; production startup installs once and never clears)" so a future maintainer doesn't assume idempotent install.

- **Concurrency: `s.rekeyer` reuses `s.mu`, leaf-lock with no nested critical sections.** The existing mutex already guards `listener` / `closed` / `closedCh`; adding `rekeyer` to its scope is the smallest delta. `handleRekey` takes the lock for a one-pointer-load, releases, then calls into the Rekeyer — never holds the mutex across the (potentially blocking) Rekeyer call. The reviewer-noted "goroutine-safe-by-construction" alternative (rely on the `Listen` → `SetRekeyer` → `Serve` happens-before chain) is genuine but more fragile than a 5-line mutex pair and inconsistent with the rest of `Server`'s state management; the spec deliberately picked the locking shape.

- **`handleRekey` follows `handleSessionsRename`'s no-`context.WithTimeout` template.** Mirrors the shorter sibling rather than `handleSessionsRm`'s ctx-timeout variant because the rekey-trigger contract is "deliver the request to the v2 manager loop", not "complete the rekey handshake". The actual Noise handshake runs asynchronously on the conn's own state machine — a handler-level 30s timeout would lie about cancellability. `context.Background()` is passed into `Rekeyer.Rekey` so slice B's implementer can propagate its own enqueue-cancellation logic; this slice imposes no deadline.

- **Dispatcher guard order: nil-rekeyer → nil/empty payload → Rekeyer call.** Guards fire BEFORE the Rekeyer call so a misconfigured daemon or malformed client cannot trigger work or reach a nil dereference. The behaviour table (one row per branch):

  | Precondition | Server reply |
  | --- | --- |
  | `s.rekeyer == nil` | `Response{Error: "rekey: no rekeyer configured"}` (no `ErrorCode`) |
  | `payload == nil \|\| payload.ConnID == ""` | `Response{Error: "rekey: missing connID"}` (no `ErrorCode`) |
  | `r.Rekey(...)` returns `ErrConnNotFound` (via `errors.Is`) | `Response{Error: err.Error(), ErrorCode: ErrCodeConnNotFound}` |
  | `r.Rekey(...)` returns any other error | `Response{Error: err.Error()}` (no `ErrorCode`) |
  | `r.Rekey(...)` returns `nil` | `Response{OK: true}` |

- **Server uses `errors.Is`, not `==`, to map the typed sentinel.** Pinned by `TestServer_Rekey_ErrConnNotFound_Wrapped` against `fmt.Errorf("manager: %w", ErrConnNotFound)`. Slice B's manager will wrap with package context (e.g. `fmt.Errorf("v2 session manager: %w", control.ErrConnNotFound)`); the dispatcher and the client both survive that wrap unchanged. The `errors.Is` choice is the same shape `handleSessionsRm` uses for `sessions.ErrSessionNotFound` — the convention is documented in [`docs/PROJECT-MEMORY.md:20`](../../PROJECT-MEMORY.md) ("Refusal-to-wire-code mapping is the consumer's job, NOT the primitive's").

- **`control.Rekey(ctx, socketPath, connID)` client helper mirrors `SessionsRm`'s single-arg shape.** Reconstructs `ErrConnNotFound` from `Response.ErrorCode` so callers can `errors.Is` against the bare sentinel; other server errors flow through as `errors.New(resp.Error)` verbatim. No production caller exists in this slice — wired now so slice B is a one-file change in `cmd/pyry/`. Empty-response defensive shape (no `Error`, no `OK`) returns `"control: rekey response missing ok flag"`, matching the existing `sessions.rm` / `sessions.rename` clients.

## Patterns established

- **Post-construction setter to dodge a multi-call-site constructor cascade.** When a new optional dependency would force a `NewX` signature change across N call sites (here ~11 total), an `X.SetDependency(d)` method called between `Listen` and `Serve` keeps the constructor's signature frozen while still letting daemon startup install the implementer. The shape works whenever (a) the dependency is optional (the zero/nil state is a documented production state) and (b) the install is a single happens-before-Serve event (no concurrent reinstalls). Doc-comment locks the calling discipline so a future maintainer doesn't move the install into a goroutine or call it repeatedly. The structural alternative — pass the dependency through the constructor — is correct if and only if the call-site cascade is small enough to absorb; pick by counting, not by reflex.

- **Free-standing interfaces over aggregate embedding when implementers differ.** `Sessioner` embeds `Remover` / `Renamer` / `Lister` / `GetOrCreator` because they share a single producer (`*sessions.Pool`). `Rekeyer` stays free-standing because its implementer (`*relay.V2SessionManager`) is a different type. The rule: aggregate sub-interfaces only when one concrete type satisfies the aggregate; otherwise the aggregate forces stub methods or covariant adapters on the producer that doesn't naturally satisfy every embedded contract. Generalise: "interface-at-the-consumer" is the package convention, but the consumer's aggregate shape should track the producer landscape — one producer → one aggregate; two producers → two interfaces.

- **Sentinel owned at the interface, wrapped at the producer.** `ErrConnNotFound` is defined in `internal/control` (where `Rekeyer` is defined), and slice B's producer will wrap its internal not-found state with `%w` against the control-package sentinel. The pattern matches how `internal/control` consumes `sessions.ErrSessionNotFound` — except the import direction is flipped because `internal/sessions` is the producer's home. The shape generalises: the package that owns the interface owns the contract sentinels; producers in other packages wrap-or-return the sentinel directly so `errors.Is` works regardless of how many layers of wrapping the producer adds. Slice B has a small choice (return the bare control sentinel, or define a local `relay.ErrConnNotFound` and wrap-translate at the wiring layer); both shapes are byte-stable on the wire because the dispatcher uses `errors.Is`.

- **camelCase on the control socket, snake_case on the mobile-WS wire — two unrelated naming conventions, two unrelated wire formats.** `RekeyPayload.ConnID \`json:"connID"\`` (control socket; matches `SessionsPayload.ID`, `AttachPayload.SessionID`, `ResizePayload.SessionID`) coexists with `RoutingEnvelope.ConnID \`json:"conn_id"\`` (mobile-WS wire; matches `docs/protocol-mobile.md`'s spec). A future contributor might be tempted to normalise one to match the other for "consistency", but the two wires have different audiences (local operator clients vs the phone), different specs, and different historical conventions. Doc-comments on both halves of the asymmetry are the load-bearing artefact that keeps the choice locked. Generalise: when two unrelated wire formats coexist in one binary, their JSON-tag conventions are local choices for each wire — not global choices for the project — and the doc-comment naming both halves is the pattern that prevents drift.

- **Guard order: configuration check → input validation → seam call.** `handleRekey` fires the nil-Rekeyer guard before the empty-connID guard before the Rekeyer call. The ordering is load-bearing: a daemon with no Rekeyer installed should surface its config drift over an input-validation error, since the input never reached anything that could have used it. Mirrored in the test suite (`TestServer_Rekey_NoRekeyerConfigured` uses a non-empty connID to prove the guard fires on the rekeyer-nil branch, not the input-empty branch). Generalise: when multiple guards fire before a seam call, order them outside-in — configuration / wiring guards first, then per-request input validation — so the error message most accurately diagnoses the actual problem.

## Lessons learned

- **AC-mandated frozen signatures are an architectural constraint, not a code-style preference.** The original #451 wanted "verb + plumbing + manager wiring" in one slice. The architect's review counted 1 production + 10+ test `NewServer` call sites and split the work into #459 + #460 specifically to keep `NewServer`'s signature frozen. The `SetRekeyer` setter is not a stylistic pick — it is the structural mechanism that makes the split possible. A future contributor who reads "why a setter instead of a constructor arg?" and reaches for a refactor would re-introduce the cascade the split was designed to prevent. The lesson generalises: when a slice's spec names a "no-signature-change" constraint, the API shape encodes the constraint — refactoring it back to a constructor arg silently invalidates the architectural rationale. The doc-comments on `SetRekeyer` and the `rekeyer` field both cite the 34-call-site fan-out so the constraint survives a context-free read of the source.

- **`defer` order is load-bearing when the test's teardown depends on unblocking a server-side goroutine.** `TestServer_Rekey_ClientCtxTimeout` blocks the fake Rekeyer on a `<-block` channel that the test closes via `defer close(block)`. The control server's `handleWG.Wait` (in `Server.Close`) waits for every in-flight handler to complete; if `stop()` runs before `close(block)`, the test deadlocks. LIFO `defer` ordering (registering `stop()` first → registering `close(block)` second → `close(block)` runs first → `stop()` runs second) is the fix; an inline comment names this so a future contributor doesn't reorder for "readability" and re-introduce the deadlock. The lesson: any test that uses a `block`-style channel to stall a server-side goroutine MUST close the channel before the server's `Close()` waits for that goroutine — `defer`'s LIFO order is the cheapest mechanism to encode the ordering at the call site.

- **Slice A ships callable client helpers with no production caller — that is the point.** `control.Rekey` is fully tested but unused by `cmd/pyry/` in this slice. A reviewer's reflex ("dead code; remove it") would re-introduce slice B's file-count blow-up: if the client helper landed in slice B, slice B would touch 5 production files (`protocol.go`, `server.go`, `client.go`, `cmd/pyry/main.go`, `internal/relay/`) — exactly the cross-package fan-out the split was designed to prevent. The lesson: when a split makes "wire contract" and "consumer wiring" independent shippable units, the wire-side slice will land callable surfaces with no production caller, and the spec must name that production state explicitly so the reviewer doesn't flag it as dead code. The doc-comment on `Rekey` and the verb-constant doc-comment both lock the no-caller state with a forward reference to slice B.

- **`Server.SetRekeyer(nil)` is supported for tests; production never calls it that way.** The setter's contract permits clearing the previously-installed Rekeyer because some tests (a future test that exercises both states in one binary) might want to. Production startup installs once and never clears. The doc-comment names both halves so the production discipline survives a future test pattern that exercises the test-supported clear-path. The lesson: when a setter's signature naturally permits a value (here `nil`) that production never uses, doc-comment the production-vs-test posture explicitly — leaving it ambiguous invites a future production caller to start using the test-only path.

Both shapes are byte-stable here because the dispatcher uses `errors.Is`. Architect noted but not gating.

If slice B finds the asymmetry painful for operator-facing CLI output, the JSON tag can be revisited then; for this slice the camelCase match to `sessionID` / `jsonlPolicy` is the right local optimum.
