# Ticket #593 — Supervisor hosts claude via `tuidriver.Session` across the restart loop

Migrates `internal/supervisor` from a raw `pty.Start` + two `io.Copy` host loop to hosting claude
through a supervisor-owned [tui-driver](https://github.com/pyrycode/tui-driver) `Session`
(`Spawn{MirrorOutput:true}`). **Behaviour-preserving** — the conversation cursor, restart survival,
live terminal resize, and every raw-byte consumer keep working unchanged. This is task **T2** of
Phase 5 / Phase 1 in [ADR 025](../decisions/025-mobile-remote-head-interactive-session.md), the
load-bearing migration that gives reliable turn delivery ([#594], not yet built), attach rewire
([#595]), and structured streaming ([#596]) a single, supervisor-owned `Session` to build on. It
swaps the hosting mechanism only; none of that downstream work is pulled forward here.

## Implementation

### Host loop: `Spawn` + `Session` in both modes (`runOnce`)

- `tuidriver.Spawn(cmd, tuidriver.SpawnOpts{MirrorOutput: true})` replaces `pty.Start(cmd)` in both
  the foreground and service/bridge branches of `runOnce`. `MirrorOutput: true` is **required in both
  modes** — the `Session` seals the PTY `*os.File` privately (no accessor), so `MirrorOutput()` is the
  *only* output path; there is nothing to `io.Copy` a raw master from.
- **Input pump** stays `io.Copy(sessionWriter{sess}, src)` via a 5-line unexported `sessionWriter`
  adapter (`Write(p)` → `sess.AttachInput(p)`, returns `len(p), nil` on success so `io.Copy` keeps
  draining). Mirrors the old `io.Copy(ptmx, src)`.
- **Output pump** becomes `for chunk := range sess.MirrorOutput() { dst.Write(chunk) }` in a goroutine
  (sink = `os.Stdout` foreground, `bridge.Write` service), joined via the existing `done` channel +
  `goroutineDrainTimeout` net. Replaces `io.Copy(dst, ptmx)`.
- **Lifecycle** uses `sess.Wait()` (returns exactly `cmd.Wait()`'s error → `Run`'s
  crash/clean/ctx-cancel switch + backoff classification is byte-identical; no `cmd.Cancel` added, so
  SIGKILL-on-cancel / `GracefulShutdown` semantics are preserved) and `sess.Close()` (SIGTERM → grace
  → SIGKILL, close PTY, join reader; idempotent; result ignored — `waitErr` is authoritative).

### Held handle re-keyed to `*tuidriver.Session`

The per-iteration PTY handle and its helpers are renamed (honest names; **readiness choreography
byte-identical**): `ptmx *os.File`→`sess *tuidriver.Session`, `ptmxMu`→`sessMu`,
`ptmxReadyCh`→`sessReadyCh`, `setPTY`→`setSession`. `WriteUserTurn` changes exactly one line —
`s.ptmx.Write(payload)` → `s.sess.AttachInput(payload)` — keeping its full contract (validate → stamp
cursor under `convMu` → drop-when-no-session → `"supervisor: write user turn:"` error wrap).
`AttachInput` is the sanctioned raw-input seam (`writeRaw` → `pty.Write` internally), byte-for-byte
the old behaviour.

**Public signatures are unchanged** — `WriteUserTurn`, `WaitForPTY`, `CurrentConversation`,
`Bridge.Resize` keep their names (even though "PTY" is now imprecise; only doc-comments updated to
"Session is live"). So `internal/sessions`, `internal/control`, `internal/relay/handlers`, and
`cmd/pyry` compile untouched. The `setPTY`→`setSession` / `SetPTY`→`SetResizer` renames are
package-internal (`runOnce`-only callers).

### `Bridge`: `*os.File` resize coupling → a `resizer` delegate

The `Session` owns the PTY privately, so `Bridge.SetPTY(*os.File)` could not survive. Replaced by an
unexported single-method interface defined at the consumer (accept-interfaces convention), satisfied
structurally by `*tuidriver.Session`:

```go
type resizer interface { Resize(rows, cols uint16) error }
```

`SetPTY`→`SetResizer(resizer)`; `Resize` now forwards `if b.rs == nil { return nil }; return
b.rs.Resize(rows, cols)` (delegate wraps its own `"tuidriver: resize …"` error). `os` and
`creack/pty` imports dropped from `bridge.go`. **`Bridge.Write` / `SetOutputObserver` / `Attach` are
untouched** — they stay the raw-byte fan-out hub; only the byte feed into them changed (see Patterns).

### `winsize.go`: SIGWINCH → `Session.Resize`

`watchWindowSize`/`resizeOnce` take `*tuidriver.Session` and call `sess.Resize(size.Rows, size.Cols)`
instead of `pty.Setsize(ptmx, size)`. The operator's *own* terminal is still read via
`pty.GetsizeFull(os.Stdin)` (the GC-finalizer-safe form; `creack/pty` + `x/term` imports kept). The
foreground prime resizes immediately from `StartPTY`'s fixed **40×120** spawn default to the
operator's real terminal — satisfying AC #2's "real terminal size, not the spawn default." Bridge mode
reaches real geometry when a client attaches and drives `Bridge.Resize`.

### Deliberate divergence from `ptyrunner`

Unlike [`internal/agentrun/ptyrunner`](../features/ptyrunner-package.md) (the other in-repo `Session`
consumer), `runOnce` does **NOT** call `tuidriver.EnsureClaudeEnv` — it force-overrides
`TERM=xterm-256color`, which would change claude's TUI rendering versus today's *inherited* `TERM`.
That override exists for downstream screen parsing, which #593 does not do. `cmd.Env =
append(os.Environ(), helperEnv...)` is kept exactly. ptyrunner also delivers prompts via
`Session.WritePrompt` (bracketed paste); #593 stays on the raw `AttachInput` seam — the
`DeliverPrompt` rewrite is [#594], not pulled forward.

`internal/supervisor/spawn.go`'s `SpawnPTY` (test-only, zero production callers) is left untouched —
AC #1 reaches only the host loop.

## Patterns established

- **Re-key, don't reshape.** The `*os.File`→`*Session` migration preserves the `sessReadyCh`
  close-on-register / freshen-on-clear choreography exactly and keeps every public method name, so the
  blast radius is a package-internal rename + a one-line write-seam swap. Honest field/helper names
  (`sess`, `setSession`) over keeping `ptmx`; honest *public* names deferred to avoid churning
  consumers (`WaitForPTY` keeps its name though it now waits on a Session).
- **Change only the byte feed to preserve a fan-out.** `Bridge.Write` already fanned raw output to
  *both* surviving consumers — `pyry attach` (`b.output`) and the assistant-turn bridge
  (`b.outputObserver`, used by `cmd/pyry/assistant_turn.go` + `assistant_turn_v2.go`). AC #4 is
  satisfied by swapping `io.Copy(bridge, ptmx)` → the `MirrorOutput → bridge.Write` pump; neither
  consumer nor `Bridge.Write`/`SetOutputObserver` changes.
- **Accept interfaces at the consumer; document the untyped-nil footgun.** `resizer` lives in
  `bridge.go` where it is consumed. A typed nil (`(*tuidriver.Session)(nil)`) is a *non-nil* interface
  that `Resize` would dereference — `runOnce` always clears with the untyped `nil` literal; the
  doc-comment pins this.
- **Teardown ordering preserved (lessons.md `SetPTY(nil)`-before-`EndIteration`).** Registrations are
  cleared before the PTY closes: bridge `setSession(nil)` → `SetResizer(nil)` → `EndIteration` →
  `sess.Close()`; foreground `setSession(nil)` → `input.Close()` → `sess.Close()`. A racing
  `WriteUserTurn`/`Resize` then sees `nil` and drops/no-ops rather than touching a closing session.

## Lessons learned

- **`MirrorOutput()` (drop-newest, buffer 256) is now the *only* output path** — the sealed `Session`
  exposes no `*os.File` to `io.Copy` from. This trades the old `io.Copy(dst, ptmx)` backpressure for
  best-effort drop-newest: the internal reader `select`s with a `default` drop, so even a stalled
  output pump can never wedge the reader or `Close`, and a dropped chunk heals on claude's next
  repaint. **Accepted design property (not a regression):** foreground output likewise becomes
  best-effort drop-newest — under sustained stdout back-pressure (e.g. Ctrl-S) the old path *paused*
  claude; the new path drops chunks that heal on repaint. Inherent to the ticket-mandated migration
  and matching the ptyrunner model; flagged for the record.
- **The output-pump range-loop terminates without an explicit cancel.** tui-driver's reader closes
  `mirrorOut` *before* `readerDone` (LIFO defers), so by the time `sess.Wait()` returns (`<-exited`
  then `<-readerDone`) the mirror channel is already closed — the `for chunk := range
  sess.MirrorOutput()` pump is draining/done. Join it through the existing `goroutineDrainTimeout`
  safety net.
- **The old EBADF resize race is structurally defused.** `Session.Resize` returns an error on a closed
  FD instead of operating on a dangling `*os.File`, so the [lessons.md](../../lessons.md)
  `SetPTY(nil)`-before-`EndIteration` race can no longer touch a closed fd. The clear-before-close
  teardown ordering is kept anyway, for clarity.
- **Code comments drifted post-swap (known, non-blocking).** Code review (PASS, no MUST/SHOULD-FIX)
  flagged that `Bridge.Write`'s doc-comment still describes the supervisor copying from the master via
  `io.Copy(bridge, ptmx)` and frames "Write never returns an error" as load-bearing to avoid wedging
  claude — the invariant is still true but its justification no longer holds (tui-driver's internal
  reader drains the PTY independently; the mirror pump ignores `Write`'s return). Similar stale `ptmx`
  / `io.Copy(bridge, ptmx)` phrasing lingers at `bridge.go:128`, `bridge_test.go`, and
  `supervisor_test.go`. Left as-is per the spec's "don't touch `Bridge.Write`'s code" guard; a future
  comment-only sweep can clean it.
- **Pre-existing e2e flake observed (out of scope, untouched).** `internal/e2e/internal/fakeclaude`
  `TestFakeClaude_OpensInitialAndRotatesOnTrigger` races on its trigger-file rotation under `-race`.
  That helper does not import `internal/supervisor`, is unchanged on this branch, passes on re-run, and
  is `e2e`-tagged (not in the `make check` gate). Flagged for the operator.

## Testing

Test-first (RED → GREEN); stdlib `testing` only; changes contained to `supervisor_test.go` +
`bridge_test.go`.

- **Existing behaviour tests stay green unchanged** (public API only): `RestartsAfterCrash`,
  `ChildExitsCleanly`, `GracefulShutdown`, `Foreground_NoStdinReaderLeak`, and the `WriteUserTurn_*`
  set — `HappyPath` spawns a real `Session` and polls the `stdin_to_file` helper, exercising
  `AttachInput` end-to-end.
- **Mechanical call-site swaps:** the four `WaitForPTY_*` tests swap `sup.setPTY(/dev/null)` →
  `sup.setSession(&tuidriver.Session{})`. A zero-value `&tuidriver.Session{}` is a valid non-nil
  pointer; `setSession` only stores it and drives the readiness channel — **never dereferences it** —
  so it is safe for readiness-only assertions. Never pair a dummy session with `WriteUserTurn` (would
  deref a nil PTY).
- **`TestBridge_Resize*` rewritten onto a `fakeResizer` double** (records `(rows, cols)` + settable
  error): forwards verbatim → `(40,100)`; propagates a delegate error (via `errors.Is`); returns `nil`
  when no resizer is registered; returns `nil` after `SetResizer(nil)`. The "geometry reaches the
  kernel" path is covered by tui-driver's own `Session.Resize` tests.
- **New `TestSupervisor_Bridge_MirrorReachesConsumer` (proves AC #4).** A new `emit_marker`
  `TestHelperProcess` mode writes a known marker to stdout then blocks; the test runs the supervisor in
  bridge mode and asserts the marker reaches **both** raw surfaces — the attached-client writer
  (`b.output`, the `pyry attach` path) and the output observer (`b.outputObserver`, the assistant-turn
  bridge) — via the `MirrorOutput → bridge.Write` feed. A goroutine-safe `syncBuffer` guards the pump's
  concurrent writes.
- **Gate:** `make check` (vet + `-race` + staticcheck + `cmd/substrate-guard` — no claude TUI screen
  literal added; mirror bytes are opaque and never inspected) + `go test -race
  ./internal/supervisor/...`, all green. Code review verified the real-claude e2e suite
  (`internal/e2e/realclaude`) passes against the local module cache.
