# Ticket #391 — `pyry agent-run` cuts over from PTY drive to stream-json subprocess

Wires the `streamrunner` primitive (#390) into the `pyry agent-run` verb, replacing the PTY blind-typing path that caused the 2026-05-14 `/doctor` prompt-poisoning regression. The verb now JSON-encodes the user turn on stdin and forwards claude's stream-json stdout byte-for-byte, instead of typing the prompt into a PTY input buffer and scraping the on-disk JSONL.

## Implementation

- **`cmd/pyry/agent_run.go`** — `runAgentRun` body collapses from ~150 lines of errgroup orchestration to ~30 lines: `--self-check` short-circuit → `parseAgentRunArgs` → `os.ReadFile(promptFile)` → resolve `PYRY_CLAUDE_BIN` → `signal.NotifyContext(SIGTERM, SIGINT)` → one `streamrunner.Run` call → return nil on nil or `context.Canceled`, else wrap `fmt.Errorf("agent-run: %w", err)`. Stderr explicitly threaded to `os.Stderr` (the PTY drive merged it into the PTY; new path makes it a deliberate forward).
- **`buildClaudeArgs` signature change** — from `(parsed, settingsPath, sessionID) []string` to `(parsed) []string`. New argv:

  ```
  --input-format stream-json
  --output-format stream-json
  --verbose
  --dangerously-skip-permissions
  --append-system-prompt-file <sys>
  --model <m>
  --effort <e>
  --max-turns <n>
  --allowed-tools <comma-joined>
  ```

  `--verbose` is what causes assistant message events to land on stdout under stream-json (without it, only `result` is emitted) — spike-verified 2026-05-14. `--allowed-tools` is comma-joined; `splitAllowedTools` already normalises operator input to a clean slice. The three load-bearing PTY-mode flags (`--settings`, `--permission-mode`, `--session-id`) are negatively pinned by `TestBuildClaudeArgs_Shape`.
- **Removed from the verb's runtime path** (packages stay in tree per the ticket — cleanup is a follow-up):
  - Per-spawn `.pyry-agent-run-settings.json` write (`agentrun.WriteSettings`) — replaced by `--allowed-tools` as the authoritative tool gate under `-p`-style stream-json.
  - Workspace-trust mark in `~/.claude.json` (`agentrun.MarkWorkdirTrusted`) — replaced by `--dangerously-skip-permissions`.
  - `newSessionUUID` + `crypto/rand` import — claude mints its own session id under stream-json and reports it in the `system init` event.
  - JSONL tail watcher (`internal/agentrun/jsonl`, `jsonl/tail`) — claude writes events to stdout directly.
  - `streamjson.Emitter` re-emission + trailer composition (`internal/agentrun/streamjson`) — claude emits its canonical `result` event itself.
  - `agentrun.Drive` PTY path — not invoked from this codepath any more (still imported by `agent_run_selfcheck.go` for the `--self-check` mode tracked in #375).
  - `settings-file: <path>` stdout marker line — the `system init` event takes over its dispatcher-metadata role.
  - `classifyForEmitter` helper (no emitter to classify for) and the `golang.org/x/sync/errgroup` import.
- **`parseDurationEnv` survives** because `agent_run_selfcheck.go` (#336) still calls it to compress PTY-Drive sleeps for its own tests. The stream-json runtime path has no timing knobs.
- **Concurrency model** — a single goroutine (`main`'s). `streamrunner.Run` blocks until the child exits; stdlib internally manages stdin/stdout/stderr forwarders. Shutdown is fully stdlib-driven via `cmd.Cancel = SIGTERM` + `cmd.WaitDelay = 5s`. The old three-actor errgroup (drive + watcher + signal handler racing) is gone.
- **Exit-code mapping**:
  - clean run → `streamrunner.Run` returns nil → verb returns nil → exit 0.
  - non-zero claude exit → `*exec.ExitError` → wrapped `agent-run: %w` → exit 1. The exact non-zero code claude produced is preserved in the chain (`errors.As`) but not propagated as the pyry exit code; AC#4's "no changes required on the dispatcher's salvage path" is the binding constraint.
  - operator SIGTERM/SIGINT mid-run → ctx-cancel → `streamrunner` SIGTERMs the child, waits 5s, returns nil → verb returns nil → exit 0.

## Tests

`cmd/pyry/agent_run_test.go` rewritten in place. Deleted: `TestRunAgentRun_EmitsSettingsFile`, `TestRunAgentRun_MarksWorkdirTrusted`, `TestNewSessionUUID_Shape`, `writeFakeClaudeJSONL`. Kept byte-identical: parse-args tests and `TestSplitAllowedTools`.

- `TestAgentRunStreamJSONFake` — new fake claude entry point gated by `PYRY_AGENT_RUN_FAKE=1`. Drains stdin (optionally captured to `GO_AGENT_RUN_FAKE_STDIN_FILE` for envelope round-trip assertions), then per `GO_AGENT_RUN_FAKE_MODE`: `clean` emits three canned stream-json lines (`system init` / `assistant text` / `result success`) and exits 0; `exit1` exits 1 after draining; `sleep` sleeps 30s (mirrors streamrunner's helper shape; verb-level test doesn't exercise it).
- `configureFakeClaude(t)` — simplified one-liner shell wrapper that re-execs the test binary with `-test.run=^TestAgentRunStreamJSONFake$`, dropping the production argv on the floor (the Go test binary's flag parser would reject real claude flags). Sets `PYRY_CLAUDE_BIN` + `PYRY_AGENT_RUN_FAKE=1`.
- `TestBuildClaudeArgs_Shape` — two table rows × `slices.Equal` against an explicit `want`, plus named structural assertions (`--dangerously-skip-permissions` present; `--input-format`/`--output-format` each followed by `"stream-json"`; `--verbose` present; `--allowed-tools` followed by the comma-joined tools; `--max-turns` followed by `strconv.Itoa`), plus negative pins on the three banned legacy flags.
- `TestRunAgentRun_StreamJSON_Clean` — full end-to-end. Asserts (a) stream-json events appear in order `system init` → `assistant` → `result success`, (b) stdout does NOT start with `settings-file: ` (negative pin), (c) neither `<workdir>/.pyry-agent-run-settings.json` nor `<HOME>/.claude.json` is written by the verb (negative filesystem pins), (d) the prompt file's `"hello"` round-trips into the stream-json user-turn envelope on claude's stdin — confirms the verb uses `streamrunner.Run` rather than a path that bypasses the envelope.
- `TestRunAgentRun_StreamJSON_NonZeroExit` — `errors.As(err, &exitErr)` succeeds with `ExitCode() == 1`; error string starts with `agent-run: `.

### Deliberately omitted: verb-level ctx-cancel test

The streamrunner package owns the SIGTERM/SIGKILL grace behaviour (`TestRun_CtxCancelMidRun` in `internal/agentrun/streamrunner`). The verb's only ctx-cancel logic is one line (`return nil` if `errors.Is(err, context.Canceled)`). Covering it at the verb layer would require either a package-private signal-context hook for DI, or sending SIGTERM to the test process (which risks killing the test runner). Both are heavier than the bug they prevent. The omission is documented inline in the test file so it's intentional, not forgotten.

### JSONL emission assumption

Production claude under `--output-format stream-json` still writes `~/.claude/projects/<encoded-cwd>/<session-id>.jsonl`. The verb no longer reads it; the fake does not produce it. Verification is left to manual smoke against a real `claude` binary — see #375 for the self-check refactor that owns that surface. The assumption is pinned in a top-of-file comment in `agent_run_test.go`.

## Patterns established

- **Cut over to the new primitive; don't dual-path.** Once `streamrunner` (#390) was proven, this ticket replaced the entire PTY-drive orchestration in one move rather than adding a feature flag. The old packages stay compiled (kept for `--self-check`'s benefit) but the verb has one runtime path, not two. The cleanup of the unused packages is a separate follow-up — separating "stop importing" from "delete the file" keeps each PR small and the bisect surface clean.
- **Document deliberate test omissions inline.** The verb-level ctx-cancel test was rejected with a one-paragraph rationale (streamrunner owns the behaviour; verb logic is one line; DI cost exceeds bug-prevention value). Future readers see "this is intentional" instead of "someone forgot to write this test."
- **AC-trace ↔ test mapping carries through.** Each AC maps to a specific test (or a documented absence-assertion). `TestRunAgentRun_StreamJSON_Clean` alone covers AC#1, AC#3, and AC#5 via positive event-order checks plus negative filesystem/marker pins.

## Lessons learned

- **Prompt poisoning is a class, not an instance.** The `/doctor` regression on 2026-05-14 looked like a bug in the settings-file schema (an invalid `defaultMode: "deny"` value), but the load-bearing failure was the PTY drive's "sleep, then type" delivery model — *any* content claude's startup happens to queue into the input buffer concatenates with the prompt. The fix is structural: the prompt is now a JSON envelope on stdin, not bytes typed into a TUI. Belt-and-suspenders on the settings file would not have prevented this; the design itself had to change. (See [decisions/](../decisions/) for the original PTY-supervisor decision; the stream-json cutover is the principled reversal for headless runs.)
- **"Stays in tree" doesn't mean "stays imported."** The PTY-drive / trust / settings / JSONL-tail / streamjson packages all stay compiled because `--self-check` (#336) still uses parts of them, but the verb's runtime path imports none of them any more. Drawing the line at "no runtime imports from this codepath" lets the cleanup ticket delete the packages in one isolated commit once the last consumer (`--self-check`, #375) migrates.
- **`--dangerously-skip-permissions` is acceptable when `--allowed-tools` is the gate.** The flag name is alarming; the security posture isn't worse than the old design. The PTY path *defeated* the trust dialog by writing to `~/.claude.json`; the new path skips it with a flag. The authority gate moves from a deny-default settings file to the `--allowed-tools` argv, both of which the operator (dispatcher) controls end-to-end. Document the reasoning at the flag site (it lives in `buildClaudeArgs`'s doc-comment) so a future reader doesn't have to grep the spec.

## Out of scope (deferred)

- Deletion of `internal/agentrun/{drive,trust,settings}.go`, `internal/agentrun/jsonl`, `internal/agentrun/jsonl/tail`, `internal/agentrun/streamjson` — cleanup ticket, blocked on #375 migrating `--self-check` off them.
- `--permission-prompt-tool stdio` protocol handling for the mobile case — separate ticket once mobile design lands.
- Propagating claude's exact non-zero exit code as the pyry process exit code — dispatcher hasn't asked for it.
- Dispatcher submodule pointer bump to re-enable Phase C — explicit out-of-scope per the ticket.
