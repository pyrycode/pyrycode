# Ticket #670 — Daemon pre-marks its workdir trusted (confined to `$HOME`) before spawning claude

The daemon serve path (`runSupervisor`) now pre-marks the supervised claude's workdir trusted in `~/.claude.json` before any spawn, and confines that auto-trust to the operator's home directory. This fixes the **#421 clean-exit restart loop**: in a workdir claude had never trusted, the supervised claude wedged on claude's workspace-trust modal ("Is this a project you created or one you trust? … No, exit"), took the No path, and the supervisor respawned it (`--continue`) into the same modal every ~2–90s — `claude exited cleanly` forever. The bridge forwards only transcript events, so the phone never saw the modal and the conversation hung. Pre-marking the workdir trusted eliminated the loop entirely (confirmed live, pyrycode-mobile#421). The agent-run path already pre-marked at the cmd layer (#470/#475); the daemon serve path was the sibling that never did.

## Implementation

All production change is in `cmd/pyry/main.go` (the wiring layer); `internal/supervisor` and `internal/sessions` are unchanged.

- **`confineWorkdirToHome(workdir) (realpath, error)`** (`main.go:430`) — resolves `$HOME` (`os.UserHomeDir` → `filepath.EvalSymlinks`) and the workdir (`filepath.Abs` → `filepath.EvalSymlinks`), then requires `withinDir(homeReal, workReal)`. Returns the workdir realpath on success, a content-free rejection error otherwise. Empty workdir resolves to the process cwd (matching how the supervisor launches claude with `-pyry-workdir` unset).
- **`withinDir(dir, path) bool`** (`main.go:456`) — boundary-aware containment via `filepath.Rel`: rejects a `rel` of `..` or one beginning `../`. This is the security-critical primitive — a naive string-prefix check would treat `/home/userfoo` as inside `/home/user`.
- **`runSupervisor` wiring** (`main.go:509-516`): `workdirReal, err := confineWorkdirToHome(*workdir)` → `trustedWorkdir, err := trustMark(workdirReal)` (the existing `agent_run.go:28` package var = `trust.MarkWorkdirTrusted`) → `Bootstrap.WorkDir = trustedWorkdir` (`main.go:563`, was `*workdir`). Runs before `conversations.Load` / `sessions.New` / relay start.
- **AC-2 byte-identical key.** `Bootstrap.WorkDir` flows to `supervisor.Config.WorkDir` → existing `runOnce` `cmd.Dir = s.cfg.WorkDir` (`supervisor.go:639`, unchanged) and to `sessionTpl` (`pool.go:423`). The marked `projects[...]` key and the child's cwd are the same realpath, so a symlinked or wrong-case workdir cannot re-render the modal (the #470/#473 gotcha).
- **Fail-fast (AC-4).** Both errors return straight out of `runSupervisor`, before any spawn or readiness claim — a loud non-zero startup exit, never a silent restart loop. Matches the "malformed registry → fatal at startup" convention.

## Patterns established

- **`$HOME` confinement lives at the caller, not the shared helper.** `trust.MarkWorkdirTrusted` is used by *both* the agent-run path and now daemon-serve; pushing the bound into the helper would silently confine agent-run too (the ticket does not authorize that). The new bound is a strict-tightening deny-gate in `runSupervisor` only — `MarkWorkdirTrusted` is untouched, agent-run stays unconfined. (Confirmed by code-review's security pass.)
- **Canonicalise *both* sides of a containment check** (the #118/#221 "one path canonical, one not" gotcha). `EvalSymlinks` resolves `$HOME` *and* the workdir before the boundary-aware (`filepath.Rel`, not string-prefix) comparison, so a symlinked home (macOS `/var`→`/private/var`) is not a false reject and `/home/userfoo` is not inside `/home/user`. Resolving the *workdir* side to its realpath also correctly rejects a symlink that lives under `$HOME` but points outside it (claude keys trust on the realpath).
- **Mirror the agent-run cmd-layer trust pattern** ([codebase/470.md](470.md)): the cmd layer marks + resolves; the spawn primitive (supervisor, the daemon's analogue of `ptyrunner`) receives a pre-resolved realpath and stays unaware of `agentrun/trust`. `runSupervisor` is the daemon's `runAgentRunPty`.
- **Content-free rejection errors** — name the offending path and the `$HOME` boundary, never `~/.claude.json` contents (the file may carry tokens). Asserted by `TestConfineWorkdirToHome_RejectsOutsideHome` (rejection must not contain `.claude.json`).

## Lessons learned

- **The spec under-specified the security bound.** The architecture spec's design, AC-mapping, and security review covered the trust pre-mark (its AC-1/AC-2/AC-4) but did **not** address the `$HOME` confinement — which is ticket AC-3 and the whole reason the ticket is `security-sensitive`. It was implemented from the authoritative ticket body + Technical Notes (canonicalise both sides, boundary-aware containment, content-free errors). When a spec and ticket diverge, the ticket body is authoritative; the developer flagged the gap in the PR so the bound got explicit review and doc coverage.
- **The new confinement broke the e2e harness's implicit workdir.** The harness ran the daemon with `-pyry-workdir` unset, so its workdir resolved to the **test process cwd (the repo dir)** — outside the isolated `HOME=t.TempDir()`. Under the new bound that fail-fasts *every* serve-path test. Fixed by defaulting the daemon workdir to its `$HOME` in `spawnWith` (last-wins, overridable) and in the four bespoke daemon-mode spawn sites (`auto_attach.go` ×3, `attach_pty.go` ×1). **Any future e2e helper that spawns a serve-mode daemon must set `-pyry-workdir` within the test HOME.** (`attach`-verb spawns have no serve path and are correctly left alone.)
- **Code-review NITs (none blocking, no change made):** (1) `confineWorkdirToHome` resolves the path, then `trustMark`→`MarkWorkdirTrusted` resolves it again — idempotent on an already-canonical path, AC-2 still holds; the confinement legitimately needs its own resolution to apply the bound. (2) The `confineWorkdirToHome` doc comment cites "case-folding" as handled by `EvalSymlinks`, which it does not do on case-insensitive filesystems — but it isn't load-bearing, because the marked key and the spawned `cmd.Dir` are the same bytes (`trustedWorkdir`), so claude resolves both identically and the modal can't re-render regardless.

## Out of scope / forward notes

- **Runtime trust-modal safety net on the bridge path — deferred.** `ptyrunner` *detects + aborts* the modal (`ErrTrustModalDetected`); it does not *dismiss* it, and relies on a dispatcher retry the long-lived supervised host doesn't have. The pre-mark eliminated the observed loop entirely, so a runtime net would defend an unobserved post-fix mode (Evidence-Based Fix Selection). File a follow-up only if the pre-mark proves insufficient in practice.
- **Per-conversation pre-marking (#672).** Once conversation-sessions give each conversation its own (phone-supplied) `conversation.Cwd`, the same canonicalise-and-confine check applies per workdir, with an out-of-`$HOME` path rejected with a clear error *to the phone* rather than at startup. That phone-facing rejection path is **not built here** — no live caller until #672.
- **`claudeSessionsDir` realpath alignment for symlinked daemon workdirs.** `resolveClaudeSessionsDir` still encodes `Abs(*workdir)` (no symlink resolution); for a symlinked daemon workdir this can diverge from claude's realpath-encoded JSONL dir (affecting #668's transcript resolver / rotation watcher). Pre-existing, unobserved, neither created nor worsened here.
