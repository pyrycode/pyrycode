# Ticket #347 — `EncodeProjectDir` helper for `~/.claude/projects/` dashed name

Adds `internal/agentrun.EncodeProjectDir(workdir) (string, error)`: chains `ResolveWorkdir` then maps `/` AND `.` to `-` in the resolved absolute path. Centralizes the on-disk encoding rule under `~/.claude/projects/<encoded-cwd>/<sid>.jsonl` so the JSONL watcher (#333) and other consumers do not inline `strings.ReplaceAll` at each call site. No caller in this diff. See [agentrun-package.md](../features/agentrun-package.md) for the package surface.

## Implementation

- `internal/agentrun/trust.go` — adds package-level `projectDirReplacer = strings.NewReplacer("/", "-", ".", "-")` and `EncodeProjectDir(workdir) (string, error)`. Body is two lines: chain `ResolveWorkdir`, return its error verbatim on failure, otherwise return `projectDirReplacer.Replace(resolved)`.
- `internal/agentrun/trust_test.go` — four new rows alongside the existing `ResolveWorkdir` tests:
  - `TestEncodeProjectDir_DarwinRealpath` (darwin-only via `runtime.GOOS` skip) — `t.TempDir()` resolves through `/private/var/folders/...`; asserts the result starts with `-private-var-folders-`. Pins the realpath step; a non-resolved encoding would produce `-var-folders-...` and the test would fail.
  - `TestEncodeProjectDir_LiteralSubstitution` — pre-resolves a `t.TempDir()` via `filepath.EvalSymlinks`, then asserts the helper output equals the same replacer applied to the resolved path. Platform-agnostic pin on the substitution rule.
  - `TestEncodeProjectDir_DotInPathSegment` — creates `t.TempDir() + "/.hidden"`, asserts the result has suffix `--hidden` (the `/.` sequence becomes `--`). Pins the `.` → `-` half of the rule.
  - `TestEncodeProjectDir_MissingPath` — asserts `errors.Is(err, fs.ErrNotExist)` against a non-existent path, mirroring `TestResolveWorkdir_MissingPath`. Pins the error pass-through contract.

## Patterns established

- **`strings.NewReplacer` over chained `ReplaceAll` for multi-char → single-target rules.** Single-pass; the "both characters map to dash" intent is visible at the declaration site instead of being implied by the order of two sequential calls. Package-level `var` so the replacer is constructed once.
- **Pass-through error wrapping is non-wrapping.** The ticket AC explicitly required `errors.Is(err, fs.ErrNotExist)` to keep working through the new helper. The implementation returns `ResolveWorkdir`'s error unchanged — wrapping would double-prefix the `"agentrun: resolve workdir %q: %w"` text without adding information. Convention for any thin wrapper that only adds a post-success transform: chain the inner, surface its error verbatim.
- **Sibling helpers, not a configurable one.** `ResolveWorkdir` (resolved abs path — key shape for `~/.claude.json`'s `projects` map) and `EncodeProjectDir` (dashed name — directory shape under `~/.claude/projects/`) are two distinct output shapes consumed by different call sites. Folding the dash transformation into `ResolveWorkdir` via a bool/enum would break the existing `MarkWorkdirTrusted` caller. Keep one function per output shape; share the resolve half by chaining.

## Lessons learned

- **Ticket ACs on third-party filesystem layout are advisory; observation wins.** The #347 issue body specified only `/` → `-`. Direct inspection of `~/.claude/projects/` on the working host showed entries like `-Users-juhanailmoniemi-Workspace-Projects--pyrycode-worktrees-architect-1` — `.` is ALSO mapped to `-`, so a workdir containing `.pyrycode-worktrees` encodes with a doubled dash (`--pyrycode-worktrees-`). The architect spec called this out from observation and `docs/lessons.md:53` ("Don't trust ticket bodies on filesystem layout — observe."); shipping the AC-literal rule would have produced keys that never match claude's actual directory names — silently broken for any workdir containing a `.` segment (extremely common: `.git`, `.venv`, `.pyrycode-worktrees`). The `TestEncodeProjectDir_DotInPathSegment` row pins the observed rule so a future refactor can't quietly drop it.
- **The macOS realpath trap recurs.** `t.TempDir()` under `/var/folders/...` only encodes correctly after `EvalSymlinks` resolves it to `/private/var/folders/...`. Same trap as the #341 trust-mark key shape and `internal/sessions/rotation/watcher.go`'s path comparison. Any future helper that produces a key claude will look up must resolve symlinks first; the darwin-only test row is the cheap, durable pin against accidental regressions.

## Out of scope (deferred to siblings)

- JSONL watcher consumption (#333) — the first caller; will pass `~/.claude/projects/` + `EncodeProjectDir(workdir)` to fsnotify.
- Any cross-platform encoding-rule audit. The rule is darwin- and linux-only since pyrycode does not target Windows; if claude ever changes the rule, the four pinning tests fail loudly.
