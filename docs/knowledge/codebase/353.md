# Ticket #353 — `agent-run/jsonl`: surface every line kind with raw bytes and usage on Event

Additive extension to the `jsonl.Reader` contract for a future stream-json emitter (split from #335). The reader now surfaces an `Event` for every well-formed line — not just assistant entries — and each `Event` carries the verbatim line bytes (`Raw`) plus a classified `Kind` and an optional `*UsageBlock`. Existing `StopReason` / `TextChars` / `EndOfTurn` semantics for assistant entries are preserved verbatim. The only in-tree consumer (`tail.Watcher`) already forwarded every Event unconditionally — no code change there, just test-assertion updates.

See [features/jsonl-reader.md](../features/jsonl-reader.md) for the full updated surface and behaviour matrix.

## Implementation

- `internal/agentrun/jsonl/reader.go` — `Event` gains three trailing fields (`Raw json.RawMessage`, `Kind string`, `Usage *UsageBlock`); existing three (`StopReason`, `TextChars`, `EndOfTurn`) keep their contracts and field positions. New exported type `UsageBlock{InputTokens, OutputTokens, CacheCreationInputTokens, CacheReadInputTokens int}` mirrors the assistant `message.usage` JSON object. `rawAssistantMessage` extended with a pointer-typed `Usage *struct{…}` so `encoding/json` leaves it `nil` when the field is absent. Closed whitelist `knownKinds = {"assistant","user","tool_use","tool_result","system","attachment"}` — any other `type` value (and a missing field) maps to `Kind == ""`.
- **`Next` loop rewrite, single semantic flip.** The previous `if raw.Type != "assistant" { continue }` is replaced by classification + early return for non-assistant kinds (only `Raw` + `Kind` populated; everything else zero/nil). For `assistant`, the inner `rawAssistantMessage` decode runs as before, and the loop additionally builds a `*UsageBlock` from `msg.Usage` when present. Malformed-JSON lines (outer or inner) remain Warn-logged-and-skipped without surfacing an Event — the contract that one broken line must not poison the stream is preserved.
- **Buffer-aliasing fix for `Raw`.** The fresh-allocation requirement is the single sharpest implementation detail: `lineCopy := make([]byte, i); copy(lineCopy, r.buf[:i])` BEFORE advancing `r.buf = r.buf[i+1:]`. A naive `lineCopy := r.buf[:i]` slice would share the backing array with the reader's growing buffer; a subsequent `append(r.buf, ...)` can write into that same memory and silently mutate `Event.Raw` after the caller has it. Pinned by `TestReader_RawByteEquivalence`, which calls `Next` once more before inspecting earlier `Raw` slices.
- **`tail.Watcher` no-op.** `drain` already forwards every `(Event, nil)` to `cfg.OnEvent` and only stops on `ev.EndOfTurn`. The acceptance-criterion line "the watcher adds no filtering of its own beyond what the reader does" was already satisfied; only test-assertion counts changed.
- `internal/agentrun/jsonl/reader_test.go` — `TestReader_NonAssistantLinesSkipped` repurposed to `TestReader_NonAssistantLinesSurfaced` (same fixture: user + system + summary + assistant; asserts 4 Events in source order with the expected `Kind` per line, only the last has `EndOfTurn == true`, `AssistantCount() == 1` unchanged). New: `TestReader_RawByteEquivalence` (three lines incl. nested JSON / whitespace / unicode, Next-then-inspect-earlier-Raw to catch the aliasing regression), `TestReader_KindClassification` (table over the seven known + unknown + missing-`type`), `TestReader_UsageParsedOnAssistant` (distinct non-zero values for all four fields), `TestReader_UsageNilOnAssistantWithoutUsage`, `TestReader_UsageNilOnNonAssistant` (defensive: even a `user` line carrying a `usage`-shaped sub-object must yield `Usage == nil`).
- `internal/agentrun/jsonl/tail/watcher_test.go` — `TestWatcher_LateCreate` event count 2 → 3 (assistant + user + assistant), assertion shifted from `events[1].EndOfTurn` to `events[len(events)-1].EndOfTurn` so future fixture changes don't re-break it. `TestWatcher_FixtureIntegration` event count 25 → 62 — `clean.jsonl` has 64 total lines, the sole `end_turn` assistant entry sits at line 62, so the watcher drains exactly 62 Events before stopping (the previous 25 was the assistant-only subset of the same prefix). The end-of-turn count + last-event `EndOfTurn` assertions are unchanged.

## Patterns established

- **Fresh-allocate before exposing a slice of an internal growable buffer.** Whenever a struct field handed to callers points into a buffer the package will later `append` to, copy first. The compiler / `go vet` will not catch this; it manifests only when the caller retains the slice across the next operation that triggers a backing-array realloc. The pattern to mirror in future ticket work: `lineCopy := make([]byte, i); copy(lineCopy, src[:i])` immediately before any state advance that could reuse the source memory.
- **Closed whitelist + verbatim-bytes fallback for forward compatibility.** `Kind` classifies a known set into a typed string; everything else collapses to `""` and the consumer recovers the original via `Raw`. New claude line types added in future versions land in the unrecognised bucket until the whitelist is widened, and downstream re-emitters keep working unchanged. Prefer this shape over a `map[string]any`-style open enum any time the protocol is owned by an external producer.
- **Pointer-typed optional JSON sub-objects + pointer-typed exported field.** `rawAssistantMessage.Usage *struct{...}` distinguishes "field absent" from "field present with all zeros" at decode time; `Event.Usage *UsageBlock` propagates that distinction to consumers. The exported type itself is a value (not a pointer-to-anonymous-struct) so consumers get a stable named type with no nil-deref surprises inside the struct.
- **Additive API extension via trailing struct fields.** New fields go at the end of `Event` so `gofmt` diffs stay minimal and any (hypothetical) struct-literal consumers using positional initialisation would fail to compile rather than silently bind to the wrong field. Existing field semantics are preserved verbatim; the doc-comment per field carries the per-Kind contract.

## Lessons learned

- **"Skip" → "surface" is a one-line semantic flip that ripples through `len(events)` test assertions.** Before #353 the reader's contract was "assistant-only Events"; after #353 the watcher's `OnEvent` callback fires for every line kind. The watcher's `drain` loop didn't change — but two tail tests asserting `len(events) == <assistant-count>` had to flip to the full line count. Future contract flips of this shape should grep for `len(events)` (and equivalents) across all consumers before declaring the change additive.
- **Counting from the fixture, not from intuition, catches off-by-many bugs.** `TestWatcher_FixtureIntegration` against `clean.jsonl` was previously `len(events) == 25` (assistant-only entries). After surfacing every line the correct count is 62, not 64 — the file has 64 lines total, but the sole `end_turn` assistant entry sits at line 62 and the watcher stops draining once `EndOfTurn` fires. The `2` "missing" lines (a user + a summary tail) sit past the stop point; the recorder never sees them. Anchoring the assertion to the actual stop-point position (and noting it in the comment) prevents a future fixture refresh from silently breaking the test.
- **`json.RawMessage` is `[]byte` under the hood — buffer aliasing applies.** The architect spec called the aliasing trap out explicitly and the developer mitigated it; without that callout it would be easy to land `lineCopy := r.buf[:i]` and watch tests pass single-line but break under back-to-back `Next` calls. Worth pinning in code review checklists: whenever a `json.RawMessage` (or any `[]byte`) is handed to a caller from a reader that owns a growable internal buffer, verify a `make`+`copy` sits between the slice and the caller.

## Out of scope (deferred to siblings / future)

- **The stream-json emitter itself.** #353 only extends the reader contract; no downstream consumer of `Event.Raw` / `Event.Kind` / `Event.Usage` exists yet. The next slice (still split from #335) wires those into the `pyry agent-run` verb's `--output-format stream-json` path.
- **Usage aggregation across a run.** The reader surfaces per-Event `Usage`; summing across all assistant Events to compose a result trailer is the emitter's job.
- **Stripping `'\r'` from CRLF.** Decision recorded in the spec: leave it. Inventing CRLF semantics here would risk byte-equivalence guarantees against claude's actual output, which is LF-only in observed fixtures. If claude ever writes CRLF, the consumer normalises.
- **Debug-log for unrecognised `Kind`.** Decision recorded in the spec: no. The whole point of the `""` fallback is forward compatibility; logging every unknown line would be noise the moment claude adds a new kind. Bytes survive in `Raw`.
