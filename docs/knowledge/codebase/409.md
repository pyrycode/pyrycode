# Ticket #409 — `realclaude.WithWorktreeAuthenticated` opt-in real-API fixture

`internal/e2e/realclaude` gained a `WithWorktreeAuthenticated(t) string` sibling to `WithWorktree(t)` that preserves the `$HOME`-pinned worktree but additionally re-pins `ANTHROPIC_API_KEY` from the outer test environment so the spawned `pyry agent-run` subprocess can authenticate against the real Anthropic API. Unblocks #383 (the permission-protocol spike that needed a real model response and hit the synthetic `"Not logged in"` envelope) and any future realclaude test that needs more than an argv-shape probe.

## Implementation

- **`internal/e2e/realclaude/fixtures.go` — new exported `WithWorktreeAuthenticated(t *testing.T) string`** (`fixtures.go:39-54`, ~12 LoC). Six-step body, no helper extraction, no abstraction:
  1. `t.Helper()`.
  2. `key := os.Getenv("ANTHROPIC_API_KEY")` — read from the outer test-runner env.
  3. If `key == ""`, `t.Skipf(...)` with a named-variable message — **skip precedes `t.TempDir()`** so the framework doesn't allocate an unused temp dir on skip. AC#5 mandates skip-not-fail so the suite stays green on contributor machines without API keys; AC#4 reconciles with #5 only by routing the absent-credential branch through `t.Skip`.
  4. `dir := WithWorktree(t)` — delegates to the existing helper for `t.TempDir()` + `t.Setenv("HOME", dir)`.
  5. `t.Setenv("ANTHROPIC_API_KEY", key)` — explicit re-pin into the test process env. Mechanically redundant with `RunPyryAgentRun`'s `cmd.Env = append(os.Environ(), ...)` inheritance at `fixtures.go:169`, but documents intent at the call site and forecloses silent regression if a future refactor inserts an env-scrubbing helper between this call and the subprocess spawn. The `t.Setenv` cleanup ordering also restores the value on test exit even if other in-test code mutates it.
  6. Returns the workdir path — same signature shape as `WithWorktree`.
- **No imports added.** `os` and `testing` were already in the file.
- **Doc comment (5 lines)** names `ANTHROPIC_API_KEY` literally, restricts the helper to real-API tests (not argv-shape probes), and surfaces the skip path. Searchable via `go doc github.com/pyrycode/pyrycode/internal/e2e/realclaude.WithWorktreeAuthenticated` under `-tags=e2e_realclaude`.
- **No credential-shape validation** (no `strings.HasPrefix("sk-ant-")`). Claude is the only authority on whether a key is valid; the helper propagates any non-empty value and lets the subprocess surface bad values as a non-zero exit. Same one-place-to-update discipline `validateRunOpts` uses for `--effort` (`fixtures.go:207`).
- **`internal/e2e/realclaude/fixtures_test.go` — new `TestWithWorktreeAuthenticated_RealAssistant`** (`fixtures_test.go:60-102`, ~43 LoC). Five-phase shape that mirrors `runRoleSmokeTest` (`per_agent_test.go:83-133`):
  1. `WithWorktreeAuthenticated(t)` — self-skips when `ANTHROPIC_API_KEY` is unset.
  2. `RunPyryAgentRun(t, RunOpts{...})` with minimum viable shape — `Prompt: "Reply with the single word 'pong' and nothing else."`, `SystemPrompt: "You are a minimal e2e authentication probe. Keep replies under 10 words."`, `AllowedTools: []string{"Read"}` (minimum non-empty per `validateRunOpts`), `MaxTurns: 1`, `Effort: "low"`, `Model: "claude-haiku-4-5"`. No `Timeout` override — 5-min default from `RunPyryAgentRun`.
  3. `ExitCode == 0` (failure dumps truncated stderr).
  4. `SessionID != ""` (failure dumps truncated stdout).
  5. **Negative-pin** against the #383 failure mode: `bytes.Contains(result.Stdout, []byte("\"model\":\"<synthetic>\""))` and `bytes.Contains(result.Stdout, []byte("\"error\":\"authentication_failed\""))` both rejected. The structural assertions below would already fail on a synthetic-auth envelope (`ExitCode != 0`, no `EndOfTurn` text), but the substring check produces a clearer diagnostic for the auth-failed mode specifically. Stdout in the diagnostic is capped via `truncate` (1 KiB) — bounded by design, not as a leakage defense against today's stream-json (which does not echo the API key), but as cheap insurance against any future claude diagnostic change that might.
  6. `ReadJSONL(t, workdir, result.SessionID)` walked for the first `Kind == "assistant" && EndOfTurn && TextChars > 0` event; if absent, failure cites the resolved JSONL path via the file-local `jsonlPathFor` (`prompt_fidelity_test.go:79`, fifth consumer now) and truncated stderr.
- **Cost**: one haiku/low/`max-turns=1` call per `make e2e-realclaude` run when `ANTHROPIC_API_KEY` is exported, ~$0.01; wall time ~10–30 s. When the credential is absent the test reports `SKIP` via Go's standard output and contributes zero cost / time.
- **`t.Parallel()` is NOT called** — matches the existing realclaude convention (cost-predictable, no API rate-limit interaction). The shared `truncate` and `jsonlPathFor` helpers came along as file-local symbols rather than fixture-exports — YAGNI bar for promoting to fixture-package symbols is "third consumer" by the established pattern; both are already past that, but the realclaude convention has so far kept them file-private and no test in this PR forces the change.
- **No change to `RunPyryAgentRun`.** The subprocess already inherits the full outer env via `cmd.Env = append(os.Environ(), opts.ExtraEnv...)` (`fixtures.go:169`); the helper's only contract change vs. plain `WithWorktree` is the credential-presence gate plus the explicit `t.Setenv` re-pin. AC#1 satisfied by the inheritance path alone; AC#3/#4 satisfied by the gate; AC#5 satisfied by the test's pass-on-credentials / skip-on-absence behaviour.

## Patterns established

- **Skip-on-missing-prereq belongs inside the fixture, not in each test.** AC#3 mandates the doc comment name the prerequisite and AC#5 mandates skip-not-fail; combining those into a single gate at the helper means (a) the doc comment and the gate co-locate, (b) future consumers don't duplicate the `os.Getenv("ANTHROPIC_API_KEY") == ""` check at every call site, and (c) the helper IS the gate — a test calling `WithWorktreeAuthenticated` self-elects to require auth, and the helper handles that election. Sibling realclaude tests that don't need auth keep calling `WithWorktree(t)` and don't pay any credential-gate cost.
- **`t.Setenv` redundant-with-inheritance is intent-documenting, not bug-fixing.** Step 5 of the helper would be a no-op today: `RunPyryAgentRun` already inherits `ANTHROPIC_API_KEY` via `os.Environ()`. The redundancy is load-bearing for *future* maintenance — if anyone later inserts an env-scrubbing helper, or refactors `WithWorktree` to wrap with a different env-isolation primitive, the explicit re-pin documents the contract at the call site and makes the silent regression a compile-or-test failure rather than an undetected `authentication_failed` run. Mirror this pattern when a fixture's correctness depends on env inheritance that could plausibly be scrubbed in a future refactor.
- **Doc-comment env-var literal naming is the discoverability surface.** Naming `ANTHROPIC_API_KEY` in the doc comment (not in a separate doc page, not in a constant declared elsewhere) is the only documentation deliverable per AC#3. `go doc` is the search index. Future fixtures that gate on outer-env state should follow the same pattern: literal env-var name in the doc comment, no separate prose doc.
- **Negative-pin substring assertions complement structural ones.** The structural assertions (`ExitCode == 0`, `EndOfTurn` text present) would already fail on a synthetic-auth envelope; the substring checks against `"model":"<synthetic>"` and `"error":"authentication_failed"` produce a clearer diagnostic for the auth-failed mode specifically. This is belt-and-suspenders against a *named* failure mode (#383's trace), not generic defensive coding — the substring set is the literal fingerprint of one bug class, and the assertion's failure message names that class to the operator.

## Lessons learned

- **The fixture's authentication failure mode looks like a model failure.** A `$HOME`-isolated subprocess that can't find credentials returns a synthetic `assistant` envelope with `"text":"Not logged in · Please run /login"`, `error:"authentication_failed"`, exit 1, and `duration_ms` in the low hundreds — zero `tool_use` frames, zero permission events. A test author seeing this without the credential-gate context will spend time chasing a "model returned wrong text" hypothesis. The negative-pin substring assertions in `TestWithWorktreeAuthenticated_RealAssistant` exist to surface the auth-failed shape as itself in any future regression. Pattern lesson: when a missing-credential failure mode can masquerade as a logic-level failure, the test should pin against the masquerade's fingerprint, not just rely on structural asserts to fail elsewhere.
- **Skip-before-`t.TempDir()` matters.** If the helper called `t.TempDir()` first and *then* checked the credential, a skipped run would still allocate the temp directory; `t.TempDir` registers cleanup that runs even after `t.Skip`. Cheap and harmless in isolation, but `t.Skip` ending up with an allocated-but-empty workdir is a code smell that suggests the gate was added as an afterthought rather than as the first thing the fixture does. Order matters: cheap checks first, allocations after.
- **AC pairs that look contradictory often pin the design.** AC#4 ("architect picks skip-or-fail when credential absent") read in isolation looked like a judgement call; AC#5 ("test must be skipped, not failed, on contributor machines without API keys") reads in isolation like a test-author concern. Together they force the gate into the helper (the only way to satisfy both is to make the helper itself call `t.Skip`). When two acceptance criteria look orthogonal, check whether they actually constrain the design jointly — the constraint often picks the simplest implementation by elimination.
- **No credential-shape validation is the correct choice even when it feels lazy.** Claude is the only authority on key validity, and any prefix-check in the helper is (a) a maintenance burden (Anthropic can change the format), (b) a confusion vector (a "real but rejected" key gets `t.Skip`'d-with-no-key-present message instead of the actual claude error), and (c) zero practical value over letting claude exit non-zero. `validateRunOpts` makes the same call for `--effort`. Generalisable rule: if a downstream system already validates X and surfaces a clear error on bad X, don't re-validate X upstream.

## Out of scope (deferred)

- **CI secret wiring** is the operator's responsibility, not a code change. The helper's contract is "if `ANTHROPIC_API_KEY` is exported in the outer env, propagate it; otherwise skip." Whether GitHub Actions, the dispatcher's code-review phase, or a developer's shell provides the value is upstream of this fixture.
- **`CLAUDE_CONFIG_DIR`-based alternative isolation** (spec §"Why option A", option C) deferred without prejudice. Picking it would require verifying claude actually reads such a var across platforms and versions; the cost of getting it wrong is exactly the failure mode this helper exists to eliminate. If a future need lands where env-passthrough doesn't fit (e.g. testing flows that explicitly suppress `ANTHROPIC_API_KEY`), revisit then.
- **A second real-API consumer.** #383 is the first known caller; if no second test materialises, the helper stays a one-call fixture. If a second arrives, the file-local `jsonlPathFor` / `truncate` helpers may finally cross the YAGNI bar for promotion to fixture-package exports — defer until that need is real.
