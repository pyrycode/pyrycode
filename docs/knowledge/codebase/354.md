# Ticket #354 — agent-run: stream-json stdout emitter + result trailer

Lands the JSONL → stdout bridge that makes `pyry agent-run --output-format stream-json` look on the wire like `claude -p --output-format stream-json` to the dispatcher's existing parser. New leaf package `internal/agentrun/streamjson` re-emits each watcher `Event.Raw` verbatim, aggregates per-event `Usage`, and composes a single `type:"result"` trailer on close. `cmd/pyry/agent_run.go` now mints a UUIDv4, passes it via `--session-id`, and runs the JSONL watcher + `agentrun.Drive` under one `errgroup` with end-of-turn cancellation wired through.

See [features/streamjson-package.md](../features/streamjson-package.md) for the full Emitter contract and wire-shape table.

## Implementation

- **`internal/agentrun/streamjson/emitter.go`** (~270 lines). `Emitter` owns aggregation state (`numTurns`, `endOfTurnSeen`, `lastStopReason`, `aggUsage`, `exitReason`, `closed`/`closeErr`, sticky `writeErr`) under a single mutex. `Emit` writes `ev.Raw + '\n'` in one buffered op (`append([]byte(nil), ev.Raw...); append(line, '\n')` + one `Writer.Write`) — no re-encoding under any circumstance. State updates run first so totals stay coherent even if the write fails. `Close` marshals a typed `trailer` struct with explicit JSON tags pinning field order (`type, subtype, is_error, duration_ms, num_turns, result, stop_reason, session_id, total_cost_usd, usage, terminal_reason`) so the captured-fixture byte-equivalence test diffs cleanly. `wireFields(ExitReason) (subtype, terminal, isErr)` is the one-place translation table between internal `ExitReason` and the wire trio.
- **Sticky write error + idempotent Close + Emit-after-Close no-op.** Three orthogonal safety properties of the same mutex. A broken pipe during `Emit` doesn't unwind the watcher: subsequent `Emit`s short-circuit returning nil, the watcher drains to EOT or ctx-cancel, then `Close` makes a best-effort trailer write. `Close` called twice returns the first call's error without re-writing. `Emit` after `Close` returns nil — spurious late events from a slow watcher goroutine cannot panic.
- **Default classification in `Close`.** When `SetExitReason` was never called: EOT observed → `ExitReasonCompletion`; EOT not observed → `ExitReasonError`. The driver-side `classifyForEmitter` in `agent_run.go` deliberately does NOT call `SetExitReason` for `nil` or `context.Canceled` errgroup returns — it leaves the EOT-observed branch to handle them, which correctly produces `completion` for clean EOT-then-SIGTERM and `error` for operator-SIGINT-before-EOT.
- **`cmd/pyry/agent_run.go` orchestration.** `runAgentRun` grew from ~45 to ~90 lines, adding (in order): `newSessionUUID()` mint, `streamjson.New(Config{Writer: stdout, SessionID: <uuid>})`, `signal.NotifyContext`, `tail.New` with `OnEvent → emitter.Emit` and `OnEndOfTurn → cancel`, `errgroup.WithContext(ctx)` running watcher and Drive concurrently, then `classifyForEmitter(em, runErr)` → `emitter.Close()` → wrapped-or-nil return. The end-of-turn-cancels-ctx wiring is what propagates EOT → claude SIGTERM → Drive returns nil → errgroup unblocks; the architect spec called it out and it works as written. Signature change: `runAgentRun(stdout io.Writer, args []string)` — `main.go` passes `os.Stdout`, tests pass `&bytes.Buffer{}`.
- **`buildClaudeArgs` adds `--session-id <uuid>`** as the last pair. The two pre-existing security invariants (`--permission-mode default` MUST appear, `--allowedTools` MUST NOT appear) are untouched; `TestBuildClaudeArgs_Shape` grew one assertion stanza.
- **`newSessionUUID`** mirrors `internal/conversations/id.go:NewID`'s 7-line pattern verbatim: `crypto/rand` 16 bytes → set version-4 nibble in byte 6, RFC-4122 variant nibble in byte 8 → `fmt.Sprintf` the 8-4-4-4-12 hex layout. Not extracted to a shared helper — one call site, the spec was explicit about not factoring prematurely.
- **`testdata/captured_run.jsonl`** is a synthesised fixture (claude was unavailable in the dev environment) that matches the captured shapes embedded in the spec: `system init` → `user` → `assistant tool_use` (with usage) → `tool_use` → `tool_result` → `assistant end_turn` (with usage) → `result` trailer. The trailer matches `claude -p`'s richer shape (extra `api_error_status`, `duration_api_ms`, `modelUsage`, `permission_denials`, `fast_mode_state`, `uuid`, `server_tool_use` inside `usage`), and the test asserts pyry produces a structurally-equivalent subset rather than byte-identical output.
- **Test coverage.** `emitter_test.go` is 528 lines, 17 tests. The fixture-compare test (`TestCapturedFixture_ByteEquivalence`) is the load-bearing one: it replays every non-result line through `Emit` and asserts byte-equivalence against the fixture, then `Close`s and asserts the trailer's `type` / `subtype` / `is_error` / `num_turns` / `stop_reason` / `terminal_reason` match the fixture exactly and the four `usage` int fields match. `cmd/pyry/agent_run_test.go` grew `TestNewSessionUUID_Shape` (UUIDv4 nibble layout) and extended `TestRunAgentRun_DrivesFakeClaude` so the fake-claude binary writes a canned `end_turn` JSONL line into `<home>/.claude/projects/<encoded(workdir)>/<sid>.jsonl`, satisfying the watcher's EOT trigger so the run terminates cleanly and a `subtype:"success"` trailer lands as the last stdout line.

## Patterns established

- **`io.Writer` parameter over `os.Stdout` global for verb implementations.** `runAgentRun(stdout io.Writer, args []string)` lets tests capture stdout via `&bytes.Buffer{}` without the `os.Pipe()` / dup ceremony the older `TestRunAgentRun_EmitsSettingsFile` uses. `main.go` passes `os.Stdout` literally. Future verbs that produce stdout output (especially anything stream-shaped) should adopt this signature shape; existing verbs with `os.Pipe` capture can migrate when next touched.
- **End-of-turn → `cancel`, errgroup, ctx-cancel-as-success.** The watcher's `OnEndOfTurn` callback IS the parent-ctx cancel func, and `agentrun.Drive` already maps ctx-cancel-driven exits to nil. Result: clean teardown without an additional signalling channel. Any future end-of-something detector that needs to gracefully stop a long-running child should reuse this shape (cancel-from-callback → child SIGTERM via `cmd.Cancel` → `cmd.Wait` returns → errgroup unblocks).
- **Pluggable setter for future cross-package wire-up.** `SetExitReason(ExitReasonMaxTurns)` is the seam where #334's budget Counter will plug in without re-architecting this code. The pattern is: idempotent setter on the consumer side, single explicit `SetX(...)` call from the producer's existing hook (Counter's `Terminate`). Cleaner than a callback-injection or a shared channel because the producer (#334) and consumer (#354) ship in separate tickets and the seam stays one method on a value the producer already holds.
- **Wire shape derived from captured fixture, not invented.** The trailer's exact field set, types, value vocabulary, and the dual-emit of `subtype`+`terminal_reason` come from real captured `claude -p` output. The "why both": v1 dispatcher reads `subtype` (log) and `terminal_reason` (store); v2 fixture-compare test asserts on `subtype`. Either alone breaks one consumer. **Pattern for any future emitter mirroring an external tool's wire shape:** capture the real output first, document the full field set in the spec, pin the subset you emit with a byte-equivalence test against the capture, document the known-expected diff list inline.
- **Mutex placement — fine-grained, no nested locks.** The emitter mutex is held only for the duration of one `Emit` (one write + state update) or one `Close` (one write + flag flip). Three call sites (`Emit` from watcher goroutine, `SetExitReason` from future budget Counter goroutine, `Close` from driver goroutine), zero callbacks-under-lock, zero nested acquisition.

## Lessons learned

- **`Event.Raw` already has the trailing `'\n'` stripped — re-append it, don't try to be clever.** The reader (`internal/agentrun/jsonl/reader.go`) strips the LF before handing `Raw` to the consumer. The emitter's job is exactly one `Write(append(ev.Raw, '\n'))`. Anything more (re-marshalling for "consistency", trimming whitespace inside, normalising) breaks byte-equivalence — the whole point of the contract.
- **`Emit` after `Close` must be a no-op, not a panic.** When the watcher goroutine cancels and the driver goroutine calls `Close`, there's a small window where the watcher's last batch of buffered events can still fire `OnEvent` → `Emit`. If `Emit` panicked or wrote to a closed writer, the watcher unwind would surface a goroutine panic instead of returning ctx.Err cleanly. The fix is one branch at the top of `Emit`: `if e.closed { return nil }`. Cheap insurance against a race that's annoying to reproduce.
- **End-of-turn cancels the parent ctx — this also cancels Drive's PTY write goroutine, which is exactly what we want.** Drive's `sleepOrCancel` honours ctx; once `cancel` fires from `OnEndOfTurn`, any pending defensive prompt-typed-but-not-flushed write skips and the child's `cmd.Wait` returns ctx-aware. The same `cancel` also tears down the watcher (via gctx propagation). Single cancel, three cleanup paths — relies on Drive and the watcher both being correctly ctx-aware (#332 and #349 verified this respectively).
- **Synthesised fixtures are fine when the real tool isn't reachable, provided the shape comes from a captured spec.** The captured-fixture test asserts byte-equivalence for the non-result lines (which pyry only re-emits, so the equivalence is automatic from `Raw` passthrough) and structural equivalence for the trailer (which pyry composes, so the field set, types, and values are the contract). Both halves test what matters; the synthetic provenance doesn't reduce the test's signal because the architect spec already pinned the shape from real captures.
- **`bytes.Buffer{}` for stdout capture is a step-change over `os.Pipe()` redirection.** The existing `TestRunAgentRun_EmitsSettingsFile` uses `os.Pipe` to capture the marker line; the new test pattern just passes a `*bytes.Buffer` as the `stdout io.Writer` parameter. No goroutine to drain the pipe, no `t.Cleanup` to restore `os.Stdout`, no risk of leaking a half-read pipe on test failure. Cleaner enough that it's worth migrating the older tests when next touched.

## Out of scope (deferred)

- **Budget Counter integration call site.** This ticket exposes `SetExitReason(ExitReasonMaxTurns)`; the actual call from `budget.Counter.Terminate` is a sibling ticket that imports the streamjson package into the budget integration layer.
- **Final assistant text in `result`.** Always `""`. Re-parsing the last `Kind == "assistant"` `Event.Raw` on `Close` to extract the final text is an obvious extension if a consumer wants it; v1 dispatcher reads `r.result || ""` so empty is functionally fine today.
- **Richer `usage` fields** (`server_tool_use`, `service_tier`, `cache_creation`, `iterations`, `speed`, `inference_geo`). Producer side (`jsonl.UsageBlock`) only mirrors the four ints. Widening starts there, not here.
- **`duration_api_ms`.** No clean source from pyry's vantage point; omit and let the dispatcher derive if needed.
