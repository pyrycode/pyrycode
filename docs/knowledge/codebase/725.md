# Ticket #725 — deny-on-timeout for an unanswered modal: safe-deny, `modal_dismissed{timeout}`, audit

The **fail-closed safety net** of the daemon-side modal bridge ([EPIC #597] Phase 3,
[ADR 025] § Security model). A surfaced permission/trust modal blocks claude until
resolved. #727 lets a phone *cancel* it and #717 lets a gated phone *answer* it — but
if **no authorized device answers** (phone offline, push missed, user asleep) the
modal must not linger forever and must **never** be silently granted. This slice arms
a bounded timeout when a modal is surfaced; on expiry the daemon **safe-denies** it
(the ESC keystroke), broadcasts `modal_dismissed{source: timeout}` to every
interactive conn, and audits exactly one `denied_timeout` decision. It is the last leg
of #717's 3-way split (#726 safe-answer seam, #727 dismiss/broadcast/interception, #725
deny-on-timeout) and **blocks #708**, the live-wiring capstone.

It introduces **no new wire type, no new package, no new crypto, no new exported
type** — it reuses machinery already built: the registry one-shot `Resolve` (#716), the
`broadcastModalDismissed` fan-out + `ModalResolver` seam (#727), the supervisor ESC
keystroke (#726), and `audit.Log` with `OutcomeDeniedTimeout`/`SourceTimeout` (#712).
It does **not** consult #717's per-device answer gate — a timeout has no answering
device to gate, and the action is unconditionally DENY, so there is nothing to
authorize.

## Implementation

Three production files change (~95 LOC), additive — no new files, no consumer cascade.
`security-sensitive` (a security-critical timer on the permission surface; spec-stage
security review verdict **PASS**, 9 categories).

| File | Change |
|---|---|
| `internal/relay/v2session.go` | **The timer lifecycle + fire.** New package var `modalDenyTimeout = 2 * time.Minute` (lowercase, test-overridable via save/restore, mirrors `rekeyReplyTimeout`). New `modalTimeout chan string` field on `V2SessionManager` (daemon-global, keyed by `modal_id`, buffered `wakeBufferSize`=16; allocated in `NewV2SessionManager` beside `wake`). New `Run` select arm `case modalID := <-m.modalTimeout: m.handleModalTimeout(runCtx, modalID)`. New exported `ArmModalTimeout(ctx, modalID)` — `time.AfterFunc(modalDenyTimeout, cb)` whose `cb` funnels the id onto `m.modalTimeout` (`select { case m.modalTimeout <- id: case <-ctx.Done(): }`, the `armRekeyTimer` callback shape). New `handleModalTimeout` (near-copy of `handleModalCancel`): nil-resolver guard → `ResolveTimeout` → `if !ok return` → `broadcastModalDismissed` (**unchanged**, reused). New `ResolveTimeout(modalID string) (ModalDismissal, bool)` method on the existing `ModalResolver` interface. |
| `cmd/pyry/modal_resolve_v2.go` | **The resolver.** New `ResolveTimeout` on `modalResolverV2`, structurally identical to `ResolveCancel`: `reg.Resolve(id)` (one-shot consume) → best-effort `kb.SendEsc()` → `audit.Log({denied_timeout, timeout})` → return `{denied_timeout, timeout}, true`. Three differences from `ResolveCancel`: **no `dev` param** (a timeout has no answering device → empty `DeviceHash`/`DeviceLabel`), outcome/source = `OutcomeDeniedTimeout`/`SourceTimeout`, and `ok=false` on an unknown/already-consumed id (the AC-2 loser path). No `modalKeystroker` change — `SendEsc` already exists (#726). |
| `cmd/pyry/interactive_modal_v2.go` | **The arm-on-surface.** New unexported `modalTimeoutArmer` interface (`ArmModalTimeout(ctx, modalID)`) defined consumer-side — `*relay.V2SessionManager` satisfies it structurally; deliberately **not** a widening of the shared `interactiveBroadcaster` (three emitters implement that; only this surfacer arms timeouts). New `armer` field + constructor param on `interactiveModalEmitterV2`. The arm call is placed **immediately after a successful `reg.Record`**, before the marshal/broadcast loop. |

### Data flow (production-live in #708)

```
SURFACE (producer goroutine, cmd/pyry)
  interactiveModalEmitterV2.Handle:
    payload, _ := reg.Record(req, class)        // mints modal_id, records Outstanding
    armer.ArmModalTimeout(ctx, payload.ModalID) // arm BEFORE marshal/broadcast
    ... marshal + per-conn modal_shown push ...

FIRE (AfterFunc goroutine → Run goroutine, internal/relay)
  AfterFunc cb: select { case m.modalTimeout <- id: case <-ctx.Done(): }
  Run select arm:  case id := <-m.modalTimeout: m.handleModalTimeout(runCtx, id)
  handleModalTimeout: nil-guard → d,ok := ResolveTimeout(id); if !ok return
                      → broadcastModalDismissed(runCtx, id, d)   // UNCHANGED (#727)

RESOLVE-TIMEOUT (Run goroutine, cmd/pyry)
  modalResolverV2.ResolveTimeout(id):
    out,ok := reg.Resolve(id); if !ok return zero,false   // AC-2 loser path
    _ = kb.SendEsc()                                       // safe-deny, best-effort
    audit.Log({modal_id, out.Class, denied_timeout, timeout})  // empty device
    return {denied_timeout, timeout}, true
```

## Patterns established

- **The timer-callback → `wake` channel → `Run` precedent generalised to a
  daemon-global signal.** #725 mirrors `armRekeyTimer`/`armRekeyReplyTimer` (per-session
  `time.AfterFunc` → `wake` channel → `Run`), but keyed by `modal_id` on a new
  daemon-global `modalTimeout` channel rather than by `*V2Session` — because a modal is
  not bound to one conn. The timer is **armed off the `Run` goroutine** (the surfacer
  calls `ArmModalTimeout`, which only calls `time.AfterFunc` and touches no `Run`-owned
  state) and **fired on the `Run` goroutine** (the single owner of the resolution). The
  `-race` run across this off-Run-arm → on-Run-fire crossing is the real correctness
  proof.
- **Exactly-once safe-deny is *structural*, not lock-defended.** Answer/cancel-resolution
  (`handleModalAnswer`/`handleModalCancel`, via `m.cfg.Frames`) and timeout-resolution
  (`handleModalTimeout`, via `m.modalTimeout`) are **both arms of the same `Run`
  `select`**. A `select` services one arm to completion, so the two never run
  concurrently — whichever `Run` services first calls `reg.Resolve` and consumes the
  modal; the loser sees `ok=false` and takes the no-keystroke / no-audit / no-broadcast
  path. The registry's one-shot `Resolve` (delete-on-resolve under its own leaf mutex)
  is the idempotency gate; the `Run` serialization means there is no actual
  answer-vs-timeout data race to lose. Same belt-and-suspenders-of-different-fabric
  shape #727 established for two racing cancels.
- **The timer is deliberately not tracked or `Stop`ped — and that asymmetry with its
  two sibling arm-methods is intentional.** `ArmModalTimeout` discards the `*time.Timer`
  that `time.AfterFunc` returns (unlike `armRekeyTimer`/`armRekeyReplyTimer`, which
  return it). Cancelling the timer when an answer wins would need a
  `map[modalID]*time.Timer` mutated by both the surfacer (arm) and `Run` (cancel) — new
  cross-goroutine state and a new lock — for **zero** correctness gain: an un-stopped
  timer that fires after the modal was already answered simply runs `ResolveTimeout` →
  `Resolve`-miss → no-op (one `Run` iteration + one registry lock, after the window).
  An **un-fired `AfterFunc` holds only a heap entry, not a parked goroutine**, so
  leaving it un-`Stop`ped leaks nothing. (Code review flagged the sibling asymmetry as a
  NIT; the doc comment already pre-empts it — no change requested.)
- **Arm before marshal/broadcast — the fail-closed guarantee is independent of who is
  watching.** The arm is unconditional on a successful `reg.Record`, placed before the
  payload marshal and the broadcast loop. So a modal that records but **fails to
  marshal** (the existing defensive drop path), or one surfaced to **zero** connected
  interactive conns, is still safe-denied on the window. claude is blocked on the prompt
  regardless of who is watching, so it must be denied if nothing resolves it.
- **ESC is the layout-independent fail-closed deny for *both* modal classes.** For a
  permission modal ESC dismisses the prompt (claude treats it as deny) — the exact
  keystroke #727's `ResolveCancel` already routes. For a trust modal the deny option is
  `exit`, which `classifyAnswer` already maps to `verbEsc`. So ESC = "exit" = deny
  uniformly, needs no `modalKeystroker` change, and only the audit classification
  differs (`denied_timeout`/`timeout` vs `cancelled`/`remote`). Routing the explicit
  reject digit (`Answer("3")`/`("4")`) was **considered and rejected**: it depends on
  the option's 1-based render position and adds a class-conditional branch with no
  security benefit over ESC.
- **Best-effort keystroke, idempotency committed first.** A `SendEsc` error (no live
  session / teardown) is `Warn`-logged with the supervisor sentinel and **tolerated** —
  the modal is already consumed (`Resolve` committed above) and moot, so the dismissal
  must still broadcast and the audit must still be written. Aborting would orphan a
  consumed modal. Identical discipline to `ResolveCancel`.
- **No answering device ⇒ empty audit identity by construction.** `ResolveTimeout` takes
  no `dev` param, so `audit.Entry.DeviceHash`/`DeviceLabel` are empty — the documented
  no-device-timeout case (`audit.go:28`). `devices.AuthorizeRemotePermission(d,
  OutcomeTimeout)` already encodes timeout → DENY; the timeout leg only ever drives the
  deny keystroke, never a grant.

## Lessons learned

- **Production-inert until #708.** Nothing `Record`s a modal in production until #708
  live-wires the surfacer into the PTY event stream, so **no timer arms in production
  today** (the same harmless state as #727's interception). The scripted tests are the
  proof of correctness; no live two-phone e2e is required here — that is #708's capstone,
  which this slice blocks.
- **AC-3's answer-vs-timeout race is proven by composition, not one e2e.** The
  no-keystroke/no-audit half lives in the cmd/pyry resolver test
  (`TestModalResolverV2_Timeout_AlreadyConsumed`); the no-broadcast half lives in the
  relay manager test (`TestV2Session_ModalTimeout_AlreadyResolved_NoBroadcast`). The two
  halves can't disagree given the single-`Run`-goroutine serialization, so composition is
  the correct call under production-inertness — but **#708's capstone test should
  exercise the real answer-then-fire race once** through the live `Run` select (code
  review NIT, noted for the capstone).
- **The AC-3 assertions span two packages by the #727 seam-split.** Keystroke + audit
  live in the cmd/pyry resolver; the broadcast lives in the relay manager (the relay
  can't import supervisor/audit/registry). So no single test covers all three AC-3
  assertions — that is the seam boundary, not a coverage gap.

## Out of scope (siblings / deferred)

- **Live producer wiring — [#708]** (this slice blocks it). #708 live-wires
  `interactiveModalEmitterV2` to the daemon manager (passing it as the `modalTimeoutArmer`)
  and provides the two-phone e2e capstone. Until then every production modal path is inert.
- **The timeout-window default value is a policy knob, not load-bearing.** `2 * time.Minute`
  balances "long enough for a human to react to a push notification and tap" against "short
  enough not to leave claude blocked." ADR 025 specifies "a bounded window" but no number.
  Making it config-driven (vs the package var) is deferred to #708 or a follow-up; tests
  shrink it via save/restore.
- **No `conversation_id` on `modal_dismissed`.** The payload carries only `modal_id` +
  `outcome` + `source` (the opaque-nonce-is-the-sole-key contract). No id is threaded.
- **Incidental diff noise:** the merged branch re-captured
  `internal/e2e/realclaude/testdata/permission_protocol_v2.1.158.json` (new random
  `session_id`/`uuid`/timestamps/tmp paths from a real-claude e2e run, auto-committed by
  the code-review step). A regenerated golden capture, unrelated to the timeout feature —
  the same fixture churns on #716/#726/#727 and any realclaude run.

## Testing

Scripted, deterministic, `-race`-clean; no live e2e (deferred to #708). AC-3 is satisfied
by the combination across two packages.

- `cmd/pyry/modal_resolve_v2_test.go` (reuses `auditLogger()`/`auditRecords()` + the
  `ResolveCancel` shapes): **happy path** (`TestModalResolverV2_Timeout_HappyPath` — one
  ESC and nothing else, exactly one audit `{denied_timeout, timeout}` with **empty**
  device identity and **no modal body** in any field, returns `{denied_timeout, timeout},
  true`); **already-consumed no-op** (`..._AlreadyConsumed` — consume via the registry
  first → `ok=false`, no keystroke, zero audit — AC-2); **keystroke error tolerated**
  (`..._KeystrokeError` — still consumed, `Warn` with non-secret err, audit + `(…,true)`
  still happen); **unknown id no-op** (`..._UnknownID`).
- `internal/relay/v2session_modal_test.go` (extends `fakeModalResolver` with
  `ResolveTimeout`; shrinks `modalDenyTimeout` via save/restore): **arm → fire → fan-out**
  (`TestV2Session_ModalTimeout_FanOut` — three heads, two interactive + one not;
  `ResolveTimeout` fired exactly once with the armed id, `modal_dismissed{denied_timeout,
  timeout}` at **both** interactive heads and **zero** at the non-interactive one — the
  off-Run-arm → on-Run-fire `-race` proof); **already-resolved no broadcast**
  (`..._AlreadyResolved_NoBroadcast` — `ok=false` ⇒ no `modal_dismissed`, AC-2);
  **nil-resolver inert** (`..._NilResolver` — armed timeout firing debug-logs, no panic,
  no broadcast).
- `cmd/pyry/interactive_modal_v2_test.go` (adds `fakeArmer` recording `ArmModalTimeout`
  calls): a surfaced permission modal arms exactly one timeout for the surfaced `modal_id`;
  the existing `NonPermissionClass_NoOp` / `NonModalEvent_NoOp` scenarios additionally
  assert **zero** arm calls.

**Code review: PASS** ([PR #733]), no blocking findings — two NITs (the deliberate
discarded-`*time.Timer` asymmetry; the compose-vs-e2e race proof deferred to #708),
neither requesting a change. Real-claude e2e green (`make e2e-realclaude` → ok 147s).
`go test -race ./...` + `go vet ./...` pass.
