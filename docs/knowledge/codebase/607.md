# Ticket #607 — v2 interactive wire types + capabilities negotiation

Lands the v2 interactive wire **vocabulary** for [EPIC #596] (Phase 2 structured
streaming): five additive **binary → phone** event payloads (`turn_state`,
`assistant_delta`, `tool_use`, `tool_result`, `turn_end`) with their `Type*`
constants in the v2-only partition, plus an `omitempty` `capabilities []string`
field on the `hello` / `hello_ack` handshake payloads and a `CapabilityInteractive`
vocabulary constant. **Wire types only** — pure protocol structs and their
(de)serialization. No bridge logic, no push, no capability enforcement, no
validation, no dispatch; every trust decision (intersecting the phone's advertised
set with the daemon's supported set, malformed-advertisement handling,
capability-gated fan-out) lives in the consumer [#608]. The five payloads are the
mobile adapter's wire representation of [#606]'s neutral internal turn-event model
(`TextChunk` / `ThoughtChunk` / `ToolStart` / `ToolUpdate` / `TurnEnd`); this is the
data layer #608 maps those events **into**. Package reference (updated this ticket):
[features/protocol-package.md](../features/protocol-package.md).

## Implementation

Three production files (~68 LOC), ~478 LOC total in the feature commit:

- **`interactive.go`** (new) — five json-tagged payload structs, one doc comment
  each naming the `Type*` constant, the binary → phone direction, and the
  `docs/protocol-mobile.md §`, mirroring `messaging.go`'s per-type template:
  - `TurnStatePayload{ConversationID, State}` — `State` is `"thinking" |
    "responding" | "idle"`, a plain string (no named enum), following the
    `MessagePayload.Role` precedent.
  - `AssistantDeltaPayload{ConversationID, TurnID, Seq int, Text}` — `Seq` is a
    per-turn, non-negative delta-ordering counter that resets each turn (`int`, the
    package count-field idiom; distinct from the session-monotonic `Envelope.ID`).
  - `ToolUsePayload{ConversationID, TurnID, ToolUseID, Name, InputSummary}`.
  - `ToolResultPayload{ConversationID, TurnID, ToolUseID, IsError bool, ResultSummary}`.
  - `TurnEndPayload{ConversationID, TurnID, StopReason}` — `StopReason` carries the
    `turnevent.TurnEndReason` strings verbatim (`end_turn` / `max_tokens` /
    `max_turn_requests` / `refusal` / `cancelled`). **No omitempty on any field** —
    every field is always on the wire so the fixtures pin the full shape and
    boundary values (`seq:0`, `is_error:false`) cannot silently vanish.
- **`codes.go`** (mod) — a **new** const block of five `Type*` constants
  (`TypeTurnState` … `TypeTurnEnd`), kept separate from the `TypeRekeyRequest` block
  so the doc comment can distinguish the two v2-only flavours: `rekey_request` is a
  v2 **control** envelope intercepted before `dispatch.Route`; these five are v2
  **application events** (outbound binary → phone, never dispatched inbound). Both
  are "v2-only" for the partition — neither enters `v1TypeSet`.
- **`handshake.go`** (mod) — `Capabilities []string json:"capabilities,omitempty"`
  added as the last field of **both** `HelloClientPayload` (the phone's
  advertisement) and `HelloAckPayload` (the daemon's supported-set echo), plus the
  `CapabilityInteractive = "interactive"` vocabulary constant — the single home for
  the string so the round-trip test and #608 reference a constant, not a literal.
  Pure vocabulary, no enforcement here.

Tests + fixtures (~410 LOC, not counted toward the production budget):

- **`interactive_test.go`** (new) — five round-trip tests, one per type. A
  `roundTripEnvelope` helper re-marshals the **decoded payload struct** (not the
  original `RawMessage`) back into the envelope and asserts canonical byte-equality.
  Boundary values pinned: `Seq == 0`, `IsError == false`, `StopReason == "end_turn"`.
- **`handshake_test.go`** (mod) — two new programmatic capability round-trips (no
  new fixtures): an advertised `[CapabilityInteractive]` survives marshal→unmarshal
  on both payloads, and a nil/empty set marshals with the `"capabilities"` key
  **absent** (the omitempty assertion). The pre-existing `TestHelloClientPayload_RoundTrip`
  / `TestHelloAckPayload_RoundTrip` are left **unchanged** as the byte-stability
  regression guard against the unedited fixtures.
- **`compat_test.go`** (mod) — five `ErrUnknownType` rejection rows in
  `TestIsV1Compatible`; the five constants added to the test-local `v2OnlyTypes`
  allowlist (doc comment broadened to "v2 control **and** v2 interactive application
  events") and to the `all` partition slice; the size assertion becomes
  `len(v1TypeSet) + len(v2OnlyTypes) == 16 + 6 == 22`.
- Five fixtures — `testdata/{turn_state,assistant_delta,tool_use,tool_result,turn_end}.json`,
  each a complete unsolicited binary → phone envelope in the `message.json` style.
- **`docs/protocol-mobile.md`** amended: five rows in the Application-message-types
  table, a new `### Interactive events (v2, capability-gated)` subsection with a
  per-type field table (and the `stop_reason` taxonomy values noted as mirroring the
  ACP turn-end reasons), and a `### Capability negotiation (v2)` note documenting the
  `capabilities` field + that the intersection/enforcement is the consumer's (#608).
  `qmd update && qmd embed` run after the edit.

## Patterns established

- **Re-marshal the decoded struct, not the `RawMessage`, to pin wire shape.** The
  `MessagePayload`-style round-trip that re-marshals the envelope with the *original*
  `json.RawMessage` passed through cannot catch a missing or reordered json tag — the
  payload bytes never get re-encoded. `interactive_test.go`'s `roundTripEnvelope`
  marshals the decoded payload struct back into `Envelope.Payload` first, so a tag
  divergence surfaces as a byte mismatch. This is the correct shape for any future
  per-type round-trip test in this package. (Lifted from the PR's Lessons-learned.)
- **One json tag, two opposite jobs.** `capabilities` uses `omitempty` as the
  byte-identical lever (a nil/empty slice drops the key → existing fixtures stay
  byte-stable), exactly the `Token` (#308) precedent. The five interactive payloads
  do the **inverse** — *no* omitempty anywhere — so always-present boundary
  zero-values (`seq:0`, `is_error:false`) stay explicit on the wire and the fixture
  pins the full shape. Pick the tag by whether the field's absence is meaningful.
- **The v2-only partition now has two flavours.** `v2OnlyTypes` (test-local
  allowlist) holds control types (`rekey_request`, intercepted before
  `dispatch.Route`) and application events (the interactive five, outbound and never
  dispatched inbound). `TestTypeConstants_V1V2Partition` is the deterministic rail:
  every `Type*` constant must be in `v1TypeSet` **xor** `v2OnlyTypes`. Adding a
  constant without partitioning it is a compile-time-visible build failure — the
  belt-and-suspenders of different fabric (advisory doc comments + this test).
- **`protocol` stays a stdlib-only leaf.** `StopReason` carries the
  `turnevent.TurnEndReason` string values verbatim, but `internal/protocol` does
  **not** import `internal/turnevent` — the wire-value/taxonomy alignment is
  documented, not enforced by a shared type. #608 produces the field via
  `string(turnevent.TurnEnd.Reason)`. Same Role-is-string discipline that keeps the
  package import-free of its consumers.

## Lessons learned

- **A struct→wire round-trip must re-encode the decoded struct.** See the pattern
  above — the original-`RawMessage`-passthrough variant is a silent no-op against tag
  bugs. (From the PR body's Lessons-learned section.)
- **The real-claude e2e gate cannot run in the credential-less review sandbox.**
  Code-review's mandatory `make e2e-realclaude` builds `cmd/pyry` in an isolated temp
  GOPATH, which re-fetches the **private** module `github.com/pyrycode/tui-driver`;
  the review sandbox has no GitHub credentials, so the fetch dies at `git ls-remote`
  before any test logic runs. For this ticket the failure was **not** attributable to
  the diff (pure `internal/protocol/*` + `docs/*`; `go.mod`/`go.sum` untouched, and
  QA's `make build` resolved the module fine in the normal worktree), so code-review
  escalated to the operator rather than routing back to the developer or
  auto-passing. The judgment review of the diff itself was clean — no MUST/SHOULD/NIT.
  Any pure-leaf-package ticket whose review needs the e2e gate will hit the same wall
  until the sandbox gets module-cache/credential access.

## Scope (S kept)

- **Production source files:** 3 (`interactive.go` new, `codes.go` mod,
  `handshake.go` mod) — under the ≥5 split gate.
- **New exported types:** exactly **5** (the payload structs) — at, not over, the
  >5 red line. `CapabilityInteractive` and the five `Type*` are *constants*, not
  types. This is the recorded `po-data-model-taxonomy-size-red-line` shape: the count
  is domain-variant breadth (five wire event types), not coupling.
- **Consumer call sites needing simultaneous update:** 0 — the additive
  `Capabilities` field compiles unchanged against the two keyed `HelloAckPayload{…}`
  literals (`internal/relay/auth.go:100`, `internal/relay/v2session.go:688`).
