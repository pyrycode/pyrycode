# Ticket #633 — Event-stream bridge: live producer wiring in `startRelayV2` (JSONL resolver + `/clear` re-subscription)

The **wiring slice** that takes Phase 2 structured streaming live ([EPIC #596],
[ADR 025] § Phase 2). It attaches the #615 **producer** (`internal/turnbridge`) to
the #632 **consumer** (`interactiveTurnEmitterV2`) inside `startRelayV2`, so the
supervised session's structured turn events actually flow to the capability-gated
phone fan-out — and keep flowing across a `/clear`-rotated JSONL on the next
(restart-driven) re-subscription. Both #615 and #632 shipped deliberately unwired and
deferred this production glue here. **Not `security-sensitive`** — behaviour-preserving
plumbing over the operator's **local** transcript; the capability gate + per-conn
routing live entirely in the already-labelled #632 emitter ("label tracks design, not
lineage": plumbing that merely feeds an internet-exposed consumer is not itself the
trust boundary).

### `startInteractiveTurnStreamV2(ctx, sup, mgr, claudeSessionsDir, logger) func()`

The wiring helper (~30 LOC body):

1. `emitter := newInteractiveTurnEmitterV2(sup, mgr, logger)` — `sup` (`*supervisor.Supervisor`) is the `cursorReader`, `mgr` (`*relay.V2SessionManager`) the `interactiveBroadcaster`.
2. `tr := tuidriver.NewTracker(tuidriver.TrackerOpts{})` — `Session.Events` requires a non-nil tracker; zero opts → package defaults (drives only the dropped stall arm the mapper discards anyway).
3. `resolve := resolveLatestSessionJSONL(claudeSessionsDir)`.
4. `sub := turnbridge.NewSessionSubscriber(sup, resolve, tr, logger)` — `sup` satisfies `turnbridge.SessionHost` (`Session()` + `WaitForPTY`).
5. `prod, err := turnbridge.New(Config{Subscribe: sub, OnEvent: func(ev){ emitter.Handle(ctx, ev) }, Logger: logger})`. `New` errors only on a nil `Subscribe` (unreachable here) → fail-soft: Warn + no-op cleanup.
6. One goroutine: `go func(){ defer close(done); if err := prod.Run(ctx); err != nil { logger.Debug(...) } }()` — `Run` returns only `ctx.Err()` per its contract.
7. Return `func(){ <-done }`.

The **`OnEvent` closure captures the relay lifecycle `ctx`** — the ctx-less-`OnEvent` →
ctx-ful-`Handle` seam #632 named (keeps `context.Context` out of the emitter struct,
the #632 contract). It runs **only** on the producer's single `Run` goroutine (the
drain invokes `OnEvent` serially), so the emitter's unguarded counters
(`inTurn`/`turnID`/`seq`/`currentState`/`nextID`) never race — **#632's named
single-`Run`-goroutine assumption holds by construction.**

### `resolveLatestSessionJSONL(dir) func(ctx)(path string, off int64, err error)`

The one non-trivial sub-part — a **rotation-following** resolver. A captured-`dir`
closure; each call:

- `os.ReadDir(dir)` → on error, return a wrapped `fmt.Errorf("read claude sessions dir %s: %w", dir, err)` (a path/errno, never file bytes — AC#5).
- Skip directories, names not ending `.jsonl`, and names whose stem fails `jsonlStemPattern` (the canonical 36-char lowercase UUIDv4).
- `os.Stat` each candidate; track the latest `ModTime()`, **lexicographically-larger filename** as the tie-break (deterministic — matches `internal/sessions.mostRecentJSONL`). Per-entry Stat failures are skipped (raced/vanished between ReadDir and Stat), not fatal.
- No candidate → `fmt.Errorf("no session jsonl found in %s", dir)`. Otherwise → `(filepath.Join(dir, name), size, nil)`.

`startOffset = info.Size()` is **load-bearing**: the tail starts at EOF, so a
(re)subscription streams **only new** events and never replays the historical
transcript to the phone (which would re-emit old turns).

**Why a most-recently-modified scan, not `tuidriver.SessionJSONLPath`** — the
supervised relay (bootstrap) session spawns with `--continue`, **not** `--session-id`
(`supervisor.go` leaves `JSONLPath` empty), so pyry holds **no stable claude session
UUID**, and `*tuidriver.Session` exposes no path accessor. `SessionJSONLPath(home,
cwd, sessionID)` needs a UUID that does not exist. The PO's validation pass caught the
original ticket framing this as trivial DI ("resolve from the source the daemon already
uses") — there is no such ready-made source; the resolver is **must-build** work.

### The two-part gate + teardown ordering (`startRelayV2`)

After the existing coarse-bridge block:

```go
var streamCleanup func()
if bridge != nil && claudeSessionsDir != "" {
    streamCleanup = startInteractiveTurnStreamV2(ctx, sup, mgr, claudeSessionsDir, logger)
} else if bridge != nil {
    logger.Info("relay: interactive turn stream disabled; claude sessions dir unresolved",
        "event", "interactive_turn_stream.no_sessions_dir")
}
```

- **`bridge != nil`** — the foreground gate, identical to the coarse v2 bridge (AC#4). Foreground has no phone-mirroring surface; inbound paths (`send_message`, …) are unaffected.
- **`claudeSessionsDir != ""`** — a deterministic guard against the resolver perpetually erroring (and Warn-spamming every `subscribeRetryDelay`). `""` already disables reconcile + the rotation watcher per `DefaultClaudeSessionsDir`'s contract, so disabling the producer too is coherent.

Drain ordering (AC#3): `bridgeCleanup() → streamCleanup() (<-done) → <-mgrDone`. **Stop
both producers before waiting on the manager** so no fan-out races a winding-down
manager. By drain time `ctx` is already cancelled (`main.go` defers `relayCleanup`
after `pool.Run(ctx)` returns), so `prod.Run` returns promptly and `streamCleanup`'s
`<-done` unblocks.

## Patterns established

- **Producer ⇄ emitter bridged by a ctx-capturing closure.** The ctx-less `OnEvent` →
  ctx-ful `Handle(ctx, ev)` seam is closed by `func(ev){ emitter.Handle(ctx, ev) }`
  capturing the relay lifecycle ctx — `context.Context` stays out of the emitter struct
  (#632's contract) while every `Handle` carries the live ctx that lets `ActiveConns` /
  `Push` short-circuit on teardown. The closure fires only on the producer's single
  `Run` goroutine → #632's named single-`Run`-goroutine assumption is satisfied **by
  construction**, not by a lock.
- **The `/clear` AC = #615's restart-driven loop + a per-call rotation-following
  resolver, with zero new loop/watcher code.** `Session.Events` tails a *fixed*
  path/offset; its channel closes only on ctx-cancel / tail-close / `sess.Wait()`. A
  live `/clear` does **not** exit claude (the same process rotates its on-disk UUID).
  So re-subscription is **restart-driven**: subscription N tails JSONL `A`; the process
  exits → `sess.Wait()` returns → the #615 watcher cancels `sessCtx` → the channel
  closes → `Run` loops → subscription N+1 calls `resolve` **fresh** → it returns the
  current newest (post-`/clear`) JSONL `B` at offset `size(B)`. The resolver returning
  the most-recently-modified file *on each call* is the load-bearing half — a
  captured/static path would keep tailing the stale `A`. Verified, **not**
  re-implemented (AC#2).
- **Reuse the daemon's `claudeSessionsDir`, do not recompute via `tuidriver.EncodeCwd`.**
  Threading the already-computed dir down (over a second cwd-encoder in `cmd/pyry`)
  makes the resolver scan the *exact same string* reconcile + the fsnotify rotation
  watcher use — coherent-by-construction. It also keeps the resolver **pure**
  (`dir → newest path, size`), so it unit-tests against a `t.TempDir()` with no `$HOME`
  / cwd / symlink dependency. `encodeWorkdir` (maps `/` and `.`) and `EncodeCwd` (maps
  every non-alnum byte, canonicalises symlinks) diverge on edge-case cwds; reusing the
  shipped dir keeps the producer exactly as correct as the rest of the daemon.
- **The UUID-stem regex is a deliberate third copy (coherence, not hardening).**
  `package main` cannot import the private `internal/sessions.uuidStemPattern` (or the
  `rotation` copy) without exporting it, which the spec forbids. Code-review confirmed
  the copy **byte-identical** to `internal/sessions/reconcile.go:18` and
  `rotation/watcher.go:19`, so the resolver selects the **same** file `mostRecentJSONL`
  would for the same dir. The filter is the coherence choice — a bare `.jsonl`-suffix
  match would diverge only on a stray non-session `.jsonl`, but matching the established
  scan keeps the producer exactly as correct as the shipped machinery.
- **Fail-soft for an optional surface.** `turnbridge.New` error (unreachable nil
  `Subscribe`) → Warn + no-op cleanup; `prod.Run` non-ctx error → Debug. The structured
  stream is a best-effort phone surface and never takes down the relay leg.

## Lessons learned

- **The resolver UUID-stem filter is the coherence choice, not speculative hardening.**
  The resolver scans the *same* `claudeSessionsDir` as `internal/sessions.mostRecentJSONL`
  (reconcile + the fsnotify rotation watcher). Filtering to the canonical `<uuid>.jsonl`
  stem (a `package main` copy of the private `uuidStemPattern`, not a reuse) makes the
  resolver select the *same* file those siblings would for the same dir. A bare
  `.jsonl`-suffix match would diverge only on a stray non-session `.jsonl`, but matching
  the established scan keeps the producer exactly as correct as the shipped machinery.
  (Lifted from the PR's Lessons-learned.)
- **The live `NewSessionSubscriber` + resolver can only be exercised end-to-end by the
  `e2e` oracle** (it needs a real `*tuidriver.Session`); the unit tests drive the *same*
  producer ⇄ emitter composition with a **fake `Subscriber`**, mirroring #615's
  core/glue split. The resolver itself is pure (`dir → newest path, size`) and unit-tests
  against a `t.TempDir()`. (The sandbox cannot fetch the private `tui-driver` module, so
  the `e2e` tag is verified in CI / manually — the unit tests remain the behaviour
  oracle, the inherited #589/#615 caveat.)
- **An injected "the source the daemon already uses" resolver was must-build work.** The
  PO's no-signal validation pass found the original framing hid the coupling: there is
  no stable UUID on a `--continue` relay session, so no `SessionJSONLPath` — and the
  `/clear` AC the body credited entirely to #615's upstream loop actually *depends* on
  this resolver re-evaluating per subscription.

## Concurrency model

- **One new goroutine:** the producer's `Run` (spawned by the helper). The emitter is a
  passive state machine (#632) — zero goroutines. The per-session `sess.Wait()` watcher
  inside `NewSessionSubscriber` is #615's, not this slice's.
- **Single-writer for emitter state.** `Run → drain → OnEvent → emitter.Handle` all
  execute on the one `Run` goroutine, serially — no atomics, no mutex (the #632 named
  assumption, honoured here: do not call `Handle` from anywhere else).
- **`ActiveConns`/`Push` funnel through the manager's single Run goroutine**, each with a
  `ctx.Done` escape arm — a `Handle` in flight during teardown passes a cancelled ctx →
  `ActiveConns` returns `nil`, `Push` returns `ctx.Err()` → `emit` returns early. The
  producer never blocks on a winding-down manager.
- **No watcher leak across teardown.** By drain time `pool.Run` has already returned →
  the supervisor tore down → `sess.Close()` ran at every `runOnce` exit → the current
  subscription's `sess.Wait()` returned → its watcher's `cancel()` fired. At most one
  watcher is ever live (each ends with its session before `Run` re-subscribes) — #615's
  guaranteed property, verified here.

## Testing

`cmd/pyry/interactive_turn_stream_v2_test.go` (new, `package main`, stdlib `testing`,
`-race`, done-channel synchronisation — no sleeps). Ten tests:

*Resolver (`resolveLatestSessionJSONL`) — table-driven against `t.TempDir()`:*
- `NewestWinsWithSizeOffset` — staggered-mtime `<uuid>.jsonl` files → returns the newest path and its `Size()` as offset.
- `ReEvaluatesPerCall` — call, add a newer file, call again → the second call returns the new file (the AC#2 mechanism in miniature — proves per-call freshness).
- `TieBreakLexicographic` — equal mtimes → lexicographically-larger name wins.
- `IgnoresNonSessionEntries` — non-`.jsonl`, non-UUID-stem names, and subdirectories skipped.
- `EmptyDirErrors` / `UnreadableDirErrors` — non-nil (wrapped) error, empty path.

*Wiring (producer ⇄ emitter) — drive the producer with a FAKE `Subscriber`* (the live
subscriber needs a real `*tuidriver.Session`):
- `EventsReachHandle` — a `[Thought, Text, Tool, TurnEnd]` sequence reaches the real `emitter.Handle` closure → the `fakeInteractiveBcast` records `turn_state` / `assistant_delta` / `tool_use` / `turn_end` in order (proves the *bridge*, not the mapper — reuses #632's assertions).
- `RotationReSubscribes` — a fake `Subscriber` returning channel-1 then channel-2, with a resolve-spy; close channel-1 (simulated session end) → the subscriber is called again, channel-2 events reach the broadcaster, the spy shows `resolve` re-evaluated; no goroutine leak across the transition.
- `TeardownUnblocks` (AC#3) — cancel `ctx` → `prod.Run` returns and the helper's `<-done` cleanup unblocks within a deadline; `-race` clean.
- `NoAppOutputLogLeak` (AC#5) — distinctive thought/assistant/tool strings fed through the wiring with a capturing `slog` handler → none appear in any log line *or* the wire payloads emitted by the helper/resolver/producer path.

`make check` (vet + `-race` + staticcheck + substrate-guard) and `make build` green
(QA). **Code review PASS — all five ACs satisfied, no MUST FIX / SHOULD FIX.**

## Out of scope (deferred)

- **Live `/clear` *without* a following restart** (gap-free in-process rotation). The
  producer re-subscribes only on a supervisor restart and has **no live rotation signal**
  (the fsnotify watcher feeds `Pool.RotateID`, not the producer), so a `/clear` followed
  by continued chatting leaves it tailing the pre-`/clear` JSONL until the next restart —
  an inherited #615 property. A follow-up could bridge the rotation watcher's signal into
  a `sessCtx` cancel. **Phase 2 follow-up** (spec Open Question #1).
- **Encoder unification.** `sessions.encodeWorkdir` (maps `/` and `.`) vs tui-driver's
  `EncodeCwd` (maps every non-alnum byte, canonicalises symlinks) diverge on cwds with
  `_`/spaces/symlinks — a pre-existing property that already governs reconcile, the
  rotation watcher, **and** this resolver alike. Reusing `claudeSessionsDir` keeps the
  producer exactly as correct as the shipped machinery; unifying the two encoders (so all
  three use `EncodeCwd`) is a broader, separate fix. **Recommend a follow-up issue** (Open
  Question #2).
- **Cold-start Warn cadence.** On a fresh cwd with no prior transcript, `resolve` errors
  until claude writes its first JSONL → a Warn per `subscribeRetryDelay` (500 ms). Not
  built (evidence-based: no observed failure — Open Question #3). For a `--continue`
  resumed session the JSONL exists immediately, so this window is normally nil.
- **Retiring #589's coarse `message` path** — its own sibling slice re-targets that;
  untouched here.
