# Ticket #571 — `internal/relay`: concurrency-safe v2 session push surface for unsolicited `noise_msg`

The missing primitive behind every server-initiated delivery to a v2 (Noise) phone. Until this slice the daemon's [`V2SessionManager`](../features/v2-session-manager.md) could only reply to a phone *synchronously* — one reply per inbound frame, drained inside `dispatchAppFrame`. There was no way to send an **unsolicited** frame to an open session; the code flagged exactly this gap at the `V2Session` / `State()` doc-comments ("a broadcast layer added in a later slice"). `(*V2SessionManager).Push` closes that gap: it seals a caller-owned `protocol.Envelope` under an addressed session's send CipherState, wraps it as the existing `noise_msg` transport frame, and forwards it to the phone — safe to call from a producer goroutine other than the dispatch goroutine. The consumer is #572 (the v2 assistant-turn bridge, `blocked-by` this ticket), which fans the assistant's reply to `send_message` (today only an `ack`) to connected phones. Purely additive: ~85 LOC in one production file, no new exported types, no new wire shapes, no goroutines, no change to v1 or the inbound dispatch path.

## Implementation

**One production file: `internal/relay/v2session.go` (+~85 LOC).** The whole change is the structural twin of the [#462](462.md) `Rekey`/`manualRekey` funnel — same funnel-then-handle split, same reused `ErrConnNotFound` / `ErrSessionNotOpen` error contract.

- **`pushReq` struct** — `{connID string, env protocol.Envelope, reply chan error}`. Mirrors `manualRekeyReq` byte-for-byte. `reply` is `cap=1` so `Run`'s reply send is non-blocking even if the caller's ctx fires between enqueue and reply.
- **`push chan pushReq` field on `V2SessionManager`** — unbuffered, allocated in `NewV2SessionManager` alongside `manualRekey` (`push: make(chan pushReq)`). Doc-comment mirrors `manualRekey`'s. Not closed on `Run` exit; in-flight callers unblock via `ctx.Done`.
- **One new `Run` select arm** — `case req := <-m.push: req.reply <- m.handlePush(runCtx, req.connID, req.env)`, sitting next to the existing `manualRekey` arm.
- **Public `Push(ctx, connID, env)` method** — the structural twin of `Rekey`. Does only channel I/O on the **caller's** goroutine: a `select` send onto `m.push` (or `ctx.Done`), then a `select` receive on `req.reply` (or `ctx.Done`). No session-state access happens here.
- **Private `handlePush(_, connID, env)` handler** — runs on `Run`'s dispatch goroutine. Six short steps, reusing `emitRekeyRequest`'s seal sequence (minus the rekey bookkeeping):
  1. `s, ok := m.sessions[connID]`; `!ok` → `ErrConnNotFound`. A torn-down session was already `delete`d from the map by `closeWith`, so "closed" collapses into this same branch — no separate closed-state check needed.
  2. `s.state != V2StateOpen` → `ErrSessionNotOpen`. **Security gate:** a `V2StateHandshakeComplete` session holds CipherStates but has not passed the token check; refusing the push keeps server output away from an un-authenticated peer.
  3. `json.Marshal(env)` → `fmt.Errorf("marshal push envelope: %w", err)` (defensive; a well-typed `message` envelope does not fail to marshal in practice).
  4. `s.send.Encrypt(envJSON)` → `fmt.Errorf("seal push envelope: %w", err)` (realistically unreachable under correct flynn/noise). **Reads `s.send` at execution time on the dispatch goroutine** — always the current CipherState, so it composes with a re-key swap for free.
  5. `marshalInnerFrameV2(protocol.TypeNoiseMsg, ciphertext)` → wrap-and-return on error.
  6. `m.send(protocol.RoutingEnvelope{ConnID: s.connID, Frame: frame})`; return `nil`.

- **Input is a caller-owned `protocol.Envelope`** — not pre-marshaled bytes, not `(type, payload)`. The ticket's verb is "seals an application envelope," so the envelope is the input. `Push` performs **no** envelope validation — it is a pure transport primitive. #572 builds the `message` envelope (`Type: protocol.TypeMessage`, `Payload:` a marshaled `protocol.MessagePayload`) and owns `ID`/`TS`.

- **No teardown on push failure.** A failed push never transitions the session out of `V2StateOpen` — best-effort delivery, exactly like `emitRekeyRequest`'s drop-and-stay-open posture. A transport-level drop (relay disconnected) is logged at debug inside `m.send` and returns `nil`, matching v1 reconnect semantics.

## Patterns established

- **The single-writer funnel, reused a fourth time, is the package's concurrency idiom — not a mutex.** Every `s.send.Encrypt` in the package (in-flight `dispatchAppFrame` reply, `emitRekeyRequest`, `sealError`, and now `handlePush`) runs on the single `Run` goroutine, serialised by `Run`'s `select`. The unbuffered `m.push` channel forces a cross-goroutine push to *wait its turn* rather than race the 64-bit nonce counter. A mutex on `s.send` was rejected: it would contradict the documented no-mutex contract and have to be threaded through *every* `s.send.Encrypt` site — a cross-cutting refactor of the in-flight reply path, violating AC#5 ("no change to the inbound dispatch path's existing semantics"). The funnel touches nothing existing: one channel, one arm, two functions.
- **Composition with the re-key swap is free because both run on `Run`.** `handleRekeyInit`'s `s.send, s.recv = newSend, newRecv` tuple swap and `handlePush`'s `s.send.Encrypt` are both on the `Run` goroutine, serialised by the `select`. A push either seals fully under the old key or fully under the new — never a torn read, never a stale captured pointer. No extra locking.
- **Reused sentinels keep the `conn_id` error contract uniform.** `ErrConnNotFound` (wraps `control.ErrConnNotFound` via `%w`, so the wire-mapping `errors.Is` fires at both levels) and `ErrSessionNotOpen` are the same sentinels `Rekey` returns. The push surface adds no new error vocabulary.
- **Inline the seal sequence, don't extract.** The `Encrypt`→`marshalInnerFrameV2`→`m.send` sequence is already inlined at three sites; a fourth inline instance is consistent with the package's choice. Extracting a 2-line helper for one new caller would be over-DRY (cf. PROJECT-MEMORY "Resist over-DRY on duplicated registry primitives"). The existing three sites were left untouched.

## Lessons learned

- **Pre-existing gofmt drift in `v2session.go` / `v2session_test.go` was left untouched as out-of-scope.** Both files carry gofmt drift (an `openSession` struct alignment, an envelope-literal alignment, a trailing blank line) inherited from [#581](581.md)'s `code-review: auto-commit` commit. This slice's additions are gofmt-clean; the drift is **not** a CI gate (CI runs `go vet` / `staticcheck` / `go test -race`, not `gofmt -l`), so it does not block. Worth a dedicated `gofmt -w` sweep, but folding an unrelated whitespace reformat into a feature PR would muddy the diff — left for a cleanup ticket. (Lifted from the PR body's Lessons-learned section.)

## Testing

White-box tests in `internal/relay/v2session_test.go` (same package), stdlib `testing` only, all `-race`-clean. Built on the existing harness (`driveToOpen`, `v2Recorder`, `waitForEnvelopes`, `sealAppFrame`, `decryptAppFrame`). A new helper `buildMessageEnvelope(t, id, text)` constructs the binary→phone `message` envelope (`TypeMessage` + `protocol.MessagePayload`) the #572 bridge will hand to `Push`; push envelopes are always built on the **test** goroutine (a `t.Fatalf` inside a child goroutine is unsafe).

- `TestV2Session_Push_InterleavedWithReply_DecryptsUnderRace` (AC#1, AC#4) — fires a `Push` from a separate goroutine while feeding an inbound sealed request that triggers a `dispatchAppFrame` reply, so the push funnel and the reply path contend for the single `Run` goroutine. Decrypts all three outbound frames (`noise_resp` + reply + push) in capture order under the phone's `initRecv`; a clean in-order decrypt across both app frames is the nonce-integrity proof (`decryptAppFrame` `t.Fatal`s on any AEAD failure). The pushed frame decodes through the **same** `decryptAppFrame` path to a valid `TypeMessage` envelope (proves no new wire shape). Order between reply and push is nondeterministic (`Run`'s `select`) — asserts presence, not order.
- `TestV2Session_Push_ConcurrentWithReplies_NoNonceCorruption` (AC#2) — the stress version: N=8 concurrent pushes + M=8 in-flight request/reply dispatches on the same open session; all N+M outbound frames decrypt in capture order with no AEAD failure. The proof the funnel serialises every `s.send.Encrypt` onto `Run` and the nonce never reuses.
- `TestV2Session_Push_UnknownConn_ErrConnNotFound_OtherSessionUnaffected` (AC#3) — push to a never-seen `conn_id` returns an error satisfying **both** `errors.Is(err, relay.ErrConnNotFound)` and `errors.Is(err, control.ErrConnNotFound)`; an unrelated open session's subsequent solicited round-trip still decrypts cleanly (pins "no mutation of another session's state").
- `TestV2Session_Push_NotOpen_ReturnsErrSessionNotOpen` (AC#3) — table-driven over `{awaiting_init, handshake_complete}`; white-box session injection (`handlePush` returns at the state check before touching `s.send`, so nil CipherStates are safe). Asserts `ErrSessionNotOpen` and zero outbound frames — the security gate against pushing to an un-authenticated peer.
- `TestV2Session_Push_ClosedSession_ReturnsErrConnNotFound` (AC#3) — drives an AEAD-failure 4421 teardown (flips a ciphertext byte) that deletes the session from the map, then asserts a push to that `conn_id` collapses into the `ErrConnNotFound` branch.
- `TestV2Session_Push_CtxCancelled_ReturnsCtxErr` — a `Push` whose ctx is already cancelled returns `ctx.Err()` without blocking; with no `Run` goroutine draining `m.push`, the `ctx.Done` arm is the only ready case (deterministic).

AC#5 (no regressions) is covered by the existing `internal/relay` v2 suite passing unmodified — the change is additive. `go test -race ./...` and `go vet ./...` pass; `go build ./cmd/pyry` OK.

## Out of scope (deferred — owned by #572)

- **Enumeration / broadcast.** `Push` is addressable-by-`conn_id` only, per the AC. Fanning out to *all* connected phones needs a snapshot of currently-open `conn_id`s — the v2 analog of v1's `dispatch.Dispatcher.ActiveConns()`. That is itself a cross-goroutine read of `m.sessions` requiring its own snapshot funnel (e.g. an `ActiveConnIDs() []string` method through the same `select`), and bundling it here would over-scope #571. #572's own architect run designs it.
- **Production wiring.** Like `Rekey` before its wire-up, `Push` is reachable only from `internal/relay` tests until #572 wires the assistant-turn bridge. `cmd/pyry/relay.go:startRelayV2` is where #572 obtains the `*V2SessionManager` handle.
- **Outbound envelope-ID policy.** `Push` leaves `env.ID` to the caller. v2 has no per-session outbound counter (manager-internal frames use fixed IDs 1/2). #572 decides whether a per-session monotonic ID is needed or whether `MessagePayload.MessageID` (a UUID already present) suffices for phone-side ordering/dedup — recommend starting with a caller-side counter, escalating only if the phone protocol requires monotonic envelope IDs.
- **E2E oracle extension.** No production caller of `Push` exists until #572, so the real `send_message → message` round-trip is #572's e2e deliverable, extending `internal/e2e/relay_v2_daemon_test.go`. No e2e change in this slice.
