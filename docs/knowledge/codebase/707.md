# Ticket #707 — `interrupt` v2 wire type + Esc routing (`Cancel` neutral command + `Interrupter` seam)

The **remote interrupt** control ([EPIC #597] Phase 3, [ADR 025]): a paired phone's
equivalent of pressing **Esc** at the local terminal. A phone sends a bare
`interrupt` v2 control frame; the daemon maps it to the internal neutral
`turnevent.Cancel` command and routes it to the supervised claude as a single Esc
keystroke — claude's own interrupt. Maps adapter-neutrally — epic #600 later maps
ACP `session/cancel` onto the same `Cancel` shape.

Every dependency already existed in `main`, so this slice is **~30 lines of
pattern-echo across four production files**: the keystroke surface is sealed
end-to-end (`supervisor.SendEsc()`, #726, already consumed by `modal_cancel` since
#727 — **this ticket touches neither tui-driver nor supervisor**), the neutral
`Cancel` slot was the already-reserved next member of `turnevent.Inbound`
(`permission.go:43`), and the inbound v2 control-frame interception pattern
(`dispatchAppFrame` before `dispatch.Route`) is established by `rekey_request` /
`request_snapshot` / `modal_cancel` / `dequeue_message`. Held at **S**, no split.

The one **genuinely new** thing: the **inbound `interactive` capability gate**.
Existing inbound control frames gate *outbound* emission on `s.interactive`, but
`interrupt` is the first whose *authorization* is the `interactive` capability.
`security-sensitive` (the new inbound trust decision on an internet-exposed
surface); architect spec-stage security review verdict **PASS**.

## Implementation

Four production files, three packages + the protocol doc + one test file. All
changes **additive** — no signature changes, no consumer cascade (the new
`V2SessionConfig.Interrupter` is an optional nil-safe field, the same shape that
held `ModalResolver`/#727 and `Snapshotter`/#618 at S).

### `handleInterrupt` — the new logic (`internal/relay/v2session.go`)

Runs on the manager's **single Run dispatch goroutine**, so the `s.interactive`
read is lock-free under the package's single-owner invariant. Signature takes
**only `s`** (no `ctx`, no `env`) — a documented deviation from the `(ctx, s, env)`
sibling handlers: the frame carries no payload to decode and the handler does no
cancellable work. Order is load-bearing — **capability gate first**:

1. **`if !s.interactive` → return** (no Esc). The new inbound capability gate (AC-2
   negative path). The `s.interactive` flag is server-authoritative — set
   fail-closed in `handleNoiseInit`'s token-OK path from the daemon's
   `negotiateCapabilities` (#626), never from the phone's raw advertisement.
2. **`if m.cfg.Interrupter == nil`** → debug-log `v2.interrupt.inert`, return
   (foreground / pre-wire; mirrors `handleModalCancel`'s nil-resolver guard).
3. **`m.cfg.Interrupter.SendEsc()`** — best-effort. An error (no live session /
   teardown) is `Warn`-logged (`v2.interrupt.keystroke_err`, `conn_id` + the
   supervisor sentinel) and tolerated — nothing to roll back, no reply owed
   (fire-and-forget, no broadcast).

### Data flow

```
phone ── interrupt (noise_msg, AEAD) ──> relay ──> V2SessionManager.Run
  └─ handleFrame → handleNoiseMsg → dispatchAppFrame (probe decode)
       └─ case TypeInterrupt → handleInterrupt(s)
            ├─ !s.interactive          → return (inert)        [AC-2 negative]
            ├─ cfg.Interrupter == nil  → debug log, return     [foreground/unwired]
            └─ cfg.Interrupter.SendEsc() → supervisor.SendEsc() [AC-2 positive]
                 └─ tui-driver Session.SendEsc() → one Esc to claude
```

## Patterns established

- **The first inbound v2 control frame whose authorization IS a capability — a
  one-line gate, not an abstraction.** Existing inbound controls each authorize
  differently: `modal_cancel` consumes a one-time nonce, `modal_answer` uses the
  per-device gate (#702), `dequeue_message` is deliberately ungated. `interrupt`
  is the first to gate on `s.interactive` itself. The handler uses a **one-line
  `if !s.interactive` check, NOT a reusable inbound-gate abstraction** — `interrupt`
  is its only consumer, so a shared gate would be a one-consumer abstraction
  (YAGNI / Simplicity First). The architect's body explicitly authorized this
  minimal-check shape for an Esc-only frame.
- **`Cancel` is declared vocabulary, not the live path — the `modal_cancel`
  precedent.** The mobile `interrupt` frame routes to Esc **directly** via the relay
  seam (`handleInterrupt`), it does **not** construct a `turnevent.Cancel` value —
  exactly as the mobile `modal_cancel` frame routes to `ModalResolver.ResolveCancel`
  rather than building a `PermissionResponse`. The mobile-wire → neutral-`Cancel`
  translator is future work (the ACP adapter, #600); `Cancel` exists *now* so #600
  has its target and the `Inbound` sum is closed against ad-hoc additions.
- **The optional nil-safe `V2SessionConfig` seam is what holds the slice at S.**
  A third instance of the `Snapshotter`(#618)/`ModalResolver`(#727) precedent: the
  relay declares the one-method interface it consumes (consumer-side, CODING-STYLE
  "define interfaces where consumed"), the composition root supplies the impl, and
  the optional field (nil ⇒ inert no-op) keeps every existing construction site
  compiling unchanged.
- **The interface is named for its relay-domain role; the method keeps the sealed
  surface's name.** `Interrupter` (role) with method `SendEsc` (the existing sealed
  `supervisor.SendEsc`, #726) — so `*supervisor.Supervisor` satisfies it with
  **zero new supervisor code**, mirroring `ScreenSnapshotter.ScreenSnapshot` /
  `ModalResolver.Resolve*`.
- **Idempotency by effect, not machinery.** A replayed `interrupt` simply sends
  another Esc; an Esc with no running turn is a no-op in claude, so the frame needs
  **no nonce / dedup / `interrupt_id`** (unlike modals). A bare control frame.

## Lessons learned

*(Lifted from the PR #736 Lessons-learned note.)*

- **`internal/relay/v2session.go` is gofmt-dirty at HEAD under local Go 1.26.2** (a
  single trailing-EOF blank line vs CI's pinned older Go). Left untouched —
  reformatting would spray a spurious whole-file diff and fight CI. New code is
  gofmt-clean. (The general [[pyrycode-gofmt-dirty-at-head-go1.26]] caution, observed
  again in this exact file.)
- **A fire-and-forget inbound control handler that emits no outbound envelope needs
  a non-obvious test sync barrier: open a *second* conn after sending the control
  frame.** The single FIFO `Frames` channel + single `Run` goroutine guarantees the
  earlier frame is fully handled before the barrier conn opens, so the fake's
  counter is final — and it works even when the seam is nil (barrier-open *is* the
  proof of no panic / no hang on the nil path). This is the reusable idiom for
  testing the next no-reply inbound control handler.

## Out of scope (flagged, not designed here)

- **Multi-phone concurrency.** Any interactive paired phone can interrupt the single
  live claude — there is **no per-conversation / per-connection binding** for
  interrupt, consistent with the existing broadcast fan-out model (a user's paired
  devices are one trust domain, per the [Security model]). Per-conversation interrupt
  scoping in a future multi-session world is a later ticket — flagged in the spec's
  Open questions, not a vulnerability today (the daemon supervises one live claude).
- **`Cancel` payload for #600.** `Cancel` is fieldless now; if ACP `session/cancel`
  needs a session/turn correlation id, #600 adds the field additively.
- **The mobile consumer** — the phone-side interrupt UI (pyrycode-mobile).
