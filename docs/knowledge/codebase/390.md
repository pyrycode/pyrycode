# Ticket #390 — `internal/agentrun/streamrunner` stream-json subprocess primitive

Introduce the no-PTY sibling of `agentrun.Drive` as a leaf primitive. `Run(ctx, Config) error` spawns claude with a caller-supplied argv, writes one stream-json user-turn JSON envelope to its stdin, forwards stdout/stderr to caller-supplied writers, and maps the child's exit to the verb-level contract (nil on clean exit or ctx-cancel; `*exec.ExitError` on non-zero exit). No production consumer in this ticket — `pyry agent-run` wiring is deferred to #391; the AC-trace and behaviour are pinned by a `TestHelperProcess` fake-claude suite.

## Implementation

- **`internal/agentrun/streamrunner/runner.go`** — single ~195-line production file. `Config` (required: `ClaudeBin` / `WorkDir` / `Stdout` / `Stderr`; optional: `Args` / `PromptBytes` / `Env` / `Logger`). `Run` validates → marshals envelope via unexported `userTurn` / `userTurnMessage` / `userTurnContentText` structs → `exec.CommandContext` → sets `cmd.Cancel = SIGTERM-signal` and `cmd.WaitDelay = 5s` (the `killGrace` constant) → opens stdin via `cmd.StdinPipe()` → `cmd.Start()` → synchronous `stdin.Write(envelope)` + `stdin.Close()` with Warn-logged-and-continued failures → `cmd.Wait()` → return `nil` if `ctx.Err() != nil`, else verbatim wait error.
- **`internal/agentrun/streamrunner/helper_test.go`** — `TestStreamRunnerHelperProcess` re-exec fake claude with four modes keyed by `GO_STREAMRUNNER_HELPER_MODE`: `clean` (three-line stream-json sequence + exit 0), `exit1` (drain + exit 1), `sleep` (SIGTERM handler prints `"got SIGTERM"` to stderr + exits 0; 30s timeout fallback), `echo_stdin` (copies stdin to `GO_STREAMRUNNER_HELPER_STDIN_FILE` mode 0o600 + exit 0).
- **`internal/agentrun/streamrunner/runner_test.go`** — `helperRunCfg(t, mode, stdout, stderr, extraEnv...)` constructor mirroring `agentrun.helperDriveCfg`. Four `t.Parallel` tests:
  - `TestRun_CleanExit` — substring-checks `"type":"system"` / `"type":"assistant"` / `"type":"result"` in stdout; asserts stderr empty.
  - `TestRun_NonZeroExit` — asserts `errors.As(&exitErr)` succeeds and `exitErr.ExitCode() == 1`.
  - `TestRun_CtxCancelMidRun` — parent cancels ctx 100ms after Run starts; asserts Run returns nil, elapsed < 6s, stderr contains `"got SIGTERM"` (sanity-check that SIGTERM, not SIGKILL, was the trigger).
  - `TestRun_StdinEnvelopeRoundTrip` — prompt is `"hello \"world\"\n\\backslash\x01end"`; helper writes received stdin to a tempfile; test unmarshals into a local `userTurn` mirror and asserts byte-for-byte text equality + newline-terminated envelope.

## Design choices

- **Stdlib `cmd.Cancel` + `cmd.WaitDelay` over a hand-rolled goroutine.** Go 1.20+ added these fields precisely for the SIGTERM-grace-SIGKILL pattern. Using them removes a goroutine + timer + state machine the package would otherwise own and makes the contract reader-visible in the `os/exec` docs. The 5s grace is hardcoded per AC ("no timing knobs") and mirrors `budget.GracePeriod` for symmetry. The two paths are orthogonal — `budget.Counter` calls `cmd.Process.Signal(SIGTERM)` directly from its own `Terminate` hook on a different signal (turn count), independent of ctx cancellation.
- **JSON-encoded prompt, not shell-escaped.** Verified by `TestRun_StdinEnvelopeRoundTrip` with a deliberately tricky prompt containing `"`, `\n`, `\\`, and `\x01` — all four round-trip through `encoding/json` cleanly. This is the load-bearing security property: the spec's "stdin write is a single JSON line followed by stdin close" AC pins the encoding path so callers can't accidentally trigger argv injection by routing operator strings through shell escapes.
- **Stdin write failures Warn-logged-and-continued, not returned.** Identical pattern to `drive.go`'s PTY writes (`drive.go:93-104`). Reason: the child may have already exited (especially in fast-exit modes); its exit code is the authoritative outcome, not whether stdin made it through. The eventual `cmd.Wait()` surfaces the child-exit error verbatim.
- **No `New() *Runner` constructor.** Pure function call shape. The package has no state to thread between calls and no resources to release beyond what `cmd.Wait` already handles.
- **Sibling, not a replacement.** `agentrun.Drive` (PTY-driven) stays as the interactive bridge mode primitive. The new package is its no-PTY sibling for the headless `pyry agent-run` verb path. Splitting them — rather than overloading `Drive` with a no-PTY flag — preserves a clean test surface and a clean failure-mode story for each.

## Patterns established

- **Stdlib over hand-rolled timers when the lifecycle matches.** Before reaching for a goroutine + `time.AfterFunc` to manage SIGTERM-grace-SIGKILL, check whether `cmd.Cancel` / `cmd.WaitDelay` already covers the case. They do, for any "ctx-cancel = graceful then forceful child kill" pattern. The `budget` package's hand-rolled timer is correct for *its* case (mid-run terminate on turn count, not ctx cancel), but `streamrunner`'s ctx-cancel path is the textbook stdlib case.
- **Logging-discipline pinned in the package doc-comment + asserted by absence.** The package never receives the stream-json content in a parseable form — writers are opaque `io.Writer` values. There is therefore no need for a test specifically guarding "doesn't log prompt bytes"; the type system enforces it. The package doc-comment still pins the discipline so future contributors don't add logging on the way through.
- **AC-trace table in the architect spec carries through to the test names.** Each row of the spec's acceptance trace maps to a single test in `runner_test.go` (or an absence-assertion in the doc-comment). When the developer reads "AC#3 (single-line JSON envelope, stdin close, JSON-encoded prompt)" and looks at `TestRun_StdinEnvelopeRoundTrip`, the mapping is one-to-one. Generalisable: spec-side AC traces are the right contract surface for "this test pins this AC" callouts.

## Out of scope (deferred)

- **Caller wiring in `pyry agent-run` (`cmd/pyry/agent_run.go`)** — deferred to #391. That ticket will assemble the full claude argv (`--input-format stream-json --output-format stream-json --verbose --dangerously-skip-permissions --allowed-tools … --model … --effort … --max-turns … --append-system-prompt-file …`), thread the verb's stdout/stderr writers, and compose `streamrunner.Run` with the existing `streamjson.Emitter` shape. The current `Drive`-based PTY path remains until #391 lands.
- **Multi-turn drives.** AC#3 pins single user-turn-then-close-stdin. A future ticket can add a multi-turn primitive (or extend this one) when a consumer needs it.
- **Operator-tunable timing knobs.** None apply: stream-json mode has no trust dialog and no TUI-write timing, and the 5s SIGTERM grace matches `budget.GracePeriod` by design. Defer until evidence-based-fix-selection forces a knob.
- **Lint-time `go list -deps` enforcement of the "no `internal/supervisor` import" rule.** Doc-commented + verifiable on demand; not wired into CI. A regression would be a code-review catch.
