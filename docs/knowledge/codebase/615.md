# Ticket #615 ‚Äî Event-stream bridge (producer): drain `Session.Events()` into the internal turn-event model

Introduces `internal/turnbridge`, the **producer half** of the Phase 2
structured-event bridge ([EPIC #596], [ADR 025] ¬ß Phase 2). It drains the
supervised claude session's unified tui-driver `Events()` stream and maps each
event into the neutral internal turn-event model ([#606], `internal/turnevent`),
plus an additive `Supervisor.Session()` accessor for the subscription seam.
Deliberately **unwired to a live fan-out** ‚Äî the consumer slice ([#616]) attaches
the `OnEvent` callback and fans the mapped stream out to capability-advertising
phones. Full package reference: [features/turnbridge-package.md](../features/turnbridge-package.md).

#615 + #616 are the two halves of the originally-combined #608. Blocked on [#619]
(tui-driver **v1.3.0** bump, merged before this ran) ‚Äî `ParseToolUse` /
`ParseToolResult` and the `Tracker`-param `Session.Events` signature exist only
from v1.3.0.

## Implementation

Two new production files in a new package + one additive supervisor accessor
(3 production source files ‚Äî `producer.go` ~207, `mapper.go` ~182, the
supervisor accessor +16; 4 exported types; tests ~476 LOC across `mapper_test.go`
~277 + `producer_test.go` ~199, not counted toward the production budget):

- **`producer.go`** ‚Äî the lifecycle. A **testable core** (`Run`'s outer
  re-subscribe loop + `drain`'s inner `select`, driven by an injected
  `Subscriber`) and **live glue** (`NewSessionSubscriber`, the production
  `Subscriber` calling `Session.Events`). `New(Config)` errors iff `Subscribe` is
  nil. `Config.OnEvent` is the single output callback (nil ‚áí no-op beyond
  draining). The live subscriber owns a per-session ctx cancelled by a
  `go func(){ sess.Wait(); cancel() }()` watcher ‚Äî the linchpin of "no leaked
  goroutine across a session restart" (see Lessons learned).
- **`mapper.go`** ‚Äî `mapEvent`, a **pure** tui-event ‚Üí `turnevent.Event` type
  switch returning `(event, ok)`; `ok == false` drops. Robust JSONL-sourced kinds
  map (assistant `text`‚Üí`TextChunk`, `thinking`‚Üí`ThoughtChunk`, `tool_use` via
  `ParseToolUse`‚Üí`ToolStart`, `tool_result` via `ParseToolResult`‚Üí`ToolUpdate`,
  `JsonlEndOfTurn`‚Üí`TurnEnd{end_turn}`); every PTY-state kind, `StallDetected`,
  and `Unknown` drop because the internal model (#606) has no type for them
  (ADR 025 brittleness split). Pure helpers: `thinkingText`, `toolResultText`,
  `toolKind`, `rawInput`, `toolStatus`, `toolResultContent`, `messageID`.
- **`supervisor.go`** (mod, ~8 LOC) ‚Äî `Session() *tuidriver.Session`, nil-safe,
  captured under `sessMu`, mirroring `ScreenSnapshot`. Additive: zero call-site
  blast radius. `*supervisor.Supervisor` thereby satisfies `turnbridge.SessionHost`
  structurally (the package does not import `internal/supervisor`).

Dependency direction stays clean: `cmd/pyry ‚Üí internal/turnbridge ‚Üí
{tuidriver, turnevent}`. No import of `internal/supervisor` (own `SessionHost`
seam) or `internal/sessions` (the JSONL resolver is an injected
`func(ctx)(string,int64,error)` param). `make check` (vet + `-race` + staticcheck
+ substrate-guard) and `make e2e-realclaude` (`ok ‚Ä¶ 144.876s`) both green.

## Patterns established

- **Core/glue split makes lifecycle guarantees unit-testable without a live
  claude.** A real `*tuidriver.Session` can only be `Spawn`ed, so the drain +
  re-subscribe core is driven by an injected `Subscriber` (a function closing over
  scripted channels) and tested directly; the live `NewSessionSubscriber` is
  verified downstream (#616 wiring + v2 e2e oracle). The fake `Subscriber` is the
  only double needed ‚Äî no `*tuidriver.Session` mock.
- **Pure mapper, drop-logging in the caller.** `mapEvent` has no logger and no
  I/O, so it is trivially table-testable; the drain owns the single
  `log.Debug("‚Ä¶dropping unrepresentable event", "kind", ev.Kind)`. One debug call,
  not a per-branch log fan-out. Follows #606's "primitive drops what the model
  can't hold; it does not invent error envelopes" posture.
- **Single output callback, not a channel.** `OnEvent` (nil-allowed, set once at
  construction) models "no consumer = no-op beyond draining" precisely and pushes
  backpressure / queueing / drop-policy entirely to #616 (ADR 025 ¬ß Backpressure).
  Runs on the single `Run` goroutine.
- **Synthetic JSONL fixtures populate both `Message` and `RawLine`.** The test
  `entry(...)` helper builds the typed `Message` (read by `AssistantText` /
  `thinkingText`) **and** marshals `RawLine` (re-parsed by `ParseToolUse` /
  `ParseToolResult`) from the same blocks, so the two views never drift. `RawLine`
  is nil by default on a synthetic `JSONLEntry` ‚Äî the parsers read it, so the test
  must populate it. Fixtures avoid substrate-guard-banned tokens (the guard scans
  test files too).
- **Additive accessor that doesn't breach the substrate seal.** `Session()` hands
  out the `*tuidriver.Session` the supervisor already holds; the producer only
  calls `sess.Events()` (typed events), never `MirrorOutput()`/`Snapshot()` (raw
  bytes), so no claude-screen literal enters pyrycode. Same seal logic as #618's
  `ScreenSnapshot` (which renders the snapshot inside the supervisor).

## Lessons learned

- **The per-session ctx (cancelled by `sess.Wait()`) is the *only* thing that
  closes the `Events()` channel on a supervisor restart.** tui-driver's internal
  `TailJSONL` tails a file that **persists on disk after claude exits** (EOF ‚Üí
  wait ‚Üí retry), so it never closes on process exit alone; the merge loop would
  keep running and the channel would never close. Spawn the `sess.Wait()` watcher
  **only on the `Events()` success path**, so an open-error retry leaks no
  watcher. (Lifted from the PR's Lessons-learned section.) This is the same
  per-session-ctx Wait discipline `ptyrunner.Run` uses on the same unified stream
  ([codebase/513.md](513.md)).
- **Comment-provenance accuracy (code-review NIT).** `thinkingText`'s comment
  originally claimed the `{"type":"thinking","thinking":"‚Ä¶"}` shape was "confirmed
  against tui-driver's JSONL fixtures" ‚Äî but v1.3.0 ships no thinking fixtures in
  `testdata/`. The field name is correct (it's the documented Anthropic
  extended-thinking block shape, mirroring `AssistantText`'s `Raw["text"]` access);
  the provenance claim was the only issue. Resolve open questions by citing the
  *documented shape*, not an unverifiable fixture source.
- **`EventKind` has no `String()` (code-review NIT, non-blocking).** The drop log
  renders `ev.Kind` as a bare integer. Functional and debug-level; flagged only so
  the int‚Üîname lookup is a known cost if these logs are ever read in anger.

## Scope (S confirmed)

3 production files (under the ‚â•5 split gate), 4 exported types (`Producer`,
`Config`, `Subscriber`, `SessionHost` ‚Äî under the >5 red line), 0 consumer call
sites (additive accessor + greenfield unwired package). The production slice
(producer + mapper + the ~8-line accessor) sits well under the production red
line; tests scale linearly with the mapping table and don't count toward the
production budget. One cohesive slice: accessor ‚Üí drain ‚Üí map ‚Üí output ‚Üí tests.

## Out of scope (deferred to siblings)

- **v2 wire-envelope mapping, the per-phone push, capability intersection /
  enforcement, the capability-gated fan-out, and the production JSONL resolver +
  live-`/clear`-rotation re-subscription trigger** ‚Äî all [#616] (the consumer;
  carries `security-sensitive`).
- **`ToolStart.Locations`** (deriving touched files from tool input) ‚Äî #616/refinement.
- **Idle / modal / stall / thinking-state events** ‚Äî the dropped PTY-state kinds
  have no internal-model type yet; later #596/#597 children.
- **ACP `stopReason` return + the `pyry acp` adapter** ‚Äî [#600].
