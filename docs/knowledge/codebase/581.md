# Ticket #581 — decouple e2e test readiness from the binary↔relay hello (migrate `LastBinaryHello` → `WaitBinary`)

The e2e suite proved "the binary's outbound relay connection is up" by polling
`fr.LastBinaryHello(serverID)` — a fakerelay map populated only when the binary
*sends* a relay-leg `hello` envelope. That coupled every readiness gate to an
application-level handshake. #582 will retire that handshake; if the suite still
keyed readiness off the hello, retiring it would cascade `t.Fatal`s across nine
files. This ticket migrates every readiness gate onto `fr.WaitBinary(ctx, serverID)`,
which detects the same condition independently of the hello: it returns once the
binary's WebSocket upgrade has registered its server-id (read from the
`x-pyrycode-server` header) into `s.binaries`. **Nothing on the wire changes** —
the binary still sends the hello and the fakerelay still records it and synthesizes
`hello_ack`; only the *test's readiness probe* moves off the hello. This is the
consumer-migration slice of #569's Strangler Fig: **#581 migrates readers → #582
retires the handshake → #583 removes the now-dead `LastBinaryHello` accessor.**

Pure test-only change. No production code touched (`internal/relay/connection.go`
unchanged); `security-sensitive` intentionally omitted despite parent #569 carrying
it — no production code, no wire-behaviour change.

## Implementation

Nine files, nine contiguous edit regions, each the same swap. Every readiness gate
was a multi-line block — a `deadline`, a poll loop that breaks on a positive
`LastBinaryHello`, and a companion negative pre-check that `t.Fatal`s on timeout —
replaced by a single `WaitBinary` call carrying its own timeout context:

```go
// Before (representative)
deadline := time.Now().Add(5 * time.Second)
for time.Now().Before(deadline) {
    if _, ok := fr.LastBinaryHello(serverID); ok { break }
    time.Sleep(20 * time.Millisecond)
}
if _, ok := fr.LastBinaryHello(serverID); !ok {
    t.Fatal("binary hello not observed within 5s")
}

// After (the contract every site converges to)
ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
defer cancel()
if !fr.WaitBinary(ctx, serverID) {
    t.Fatal("binary connection not registered within 5s")
}
```

Net: −88 / +46 lines across nine files. The ~17 readiness sync points the suite
had collapse into nine edit regions because two are centralized (one edit covers
several call sites):

- **Five uniform inline sites:** `register_push_token_test.go`,
  `relay_assistant_turn_test.go`, `relay_auth_test.go`, `relay_roundtrip_test.go`,
  `relay_send_message_test.go` (all 5s).
- **`relay_test.go` — `TestRelay_1011`** (4s, the lone non-5s site — duration
  preserved). This file did **not** import `context`; the migration adds the
  `"context"` import.
- **`relay_v2_daemon_test.go` — the `waitBinaryHello` helper body** migrated once;
  its 2 callers stay verbatim. Name kept (rename would fan out call sites for no
  behavioural gain); doc comment updated to describe WS-registration semantics.
- **`relay_v2_handshake_test.go` — the `startV2Harness` readiness block**, so one
  edit covers every v2-handshake subtest. `serverID` is a typed value here, so the
  existing `string(serverID)` cast is kept: `WaitBinary(ctx, string(serverID))`.
- **`respawn_after_eviction_test.go` — only the `LastBinaryHello` readiness poll**
  migrated; the unrelated idle-eviction WARN log-scanner above it is left alone.

`WaitBinary` has **no internal deadline** — it blocks until the binary registers
*or* `ctx` is done, so each site constructs a fresh `context.WithTimeout` (no usable
`ctx` is in scope at the readiness point; the dial `ctx` is always created later).
`defer cancel()` is safe in both test bodies and helpers — `WaitBinary` returns
synchronously before the defer runs. Per-site timeout durations preserved (no
behaviour change); the `t.Fatal` messages were updated to describe WS registration
rather than the hello (cosmetic).

### Deliberately left in place (AC)

- **`relay_test.go:72` — `TestRelay_Hello`'s `LastBinaryHello` read.** It captures
  the hello envelope to assert its payload (`role=server` / `server_id`) — it is the
  *subject* of that test, not a readiness gate, and a bool `WaitBinary` cannot
  replace it.
- **The `LastBinaryHello` accessor and the `lastBinaryHello` recording in
  `fakerelay.go`.** #583 removes them once nothing reads them.
- **`internal/relay/connection_test.go` — out of scope.** It uses its own in-file
  `testRelay` double (with `HelloEnv` accessors), **not** the e2e `fakerelay`;
  `WaitBinary`/`LastBinaryHello` aren't even importable there. Its hello/hello_ack
  cases are handshake-*subject* tests that #582 updates when retiring the handshake.

## Patterns established

- **Migrate the readiness consumers before retiring the thing they key off
  (Strangler Fig).** Decoupling the suite's readiness probe from the hello is a
  pure mechanical refactor with zero wire change, which makes it a safe, reviewable
  standalone slice. Retiring the handshake (#582) then can't cascade `t.Fatal`s,
  because no readiness gate still depends on it. Sequencing readers-first turns a
  risky cross-cutting removal into three small, independently-green steps.
- **Pick the readiness signal at the lowest layer that proves the condition.**
  "The binary's relay connection is up" is genuinely a WS-registration fact
  (`x-pyrycode-server` → `s.binaries`), not an application-handshake fact. Keying
  the gate on the hello conflated transport readiness with an application-level
  envelope; `WaitBinary` reads the transport fact directly, so the gate survives any
  change to the application handshake above it.

## Lessons learned

- **A reused `deadline` variable turns a clean deletion into a build break the CI
  gate can't see.** In `relay_assistant_turn_test.go` and `relay_send_message_test.go`
  the migrated block's `deadline :=` declaration was *reused* by a later
  `deadline = …` assignment elsewhere in the same test. Deleting the poll loop
  orphaned that later assignment (assign-to-undeclared). Because the whole e2e suite
  is `//go:build e2e`-tagged, `go test -race ./...` (what CI runs) never compiles
  these files — only `go vet -tags e2e ./internal/e2e/...` catches it. Fix: convert
  the now-first `deadline = …` to `:=`. **Run the tagged build/vet explicitly**;
  CI green is not evidence an e2e-tagged file even compiles.
- **The `context` import is the same hidden-from-CI trap.** `relay_test.go` gained a
  `"context"` import for the new `context.WithTimeout`. A missing import there would
  also slip past `go test -race ./...` and surface only under `-tags e2e`.
- **Validation surfaced a pre-existing, unrelated flake** in the untouched
  `fakeclaude` test double — `TestFakeClaude_OpensInitialAndRotatesOnTrigger` races
  trigger-removal against rotated-JSONL creation. Out of scope for this readiness
  migration; filed as **#584**. (Noting it here so a future reader who sees that
  test flake doesn't attribute it to the `WaitBinary` migration.)

## Testing

No new tests — the migration moves the readiness probe under existing tests; validation
is "everything still passes, including the bits deliberately left alone."

- `go test -race ./...` and `go vet ./...` pass (e2e files not compiled here, but
  confirms nothing outside e2e regressed).
- **`go test -tags e2e ./internal/e2e/...` — the real gate** (the only command that
  compiles the migrated files) passes with no readiness-related flakes;
  `go vet -tags e2e ./internal/e2e/...` clean.
- **Behaviour-unchanged guards stay green, untouched:** `TestRelay_Hello`'s line-72
  payload assertion proves the binary still sends the hello and the fakerelay still
  records it; `fakerelay_test.go`'s `TestWaitBinary` (happy + timeout) and
  `LastBinaryHello` coverage prove both accessors still behave.
- **Incidental churn:** `internal/e2e/realclaude/testdata/permission_protocol_v2.1.158.json`
  changed in the code-review auto-commit — identifier-only (`session_id`, `message.id`s,
  `toolu` ids, timing/cost), no structural change. Belongs to a realclaude test, not
  to this ticket; not a #581 behaviour change.

## Out of scope

- **Retiring the binary↔relay hello handshake** — #582.
- **Removing the `LastBinaryHello` accessor / `lastBinaryHello` recording** — #583,
  once nothing reads them.
- **`internal/relay/connection_test.go`** — handshake-subject tests on a different
  relay double; #582 territory (see above).
- **Timeout normalization.** `TestRelay_1011` keeps 4s, all others 5s — per-site
  durations preserved on purpose (no behaviour change).
