# Ticket #470 — agent-run: cut over from streamrunner to ptyrunner with `PYRY_USE_STREAMJSON` fallback

Final wiring step of the PTY-drive pivot. `pyry agent-run` now dispatches to [`internal/agentrun/ptyrunner`](../features/ptyrunner-package.md) by default — pre-marking workdir trust in `~/.claude.json`, writing a per-spawn deny-default settings JSON, minting a fresh session UUID, and delegating to `ptyrunner.Run`. The legacy stream-json subprocess pipeline via [`internal/agentrun/streamrunner`](../features/streamrunner-package.md) stays as a sibling indefinitely, selected by setting `PYRY_USE_STREAMJSON=1`. Operator decision 2026-05-19: streamrunner stays so empirical billing-classification comparison is possible after Anthropic's 2026-06-15 subscription-eligibility deadline. The dispatcher submodule is unchanged — `internal/agentrun/streamjson/emitter.go` produces the same wire shape under both modes, so the four `agents` repo forks pick up the new behaviour at their next `pyry update` without code changes. Sub-issue of [#329](https://github.com/pyrycode/pyrycode/issues/329) (tracking). Composed on top of [#471](https://github.com/pyrycode/pyrycode/issues/471) (ptyrunner skeleton), [#475](https://github.com/pyrycode/pyrycode/issues/475) (trust pre-write), [#476](https://github.com/pyrycode/pyrycode/issues/476) (per-spawn settings), [#478](https://github.com/pyrycode/pyrycode/issues/478) (JSONL tail + stream-json emit + end-of-turn), and [#479](https://github.com/pyrycode/pyrycode/issues/479) (pyry-side budget + watchdog). Follow-up [#473](https://github.com/pyrycode/pyrycode/issues/473) adapts `--self-check` to exercise the ptyrunner default path.

## Implementation

- **`runAgentRun` gains a one-line env-var branch; the rest of the body is unchanged.** The `--self-check` short-circuit, flag parse, `os.ReadFile(promptFile)`, `PYRY_CLAUDE_BIN` resolution, and `signal.NotifyContext(SIGTERM, SIGINT)` all stay verbatim. After ctx setup, `os.Getenv("PYRY_USE_STREAMJSON") == "1"` dispatches to a new `runAgentRunStreamRunner` helper (today's body, lifted out); anything else dispatches to a new `runAgentRunPty` helper that wires the four prereq primitives. The top-level wrap `fmt.Errorf("agent-run: %w", err)` is added in exactly one place — both helpers return wrapped-but-not-prefixed chains. `context.Canceled` continues to collapse to `nil` at the same guard.

- **`PYRY_USE_STREAMJSON` predicate is intentionally strict.** Only the exact string `"1"` selects the streamrunner path. `"true"`, `"yes"`, `"on"`, `"streamjson"`, `"0"`, `"false"`, and the empty string all fall through to the ptyrunner default. Mirrors how `PYRY_AGENT_RUN_FAKE` and `GO_PTYRUNNER_HELPER` work today and avoids the "did this string mean true?" ambiguity that bit the historical `PYRY_USE_LEGACY_CLAUDE` knob. `TestRunAgentRun_EnvNon1ValueDispatchesToPtyRunner` pins this predicate table-driven so a future contributor cannot quietly widen the truthy set.

- **`runAgentRunPty` is a flat sequence of three I/O calls plus one delegated `ptyRun`.** No goroutines, no locks, no nested ctx. Order: `trustMark(parsed.workdir)` → wrap as `"mark workdir trusted in ~/.claude.json: %w"` on error; `settingsWrite(parsed.allowedTools)` → wrap as `"write per-spawn settings: %w"` on error, then `defer func() { _ = os.Remove(settingsPath) }()`; `newSessionID()` → wrap as `"mint session id: %w"` on error; `ptyRun(ctx, ptyrunner.Config{…})` with the populated struct. Defer-LIFO fires on every exit path (success, sessionID error, ptyrun error, ctx-cancel), satisfying the AC #2 cleanup guarantee structurally; the defer is registered AFTER the err-check on `settingsWrite` so a settings-write failure does not try to remove an empty path.

- **`ptyrunner.Config.WorkDir` is `trust.MarkWorkdirTrusted`'s symlink-resolved realpath, NOT `parsed.workdir`.** Claude resolves the workdir before keying `projects[<realpath>]` in `~/.claude.json`; if pyry pre-marked the realpath and then handed claude a symlink path as `--cwd`, claude's modal-check would miss the entry and the trust modal would render. Passing `realpath` through to `ptyrunner.Config.WorkDir` keeps the two keys aligned and structurally satisfies the modal-skip invariant. Pinned by `TestRunAgentRun_PtyPath_WorkDirIsTrustResolvedRealpath`, which feeds a sentinel realpath through `trustMark` and asserts the captured Config carries it verbatim.

- **Four package-level seams — `trustMark`, `settingsWrite`, `ptyRun`, `newSessionID` — enable deterministic failure-surface coverage without a TestMain dispatcher.** Each is the production-wired function value; tests override via `t.Cleanup` restore-on-exit boilerplate. The alternative (a per-package TestMain that re-execs to drive real or fake claude under PTY) would be a much larger refactor parallel to the existing `TestAgentRunStreamJSONFake` pattern. Production never assigns to these — only `_test.go` files do. Block-commented in `agent_run.go:24-32` so a future contributor adding a fifth seam pauses to question whether the helper is decomposing too aggressively.

- **`buildClaudeArgs` renamed to `buildStreamRunnerClaudeArgs`; body unchanged.** Documents intent: the argv shape with `--input-format stream-json --output-format stream-json --verbose --dangerously-skip-permissions --allowed-tools …` is specific to the streamrunner stream-json subprocess invocation. ptyrunner's argv (`--session-id`, `--settings`, `--permission-mode default`) is owned by `internal/agentrun/ptyrunner/runner.go`'s private `buildArgs` and is intentionally NOT exposed at the cmd layer. The renamed function is referenced exactly twice: production call site in `runAgentRunStreamRunner` and the renamed `TestBuildStreamRunnerClaudeArgs_Shape`.

- **`agentRunUsageDescription` rewritten to describe the PTY default and the env-var rollback.** New required substrings asserted by the extended `TestAgentRunUsageDescription`: `stream-json` (the wire shape both modes emit), `--max-turns` and `--allowed-tools` (behavioural anchors), `PYRY_USE_STREAMJSON` (operator rollback knob), and `PTY` (new-default disclosure). The pre-existing scaffold-only stale-disclaimer check stays as a regression guard.

- **`configureFakeClaude` pins `PYRY_USE_STREAMJSON=1` so the two existing fake-claude wiring tests (`TestRunAgentRun_StreamJSON_Clean`, `TestRunAgentRun_StreamJSON_NonZeroExit`) exercise the streamrunner branch unchanged.** Single edit on the helper covers both consumers — the test logic itself is byte-identical to its pre-cutover shape. The ptyrunner default path is exercised via the package-level seams, not the fake, because the fake is keyed off `--input-format stream-json` argv that ptyrunner does not emit.

- **Thirteen new tests cover the env-var branching, the ptyrunner-path failure surfaces, the settings cleanup, the config wiring, and the allowed-tools round-trip.** `TestRunAgentRun_DispatchesToPtyRunnerByDefault` (env unset → ptyRun called once with all required Config fields populated). `TestRunAgentRun_EnvSet1DispatchesToStreamRunner` (env=`"1"`; ptyRun seam `t.Fatal`s if reached; clean exit via fake-claude). `TestRunAgentRun_EnvNon1ValueDispatchesToPtyRunner` (table over 7 non-`"1"` values, each routes to ptyrunner). `TestRunAgentRun_PtyPath_TrustFailure_MentionsClaudeJson` / `_SettingsFailure_NamesSettingsStep` / `_PtyRunError_Wrapped` (failure-surface pins for AC #3). `TestRunAgentRun_PtyPath_SettingsRemovedOnSuccess` / `_SettingsRemovedOnFailure` (exercise the production `settings.WriteSettings` via capture-wrapped seam, then `os.Stat` for `fs.ErrNotExist`). `TestRunAgentRun_PtyPath_WorkDirIsTrustResolvedRealpath` (sentinel realpath round-trip). `TestRunAgentRun_PtyPath_SessionIDIsUUIDv4` (asserts `sessions.ValidID(captured.SessionID)`). `TestRunAgentRun_PtyPath_ConfigWiring` (table over two fixtures: model `sonnet-4-6` / `opus-4-7`, max-turns `3` / `12`, distinct allowed-tools). `TestRunAgentRun_PtyPath_AllowedToolsPassedToSettings` (asserts the slice handed to `settingsWrite` equals `parsed.allowedTools` byte-for-byte — the deny-default allowlist's load-bearing path).

- **Real-claude smoke gated by `PYRY_E2E_REAL_CLAUDE=1`** (`TestRunAgentRun_RealClaude`, env-gated `t.Skip`). Two subtests share the same prompt + budget: `default_pty_path` runs with the env unset; `fallback_streamrunner_path` sets `PYRY_USE_STREAMJSON=1`. Both assert the stream-json trailer contains `type:"result"` and at least one `system` / `assistant` event. Gate is an env var (not a build tag) so CI's default `go test ./...` skips cleanly while contributors can run `PYRY_E2E_REAL_CLAUDE=1 go test ./cmd/pyry -run TestRunAgentRun_RealClaude` without flipping build flags. Per-PR CI continues to exercise the real-claude byte-equivalence check at the ptyrunner boundary via [#482](https://github.com/pyrycode/pyrycode/issues/482); this smoke covers the cmd→helpers→ptyrunner wiring at a different scope.

## Patterns established

- **Strict-`"1"` env-var predicate for operator-facing rollback knobs.** Boolean-shape switches keyed by `os.Getenv(NAME) == "1"` are unambiguous and grep-able; widening to `"true" | "yes" | "on" | …` is a slippery slope that historically bit `PYRY_USE_LEGACY_CLAUDE`. The pattern carries a sibling: a table-driven test of non-`"1"` strings that pins the predicate against future drift. Mirrors `PYRY_AGENT_RUN_FAKE`, `GO_PTYRUNNER_HELPER`, and now `PYRY_USE_STREAMJSON`.

- **Package-level function-typed seams over a TestMain dispatcher when the cmd layer composes four sibling primitives.** Four unexported `var` assignments at package init (one per dependency) plus per-test `t.Cleanup` restore-on-exit boilerplate is the minimum-surface seam that enables deterministic failure-surface coverage without spawning the wrapped binary. Cheaper than a TestMain re-exec for surfaces this thin; documented in a block comment so a future contributor adding a fifth seam pauses to question the decomposition.

- **`defer os.Remove(settingsPath)` registered AFTER the err-check on `settingsWrite`.** Defers fire even when the wrapped function returned an error, so a defer registered before the err-check would try to remove a path that was never written. The discipline: register the defer immediately AFTER the err-check, on the success path. Trust-write failure returns BEFORE `settingsWrite` is called, so no settings-file leak on that branch either. Pinned by `TestRunAgentRun_PtyPath_SettingsRemovedOnSuccess` / `_SettingsRemovedOnFailure`.

- **Realpath-not-parsed-workdir contract pinned with a sentinel.** When two systems independently resolve a workdir to a key (here: pyry's trust pre-write and claude's projects-dict lookup), make pyry surface the resolution to its downstream caller verbatim rather than re-resolving — a sentinel-path test pins the contract structurally. Generalises whenever a setup primitive's return value is load-bearing for a downstream lookup; the alternative (each consumer re-running `filepath.EvalSymlinks`) couples them to the resolver's algorithm.

## Lessons learned

- **The default in the per-spawn settings cleanup defer must be registered AFTER the err-check on the write.** During architecture review the contract note `defer os.Remove(settingsPath) runs before ptyRun returns because Go's defer LIFO fires at function exit` is correct but incomplete — it doesn't say *where* in the source order to register the defer. The correct shape is: err-check on `settingsWrite` returns FIRST on failure (no path to remove); on the success branch the defer registers immediately and fires on every subsequent exit path. A defer registered BEFORE the err-check would try to `os.Remove("")` on a settings-write failure, which is benign (returns ENOENT) but obscures the bug class for the next contributor reading the code. The successful-write-then-remove invariant is also the easier shape to test — both cleanup tests exercise the production `settings.WriteSettings` via a capture-wrapped seam rather than a string-returning fake, so the path that comes back actually exists at the moment `ptyRun` is invoked.

## Operator rollback procedure

Set `PYRY_USE_STREAMJSON=1` in the dispatcher's `.env` (or in the per-agent service unit). Pyry will dispatch to `streamrunner.Run` for that process's lifetime, byte-equivalent to the pre-cutover behaviour. The dispatcher's stream-json parser is satisfied by either mode — no agents-repo code change required. Unset (or set to anything other than the exact string `"1"`) restores the ptyrunner default at next spawn.
