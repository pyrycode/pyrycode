# Ticket #544 — decouple `TestResolveWorkdir_DarwinRealpath` from `$TMPDIR`

`TestResolveWorkdir_DarwinRealpath` (`internal/agentrun/workdir_test.go`) asserted a hardcoded `/private/var/` prefix on `ResolveWorkdir(t.TempDir())`. That literal only holds under a **default** `$TMPDIR` on macOS, where `t.TempDir()` lands under `/var/folders/...` → `/private/var/folders/...`. In the dispatch sandbox `TMPDIR=/tmp/claude-501`, so `t.TempDir()` resolves to `/private/tmp/claude-501/...` (`/tmp → /private/tmp`) and the prefix check failed. The failure was **pre-existing and not a product bug** — `ResolveWorkdir` was correct; the test asserted an environment-specific *path* instead of the *behaviour* it meant to pin. This ticket rewrites the assertion to derive the expected value from the input at runtime. Test-only: no production code under `internal/agentrun` changed. Surfaced by the QA agent during the out-of-scope-red path of PR #543 ([#542](542.md)).

## Implementation

- **Assert the property, not a fixed literal.** The `const want = "/private/var/"` prefix check is replaced with two runtime-derived assertions:
  - `got == filepath.EvalSymlinks(wd)` — `ResolveWorkdir` returns the symlink-resolved form of its input. This is the same canonical-form property the sibling `TestResolveWorkdir_AlreadyResolved` already asserts. Because `t.TempDir()` is already absolute, `filepath.Abs(wd) == wd`, so `EvalSymlinks(wd)` reproduces `ResolveWorkdir`'s output exactly (no `Abs` mismatch to account for).
  - `got != wd` — resolution actually **fired**. This delta is the darwin-only test's reason to exist: it proves the macOS `/private` symlink hop changed the path, rather than `ResolveWorkdir` being a no-op. It is the only place that asserts resolution *fired*; without it the test would be near-redundant with `TestResolveWorkdir_AlreadyResolved`.
- Both error paths use `t.Fatalf("...(%q): %v", wd, err)` — paths and errors only, never file contents (the `MUST NOT log file contents` note in `workdir.go:6`).
- The `runtime.GOOS != "darwin"` skip is preserved (non-darwin CI still skips); the now-unused `strings` import was removed.
- **`t.Parallel()` intentionally not added.** The pre-existing test omitted it; adding it is out of scope for a minimal diff. Flagged as a non-blocking NIT in code review (its three siblings carry `t.Parallel()`), left off deliberately per the spec.

The delta holds in both target environments: default `$TMPDIR` crosses `/var → /private/var`; the dispatch sandbox crosses `/tmp → /private/tmp`. Verified with `go test -race -run TestResolveWorkdir ./internal/agentrun/` under default `$TMPDIR`, under `TMPDIR="$(mktemp -d /tmp/claude-XXXX)"` (reproduces the sandbox condition), plus the full `go test -race ./internal/agentrun/` + `go vet` gate.

## Patterns established

- **A test for a canonicalisation behaviour must assert the canonical-form *property* derived from its input, never a hardcoded environment-specific path.** `t.TempDir()` lands wherever `$TMPDIR` points; any literal prefix assertion is implicitly coupled to one `$TMPDIR` value and breaks the moment the environment differs (here: the dispatch sandbox). Derive `want` from the input at runtime (`EvalSymlinks(wd)`) so the test pins *what the function does*, not *where this machine puts temp files*. When such a test also needs to prove the transform was non-trivial (not a no-op), pair the property assertion (`got == f(input)`) with a delta assertion (`got != input`) — the delta is what gives an otherwise-redundant test its distinct purpose.

## Lessons learned

- **A green test on the default macOS box can be silently coupled to `$TMPDIR`.** This is the same `docs/lessons.md` class — *"two sources of paths, one canonical and one not"* (rotation watcher [#118](https://github.com/pyrycode/pyrycode/issues/118)/#221, `lessons.md:219-223`): `t.TempDir()` exercises the macOS `/var → /private/var` (or `/tmp → /private/tmp`) default symlink. The original test happened to pass on every default-`$TMPDIR` dev box, so the coupling stayed invisible until the dispatch sandbox set `TMPDIR=/tmp/claude-501`. Generalises: a hardcoded `/private/var/` (or any temp-dir-rooted) literal in a test assertion is a latent environment dependency — audit `t.TempDir()`-based assertions for fixed path prefixes the same way you audit cross-source path comparisons for the canonical/non-canonical mismatch.
- **Don't guard a failure mode you haven't observed.** A pathological already-canonical `TMPDIR` (e.g. `/private/...`) would make the `got != wd` delta a genuine no-op and fail the test. Neither default macOS nor the dispatch sandbox produces it, so per the pipeline's evidence-based-fix principle no skip/guard was added — the case is documented in the spec only so a future failure under such a `TMPDIR` is immediately diagnosable rather than mysterious.
