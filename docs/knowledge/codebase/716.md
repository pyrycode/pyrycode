# Ticket #716 — surface a tui-driver modal to interactive phones (`modal_shown` + outstanding-modal registry)

The **outbound half** of the daemon-side modal bridge ([EPIC #597] Phase 3,
[ADR 025] no-raw-bytes invariant). When tui-driver detects a permission/trust modal
on claude's screen, the daemon turns it into a typed `modal_shown` event and pushes it
to interactive phones — never raw PTY bytes. This slice (split from [#703]) establishes
the **outstanding-modal registry**, keyed by a one-time `modal_id` nonce, that [#717]'s
inbound resolution half consumes to route answers back and reject stale/replayed
answers. Shipped as a self-contained, unit-tested component; live `Session.Events()`
wiring is deferred to [#708].

## Implementation

Two additive production files in two packages — **zero existing production files
touched** (additive; reuses the existing `interactiveBroadcaster` interface verbatim).

| File | Change |
|---|---|
| `internal/modalbridge/modal.go` | **NEW relay-free modal domain.** `Registry` (`sync.Mutex` + `map[string]Outstanding`) with `New`/`Record`/`Lookup`/`Resolve`; `Outstanding` entry; `PermissionRequestForClass` (class → internal `turnevent.PermissionRequest` + wire class string + `ok`); `buildPayload`/`boundPrompt`; `newModalID` (`crypto/rand` UUIDv4). `Record` is the single nonce-mint site — mints once, stores the `Outstanding`, returns the id-stamped `protocol.ModalShownPayload`. |
| `cmd/pyry/interactive_modal_v2.go` | **NEW surfacer** `interactiveModalEmitterV2` — passive state machine (no goroutine, no queue), mirroring `interactiveTurnEmitterV2`. `Handle(ctx, ev, screenText)`: gate on `EventKindPtyModalShown` + permission/trust class → `Record` → marshal once → capability-gated fan-out (`Interactive` conns only, per-conn monotonic `env.ID`, `EventID` nil). Modal body never logged. |

Class mapping (the **minimal fixed-option-set**, design option (a) — *not* screen-scraping;
tui-driver v1.3.0 has no permission/trust option extractor and `EventKindPtyModalShown`
carries only the `ModalClass`):

- `ModalClassPermission` → wire `"permission"`, options `allow_once`/`allow_always`/`reject_once`/`reject_always` (ids = `turnevent.PermissionOptionKind` strings), default `reject_once`.
- `ModalClassTrustFolder` → wire `"trust"`, options `proceed`/`exit`, default `exit`.
- every other class → `ok=false` ⇒ no `modal_shown` (AC1).

Evergreen docs: new [`features/modalbridge-package.md`](../features/modalbridge-package.md)
(full package + surfacer surface); [`protocol-mobile.md` § Modal](../../protocol-mobile.md)
updated to name the first concrete `class` values (`permission`/`trust`) and the
fail-safe-deny `default_option_id` convention. Spec:
[`specs/architecture/716-modal-surface-producer.md`](../../specs/architecture/716-modal-surface-producer.md).

## Patterns established

- **Relay-free domain package to break a cycle with a future consumer.** [#717] will
  import this registry from inside `internal/relay` (it intercepts `modal_answer` at
  `dispatchAppFrame`), so `internal/modalbridge` must **not** import `internal/relay`.
  The package imports only `internal/protocol`, `internal/turnevent`, and the **typed**
  `pkg/tuidriver` `ModalClass` API — no raw-screen substrate literal enters, so
  `cmd/substrate-guard` stays green. The `relay.ActiveConn` fan-out stays in `cmd/pyry`
  (`package main` imports both), the same split as `interactiveTurnEmitterV2`.
- **Fail-safe deny default, decoupled from display order.** `default_option_id` is the
  **deny** option (`reject_once`/`exit`), not `options[0]`, so a careless confirm on a
  *remote* surface denies rather than allows. `options` stays in claude's display order
  (allow-first); display order and the highlighted default are deliberately separate,
  keeping `default_option_id ∈ options[].id` true by construction. UI pre-selection only
  — answering is gated ([#702]) and deny-on-timeout is [#717]'s. (Security-review MUST-FIX,
  applied inline.)
- **Single mint site = single-writer nonce.** `modal_id` (`crypto/rand` UUIDv4, 122 bits,
  *not* `math/rand`) is minted **only** in `Record`, called **only** from the surfacer's
  single goroutine, **once** per `EventKindPtyModalShown` (rising-edge ⇒ one event = one
  modal = one mint). `Record` mints **and** records atomically under the mutex — no
  `modal_shown` without a registry entry, no entry without a minted id.
- **Registry mutex = deterministic safety net, different fabric.** The registry is the one
  piece two *real* goroutines touch (surfacer `Record` + [#717]'s dispatch `Lookup`/`Resolve`),
  so it carries a leaf `sync.Mutex` — not goroutine-confinement-by-convention. Single-writer-
  nonce alone doesn't make the map read-safe against #717's concurrent reads, so the safety
  net is real deterministic code, not another convention.
- **`EventID == nil` marks a control event.** `modal_shown` is not a turn-event replay-ring
  delta, so its envelope leaves `EventID` nil — verified against `forwardEnvelope`, which
  never drops `EventID==nil` envelopes, so delivery is real (it is also therefore an
  **un-droppable control frame**, which is why `Prompt` carries a defensive
  `maxPromptBytes`=4096 rune-boundary bound).
- **Shipped unwired (component + deferred wiring).** Producer + registry + mapping land with
  a unit test through a fake push surface; the live follow-active `Session.Events()`
  subscription + bound-session `ScreenSnapshot()` is deferred to [#708] — the
  [#632 emitter → #633 wiring] precedent. Named seam: an `OnModal` callback on
  `turnbridge.Config` fired from `Producer.drain`.

## Lessons learned

- `internal/modalbridge` is deliberately **relay-free** so #717's relay-side resolver can
  import it without a cycle; the `relay.ActiveConn` fan-out stays in `cmd/pyry`. The
  registry's `sync.Mutex` is a **deterministic safety net** (two real goroutines: surfacer
  `Record`, #717 dispatch `Lookup`/`Resolve`), **not** goroutine-confinement —
  single-writer-nonce alone doesn't make the map read-safe. *(Lifted from the PR's Lessons
  learned.)*
- **Per-class knowledge lives at three sites keyed by the wire-class string** (the
  `PermissionRequestForClass` switch, `titleByClass`, `denyByClass`). Current code is clean
  and `TestRecord_PayloadInvariant` guards the two real classes, but a future class added to
  one site and not the others could silently break the `default_option_id ∈ options[].id`
  security invariant. Code-review NIT, left as-is ("Simplicity first"); a single per-class
  descriptor (options + title + fail-safe default together) is the refactor *if a third
  class ever appears*.
- `newModalID` duplicates `conversations.NewID` byte-for-byte (only the return type differs).
  This is intentional — keeping `modal_id` decoupled from the `conversations` domain is the
  right call; the spec sanctioned mirroring it. Extract a shared UUIDv4 minter only if a
  **third** minter appears.
- **Incidental:** `internal/e2e/realclaude/testdata/permission_protocol_v2.1.158.json` was
  re-captured by the real-claude e2e run (`session_id`/`cwd`/`uuid` churn + a dropped
  `rate_limit_event`). It is a golden-file capture, unrelated to this feature — expect this
  fixture to show up in unrelated diffs whenever the realclaude suite runs.

## Out of scope (siblings)

- **Inbound resolution — [#717]** (blocked-by #716). Intercepts `modal_answer`, validates
  the phone-asserted `modal_id` against the registry (`Lookup`/`Resolve`, defined here),
  routes the answer back into claude, rejects stale/replayed answers, owns deny-on-timeout
  + registry removal (bounding unbounded growth). The untrusted inbound boundary is entirely
  #717's.
- **Live daemon wiring — [#708]** (capstone, or a thin wiring slice). Subscribes the surfacer
  to the follow-active `Session.Events()` stream, resolves the bound session's
  `ScreenSnapshot()` as `screenText`, and runs the two-phone e2e. Mirrors #633.
- **Answer gate — [#702]** (merged). The per-device authorization bit that gates *answering*
  a modal; this slice only *surfaces* (viewing), takes no allow/deny decision.
