# Ticket #638 ‚Äî Stall vocabulary: internal `turnevent.Stall` + v2 mobile `stall` wire type

Adds the **data vocabulary** for a "stall" in two decoupled layers ‚Äî the neutral
internal turn-event model ([#606]) and the v2 mobile wire ([#607]) ‚Äî with **no
bridge wiring**. Pure additive value types: the slice compiles and tests
standalone, no consumer needed. `tui-driver` raises a one-shot `stall_detected`
signal (`EventKindStallDetected`, shipped in tui-driver v1.3.0) and the mobile UI
already knows how to surface a stall ([#373]), but the signal had nowhere to live
‚Äî [#606] explicitly deferred the `Stall` variant and [#607] added only the five
ACP-shaped turn events. **Part of EPIC [#596]** (Phase 2 structured streaming),
split from [#624] (child A of 2). The sibling bridge slice ([#624] child B ‚Äî
which the package doc comments label [#608], the established consumer-ticket name
in this code) depends on this one and carries the `security-sensitive` label;
**this slice does not** ‚Äî no trust boundary, no inbound parsing, no fan-out.

**Onset-only ‚Äî no recovery edge.** The producer is a one-shot rising-edge marker
with no payload and no clearing event, so `Stall` models the stall **onset**
only: there is no "cleared" state because there is no producer for one. The phone
self-clears on the next turn activity.

## Implementation

Two layers mirroring the two their five-event predecessors already occupy. The
internal layer carries **no** conversation identity (consistent with every
`turnevent` variant ‚Äî the bridge injects `conversation_id` at wire-mapping time);
the wire layer carries `conversation_id` only.

**Layer 1 ‚Äî internal neutral model (`internal/turnevent/event.go`).** `Stall` is
a new **empty marker struct** added to the sealed `Event` sum:

```go
type Stall struct{}        // onset marker; no fields, no clearing state
func (Stall) isTurnEvent() {}
var _ Event = Stall{}      // added to the existing assertion block
```

- The empty struct is deliberate: the producer's `stall_detected` carries no
  payload, and ‚Äî like `TextChunk`, `TurnEnd`, etc. ‚Äî the internal variant carries
  no `conversation_id`. Onset-only ‚áí no fields.
- `Stall` stays a first-class member of the same `Event` sum. The sum is the
  stable internal contract; per-variant emit/drop decisions belong to the
  **adapters**. The mobile adapter **sends** `Stall`; the future ACP adapter
  ([#600]) **drops** it (no ACP equivalent) ‚Äî exactly the internal-only asymmetry
  [#606]'s package doc anticipated.
- Two doc edits in `event.go`: the package doc's "out of scope ‚Ä¶ get a home in a
  later ticket" line **drops `Stall`** (keeps `BusyState`) and notes `Stall` now
  lives here (mobile-sent, ACP-dropped); the `Event` interface doc enumeration is
  extended with `Stall`, annotated internal-only.
- Optional intent-documentation (not an AC): `event_test.go`'s `eventKind`
  type-switch and the `TestEvent_StreamTypeSwitch` stream gained a `Stall` case so
  the switch stays exhaustive-by-intent. The AC is satisfied at compile time by
  the marker + `var _ Event = Stall{}` assertion alone.

**Layer 2 ‚Äî v2 mobile wire (`internal/protocol`).**

- `codes.go` ‚Äî `TypeStall = "stall"` appended at the end of the v2-only
  interactive partition const block (after `TypeTurnEnd`); the block's trailing
  count comment bumped "these **five** ‚Ä¶ ‚Üí **six**".
- `interactive.go` ‚Äî `StallPayload{ConversationID string json:"conversation_id"}`,
  the peer of `TurnStatePayload` minus the `state` field. **No `omitempty`** (the
  file invariant). Doc comment records: wire form of the internal-only
  `turnevent.Stall`; conversation identity only; onset-only (no clearing field);
  **no `turn_id`** ‚Äî like `turn_state`, a stall is a coarse conversation-level
  signal, not turn-scoped; the bridge supplies `ConversationID` because the
  internal `Stall` marker carries none.
- `testdata/stall.json` ‚Äî full-shape single-line fixture
  (`{"id":306,"type":"stall","ts":"2026-05-08T10:33:15Z","payload":{"conversation_id":"c1"}}`).
- `interactive_test.go` ‚Äî `TestStallPayload_RoundTrip` mirrors
  `TestTurnStatePayload_RoundTrip`: decode the envelope, assert `Type ==
  TypeStall`, decode the payload, assert `ConversationID == "c1"`, then
  `roundTripEnvelope` re-marshals the **decoded struct** byte-equal to the fixture
  (this is what pins the json tag and the no-`omitempty` shape).
- `compat_test.go` ‚Äî the three-spot v1/v2 partition lockstep: a `stall-rejected`
  `ErrUnknownType` row in `TestIsV1Compatible`, `TypeStall: true` in
  `v2OnlyTypes`, and `TypeStall` added to `TestTypeConstants_V1V2Partition`'s `all`
  list. The union-count assertion rebalances automatically.
  `TestV1TypeSet_CoversAllExportedTypeConstants` (`len == 16`) is correctly left
  **unchanged** ‚Äî `stall` is v2-only and must never enter `v1TypeSet`.

**Doc (`docs/protocol-mobile.md`).** A `stall` row in the application-message-types
table; the "Interactive events (v2, capability-gated)" intro bumped five ‚Üí six; a
`#### stall` one-field subsection noting it is the wire form of an internal-only
signal (no ACP equivalent) and, like `turn_state`, carries no `turn_id`.

`internal/protocol` does **not** import `internal/turnevent` ‚Äî the two layers stay
decoupled, bridged only by [#624]-B at the string value `"stall"`.

## Patterns established

- **Adding a variant to the sealed `Event` sum is non-breaking by construction
  here.** The two existing exhaustive-ish type-switches over `Event` both
  degrade gracefully: `turnbridge.MapEvent` (`internal/turnbridge/outbound.go`)
  has a `default: return "", nil, false` that drops any unknown/nil event, and
  `cmd/pyry`'s `interactive_turn_v2.go` `Handle` (+ its private `eventKind`)
  debug-logs-and-drops on `default`. So `Stall` is silently **dropped** ‚Äî not
  mishandled ‚Äî until [#624]-B wires the `case turnevent.Stall` arms, exactly as
  this slice's "compiles and tests standalone, no consumer" scoping intends.
  Correctly **not** touching `cmd/pyry`'s emitter here ‚Äî that is bridge-wiring
  scope. (Confirmed in code review's blast-radius check.)
- **An internal-only event is still a first-class member of the neutral sum; the
  emit/drop asymmetry lives in the adapters.** `Stall` is `sent` by the mobile
  adapter and `dropped` by the ACP adapter ‚Äî the same containment idiom [#606]
  established for the model as a whole. Don't gate the variant out of the sum;
  gate it per-adapter.
- **Reuse the count-drift checklist when growing a parallel-list family.** "Five
  ‚Üí six" has to move in lockstep across `codes.go`'s partition comment,
  `protocol-mobile.md`'s intro, and the three `compat_test.go` lists; the
  partition union-count and `TestV1TypeSet_‚Ä¶` (`len == 16`) assertions are the
  deterministic rails that fail loudly if a spot is missed. Uniform **append at
  the end** of every parallel structure keeps the edit mechanical.

## Lessons learned

- **A fixture timestamp with a trailing-zero fractional second breaks the
  re-marshal round-trip.** `"‚Ä¶:15.0Z"` fails `TestStallPayload_RoundTrip`: Go's
  `time.Time` JSON encoder (RFC3339Nano) **drops** the trailing-zero fraction, so
  the decoded-struct re-marshal diverges from the fixture bytes. Use the canonical
  no-fraction form (`"‚Ä¶:15Z"`, as `stall.json` does) ‚Äî or a genuinely non-zero
  fraction (`.5Z`) when sub-second ordering matters. This is the same
  `time.Time`-RFC3339Nano gotcha that already governs the envelope `ts` and the
  `screen_snapshot` fixture ([#617]); it bites every new full-shape fixture that
  carries a `ts`. (From the PR body's Lessons-learned section.)

## Scope (S kept)

Three production `.go` files, ~60 LOC of written production work, two new exported
symbols (`Stall` type + `StallPayload` type) plus one exported constant
(`TypeStall`), zero consumer cascade. The S‚ÜíXS hedge in the ticket body was
declined at the architect stage: the production diff is trivial, but the wire
ceremony (full-shape fixture + round-trip test + three-spot compat-partition
lockstep + doc in two places + the count-drift comment bumps) spans seven files,
which is more than an XS. The `security-sensitive` label does not apply (pure
wire/data vocabulary) and does not change dispatch, so S is the honest call.
Code-review decision: **PASS**, no MUST/SHOULD/NIT findings.

## Out of scope (the bridge ‚Äî [#624]-B / [#608])

- **`stall_detected` ‚Üí `Stall` ‚Üí `stall` wiring.** The producer drain that emits a
  `turnevent.Stall` on the rising edge, the outbound adapter arm that maps it to a
  `StallPayload`, the `conversation_id` injection, and the capability-gated push to
  the phone all live in the sibling bridge slice ([#624] child B). That slice
  carries `security-sensitive`; this one does not.
- **A "cleared"/recovery state.** No producer exists for one ‚Äî do not
  speculatively add it (evidence-based design). If a future producer ever attaches
  onset detail or a clearing edge, that is a new ticket.
