# Ticket #727 — inbound modal-control interception + `ModalResolver` seam + `modal_dismissed` broadcast (resolve `modal_cancel`)

The **seam-introduction** slice of the daemon-side modal bridge ([EPIC #597]
Phase 3, [ADR 025] § Security model). The outbound half already surfaces a modal
to phones (`modal_shown` + the outstanding-modal registry, #716); the **inbound**
half — a phone telling the daemon to *resolve* that modal — starts here. This
slice lands the two net-new relay seams that #717 (gated `modal_answer`) and #725
(deny-on-timeout) both consume, and proves them end-to-end via the **simplest
terminal decision**, `modal_cancel` (dismiss = fail-safe deny). `modal_answer` is
intercepted and routed through the same seam **but resolves to a deferred no-op**:
no keystroke, no modal mutation, no dismissal, no audit — the escalating ALLOW
path stays inert until the per-device gate (#702/#717) exists, the central
fail-safe property of this slice.

## Implementation

Three production files (≈157 relay + 103 cmd/pyry LOC), additive — the new
`V2SessionConfig.ModalResolver` is an optional nil-safe field, so **no consumer
cascade** (every existing `V2SessionConfig{…}` keeps compiling and behaves
identically: modal frames inert). That nil-safety is why the slice stayed **S**.

| File | Change |
|---|---|
| `internal/relay/v2session.go` | **The two seams + interception.** `ModalDismissal` (`{Outcome, Source}` — the wire outcome/source the manager broadcasts; `modal_id` is already held) + the consumer-declared **`ModalResolver`** interface (`ResolveCancel` / `ResolveAnswer`), both declared beside `ScreenSnapshotter` so `internal/relay` imports neither `internal/supervisor` nor `cmd/pyry`. New optional `V2SessionConfig.ModalResolver` field (nil ⇒ inert). Two new arms in `dispatchAppFrame`'s control discriminator (`TypeModalCancel`→`handleModalCancel`, `TypeModalAnswer`→`handleModalAnswer`), modeled on `handleRequestSnapshot`. New manager-internal `broadcastModalDismissed`. |
| `cmd/pyry/modal_resolve_v2.go` (NEW) | `modalResolverV2` — the composition-root implementation: `ResolveCancel` (registry `Resolve` → ESC → audit) + `ResolveAnswer` (deferred no-op). The `modalKeystroker` interface (`SendEsc() error`, satisfied by `*supervisor.Supervisor`) keeps the resolver decoupled from the supervisor concrete type; #717 extends it with `Answer`/`AcceptTrust`. |
| `cmd/pyry/relay.go` | Wiring in `startRelayV2`: construct the **daemon-singleton** `modalReg := modalbridge.New()` and pass `ModalResolver: newModalResolverV2(modalReg, sup, logger)` into the `V2SessionConfig` literal (`sup` satisfies `modalKeystroker`). `startRelayV2`'s signature is unchanged (the registry is a local). |

**Dependency direction** (the seam's whole point): `internal/relay` declares
`ModalResolver` at the consumer (CODING-STYLE "define interfaces where
consumed"); `cmd/pyry` supplies the impl (`internal/{modalbridge,supervisor,
audit,devices}`). The relay never learns about the registry, tui-driver, or the
audit sink.

### Inbound flow (`modal_cancel`)

`dispatchAppFrame` (Run goroutine) → `handleModalCancel` → `ResolveCancel(modal_id,
s.device)`:
1. `Registry.Resolve(modal_id)` — the **single atomic consume-and-retire**
   idempotency gate. Unknown / already-resolved ⇒ `(zero, false)` before any
   side effect (AC-4).
2. `supervisor.SendEsc()` — **best-effort** ESC (the constant fail-safe dismiss,
   no user-controlled bytes). A keystroke error (no live session / teardown) is
   `Warn`-logged and **tolerated**: the modal is already consumed and moot, so
   the phone must still learn the dismissal and the forensic record must still
   exist (aborting would orphan the consumed modal — an unobserved-mode defense).
3. `audit.Log({…, OutcomeCancelled, SourceRemote})` — one record per resolved
   cancel; **one source vocabulary** feeds both the wire dismissal and the audit
   entry (`audit.go`'s documented contract).
4. return `({cancelled, remote}, true)` ⇒ `broadcastModalDismissed` fans
   `modal_dismissed{modal_id, cancelled, remote}` to **every** interactive-capable
   open conn. `modal_answer` follows the same path into `ResolveAnswer`, which
   returns `(zero, false)`, so the flow stops at the resolver.

## Patterns established

- **Consumer-declared resolver seam mirrors `ScreenSnapshotter` (#618).** A second
  instance of the same precedent: the relay declares the one-purpose interface it
  consumes, the composition root implements it, and an **optional nil-safe
  `V2SessionConfig` field** (nil ⇒ inert no-op) keeps the field additive with no
  cascade. This is what holds the slice at S — the same shape that let #618 add
  `Snapshotter`/`KnownConversation` without touching ~42 construction sites.
- **The broadcast MUST read `m.sessions` directly, NEVER `ActiveConns` — the
  load-bearing design fact.** The broadcast fires from inside `dispatchAppFrame`,
  on the manager's single `Run` goroutine. The public `ActiveConns` funnels its
  request *back* onto that same goroutine via `m.snapshot`, so calling it from the
  interception would **deadlock**. `broadcastModalDismissed` therefore enumerates
  `m.sessions` directly (the `handleActiveConns` pattern) and `Push`es per conn.
  `Push` is `Run`-goroutine-safe — it touches only `m.queues` under `pushMu` and
  returns immediately (the seal+forward happens on a later `Run` iteration via
  `drainOnce`). The fan-out test drives the cancel through the **real `Frames`/`Run`
  loop**, so an accidental `ActiveConns` call would hang the test — a *structural*
  no-deadlock proof, not an assertion.
- **The seam is introduced *dead* on the escalating arm — fail-safe by
  construction.** `ResolveAnswer` is a deferred no-op (always `ok=false`), and it
  does **not** call `Resolve` (no mutation), routes no keystroke, writes no audit.
  The `handleModalAnswer` broadcast line is reachable-by-type but unreachable in
  this slice — present so #717 needs **no** manager change (it replaces only the
  resolver impl body + extends `modalKeystroker`). The escalating ALLOW path
  cannot bite until the gate exists.
- **The registry consume is the *single* idempotency gate.** `Registry.Resolve`'s
  atomic consume-and-retire (its own mutex + map delete) makes the *first* cancel
  win and every replay/stale/unknown id a `(zero,false)` no-op — no second
  keystroke, no second audit, no second broadcast (AC-4). Inbound frames are
  serialized on the `Run` goroutine, so two cancels for the same id can't race;
  the registry mutex is the belt-and-suspenders of different fabric.
- **One timestamp, one source value, fan-out by capability.** `broadcastModalDismissed`
  shares one `time.Now().UTC()` across the logical dismissal; the `s.interactive`
  filter is the same #607 capability gate `modal_shown` rides (an old phone never
  receives v2 modal events). The fan-out reaches *every* interactive conn (incl.
  ones that never saw this modal's `modal_shown`); the payload carries only the
  opaque `modal_id` + `cancelled`/`remote` — no modal body — so a conn with no
  matching outstanding modal simply ignores it (discloses nothing).
- **Envelope `ID: 1` is non-load-bearing on a manager-internal push.** Matches
  `handleRequestSnapshot` / `emitRekeyRequest`: the phone correlates `modal_dismissed`
  on `modal_id`, and a `modal_dismissed` is a control event (`EventID == nil`,
  never in the #647 replay ring). Deliberately avoids adding a per-session
  outbound counter to `V2Session` (Simplicity First); the producer-side
  `interactiveModalEmitterV2.nextID` counter is a separate producer-owned concern,
  not reused here.
- **No new sentinel error.** `ResolveCancel`/`ResolveAnswer` signal "nothing to do"
  with the `(zero, false)` tuple, mirroring `Snapshotter.ScreenSnapshot`'s
  `(text, live)` shape — not an error.

## Lessons learned

- **The `internal/relay` v2 test harness was single-connection.** `driveToOpen` /
  `sealAppFrame` hardcoded `v2TestConnID`, but the dismiss fan-out needed ≥2
  interactive heads on **one** manager. The fix is a conn-aware handshake helper
  (`openModalConn`) that drives each device's IK handshake on its own `connID` and
  recovers that conn's `noise_resp` from the **shared** `v2Recorder` by filtering
  on `ConnID` (`findNoiseRespForConn`) — `waitConnOpen` guarantees the resp was
  forwarded-and-recorded before the conn is enumerable, so the scan is race-free.
  Driving the cancel through the real `Frames`/`Run` loop is *itself* the
  no-deadlock proof for `broadcastModalDismissed` (an accidental `ActiveConns`
  call would hang the test). *(Lifted from the PR's Lessons-learned note.)*
- **Incidental diff noise:** the merged branch also re-captured
  `internal/e2e/realclaude/testdata/permission_protocol_v2.1.158.json` (new random
  `session_id`/`uuid`/timestamps/tmp paths from a real-claude e2e run, auto-committed
  by the code-review step). It is a regenerated golden capture, unrelated to the
  modal feature — the same fixture churns on #716/#726 and any realclaude run.

## Out of scope (siblings)

- **Gated `modal_answer` resolution — [#717]** (blocked-by #727). Replaces
  `ResolveAnswer`'s body: validate `option_id` against the recorded
  `Outstanding.Options`, pass the per-device gate (#702), route `Answer`/`AcceptTrust`,
  audit, return a real `ModalDismissal`. Extends `modalKeystroker` with
  `Answer`/`AcceptTrust`. The relay manager code (`handleModalAnswer` +
  `broadcastModalDismissed`) needs **no** change — that is the proven seam this
  slice delivers.
- **Deny-on-timeout — [#725]**. Reuses `broadcastModalDismissed` + the audit path
  with `{denied_timeout, timeout}`, driven by the daemon's own timer rather than
  an inbound frame.
- **Live producer wiring — [#708]**. This slice constructs the singleton
  `modalReg` but wires only the *consumer*. Nothing `Record`s into it in
  production until #708 live-wires `interactiveModalEmitterV2` to the same
  instance, so until then **every production `modal_cancel` takes the unknown-id
  no-op path** (harmless — the intended seam-first phased rollout).
- **Cancel is intentionally ungated** (design decision, not a gap). Cancel is the
  fail-safe terminal action (dismiss = deny; worst case is a nuisance dismissal,
  never escalation), so it sits **outside** the #702 answer gate. If a future
  ticket decides cancel must also gate, it slots into `handleModalCancel` the same
  way #717's gate slots into the answer arm.

## Testing

The seam boundary splits the AC-5 assertions across two files (the relay can't
import supervisor/audit/registry; `cmd/pyry` is the composition root).

- `internal/relay/v2session_modal_test.go` (NEW, fake `ModalResolver`): **fan-out
  to ≥2 interactive heads** (`TestV2Session_ModalCancel_FanOut` — three heads, two
  interactive + one not; asserts `ResolveCancel` routed once with the right
  `modal_id` + per-conn device, `modal_dismissed{cancelled,remote}` decrypted at
  **both** interactive heads, **zero** at the non-interactive one — AC-1/AC-2);
  `modal_answer` no-op (routed through the seam, no dismissal — AC-3); unknown-id
  no-op (AC-4); nil-resolver inert (both frames debug-log, no outbound, session
  stays `V2StateOpen`).
- `cmd/pyry/modal_resolve_v2_test.go` (NEW, fake `modalKeystroker` + real
  `modalbridge.Registry` + JSON `slog`): cancel happy path (one ESC, modal
  consumed, one audit `{cancelled,remote}` with the non-secret identity fields,
  **no modal body in any log field** — AC-1/AC-5); unknown id + already-resolved
  (no keystroke, no audit — AC-4); keystroke error tolerated (modal still consumed,
  `Warn` with non-secret err, audit + `(…,true)` still happen); answer no-op
  (no keystroke, no audit, modal **not mutated** — still `Lookup`-able — AC-3/AC-5).
