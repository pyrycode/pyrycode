# Ticket #479 — agent-run: ptyrunner pyry-side max-turns budget + watchdog with shared ctx-cancel teardown

Layers the two pyry-side safety nets on top of [#478](https://github.com/pyrycode/pyrycode/issues/478)'s JSONL tail + stream-json emit: the `internal/agentrun/budget` Counter for the assistant-entry cap (interactive claude does not honor `--max-turns`), and the `tui-driver` two-arm Tracker (PTY-heartbeat + spinner-freeze) for stuck-claude detection. Both arms cancel a single shared `runCtx` and propagate the right exit-reason through `streamjson.Emitter`, so the dispatcher receives the same `result` trailer shape `streamrunner` emits today — `subtype:"error_max_turns"` on budget hit, `subtype:"error_during_execution"` on watchdog fire. Sub-issue of [#329](https://github.com/pyrycode/pyrycode/issues/329) (tracking). Siblings: [#469](https://github.com/pyrycode/pyrycode/issues/469) (trust + settings helpers), [#475](https://github.com/pyrycode/pyrycode/issues/475) (trust pre-write follow-up), [#482](https://github.com/pyrycode/pyrycode/issues/482) (real-claude byte-equivalence smoke test, blocked on this), [#470](https://github.com/pyrycode/pyrycode/issues/470) (`cmd/pyry/agent_run.go` cutover, blocked on this + #482 + #469). Scope stays surgical: two production files (`runner.go` extended, `watchdog.go` created), test extensions, zero edits in `cmd/pyry/`, `internal/dispatch/`, or anywhere downstream of `streamjson.Emitter`.

## Implementation

- **`Config.MaxTurns` flips from forward-compat-only to required; two new optional watchdog-tuning fields land.** The field comment now names the budget Counter as the load-bearing enforcement point and notes that `--max-turns` is intentionally absent from `buildArgs` (interactive claude doesn't honor it). The `cfg.MaxTurns <= 0` check joins the existing validation chain at the top of `Run`, between `Stderr` and the logger resolution. Two new optional fields slot after `Env`, before `Logger`: `WatchdogTick time.Duration` (zero → `defaultWatchdogTick = 1 * time.Second`, matching the tui-driver spike binaries; tests use 50ms) and `WatchdogTrackerOpts tuidriver.TrackerOpts` (forwarded verbatim to `tuidriver.NewTracker`; zero values pick the tuidriver-package defaults of `PTYQuietLimit = 30s` / `SpinnerFreezeLimit = 30s`; tests use ~200ms to fire the watchdog within the test deadline). The wrapped-not-embedded shape (`tuidriver.TrackerOpts` as a typed field) keeps a single named slot for future ptyrunner-specific watchdog tuning.

- **`Run` constructs a single shared `runCtx` derived from the caller's `ctx`.** `runCtx, cancel := context.WithCancel(ctx)` slots immediately after `WritePrompt` and before the emitter / budget / tail / watchdog construction. Both the budget Terminate hook AND the watchdog goroutine call `cancel()` on their respective trigger; the tail watcher's `Run(runCtx)` returns `ctx.Err()` once either fires. Parent ctx cancellation (operator shutdown) propagates to runCtx because runCtx is a child of ctx, so the post-Run-return `runCtx.Err() != nil` collapse check covers all three operator-shutdown-equivalent cases (parent cancel, budget hit, watchdog fire) in one branch. **This is the explicit AC choice — single shared ctx with a single cancel point, NOT separate teardown ordering** — rationale: tail watcher and watchdog goroutine observe disjoint signals (file events vs PTY state) but compose into one "did this turn end cleanly?" question; wiring both to one cancel keeps the answer atomic and avoids a fragile first-to-fire-wins race protocol across two cancel funcs.

- **Budget Counter Terminate closure has four idempotent side effects in fixed order.** Constructed via `budget.New(budget.Config{MaxTurns, Terminate, Kill, Logger})`; `budget.New` failures wrap as `fmt.Errorf("ptyrunner: budget: %w", err)`. Terminate body (fires from the tail watcher goroutine that invoked `counter.OnEvent`):
  1. `emitter.SetExitReason(streamjson.ExitReasonMaxTurns)` — first SetExitReason wins, so a concurrent watchdog fire that lost the race still observes the `max_turns` classification at trailer-composition time.
  2. `tracker.RecordTransition("budget-hit")` — stamps a useful last-state string into any subsequent watchdog error (if PTY stays quiet after budget-SIGTERM and the watchdog fires before the watchdog goroutine exits via runCtx-cancel).
  3. `cancel()` — signals runCtx; the tail watcher's `Run(runCtx)` returns `ctx.Err()` on its next select iteration; the watchdog goroutine exits its select loop.
  4. `cmd.Process.Signal(syscall.SIGTERM)` — returned to the budget package as the Terminate hook's error so the budget package logs terminate-failure cases. Without this, the budget's "I'm done at MaxTurns" signal would reach the emitter trailer but interactive claude would keep generating tokens until `sess.Close`'s own SIGTERM at defer time.

  Kill closure (one-liner `cmd.Process.Signal(syscall.SIGKILL)`) fires from the budget's 5s grace timer if SIGTERM did not produce an exit. `defer counter.Stop()` cancels this timer on the normal exit path so SIGKILL is not double-fired against an already-dead process.

- **`OnEvent` closure body grows by one line; `OnEndOfTurn` flips from no-op to one-line.** The Emit-error capture from #478 is unchanged (`if err := emitter.Emit(ev); err != nil && emitErr == nil { emitErr = err }`); the new `counter.OnEvent(ev)` call appends. Order matters — Emit first so the trailer reflects what was actually written; counter second so the budget hit fires AFTER the budget-trigger event was emitted. The `OnEndOfTurn` closure body is `counter.OnEndOfTurn()` — informational only, since the emitter's default classification already handles the success-trailer composition (#478 already covers that path); `counter.OnEndOfTurn` just records natural completion in the budget so `counter.Reason()` returns `ReasonCompletion` for any future caller.

- **`watchdog.go` is a new self-contained file (~88 LOC) with one regex, one parser helper, one goroutine entry.** `spinnerSecondsRe` matches the class-A spinner rendering (`✻ Verb for Ns` or `✻ Verb for Nm Ks`) and captures the minutes/seconds components; class B (`✻ Channeling…`) and class C (`✻ Actualizing… (2s · ↓N tokens)`) do not match, so the watchdog falls back to the PTY-quiet arm only on those renderings. `parseSpinnerSeconds(stripped []byte) (int, bool)` returns total seconds + ok. The regex is consumer-policy (which spinner classes to parse) not driver-policy, so it lives in ptyrunner rather than `tuidriver` — the spike binary's regex at `github.com/pyrycode/tui-driver/cmd/spike-one-turn/main.go:58` is the reference. Kept separate from `runner.go` for clarity — folding inline would push `runner.go` past 400 lines and obscure `Run`'s top-to-bottom narrative.

- **`runWatchdog` ticks at `cfg.WatchdogTick` (default 1s) and exits on either ctx.Done or watchdog fire.** On each tick: `buf.Snapshot()` → `tuidriver.StripANSI(snap)` → `parseSpinnerSeconds(stripped)` → `tr.ObserveSpinner(ok && tuidriver.IsThinking(stripped), total)` → `tr.CheckWatchdog(buf)`. The `ok && IsThinking` conjunction is load-bearing: the freeze arm only engages when BOTH the class-A seconds-rendering parsed AND the thinking glyph is present, preventing false positives on class B / C spinners that have no seconds counter to track progress against. On non-nil `CheckWatchdog`: `logger.Warn("ptyrunner: watchdog fired", "err", werr)` (the err string is the tuidriver-generated `watchdog: PTY quiet for Ns (last state: ...)` which carries no Event content), then `emitter.SetExitReason(streamjson.ExitReasonError)`, then `cancel()`, then return. On `<-ctx.Done()`: return immediately.

- **Defer-LIFO chain is the only place the cleanup ordering is enforced — there is no runtime assertion.** Registration order (top = registered first, fires last): `defer sess.Close()` (existing, line 264) → `defer counter.Stop()` (new) → `defer func() { _ = emitter.Close() }()` (moved from its #478 position to AFTER counter.Stop registration but BEFORE the watchdog goroutine spawn) → `defer wg.Wait()` (new) → `defer cancel()` (new, second cancel — see below). Fire order (top runs first): `cancel()` → `wg.Wait()` → `counter.Stop()` → `emitter.Close()` → `sess.Close()`. A block comment above the emitter.Close defer pins the LIFO discipline by citing the package-doc invariant; the package doc on `Run` (lines 198-208) names each step's load-bearing property.

- **Two `defer cancel()` calls satisfy both `go vet` and the LIFO discipline.** The `go vet lostcancel` check requires the `context.CancelFunc` from `WithCancel` to be called on every return path; the LIFO discipline requires `cancel()` to fire FIRST in the cleanup chain (so the watchdog goroutine sees ctx.Done before `wg.Wait` tries to drain it). Resolution: an early `defer cancel()` registered immediately after `context.WithCancel` (line 309) catches the early-return paths between WithCancel and the watchdog spawn (emitter.New / budget.New / tail.New failure); a second `defer cancel()` registered at the LIFO head after the watchdog goroutine spawn (line 373) positions cancel at the top of the fire order on the happy-path / late-error paths. `context.CancelFunc` is documented safe for concurrent + repeated calls, so the double-defer is idempotent.

- **`buildArgs` is unchanged — `--max-turns` is still NOT in argv.** Interactive claude doesn't honor it; the budget package is the enforcement point. The argv shape stays at `--session-id / --settings / --permission-mode / --append-system-prompt-file / --model / --effort`; `--allowed-tools` continues to land via the settings file produced by #469.

- **Three new test cases plus one `TestRun_MissingRequiredFields` row plus one inline regex-parser table test.** `TestRun_BudgetHitBeforeEndOfTurn` reuses the `noEotBody` fixture from #478 with `MaxTurns: 1`, asserts the verbatim JSONL prefix appears in stdout AND the trailer carries `subtype:"error_max_turns"` / `terminal_reason:"max_turns"` / `is_error:true` / `num_turns:1` / `session_id` matches. `TestRun_WatchdogFires` uses an EMPTY JSONL body with `WatchdogTick: 50ms` and `WatchdogTrackerOpts{PTYQuietLimit: 200ms, SpinnerFreezeLimit: 200ms}`, asserts Run returns nil within 5s AND the trailer carries `subtype:"error_during_execution"` / `terminal_reason:""` / `is_error:true`. `TestParseSpinnerSeconds` is a pure-function table test with 7 rows covering class A single-word / two-word / minutes+seconds, class B / C non-match, no-glyph, glyph-only. `TestRun_MissingRequiredFields` gains a `{"no MaxTurns", func(c *Config) { c.MaxTurns = 0 }, "MaxTurns required"}` row plus `MaxTurns: 1` in the base config.

- **`helperRunCfg` sets `MaxTurns: 5` as a default that's high enough for none of the existing tests to trigger a budget hit.** The existing #478 happy-path / ctx-cancel / emit-error / modal / mcp-failure / network-failure tests all stay compatible without per-test overrides; the budget test overrides to `MaxTurns: 1`; the watchdog test overrides to `MaxTurns: 10`. No existing test needs to override `WatchdogTick` / `WatchdogTrackerOpts` — the default 1s tick + 30s/30s limits mean the watchdog goroutine is wall-clock-invisible within their 10s deadlines.

- **No helper extension required.** The existing `jsonl` mode covers both new test bodies. The budget test passes a non-empty body containing one non-EOT line; the watchdog test passes an empty body, and the helper's existing `if body == "" { return }` guard makes the body-writer goroutine a no-op (the helper just idles on SIGTERM as for the modal/banner tests). `helper_test.go` is unchanged in this slice.

## Patterns established

- **Single shared `runCtx` for multi-goroutine teardown.** When a spawn primitive coordinates multiple goroutines (here: caller, tail watcher, watchdog) whose disjoint failure signals compose into a single "this turn ended" question, derive ONE child ctx from the caller's ctx and route every goroutine's cancel side-effect through it. The post-return `runCtx.Err() != nil → return nil` collapse check then captures all operator-shutdown-equivalent paths (parent cancel + budget hit + watchdog fire) in one branch — the wire-side classification rides the streamjson trailer's `subtype` field, not the Run error return. Generalises whenever ptyrunner-style primitives compose disjoint cancellation sources whose externally-visible behaviour should be uniform.

- **Defer-LIFO is structural — pin it with a block comment.** When the cleanup-order invariant of a Run-style function lives entirely in the deferred-call registration order (no runtime assertion), the next refactor that reorders defers will silently break it. Annotate the chain with a block comment naming the desired fire order and the load-bearing property of each step. Pinned for `cancel() → wg.Wait() → counter.Stop() → emitter.Close() → sess.Close()` in `runner.go` lines 319-322; this extends the `emitter.Close BEFORE sess.Close` pin established in #478.

- **Double `defer cancel()` reconciles `go vet lostcancel` with cleanup ordering.** When a `context.WithCancel` derivative must be cancelled at the LIFO HEAD of a multi-defer chain (so peer goroutines unblock before `wg.Wait` tries to drain them) but the early-return paths between `WithCancel` and the goroutine spawn would otherwise leak the cancel func, register a first `defer cancel()` immediately after `WithCancel` and a second `defer cancel()` at the LIFO head after the goroutine spawn. `context.CancelFunc` is documented safe for concurrent + repeated calls, so the double-defer is idempotent and cheap. The first defer satisfies `go vet`; the second positions cancel at the top of the fire order on the happy path.

- **First-wins exit-reason coordination across racing goroutines.** When multiple goroutines may concurrently classify a stream's terminal state (here: the tail watcher's Terminate-hook fire and the watchdog goroutine's CheckWatchdog fire), use a mutex-guarded first-write-wins setter (`streamjson.Emitter.SetExitReason`) rather than an explicit race-resolution protocol. The losing goroutine's call becomes a no-op and the trailer composition is deterministic. Combined with `wg.Wait` before `emitter.Close`, this also makes the operator log order match the wire order — no "watchdog fired" Warn lines appear AFTER the trailer was already written.

- **Driver vs consumer policy split for spinner-class regex.** `tui-driver` owns class-agnostic primitives (`IsThinking`, `StripANSI`, the Tracker mechanics, the class-C `ParseSpinnerTokens` for the `↓N tokens` counter); ptyrunner owns the class-A `for Ns` / `for Nm Ks` regex because the choice of which spinner classes to feed the freeze-arm is consumer policy. Generalises whenever a downstream package needs a stricter parse than the shared driver provides — keep the strict parse local, leave the driver class-agnostic.

## Lessons learned

- **`go vet`'s `lostcancel` check fires when a `context.CancelFunc` from `WithCancel` is not called on every return path.** Layering an early `defer cancel()` right after `WithCancel` plus a second one positioned at the LIFO head (post-goroutine-spawn) satisfies both vet and the spec's fire-order (cancel must fire before `wg.Wait` so the watchdog goroutine drains). A single `defer cancel()` at the LIFO head would leak the cancel func on the emitter.New / budget.New / tail.New early-return paths between `WithCancel` and the goroutine spawn; a single `defer cancel()` immediately after `WithCancel` would fire LAST (after `wg.Wait`), deadlocking the watchdog. Both defers are required; `context.CancelFunc` is safe for repeated calls so the redundancy is free.

- **Class-A spinner-seconds parsing belongs in ptyrunner, not in `tui-driver`.** `tuidriver.ParseSpinnerTokens` (state.go:67-96) targets the class-C `↓N tokens` counter — a different rendering and a different counter from what the freeze arm needs. The watchdog needs the `for Ns` / `for Nm Ks` parse for class A, so the regex + parser live in `ptyrunner/watchdog.go` rather than the driver. Driver vs consumer policy split: the driver stays class-agnostic, the consumer picks which class to gate the freeze arm on. If a future consumer wants the same class-A parse, the regex can lift into `tuidriver` then — but the current single-consumer state is the right place for it.
