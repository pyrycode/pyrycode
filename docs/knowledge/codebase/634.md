# Ticket #634 ‚Äî Event-stream bridge: coarse emitter re-targeted to non-interactive conns + coarse-half two-phone e2e

The slice that makes the Phase 2 dual-path **mutually exclusive per conn** ([EPIC #596],
[ADR 025] ¬ß Phase 2). #589's coarse `assistantTurnEmitterV2` (`cmd/pyry/assistant_turn_v2.go`)
fanned its finished-turn `message` envelope to **all** open v2 conns via the
capability-agnostic `ActiveConnIDs` (#588) ‚Äî so once #632/#633 took the structured
stream live, an `interactive`-granted phone received **both** the coarse `message` and
the structured stream (redundant double-delivery). This slice re-targets the coarse
emitter to **non-interactive conns only**, via #626's capability-aware `ActiveConns`
filtered to `!c.Interactive` ‚Äî the **exact complement** of #632's interactive-only
structured filter. The two paths are now disjoint: every open conn is on exactly one
of them. This completes the ADR 025 ¬ß Phase 2 v2-additive guarantee ‚Äî capable phones
get the structured stream; everyone else stays on the coarse path, unchanged.

**`security-sensitive`** ‚Äî it completes the capability-gated routing contract on the
internet-exposed mobile surface (#607's deferred gated-fan-out review, the coarse
half; #632 covered the structured half). Architect self-review per the label contract:
**PASS, no MUST FIX** (see the spec's `## Security review`).

## Implementation

The entire production delta is **~10 LOC in one file**, `cmd/pyry/assistant_turn_v2.go`.
Everything else ‚Äî the cursor read ‚Üí message-ID mint ‚Üí payload marshal, the per-conn
`Push` shape, the Envelope-ID policy, the no-application-output-in-logs contract ‚Äî is
**byte-frozen** from #589.

| File | Change |
|---|---|
| `cmd/pyry/assistant_turn_v2.go` | Import `internal/relay`; re-shape the `v2Broadcaster` interface (`ActiveConnIDs(ctx) []string` ‚Üí `ActiveConns(ctx) []relay.ActiveConn`); add `if c.Interactive { continue }` to the `broadcast` fan-out loop and key `Push` on `c.ConnID`. |
| `cmd/pyry/relay.go` | Doc-comment only: the `mgr`-as-`v2Broadcaster` wiring note now reads "ActiveConns + Push" and "non-interactive v2 session". No code change ‚Äî `mgr` (`*relay.V2SessionManager`) already satisfies the reshaped interface. |

After this change the `v2Broadcaster` interface is **structurally identical** to #632's
`interactiveBroadcaster` (same two methods). The two emitters now consume the same
capability-aware snapshot and apply complementary filters.

```go
for _, c := range e.bcast.ActiveConns(ctx) {
    if c.Interactive {
        continue // structured-stream conns get the #632 path, never the coarse message
    }
    e.nextID++
    env := protocol.Envelope{ID: e.nextID, Type: protocol.TypeMessage, TS: time.Now().UTC(), Payload: payloadJSON}
    if err := e.bcast.Push(ctx, c.ConnID, env); err != nil { /* unchanged: ctx-cancel early-return + DEBUG log, keyed on c.ConnID */ }
}
```

`relay.(*V2SessionManager).ActiveConnIDs` is **left in place** (out of scope to remove).
After this slice `cmd/pyry` no longer consumes it ‚Äî it is reachable only from
`internal/relay` tests, exactly as its doc comment anticipates. Retiring it is a future
slice's call (code-review flagged the now-slightly-stale doc comment as a NIT,
correctly **not** touched ‚Äî `v2session.go` is frozen by the spec).

## Patterns established

- **Mutual exclusivity as a structural invariant, not runtime coordination (AC3).**
  "No conn receives both paths" is guaranteed by two **complementary filters over the
  same set-once field**: coarse skips `c.Interactive`, structured
  (`interactive_turn_v2.go:221`) skips `!c.Interactive`, both over one `ActiveConns`
  snapshot. No conn satisfies both; none satisfies neither ‚Äî every open conn is on
  exactly one path. There is no lock, no handshake, no shared "which path" registry.
  The inverse unit oracle proves the coarse half; #632's interactive-only matrix
  proves the structured half. `ActiveConn.Interactive` is set-once at handshake (#626,
  never mutated, preserved across re-key), so there is **no TOCTOU on the flag** ‚Äî the
  only live transition is open‚Üíclosed, caught by `Push`'s `V2StateOpen` re-gate.
- **Fail-safe direction is the inverse of #632 and still correct (`security-sensitive`).**
  A conn whose `Interactive` flag defaults/fails to `false` still receives the **coarse**
  path ‚Äî the *less* sensitive one (finished-turn text, already shipping since #589). So
  a flag-integrity failure degrades to "coarse delivered, structured withheld," never
  "structured leaked to a non-capable phone," never "both withheld." The emitter only
  *consumes* the flag (`continue` on `== true`); it adds no path that could set or
  forge it.
- **Re-target, don't rewrite.** Only the open-conn enumeration changed. The minting /
  marshal / `Push` spine and the no-app-output-in-logs contract (logs carry only
  `chunk_len`, `conversation_id`, `message_id`, `conn_id`, transport `err`) are
  inherited verbatim from #589. A capability filter is a `continue`, not a new failure
  mode ‚Äî error handling is unchanged.

## Lessons learned

Lifted from the PR body's **Lessons learned** (the developer flagged two non-obvious
surprises that shaped the e2e):

- **The coarse-half e2e is deterministic but takes ~30 s, not the ~8 s the spec's
  ¬ß Concurrency model implies.** The `send_message` that stamps the cursor runs
  synchronously on the v2 manager's single `Run` goroutine and **blocks there** inside
  `WriteUserTurn`'s `WaitReady` gate until `sendMessageDeliverTimeout` (30 s) ‚Äî
  fakeclaude renders no idle TUI, so `WaitReady` never confirms. The cursor *is*
  stamped before that gate (so the coarse emit has a valid cursor), **but** the
  emitter's `ActiveConns`/`Push` rendezvous on that **same blocked `Run` goroutine**,
  so the fan-out is serviced only once the 30 s timeout fires. The spec's "delivers on
  the independent emitter goroutine before the ack" is the one inaccuracy ‚Äî the
  *outcome* the architect specified (B gets coarse, A gets none) holds. The test
  documents this load-bearingly and clears the 30 s floor with a 40 s deadline (~10 s
  margin ‚Üí deterministic, not flaky). Code-review noted it as the slowest single test
  in the suite (~32 s); acceptable.
- **`fakephone` closes the WS on a timed-out `Receive`** (coder/websocket cancels the
  read ctx ‚Üí connection closed; "a timed-out Client cannot be reused"). So a
  short-timeout re-arm/poll loop on **one** phone self-destructs after the first miss.
  The e2e reads phone B with a **single long deadline** (back-to-back frame reads, like
  the #603 template) and moves trigger re-arming to a **background filesystem-only
  goroutine** that never touches the phone connection.

## Concurrency model

No new goroutines, no new synchronization in production ‚Äî the change is a filter inside
the existing serial `broadcast`, which runs on the emitter's single `Run` goroutine.
`ActiveConns` is the manager's existing snapshot funnel (request/reply over
`m.snapshot`, already concurrency-safe, already consumed by the #632 structured
emitter). The snapshot-then-`Push` gap is the reviewed #588/#589 pattern; `Push`
independently re-gates `V2StateOpen`, catching a conn that left open in the gap.

## Testing

- **Inverse unit oracle** (`TestAssistantTurnEmitterV2_SkipsInteractiveConns`,
  `cmd/pyry/assistant_turn_v2_test.go`) ‚Äî the deterministic proof of AC1/AC3. A mixed
  snapshot `{a:interactive, b:non-interactive}` with a stamped cursor ‚Üí **exactly one**
  push, addressed to `b`; never a push to `a`; the envelope is a well-formed `message`
  (`Role:"assistant"`, the known `conversation_id`, valid-UUID `message_id`,
  `InReplyTo == nil`). Draining a second push slot (which times out empty) is what
  proves the interactive conn is never pushed.
- **Stub migration** ‚Äî `stubV2Broadcaster` migrated mechanically from
  `ActiveConnIDs(ctx) []string` / `[][]string` to `ActiveConns(ctx) []relay.ActiveConn` /
  `[][]relay.ActiveConn`. A terse `nonInteractive(ids ...string) []relay.ActiveConn`
  helper lets the six pre-existing tests keep their assertions verbatim
  (`newStubV2Broadcaster(nonInteractive("conn-a", "conn-b"))`) ‚Äî they now flow through
  the filter as non-interactive conns, behaviour unchanged. Single-file, no
  cross-package cascade.
- **Coarse-half two-phone fakeclaude e2e** ‚Äî `TestTwoPhoneCoarse_NonInteractiveOnly`,
  **new file** `internal/e2e/relay_two_phone_coarse_test.go` (`//go:build e2e`).
  **Runnable, not skipped.** One daemon + fakeclaude + fakerelay; two phones pair and
  handshake over real Noise sessions ‚Äî phone A advertises and is granted `interactive`
  (`buildHelloEarlyInteractive` / `driveHandshakeToOpenDaemonInteractive`, defined
  locally because `relay_v2_handshake_test.go` is off-limits to `feature/449`), phone B
  advertises nothing. A turn is driven from **B without awaiting the ack** (only the
  synchronous cursor stamp is needed); a re-armed fakeclaude chunk fans out. **Asserts
  B receives the coarse `message`** (marker text, assistant role, known conv-id,
  valid-UUID msg-id, server-initiated) and **A receives no `TypeMessage`** (the
  double-delivery guard ‚Äî drained for a bounded ~1 s window). The 40 s read deadline
  clears the 30 s `WaitReady` floor (see Lessons learned).
- Gates green: `go test -race ./...`, `go vet ./...`, `go build ./cmd/pyry`,
  `go test -tags=e2e -run TestTwoPhoneCoarse ./internal/e2e/...`, and the realclaude
  e2e column (code-review). AC5 (no application output logged at any level) intact.

## Scope (S confirmed)

Production change is XS (~10 LOC); the two-phone fakeclaude e2e is the bulk of the
effort and is **test code** (real Noise sessions, capability negotiation, turn drive,
decryption, mutual-exclusivity assertions). Tests scale linearly and don't count toward
the production budget ‚Äî hence S, not XS.

## Out of scope / deferred

- **The structured-receive capstone ‚Üí blocked follow-up [#642].** The original AC4
  demanded a live two-phone e2e in which the **`interactive`** phone *receives the
  structured envelopes*. **Not exercisable today:** `fakeclaude` writes only `{}\n` to
  its session JSONL (`internal/e2e/internal/fakeclaude/main.go`) while the structured
  producer tails the **real** claude JSONL transcript
  (`cmd/pyry/interactive_turn_stream_v2.go`), so an interactive phone under fakeclaude
  has no structured stream to receive; the `e2e_realclaude` suite has no Noise-phone
  harness; and #603 is open. This slice was **descoped in place** (kept its number,
  lineage, labels, Backlog position = the ship-now half); #642 is the
  structured-receive proof, blocked on #603 + a structured-stream-capable harness. The
  load-bearing "non-`interactive` phone never receives the structured stream" guarantee
  is still proven **here, deterministically** by the two complementary filters and the
  #626/#632 unit matrices ‚Äî #642 is a live confirmation, not this slice's only proof.
- **Retiring `ActiveConnIDs`** ‚Äî left in place (still used by `internal/relay` tests);
  a future slice's call. Its doc comment is now slightly stale (code-review NIT).
- **Per-conversation subscription routing** ‚Äî the coarse fan-out still reaches *every*
  non-interactive conn (the #589 shape), not a `conversation_id`-subscribed subset. Not
  a leak today (one supervised claude, one cursor, all devices belong to the one
  operator; payloads carry `conversation_id` for client-side filtering). Deferred to
  pyrycode-mobile#336 (inherited #589 deferral).
