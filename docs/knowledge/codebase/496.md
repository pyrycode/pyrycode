# Ticket #496 — `WithWorktreeAuthenticated` seeds `.claude.json` so ptyrunner tests skip the onboarding TUI

On the OAuth path, [`internal/e2e/realclaude/fixtures.go::WithWorktreeAuthenticated`](https://github.com/pyrycode/pyrycode/blob/main/internal/e2e/realclaude/fixtures.go) now copies the operator's real `~/.claude.json` verbatim into `<tempHome>/.claude.json` at mode `0o600` before the test subprocess spawns, so interactive (PTY) `claude` skips the onboarding theme picker on the fresh `$HOME` the fixture pins. Closes layer 3 of the validation-gate-on-Max-only-Mac peel (layer 1: [#487](https://github.com/pyrycode/pyrycode/issues/487) invalid `defaultMode:"deny"` rejection; layer 2: [#490](490.md) / [#491](491.md) / [#492](492.md) dual-credential helper + consumer flip + operator docs; layer 3: this ticket). Without the seed, ptyrunner read the picker's prompt glyph as "ready", delivered the bracketed-paste prompt into the picker's input field, `claude` never proceeded, and the 31 s ptyrunner deadline fired — the failure shape that produced **19/19 FAIL in 593 s** on the operator's Mac on 2026-05-21. With both `CLAUDE_CODE_OAUTH_TOKEN` exported AND `~/.claude.json` present in the operator's real `$HOME`, the suite now runs end-to-end (or skips with a named reason); none of those skips time out.

## Implementation

- **`internal/e2e/realclaude/fixtures.go:81-114` extends `WithWorktreeAuthenticated` with a third path on top of the existing two.** Capture-then-pin order: read `operatorHome := os.Getenv("HOME")` BEFORE delegating to `WithWorktree(t)` (which `t.Setenv`s HOME to a tempdir), `os.ReadFile(<operatorHome>/.claude.json)` into a local byte slice declared at outer scope (`var claudeJSON []byte` at line 86), then after `WithWorktree(t)` returns the per-test tempdir, `os.WriteFile(<tempHome>/.claude.json, claudeJSON, 0o600)`. Both the read and the seed are gated on `oauthToken != ""` — the `ANTHROPIC_API_KEY`-only path is byte-identical to pre-change behaviour (no `.claude.json` read, no seed; AC #2's no-regression contract holds).

- **`os.Getenv("HOME")` over `os/user.Current().HomeDir`.** The architect's spec explicitly chose env-var capture so tests can pre-pin `$HOME` to a fake operator home via `t.Setenv("HOME", fakeHome)` without mocking `os/user`. `os/user.Current()` reads from `/etc/passwd` (Unix) and is not test-overrideable via env vars; choosing it would force a package-private indirection variable just to make the test work, with no production benefit. Implementation matches the spec literally.

- **Option A chosen over Option B — verbatim copy, no JSON parse, no field filter, no normalisation.** The ticket framed two options: A (copy verbatim) vs. B (synthesise `{"hasCompletedOnboarding":true,"installMethod":"npm-global"}`). The spec picked A. The verbatim bytes carry every onboarding-related flag claude might care about now or later, which is exactly the property the fix needs; Option B's "test isolation" benefit is theoretical (the temp `$HOME` is per-test `t.TempDir()`, process-private, auto-cleaned — per-test isolation already bounds blast radius), while Option B's brittleness is empirical (drift would resurface as another 31 s timeout mystery identical in shape to the bug this ticket fixes).

- **Skip-message contract pinned by five load-bearing substrings** ([`fixtures.go:92-100`](https://github.com/pyrycode/pyrycode/blob/main/internal/e2e/realclaude/fixtures.go)). The new `t.Skipf` (fires when OAuth is set but `.claude.json` is unreadable) names: (1) `CLAUDE_CODE_OAUTH_TOKEN` — the token prerequisite verbatim; (2) `.claude.json` — the JSON prerequisite verbatim; (3) `onboarding` — the failure class; (4) `claude` — the recovery action's verb; (5) `hasCompletedOnboarding=true` — the contract the recovery action satisfies. The exact prose around them is the developer's call (the spec said so explicitly); only the substring set is pinned, so future prose tweaks don't break the test.

- **`os.ReadFile` errors are `t.Skipf`, not `t.Fatalf`.** A missing `~/.claude.json` is an operator-environment shape (fresh install, never invoked `claude` directly), not a fixture-internal bug. `t.Fatalf` would mark the suite as failed instead of skipped — wrong signal. Conversely, `os.WriteFile` on the proceed-path is `t.Fatalf` (lines 111-113): writing to the tempdir `WithWorktree` just created should never fail, and if it does that IS a fixture-internal bug.

- **The "either-or" auth contract from [#490](490.md) is preserved unchanged.** Reading both env vars, skipping only when both are unset/empty, re-pinning per-variable with `if value != ""` guards — all pre-existing behaviour. The new logic slots in as an extra step on the OAuth-only proceed path; the API-key-only and "both unset" paths are byte-identical to pre-#496 behaviour.

- **`internal/e2e/realclaude/fixtures_test.go:147-181` extends the existing `TestWithWorktreeAuthenticated_OAuthTokenOnly_RepinsAndPreservesAbsentApiKey` rather than adding a new happy-path test.** The existing test would otherwise trip the new skip path (synthetic OAuth token literal + no `.claude.json` in operator HOME). Pre-call additions: `opHome := t.TempDir()`, write a recognisable JSON content with a `_marker` field (`{"hasCompletedOnboarding":true,"installMethod":"npm-global","_marker":"#496-test"}\n`) at mode `0o600`, `t.Setenv("HOME", opHome)`. Post-call assertions: `os.ReadFile(<dir>/.claude.json)` returns nil error, `bytes.Equal` against the original bytes (verbatim-copy contract), `os.Stat(...).Mode().Perm() == 0o600` (mode contract). The `_marker` field makes assertion-failure messages diagnostic — a different file would produce a `bytes.Equal` failure that names the marker.

- **`internal/e2e/realclaude/fixtures_test.go:201-247` adds `TestWithWorktreeAuthenticated_SkipsWhenOAuthSetButNoClaudeJSON` (outer/inner subprocess re-exec).** Same pattern as [#490](490.md)'s `TestWithWorktreeAuthenticated_SkipsAndNamesBothEnvVarsWhenNeitherSet`. Sentinel env var `PYRY_REALCLAUDE_NOJSON_INNER=1` is deliberately distinct from `PYRY_REALCLAUDE_AUTH_SKIP_INNER=1` (which clears BOTH env vars — wrong shape for this test, which needs the OAuth token SET but `.claude.json` absent) and from `GO_TEST_HELPER_PROCESS=1` (which routes `TestMain` into `runFakePyry()` instead of `m.Run()`). Inner branch: tempdir `opHome` with no `.claude.json` written, `t.Setenv("HOME", opHome)`, `t.Setenv("ANTHROPIC_API_KEY", "") + os.Unsetenv("ANTHROPIC_API_KEY")` two-step (canonical "register cleanup, then actually unset" idiom from #490), `t.Setenv("CLAUDE_CODE_OAUTH_TOKEN", "test-oauth-token-not-real")`, call `WithWorktreeAuthenticated(t)`. Defensive `t.Fatalf` after the call (unreachable on the skip path) guards against silent regression. Outer branch: `cmd.CombinedOutput()` returns nil for `err` (skip is success exit), output contains `--- SKIP: TestWithWorktreeAuthenticated_SkipsWhenOAuthSetButNoClaudeJSON`, output contains each of the five load-bearing substrings.

- **`TestWithWorktreeAuthenticated_SkipsAndNamesBothEnvVarsWhenNeitherSet` is unchanged.** Both env vars unset → fixture skips on the original "neither set" path before the `.claude.json` check runs. Extending it with the new substrings would falsely couple two independent contracts.

- **`TestWithWorktreeAuthenticated_RealAssistant` is unchanged.** Operator runs it with real `CLAUDE_CODE_OAUTH_TOKEN` from Keychain AND real `~/.claude.json` present (the production case). Both prerequisites satisfied → fixture proceeds, seeds the temp HOME with the operator's real JSON, the real `pyry agent-run` invocation succeeds, existing JSONL assertions pass.

## Patterns established

- **Capture-then-pin: read the operator's real environment BEFORE `t.Setenv` clobbers it.** When a fixture needs to mediate operator state (HOME-rooted files, shell config, user-owned credentials) into a per-test sandbox, capture the operator's value at fixture entry, then let the per-test pin take over. Reading after the pin reads the pinned (empty) value, which is silently wrong — exactly the 31 s-timeout failure mode this ticket fixes. Concretely here: read `os.Getenv("HOME")` and the file it points to before calling `WithWorktree(t)` (which calls `t.Setenv("HOME", tempdir)`). Generalises to any fixture that pins an env var that downstream production code uses to resolve file paths.

- **Verbatim opaque-bytes copy over field-by-field synthesis for state files owned by an upstream tool.** When the consumer (here: `claude`) reads a JSON file at startup and only some subset of fields is load-bearing, copying the bytes verbatim is more robust than synthesising a minimal version with "just the flags that matter". The minimal version is correct on the day you write it and drifts the moment the upstream adds a required flag — and the failure mode (mysterious 31 s timeouts) is identical in shape to the bug the synthesis was supposed to fix. Verbatim copy carries forward whatever the upstream cares about, plus operator-owned state that's irrelevant to the test (e.g. project history, plugin caches) but harmless inside a per-test process-private tempdir. Reserve synthesis for cases where contamination is a measured failure mode, not a theoretical concern.

- **`t.Skipf` for operator-environment shapes; `t.Fatalf` for fixture-internal bugs.** A missing `~/.claude.json` on the operator's machine is an environment shape (operator hasn't run `claude` yet) — `t.Skipf` with a named-prerequisite diagnostic is the right signal. A failed `os.WriteFile` into a tempdir the fixture just created via `t.TempDir()` is a fixture-internal bug — `t.Fatalf` surfaces it loudly. The two failures look syntactically similar (both `if err != nil`) but mean different things; conflating them either masks fixture bugs as skips or marks operator-environment shapes as suite failures.

- **Distinct sentinel env vars for distinct outer/inner subprocess tests.** When multiple tests in the same package use the outer/inner re-exec pattern, give each one its own sentinel — reusing one sentinel forces the inner branch to handle all of the tests' state shapes, which violates one-test-one-concern and makes the inner logic harder to read. Concretely here: `PYRY_REALCLAUDE_AUTH_SKIP_INNER=1` (clears both env vars; tests the "neither set" path) and `PYRY_REALCLAUDE_NOJSON_INNER=1` (token set, HOME pinned to empty tempdir; tests the "OAuth set but no JSON" path). Both must be distinct from `GO_TEST_HELPER_PROCESS=1` (which `TestMain` routes into `runFakePyry()`). Generalises to any test file with multiple skip-message-shape contracts.

- **Pin substring contracts, not exact prose, for `t.Skipf` message tests.** A test that asserts on `t.Skipf`'s exact text couples to incidental prose choices the spec explicitly leaves open. The shape that's worth pinning is the set of load-bearing substrings (env-var names, recovery commands, contract assertions) — that gives the developer prose freedom while still catching genuine regressions like "the skip message no longer names `.claude.json`". Same shape as [#490](490.md)'s pinning of the Keychain extraction recipe substrings.

## Lessons learned

- **The validation-gate-on-Max-only-Mac peel had three layers, not one.** [#487](https://github.com/pyrycode/pyrycode/issues/487) (the invalid `defaultMode:"deny"` rejection) was the visible symptom. Its post-mortem named layer 2 (silent-skip masquerade because `WithWorktreeAuthenticated` only recognised `ANTHROPIC_API_KEY`) and produced the [#490](490.md) / [#491](491.md) / [#492](492.md) trio. The operator ran `make e2e-realclaude` again expecting green — and got 19/19 FAIL because layer 3 (interactive `claude` shows the onboarding TUI on a fresh `$HOME` regardless of auth env vars) had been invisible behind layer 2. Each peel surfaced the next one. Generalises: when a test suite has multiple gating mechanisms (auth env vars, file presence, terminal capabilities), unblocking one gate can expose a previously-hidden gate downstream; assume more layers exist until end-to-end execution confirms the suite is actually exercising the production surface. The empirical signal that the peel is complete is a green-or-named-skip outcome, not "the obvious gate is fixed".

- **The optional "Lessons learned" section in the PR body was correctly omitted.** PR [#497](https://github.com/pyrycode/pyrycode/pull/497)'s body has no such section. The work itself followed the spec line-for-line — the spec named Option A vs B with explicit rationale, the substring contract was pinned in advance, and code review was PASS with one NIT the spec explicitly allowed (the `os.Getenv("HOME") == ""` edge case treated as "almost never occurs in practice"). No genuine surprise to lift. Same pattern as [#492](492.md) and [#491](491.md) — fabricating a lesson to fill the section is anti-signal for future readers.

- **Code-review NIT on `os.Getenv("HOME") == ""` is correctly a no-fix.** The reviewer flagged that when `HOME` is empty in the outer environment, `filepath.Join("", ".claude.json")` yields the relative path `.claude.json` resolved against CWD — a syntactically valid but semantically wrong target. The architect's spec already addressed this explicitly in the "Error handling" section: an empty `HOME` "almost never occurs in practice", adding a third skip branch for it would clutter the fast path, and the natural failure mode (`os.ReadFile` returns ENOENT or EACCES) fires the named-prerequisites skip anyway. The reviewer correctly flagged for visibility and didn't block. Generalises: spec-acknowledged edge cases that resolve to the right behaviour via the natural failure path don't need defensive code; flagging them at code review keeps the decision auditable without forcing a fix that adds surface area.

- **Reporter's read on Option A was the right architectural call.** The ticket's "Reporter's read: Option A" was correct: theoretical contamination is revisitable if it ever becomes a real failure mode; brittleness to upstream onboarding-flag changes is empirically the more likely failure mode (drift would resurface as the same 31 s timeout shape this ticket fixes). The architect's spec affirmed the call rather than overriding, and code review's blast-radius check (`codegraph_callers WithWorktreeAuthenticated` returns 19 callers, signature unchanged, fall-through paths byte-identical) confirmed no consumer surface needed touching. Generalises: when a ticket's reporter has done the empirical work to verify the fix and flags a trade-off matrix with their read, architect's job is to validate the reasoning, not to second-guess the call from first principles.

## Out of scope (follow-ups)

- **Token rotation handling.** If the suite ever outlasts the access token's TTL (currently ~9 min runtime vs. hours-scale TTL), the fixture would need to re-read Keychain mid-run. Not a problem at current cadence; same deferral as [#489](https://github.com/pyrycode/pyrycode/issues/489) and [#490](490.md).
- **`claude setup-token` integration.** `claude` has a `setup-token` subcommand that may produce longer-lived tokens — potentially a cleaner long-term auth surface for headless test scenarios. Separate investigation; not blocking this ticket.
- **CI integration.** GitHub Actions runners have no macOS Keychain AND no operator-owned `~/.claude.json`. The realclaude suite has no CI workflow per the `## CI cadence` section of `e2e-realclaude.md` — it runs locally during the code-review phase of every dispatched ticket via the pipeline. The unrelated `self-check-daily` CI failure on the `ANTHROPIC_API_KEY` injection bug is tracked under [#406](https://github.com/pyrycode/pyrycode/issues/406).
- **Empty-`HOME` defensive branch.** Code-review NIT flagged that `os.Getenv("HOME") == ""` produces a relative path that `os.ReadFile` resolves against CWD. The spec explicitly accepted this edge case (almost never occurs in practice; natural failure path fires the named-prerequisites skip). If observed in practice, a one-line guard returning the skip directly is the simplest fix; until then, the fast path stays uncluttered.
- **Flipping to Option B (synthesised minimal JSON) if contamination becomes a real failure mode.** The spec is explicit: Option B is the right follow-up if cross-test contamination ever surfaces as a real failure. No code action until then; this bullet exists so the next reader knows the revisit path.
- **`permission_protocol_spike_test.go` inline `ANTHROPIC_API_KEY`-only skip + `mcp_smoke_test.go` parity.** Same deferral named in [#491](491.md) — these test files have inline skip logic that hasn't been flipped to `WithWorktreeAuthenticated`. XS in its own right; doesn't block.
