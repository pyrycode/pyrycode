# Ticket #656 — v2 `session_transition` wire type (SSOT)

The wire **vocabulary** for a session boundary ([EPIC #596] Phase 2, [ADR 025]).
The v2 wire carries message content but had **no session-lifecycle
representation** — `MessagePayload` is `{conversation_id, message_id, role, text}`
with no `session_id`, no timestamp, no transition reason. This slice adds a
dedicated outbound binary → phone event so the daemon's session rotation (a
`/clear`, an idle eviction, or a workspace change) can be decoded mobile-side as
a `ThreadItem.SessionBoundary` marker (`pyrycode-mobile#336`) instead of inferred
from message fields that do not exist. Boundary markers were de-scoped from
mobile #313 on 2026-06-01 for exactly this reason.

**Wire shape only** — the type constant, the payload struct, the round-trip test,
and the docs SSOT. The **producer** that emits the marker on session transitions
is sibling [#657] (`security-sensitive`, blocked on this slice). **Not
`security-sensitive`**: outbound wire vocabulary, no inbound trust boundary, no
producer, nothing dispatches or imports it yet. Split from [#655] (its sibling
[#657] is the producer half).

construction sites changed** — nothing fans out or imports it yet.

## Patterns established

- **A nullable wire field encodes a cross-field invariant directly.**
  `WorkspaceCwd` is `*string` **without `omitempty`** (the
  `BackfillSincePayload.ConversationID` precedent): non-nil iff
  `reason == "workspace_change"`, literal JSON `null` for `clear` / `idle_evict`.
  `omitempty` would drop the key and lose the "absent vs null vs value"
  distinction, so the *workspaceCwd-non-null-iff-workspace_change* invariant is
  decodable from the wire alone. The byte-equal round-trip is the regression
  detector for an accidental `omitempty` re-add (same guard role as
  `backfill_since.json`'s `conversation_id: null`).
- **A consumer enum may admit a value the producer cannot yet emit.** `reason`
  is the closed set `{clear, idle_evict, workspace_change}` so the mobile decoder
  stays exhaustive and the invariant is expressible — even though the producer
  ([#657]) will emit only `clear` / `idle_evict` until a server-side
  workspace-change source exists. The type child carries the full vocabulary; the
  producer child defers the unemittable value. (PO split rationale: the consumer
  enum outruns the server's emit sources, so the type stays complete while
  emission is deferred — see [#655] → #656/#657.)
- **`reason` is a plain `string`, not a named Go enum.** Matches
  `MessagePayload.Role` / `TurnEndPayload.StopReason`: `internal/protocol` is a
  stdlib-only leaf data package; the closed-set guarantee belongs at the decoder,
  not in a Go enum type. The mobile enum names (`Clear`/`IdleEvict`/`WorkspaceChange`)
  map to the lowercase snake wire values by the mobile decoder.
- **Own const block + `v2OnlyTypes`/partition lockstep is the standing recipe for
  a v2-only type.** Mirrors `resync` ([#647]), the snapshot pair ([#617]), and
  the interactive group ([#607]/[#638]): a dedicated `codes.go` block with the
  MUST-NOT-in-`v1TypeSet` comment, plus the three `compat_test.go` edits whose
  union-count check (`+1` to `v2OnlyTypes`, `+1` to `all`) self-balances. Adding a
  v2-only type without partitioning it fails the build.
- **A multi-field marker warrants a named struct; a payload-less control marker
  does not.** Unlike `resync` (inline `{conversation_id}`, no named type),
  `session_transition` carries five fields, so it earns `SessionTransitionPayload`
  in `messaging.go` (the `MessagePayload` precedent). `event_id` is **not** a
  payload field — it is an `Envelope`-level field ([#649]) stamped by the producer
  on structured-stream frames; a session boundary is not a structured event and
  carries no `event_id`.

## Lessons learned

- **Author `testdata` fixtures in struct-field order.** The round-trip helper's
  `canonical()` compacts bytes but **does not sort keys**, so the JSON key order
  in a fixture must equal the Go struct field order or the byte-equal check fails.
  Both `session_transition*.json` fixtures follow
  `previous_session_id → new_session_id → reason → occurred_at → workspace_cwd`.
- **`time.Time` round-trip discipline.** `OccurredAt` is RFC3339Nano; the marshal
  strips the monotonic clock, so the test compares with `.Equal`, never
  `==` / `reflect.DeepEqual`. The `whole-second` and `.5`-fraction fixtures both
  re-marshal byte-stably.
- **The branch diff carries an unrelated realclaude fixture re-recording.**
  `internal/e2e/realclaude/testdata/permission_protocol_v2.1.158.json` shows ~131
  changed lines (fresh `session_id` / `uuid` / temp-path values) — a golden
  fixture re-captured during the code-review phase's real-claude e2e run and swept
  in by its auto-commit safety net (commit `43f8323`). It is **not** part of the
  wire-vocabulary change; the volatile per-run values are the only delta.

## Testing

`internal/protocol/messaging_test.go` — `TestSessionTransitionPayload_RoundTrip`,
table-driven over the two fixtures, templated on `TestBackfillSincePayload_RoundTrip`:

- **cwd-unset** (`session_transition.json`, `reason: "idle_evict"`,
  `"workspace_cwd": null`): envelope `Type == TypeSessionTransition`; payload
  prev/new id + reason asserted; `OccurredAt` via `.Equal`; `WorkspaceCwd == nil`;
  byte-equal round-trip (the `omitempty`-regression guard — if a future
  contributor adds `omitempty`, the `"workspace_cwd":null` key disappears and this
  fails).
- **cwd-set** (`session_transition_workspace.json`, `reason: "workspace_change"`,
  `"workspace_cwd": "/home/user/project"`): same, but `WorkspaceCwd != nil &&
  *WorkspaceCwd == "/home/user/project"`; byte-equal round-trip.

`compat_test.go` — the partition stays green at `16 + 11 == 27`, plus the
`session_transition-rejected` `ErrUnknownType` row.
`TestV1TypeSet_CoversAllExportedTypeConstants` (`len == 16`) is **unaffected** —
`session_transition` is v2-only.

`go test -race ./...`, `go vet ./...`, `go build ./cmd/pyry` all green (PR [#658]).

## Out of scope (deferred)

- **The producer — sibling [#657]** (`security-sensitive`, blocked on this
  slice): the session-lifecycle hook that builds `SessionTransitionPayload` and
  emits `TypeSessionTransition` on a transition, capability-gated. This slice only
  defines the wire shape; nothing emits it.
- **`workspace_change` emission.** The type admits it, but the producer ([#657])
  emits only `clear` / `idle_evict` until a server-side workspace-change source
  exists. When that source lands, #657 emits `workspace_change` with a non-null
  `workspace_cwd`.
- **The mobile consumer** — `pyrycode-mobile#336`, which decodes the marker into
  `ThreadItem.SessionBoundary`.
