# Ticket #249 — relay: inbound token validation per WS connection

Phase 3 Track C — composes A5 (`devices.Registry.Validate`, #210) with the v1 handshake/control payload structs (#271) to give the binary the per-phone-conn auth decision. Adds `internal/relay/auth.go` as a sibling to `connection.go`: a single pure function that takes the first frame's `RoutingEnvelope`, the device token (as an explicit string — carrier-agnostic), the devices registry, the binary's server-id, and a logger; returns a structured `AuthOutcome` carrying the `hello_ack`-or-`error` response envelope and a `CloseConn` signal the (future) relay-conn layer will act on. The relay performs no token validation per spec § Authentication phone→relay→binary — the binary owns the entire trust check; this ticket lands the decision, not the wire wiring.

## Implementation

### `internal/relay/auth.go` (~149 LOC)

- **Single exported function `AuthenticateFirstFrame(env, token, reg, serverID, logger) (AuthOutcome, error)`** — pure call-and-return on top of `reg.Validate`. No goroutines, no channels owned. Safe for arbitrary concurrent invocations across distinct phone conns; the only shared state is `*devices.Registry` whose mutex guards `Validate`'s critical section.
- **Two-state semantics, three input pathologies fold into one outcome.** `reg.Validate(token)` returns `(Device, bool)`. `true` ⇒ build `hello_ack` (per `protocol.HelloAckPayload{ProtocolVersion:"v1", ServerID, ConnID: env.ConnID}`), wrap in an `Envelope{ID:1, Type:TypeHelloAck, TS:now, InReplyTo:&helloID, Payload:…}`, wrap again in `RoutingEnvelope{ConnID: env.ConnID, Frame:…}`, log INFO, return `AuthOutcome{Response, CloseConn:false}`. `false` ⇒ build `ErrorPayload{Code:CodeAuthInvalidToken, Message:MsgInvalidToken, Retryable:false}`, same envelope wrap with `Type:TypeError`, log WARN, return `AuthOutcome{Response, CloseConn:true}`. Empty token, never-paired token, and revoked-then-removed device all take the `false` arm and emit the *same* outcome — pinned by `TestAuthenticateFirstFrame_RevokedTokenSameUX` and `TestAuthenticateFirstFrame_EmptyToken`.
- **Carrier-agnostic by design.** The function never parses WS headers, never reads `env.Frame`'s payload for a token field, and never inspects a hello payload — it only JSON-decodes `env.Frame` into a `protocol.Envelope` to extract the outer `id` (echoed into `in_reply_to` per spec § hello_ack / § error). The relay-conn ticket (future) that wires this handler into phone traffic picks the wire mechanism — (a) extended routing envelope, (b) synthesized `connection_opened` control frame ahead of `hello`, or (c) amended `hello` payload — and forwards the extracted token here unchanged. None of those choices change this signature.
- **`AuthOutcome` carries protocol-level intent, not wire codes.** `CloseConn bool` means "close the phone WS because auth failed"; the actual close code (`4401`) lives on the exported `StatusUnauthorized websocket.StatusCode` constant. The relay-conn caller writes `Response` first then closes with `StatusUnauthorized`; this handler does not own the write/close sequencing. Parallel to `connection.go`'s `statusServerIDConflict` (which is unexported because nothing outside the package needs the 4409 value).
- **Outer envelope `ID: 1` is fixed.** This is always the binary's first outbound frame on the phone's conn (matches spec § Worked example line 731). Subsequent envelopes on the same conn are allocated by the (future) relay-conn layer from a per-conn counter starting at 2 — not this handler's concern.
- **`MsgInvalidToken` is an exported `const` for spec sentence pinning.** `"device token not recognised; re-pair via pyry pair on the binary"` — defined as a constant so future integration tests can pin it without re-typing the spec sentence, and so spec drift surfaces as a single-line edit rather than scattered string literals.
- **`ErrMalformedHelloFrame` sentinel for the malformed-frame path only.** Returned when `json.Unmarshal(env.Frame, &inner)` fails. Missing/empty token, unknown token, and revoked-then-removed device are NOT errors — they are valid `auth.invalid_token` outcomes returning `(AuthOutcome{…, CloseConn:true}, nil)`. The relay-conn caller maps `ErrMalformedHelloFrame` to its existing `protocol.malformed` handling; this package does NOT synthesize an error envelope for that path (the malformed-frame response shape belongs to a sibling future ticket). Sentinel carries no parser detail in its message — the caller decides whether to log the wrapped `json` decode error at the call site.
- **A5's `LastSeenAt` bump is the only mutator.** `reg.Validate` already advances `LastSeenAt` under `reg.mu` as part of the same critical section. The handler does NOT call any further mutator — pinned by `TestAuthenticateFirstFrame_ValidToken` reading `reg.FindByTokenHash(HashToken(token)).LastSeenAt.After(pastSeen)` after the call. Persistence (`Save`) is the supervisor's responsibility per `devices/auth.go`'s documented contract.
- **`CodeAuthTokenRevoked` is deliberately not emitted.** Spec § Error codes line 535 documents `auth.token_revoked` as having the *same UX* as `auth.invalid_token`, and A5's `Registry.Validate` two-state primitive cannot distinguish never-paired from previously-paired-then-removed (no tombstone, no revoked-hash set — `Remove` deletes the row outright). Emitting only `auth.invalid_token` is correct v1 behaviour. The `CodeAuthTokenRevoked` constant stays defined in `internal/protocol/codes.go` for a future tombstone primitive but is unused here. The `RevokedTokenSameUX` test locks this equivalence so a future regression cannot silently introduce `auth.token_revoked` here without an explicit ADR.

### `internal/relay/auth_test.go` (~221 LOC)

Six flat tests, no table-driven mega-test (each scenario asserts a different envelope shape; `t.Run` reads better than struct-of-structs):

- **`TestAuthenticateFirstFrame_ValidToken`** — paired registry with `Device{TokenHash: HashToken("plain-token"), Name:"Pixel", PairedAt:fixed, LastSeenAt:past}`. Asserts `CloseConn==false`, `Response.ConnID=="c-test"` (echoed), decoded envelope is `TypeHelloAck` with `ID==1` and `*InReplyTo==helloID`, decoded `HelloAckPayload` has `ProtocolVersion=="v1"` / `ServerID=="test-server"` / `ConnID=="c-test"`, and `LastSeenAt` advanced past the fixed initial value.
- **`TestAuthenticateFirstFrame_UnknownToken`** — empty registry, token `"never-paired"`. Asserts the reject shape via `assertRejectOutcome`.
- **`TestAuthenticateFirstFrame_RevokedTokenSameUX`** — paired registry, then `reg.Remove("Pixel")`, then call handler with the originally-paired token. Same reject shape as `UnknownToken`. Locks spec § Error codes line 535 equivalence.
- **`TestAuthenticateFirstFrame_EmptyToken`** — paired registry, token `""`. Same reject shape. Defends against a buggy relay-conn caller forwarding an empty string (`reg.Validate("")` already short-circuits without touching the registry).
- **`TestAuthenticateFirstFrame_MalformedHelloFrame`** — `env.Frame=[]byte("not-json")`. Asserts `errors.Is(err, ErrMalformedHelloFrame)` and outcome is the zero value.
- **`TestStatusUnauthorized_Value`** — single-line guard `StatusUnauthorized == websocket.StatusCode(4401)`. Catches accidental edits to the constant.

`assertRejectOutcome` is a file-local helper shared by `UnknownToken`/`RevokedTokenSameUX`/`EmptyToken`: asserts `CloseConn==true`, `Response.ConnID` echoed (relay needs it to address the close), envelope `Type==TypeError` / `ID==1` / `*InReplyTo==helloID`, and `ErrorPayload{Code:CodeAuthInvalidToken, Message:MsgInvalidToken, Retryable:false, RetryAfterS:nil}`.

`makeHelloRouting(t, connID, helloID)` is a file-local builder that marshals a `HelloClientPayload` (role="client", device_name="phone", client_version="v1", protocol_versions=["v1"]) into an `Envelope{ID:helloID, Type:TypeHello, …}` and wraps it in a `RoutingEnvelope`. Used by every test except the malformed-frame case (which supplies a raw `[]byte` frame directly).

`pairedRegistry(t)` is a helper returning a `*devices.Registry` pre-populated with one device plus the fixed initial `pastSeen` time for the bump-assertion. `testLogger(t)` is reused from `connection_test.go` (discarding `slog.Logger`).

## Security discipline (pinned by code review)

- **`token` is never logged**, never wrapped into a returned error, never echoed to the phone. The rejection `message` field is a fixed sentinel string with no interpolation.
- **The matched device's name IS logged on accept** (`event=auth.accept`, `conn_id`, `device_name`) — paired-device identification is operationally useful, and an attacker who reaches the accept path already holds a valid token.
- **No name is logged on reject** (`event=auth.reject`, `conn_id`, `code=auth.invalid_token`) — emitting "name X was not recognised" would let an attacker brute-forcing tokens enumerate paired-device names from binary logs.
- The handler computes no hash itself; it delegates to `reg.Validate`, which owns the plain→hash boundary and the constant-time concerns documented in `devices/auth.go`.
- `ErrMalformedHelloFrame` carries no parser detail in its message; by the time it returns, the handler has not yet read `token`, so a caller wrapping it has no semantic reason to include the token in any context.

## Out of scope (deferred)

- **Wire wiring** — the relay-conn ticket that picks the carrier (extended routing envelope, synthesized `connection_opened`, or amended `hello`) and forwards the extracted token into this handler, plus the actual `client.Send` of `Response` and the WS close with `StatusUnauthorized`.
- **Rate limiting** — deferred per ticket body.
- **Replay-attack defenses** — deferred per spec § Replay attacks.
- **Distinguishing revoked from never-paired** — deferred per A5's two-state primitive; revisit if/when a tombstone primitive is scoped (would enable `CodeAuthTokenRevoked`).
- **Server-id auth (binary→relay direction)** — owned by #248.
- **Per-message dispatch on `Frames()`** — the post-auth `send_message` / `list_conversations` / etc. branching is a sibling future ticket.
