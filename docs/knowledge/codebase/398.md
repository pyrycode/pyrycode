# Ticket #398 — e2e test pins `--pyry-idle-timeout` eviction-then-respawn contract

End-to-end coverage for the silent-outage shape that [#396](396.md) fixed in production: an idle-evicted bootstrap is woken by a relay-routed `send_message` and the operator gets a wire-level ack within the documented 2–15 s respawn-latency window. `pyry --help` advertises the contract; this test makes it a runtime invariant. No production code change — one test file plus a small concurrency-safety touch-up to the e2e harness.

## Implementation

- **`internal/e2e/respawn_after_eviction_test.go`** — single `TestE2E_IdleEviction_RespawnsOnSendMessage` (`//go:build e2e`). Five linear phases: capture initial supervised PID via `pyry status`; wait for eviction (registry → `evicted`, `event=session.idle_eviction` WARN on stderr with `session_id`/`idle_timeout=2s`/`bootstrap=true`, initial PID returns `syscall.ESRCH`); dial `fakephone` through `fakerelay` and complete hello; send `send_message` with `sentAt := time.Now()`; assert `TypeAck` returns within 15 s and the post-respawn `Child PID` differs from the captured initial PID. Drives `fakeclaude` only — no `realclaude`, no Anthropic API. Wall-budget ≈ 6–9 s on a warm cache.
- **File-local harness helper `startEvictionHarness`** — the existing `StartRotationWithRelay` takes only `extraEnv ...string` (no flag tail), but the test needs both relay wiring *and* a `-pyry-idle-timeout=2s` flag override. Rather than mint a third public `StartRotation*` variant for a single caller, the test inlines `spawnWith(spawnOpts{...})` directly (the same shape `StartRotationWithRelay` does — ~20 lines, no new exports). The next test that needs the same combination pays the refactor tax.
- **File-local `childPIDFromStatus` / `waitForChildPID`** — parse `Child PID:     N` out of `pyry status` text via a `(?m)^Child PID:\s+(\d+)\s*$` regexp, gated on `Phase:         running` being present so the post-eviction window (where `ChildPID == 0`, omitempty-elided from the JSON, absent from text) returns `(0, false)` cleanly. Test polls until present (phase 1) and until *different* from initial (phase 5).
- **`internal/e2e/harness.go` — `Harness.Stdout` / `Harness.Stderr` are now `*safeBuffer` instead of `*bytes.Buffer`.** A test goroutine polling `h.Stderr.String()` for the eviction WARN was racing `os/exec`'s pipe-copy goroutine writing to the same buffer. `safeBuffer` already existed in `auto_attach.go` for the foreground-stdio clients; the e2e harness now shares the same shape. Touches `spawn` / `spawnWith` return-type signatures across the package; existing callers compile unchanged because every read site uses `String()` / `Bytes()` / `Contains` (no direct `*bytes.Buffer` method use). Added `safeBuffer.Bytes()` for parity.

## Patterns established

- **Test-file-local helpers over harness-shape changes for one caller.** When a single test needs a one-off combination of existing harness primitives (here: `StartRotationWithRelay`'s wiring + a flag-tail override), inline the composition in the test file. The next caller — if it ever appears — pays the refactor tax; the current caller doesn't grow the public surface of `harness.go`. Same shape decision `auto_attach.go` made for `ForegroundAutoAttachClient`.
- **Substring-set log assertions over slog recording handlers.** `containsAll(stderr, []string{"event=session.idle_eviction", "session_id=", "idle_timeout=2s", "bootstrap=true"})` pins the operator-facing fields without coupling to slog's key-ordering or requiring the harness to inject a custom logger. Future operator-facing WARN lines should be locked in the same way — pin the keys, leave the format breathing room.
- **`pyry status` text parsing is the public-surface contract.** The PO body listed "second `claude exited` for a different PID" as the cleanest respawn signal; the spec chose the `Child PID:` status line instead because it's a documented operator surface and the supervisor exit-line wording is internal text whose key set can shift. Future "is the supervisor up?" tests should anchor on status output, not log scraping.
- **Concurrent-safe buffers everywhere any harness write surface is polled.** Once one test polls `h.Stderr` while the daemon is still emitting, every harness consumer benefits from the same race-free read. The `safeBuffer` switch is a quiet upgrade; no caller had to change.

## Lessons learned

- **`pyry status`'s `Child PID:` line is absent — not zero — during the eviction window.** `internal/control/protocol.go:280`'s `child_pid,omitempty` elides the field when `ChildPID == 0`, and `cmd/pyry/main.go:574` only prints the line when the JSON has it. The first naive parse used `Atoi` on a hard-coded column slice and crashed on the post-eviction snapshot. Lesson for future status-parsing tests: gate the parse on `Phase: running` (which is always printed) *and* on the field's presence; treat absence as "not yet" rather than "zero".
- **`os/exec`'s pipe-copy goroutine is a silent concurrent writer to `cmd.Stderr`.** Any test that polls `h.Stderr` for a log signal while the daemon is still running is racing the copier. `go test -race` will catch it. The fix is the buffer, not synchronisation around the read — synchronising at the read side leaves the writer goroutine's `Write` unsynchronised. Same race shape `auto_attach.go` solved for `ForegroundAutoAttachClient.Stderr`; promoting `safeBuffer` to the package's shared shape was overdue.
- **The documented 15 s respawn-latency upper bound *should* be encoded as a runtime invariant, not just a deadline.** `phone.Receive(15*time.Second)` already enforces the timeout, but the test also computes `time.Since(sentAt)` after the ack and asserts it stayed under 15 s. The redundancy means: if a future regression slows respawn from ~2 s to 14.9 s, the test still passes — but the latency value in the failure diagnostic is now actionable evidence that the contract is drifting before it breaks.

## Out of scope (deferred)

- **Cap-policy eviction structured log line.** [#396](396.md) flagged this; still not done. When `Session.Evict` (driven by `Pool.Activate`'s cap policy) next gets touched, mirror the `session.idle_eviction` shape there so a sibling e2e test for the cap-policy respawn path has the same operator-facing signal to assert on.
- **`pyry status` JSON output mode.** Today the test parses text. A `pyry status --json` (or `-o json`) verb would let future tests skip the regexp and `Phase: running` gate. Not blocking — the regexp is robust to the cosmetic-whitespace flex the spec called out — but worth picking up if `pyry status` ever grows another caller.
- **RED→GREEN evidence wired into CI.** Per AC the developer demonstrated test-fails-on-revert (`git revert c974d59`) and test-passes-on-main in the PR body; CI runs only the green path. A failing-revert matrix job is overkill for one regression; revisit if a second silent-outage shape appears that the same trick would catch.
