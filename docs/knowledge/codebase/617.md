# Ticket #617 — Screen-snapshot wire types (`request_snapshot` / `screen_snapshot`) + v2 partition

Lands the v2 screen-snapshot wire **vocabulary** for [EPIC #596] (Phase 2 structured
streaming): the request/response pair `request_snapshot` (phone → binary inbound
control) and `screen_snapshot` (binary → phone outbound event), plus their two
`Type*` constants in the v2-only partition. **Wire types only** — pure protocol
structs and their (de)serialization. No interception, no render, no push, no
dispatch, no validation; every trust decision — accepting the inbound frame from a
remote party and returning rendered screen content to it — lives in the **consumer**
(the screen-snapshot handler child), which carries the `security-sensitive` label.
This is the same wire-types/consumer split [#607] (interactive event wire types)
drew against [#608] (the bridge consumer), and the reason this ticket is **not**
`security-sensitive` (it tracks design, not lineage — no trust boundary is here).

The pair backs ADR 025's always-available, parser-independent **screen snapshot** —
the floor of the safe-degradation strategy (ADR 025 § Safe degradation): the phone
may ask for a one-shot text picture of the current claude screen at any time, and
because the snapshot depends on no screen parser it survives any parser break and
backs the stall fallback. Package reference (updated this ticket):
[features/protocol-package.md](../features/protocol-package.md). Split from #612.

## Implementation

Two production files (~22 LOC of structs + constants):

- **`snapshot.go`** (new) — two json-tagged payload structs in the `interactive.go`
  per-type idiom (file-level "wire vocabulary only" doc comment, per-struct comment
  pointing at the `docs/protocol-mobile.md §`, **no `omitempty` on any field**, pure
  data, single `"time"` import):
  - `RequestSnapshotPayload{ConversationID}` — phone → binary. The doc comment states
    this is an inbound v2 **control** envelope, structurally like `TypeRekeyRequest`:
    the v2 session manager intercepts it at the dispatch boundary **before**
    `dispatch.Route`. There is **no `dispatch.Route` handler** for it — so the next
    reader doesn't look for a handler that isn't there; the interception/render/push
    is the consumer's job.
  - `ScreenSnapshotPayload{ConversationID, Text, TS time.Time}` — binary → phone. The
    doc comment pins two invariants: `Text` is **plain rendered text only, NEVER raw
    terminal control codes** (this is what preserves ADR 025's no-raw-bytes invariant
    and the substrate seal — the snapshot is a literal-screen picture rendered to
    text, not a stream of escape sequences); and `TS`'s monotonic-clock reading
    strips on JSON marshal, so callers MUST compare it with `time.Time.Equal`, never
    `==` or `reflect.DeepEqual`.
- **`codes.go`** (mod) — **one new cohesive const block** of two `Type*` constants
  (`TypeRequestSnapshot = "request_snapshot"`, `TypeScreenSnapshot = "screen_snapshot"`)
  appended after the #607 interactive block. The block-level comment carries the
  load-bearing MUST-NOT: both are **v2-only** and MUST NOT be added to `v1TypeSet`
  (a `request_snapshot` leak would route the inbound control to `dispatch.Route`'s
  handler chain; a `screen_snapshot` leak would expose an outbound event to an old
  phone). Per-constant comments carry the inbound-control / outbound-event-plain-text
  nature.

Tests + fixtures (~115 LOC, not counted toward the production budget):

- **`snapshot_test.go`** (new) — two per-type round-trips plus a boundary sub-test,
  reusing the **same-package** `roundTripEnvelope` / `readFixture` / `canonical`
  helpers (not redefined):
  - `TestRequestSnapshotPayload_RoundTrip` — decode fixture → assert
    `env.Type == TypeRequestSnapshot` and `ConversationID == "c1"` → `roundTripEnvelope`
    (canonical byte-equal).
  - `TestScreenSnapshotPayload_RoundTrip` — same, plus a **multi-line** `text`
    assertion (`"line one\nline two\nline three\n"` + an explicit
    `strings.Contains(…, "\n")`) since a rendered screen is multi-line, and
    `payload.TS.Equal(wantTS)` (parsed via `time.Parse(time.RFC3339Nano, …)`) for the
    `ts` field — never `==` / `reflect.DeepEqual`.
  - `TestSnapshotPayloads_EmptyConversationID` — programmatic (no fixture) boundary for
    both payloads: marshal `{ConversationID: ""}` → assert `"conversation_id":""` stays
    on the wire (no `omitempty` drop) → unmarshal → assert the empty string
    round-trips. Mirrors the `Seq==0` / `IsError==false` boundary-pinning style of the
    #607 interactive payloads.
- **`compat_test.go`** (mod) — three edits, no production change to `envelope.go` (the
  v1 absence is the whole mechanism): two `ErrUnknownType` rejection rows in
  `TestIsV1Compatible`; the two constants added to the test-local `v2OnlyTypes`
  allowlist (now **8** entries) and to the `all` partition slice; the partition size
  assertion balances at `len(v1TypeSet) + len(v2OnlyTypes) == 16 + 8 == 24`.
  `TestV1TypeSet_CoversAllExportedTypeConstants` and its hardcoded `16` are correctly
  **left untouched** — the new types are not v1 types.
- Two fixtures — `testdata/request_snapshot.json` and `testdata/screen_snapshot.json`
  (the latter with a multi-line `text` and a whole-second payload `ts`), each a
  complete single-line `Envelope` in the `turn_state.json` style.
- **`docs/protocol-mobile.md`** amended: two rows in the Application-message-types
  table and a new `### Screen snapshot (v2)` section with per-type field tables,
  directions, and the inbound-control / plain-text-only notes, tied to ADR 025
  § Safe degradation. `qmd update && qmd embed` run after the edit.

## Patterns established

- **Cohesive themed const block over splitting the pair.** The architect left the
  grouping open: `TypeRequestSnapshot` could have gone into the `TypeRekeyRequest`
  control block and `TypeScreenSnapshot` into the interactive block. The default —
  one snapshot block grouping the request/response pair — was taken so a reader greps
  "snapshot" and finds both adjacent with their shared rationale. **Correctness is
  grouping-independent**: `TestTypeConstants_V1V2Partition` enforces the v1/v2
  partition regardless of which block a constant lives in, so block layout is purely a
  readability choice. (Same belt-and-suspenders-of-different-fabric rail #454/#607
  established — advisory doc comments + the deterministic partition test.)
- **`ts` fixture byte-stability discipline.** `roundTripEnvelope` re-marshals the
  decoded payload struct and byte-compares against the fixture, and
  `time.Time.MarshalJSON` emits RFC3339Nano with **trailing fractional zeros
  trimmed** — so a payload `ts` of `"2026-05-08T10:33:14.120Z"` would re-emit as
  `".12"` and fail the compare. The fixture uses a whole-second payload `ts`
  (`"2026-05-08T10:33:14Z"`) and a `.5Z` envelope `ts`, both re-emit-stable. The same
  constraint already governs the envelope's own `ts` and every `time.Time` payload
  field in the package.
- **Reuse the package's round-trip helpers; never redefine them.** `roundTripEnvelope`
  (re-marshals the *decoded struct*, not the original `RawMessage`, so a missing or
  reordered json tag surfaces as a byte mismatch — the #607 lesson), `readFixture`,
  and `canonical` are all in-package already; redefining any of them is a compile
  error. This is now the fixed shape for every per-type round-trip in `internal/protocol`.

## Lessons learned

- **`qmd update && qmd embed` reindexes the canonical repo path, not the worktree.**
  A doc edit made inside a dispatch worktree reports `0 updated` until the branch
  merges, because the reindex runs against the canonical checkout. Running both
  commands still satisfies the AC; the index catches up on merge. (Lifted from the
  PR body's Lessons-learned section.)
- **The real-claude e2e gate is environment-fragile for pure-leaf tickets.** Code
  review reported the gate **PASS** here (`ok internal/e2e/realclaude 150.428s`), but
  flagged an operator-facing note (not a #617 finding): `make e2e-realclaude` fails
  out-of-the-box when the harness pins `$HOME` to a tempdir (`fixtures.go:36`), which
  defaults `GOMODCACHE` to an empty `$HOME/go` and forces a re-fetch of the **private**
  `github.com/pyrycode/tui-driver` module. This is the same credential/module-cache
  wall #607 hit; it is orthogonal to any `internal/protocol`-only diff.

## Scope (XS kept)

- **Production source files:** 2 (`snapshot.go` new, `codes.go` mod) — well under the
  ≥5 split gate, and smaller than #607's three.
- **New exported types:** exactly **2** (the two payload structs) — under the >5 red
  line. `TypeRequestSnapshot` / `TypeScreenSnapshot` are *constants*, not types.
- **Consumer call sites needing simultaneous update:** 0 — purely additive; review
  grep-confirmed no symbol references to the new constants/structs outside
  `internal/protocol/`, and no stray `"request_snapshot"` / `"screen_snapshot"` string
  literals awaiting wire-up.

## Out of scope (deferred to the consumer + siblings)

- **The interception, render, and push** — intercepting `request_snapshot` at the v2
  dispatch boundary (before `dispatch.Route`), rendering the current screen to text via
  tui-driver, and pushing `screen_snapshot` back — all live in the **consumer ticket**
  (the screen-snapshot handler child), which carries `security-sensitive`. The trust
  boundary is there, not here.
- **Other v2 event/control types** — `modal_*`, `queue_state`, `stall_detected`, and
  the remaining phone → binary control verbs (`modal_answer`, `interrupt`, …) — other
  #596 children and Phase 3 ([#597]).
